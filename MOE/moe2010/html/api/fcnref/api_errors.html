MOE 2010.10

API Error Functions

Syntax

    char *svl_ErrorText(void);

    svl_error svl_SetError(svl_error e);
    svl_error svl_SetErrorText(const char *str);
    svl_error svl_SetErrorFmtText(const char *fmt, ...);

Description

There is a single buffer allocated for the API user to use for error messages. This is a read-only buffer, and must be set with svl_SetError... calls only.

svl_ErrorText Returns the text string currently in the buffer.
svl_SetError Accepts an error handle and sets the contents of the error buffer to the contents of the error. A pointer to the error buffer is returned in order to return the handle to a calling routine.

svl_SetErrorText Sets the error buffer to contain the given string and returns a pointer to the buffer. str must be a '\0'-terminated C-string.
svl_SetErrorFmtText Allows you to create an error string using the standard C's printf-style format. The caller must guarantee that the string generated with the format will be shorter than SVL_ERRORTEXT_MAXLEN.
