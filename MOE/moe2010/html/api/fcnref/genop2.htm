MOE 2010.10

API Tree Traversal

Syntax

    typedef svlReal (*svlRealFcn_R)(svlReal);
    typedef svlReal (*svlRealFcn_RR)(svlReal, svlReal);

    svl_error svl_Elwise_R_R_(svl_var v, void * /* svlRealFcn_R */ fcn);
    svl_error svl_Elwise_R_RR_(svl_var v1, svl_var v2, void * /* svlRealFcn_RR */ fcn);

    svl_error svl_UnaryOp_R_R_(svl_var var, svlRealFcn_R fcn);
    svl_error svl_BinaryOp_R_RR_(svl_var v1, svl_var v2, svlRealFcn_RR fcn);

Description

These functions implement shortcuts of special cases of the API tree traversal routines. 

svl_error svl_Elwise_R_R_(svl_var v, void *fcn);
svl_error svl_Elwise_R_RR_(svl_var v1, svl_var v2, void *fcn);

Callbacks to be given to svl_UnaryOp and svl_BinaryOp. They expect a pointer to a (scalar) function fcn as their argument and implement an element-wise extension of that function to a flat vector of elements of a numerical type. The function fcn must be of type svlRealFcn_R and svlRealFcn_RR, respectively. 

svl_error svl_UnaryOp_R_R_(svl_var var, svlRealFcn_R fcn);
svl_error svl_BinaryOp_R_RR_(svl_var v1, svl_var v2, svlRealFcn_RR fcn);

Replacements of svl_UnaryOp and svl_BinaryOp. They expect a pointer to a (scalar) function fcn as their argument and implement an element-wise extension of that function to a (arbitrarily nested) vector of elements of numerical type. The function fcn must be of type svlRealFcn_R and svlRealFcn_RR, respectively.
