MOE 2010.10

Color Widget

Description

A pull-down color picker used for choosing colors. The current color selection appears in the collapsed widget. When pulled-down, a color palette consisting of two button banks becomes available. Custom colors can be selected using a further pull-down color chart.

The Color widget consist of four main areas:

  • Standard Palette. The standard palette organizes basic colors into a button bank for quick color selection.

  • Recent Colors. Recently-selected custom colors can be retrieved using the color history button bank. The 10 most recent colors are saved. Unused history buttons are gray and cannot be selected.

  • Color Chart. The color chart displays all available colors. Specific colors can be selected by clicking in the chart. The chosen color is shown in the New color box, and is contrasted to the current color selection, shown beneath it in the Current color box. Luminosity (intensity) is controlled separately by a slider below the color chart.

  • RGB/HSL Input. Colors can be specified numerically using RGB (Red-Green-Blue) or HSL (Hue-Saturation-Luminosity) triplets. When a color is selected in the color chart, its RGB and HSL values are displayed. Similarly, when the RGB or HSL values are modified, the location of the color in the color chart is given (when typing into the text fields, press <Enter> to put the new values into effect).

    Transparency can be specified using the Trans input field. By default, this field is not available (hidden). For applications requiring transparency, e.g. coloring objects in the MOE Window, the field can be shown by setting the transparency attribute appropriately (see below). Note that there are no transparency indicators in the Recent Colors button bank.

    The Display Hex checkbox is used to toggle between decimal and hexadecimal display of the RGB, HSL, and transparency values.

Press OK to make a custom color the current color selection. This will also collapse the widget. Pressing the Preview button causes the value of the previewed color to be returned, without changing the data value of the widget. The widget will stay open. OK must still be pressed to set the current selection to the previewed color. The Cancel button collapses the widget, leaving the original color selection unchanged.

Attributes

Color inherits all core attributes and defines the following attributes:

Attribute Description Values Default
transparency Display Trans input field, allowing input of a transparency value. 0/1 0
minWidth Minimum width of collapsed widget (in characters). 0 to 127 2

Trigger

The Color widget triggers when a color selection is made, i.e. when any of the OK, Standard Palette, or Recent Color buttons is pressed. The widget also triggers when the Preview button is used, in which case the value of the previewed color is returned. Note that the data value of the widget, however, is unaffected by a preview operation.

The Cancel button does not normally trigger the widget. The exception is if the Preview button has been pressed. In this case, there will be trigger, with the value returned being that of the original current color selection. This allows the programmer to cancel the effect of a previous Preview.

Data Value

The selected color, in TRGB format. The transparency occupies the high byte, followed by 1 byte each for the red, green, and blue components:

T*2563 + R*2562 + B*256 + G
