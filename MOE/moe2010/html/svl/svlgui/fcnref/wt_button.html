Button Widget

Description

The Button widget is a set of pushbuttons. A pushbutton is a widget that performs an operation when pressed.

Attributes

Button inherits all core attributes . onTrigger defaults to 'return'.

Button defines the following attributes:

Attribute Description Values Default
text Button labels Vector of tokens []
columns Number of columns for layout 1 to 127 127
type Return value data type 'char', 'int' 'char'
minWidth Minimum width (in characters) of each of the buttons 0 to 127 0
foreground Button text color WINDOW_COLORS 'black'
background Button background color WINDOW_COLORS 'background'
font Label text font WINDOW_FONTS 'medium'
uniformRows If set, each row is as tall as the tallest widget in that row 1(True), 0(False) 0
uniformCols If set, each column is as wide as the widest widget in that column 1(True), 0(False) 0
graphics A graphics object key (see gr_create) or graphics content (see gr_data) Scalar key, or vector content 0
grPosition Layout of graphic object with respect to text, when both are present 'left', 'right', 'above', 'below', 'overlay' 'left'
grScale Scaling policy for the graphic object 'none', 'anisotropic', 'isotropic', 'isotropic_shrink' 'isotropic'

Trigger

Button pressed and released over one of the buttons in the button widget.

Data Value

The returned data value is the token or one-based index, depending on the value of the type attribute, of the last button pushed button in that set of buttons.

The data value has no meaning unless the Button triggered a return from WindowWait. WindowGetData and WindowSetData are inapplicable in this instance.

Notes

  • The sensitive attribute takes a vector of 0,1 which correspond to each button given in the text attribute.

  • The number of tokens in the bubbleHelp attribute must match the number of buttons given in the text attribute.
