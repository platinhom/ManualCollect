Introduction

The MOE Window is where molecular systems are rendered and manipulated at the atomic level. It always reflects the current state of the system, and animates calculations such as energy minimization and molecular dynamics. The 3D rendering area provides different renderings of molecular objects such as atomic nuclei, bonds and backbone ribbons. Graphic objects such as electrostatic and water-accessible surfaces (or other graphical objects specified in SVL) can be rendered and displayed with the molecular system under study. Dynamically calculated distance meters and angle measurements can also be drawn, and general visualization settings can be changed at any time.

The menu bar at the top of the MOE Window provides direct access to commonly used commands, applications and tools. The MOE Window includes a command line interface (CLI), which is used to enter SVL commands and also serves as a prompt area for atom editing. Both the menu and the right button bar are fully customizable. The Cancel menu in the top right corner of the window allows you to halt any tasks running in MOE.

MOE also offers a collection of molecular builders that build and edit systems of varying complexity, from small molecules and carbohydrates to proteins and crystals:

  • Builder
  • Carbohydrate Builder
  • Protein Builder
  • Create Sequence
  • Crystal Parameters
  • Polymer Builder

To exit MOE, choose MOE | File | Quit. MOE will display a warning if molecular data has not been saved. You can either cancel the Quit operation or quit MOE and discard unsaved data. If nothing has been modified, MOE will terminate.

Errors in MOE are signaled by an error box which will prompt you for acknowledgement (e.g. when a syntactically incorrect command is entered at the CLI, or when atoms are deleted from the system while a calculation involving atoms is on-going). MOE will be suspended until you press OK in the error box, or until you press Enter or Esc in any of the MOE windows.

Command Line

The Command Line Interface (CLI) is where SVL commands are typed and issued. While the MOE Window has the focus, all keystrokes are automatically directed to the CLI regardless of where the mouse is positioned within the window.

The command line area is also used as a prompt area for several MOE functions. This prompt area may be for entering data or simply for displaying messages. In either case, to cancel the function which is displaying the prompt, press Esc. While the MOE Window's CLI is occupied by a prompt, the CLI in other windows will function as usual.

For more information on the Command Line Interface see the SVL Commands Window.

SVL and SEQ Buttons

The SVL and SEQ buttons are shortcuts for raising the SVL Commands Window and the Sequence Editor. When <Ctrl> is held down, the SVL button also serves as a shortcut for raising the SVL Modules & Tasks panel.

3D Rendering Area

The 3D rendering area is MOE's main visualization area for viewing the current molecular system. The mouse is the main vehicle for atom manipulation in this area. A quick reference for all the default mouse actions can be found in Using the Mouse.

The right mouse button brings up the popup menu, which contains a selection of commands that overlap both the menu bar and the right button bar.

At the top of the window, messages are often displayed during calculations. For example, while running the GizMOE minimizer you might see a message such as the one shown here. The font used for these messages may be customized and saved in the MOE resource configuration file using the MOE | Render | Draw font toggle buttons. To save the current values for future MOE sessions use the Save As Default button located at the bottom of this menu. For more information on customizing MOE see Configuration Options Panel.

Button Bars

The MOE Window includes a button bar to the right of the rendering area. Abbreviated RHS (for "right-hand side"), the right button bar includes shortcuts to the commands in the menus and is entirely customizable. It is also possible to program and display a left button bar in the MOE Window. See Customizing your Menus for information on adding commands. A full desciption of the MOE right button bar is given in the chapter LigX: MOE Right Button Bar.

Footer

The footer located at the bottom of the MOE Window provides a trackball, a zoom dial, clipping and fogging sliders, atom and ribbon rendering options, and forcefield and periodic systems controls. Additionally, the footer area provides system-dependent information and, depending on the atom selection set, may also provide controls for editing atom properties, interatomic distances and angles. Bubble help in the footer area gives information about the current system.

The footer area comprises the following elements:

  • R, S, L. Buttons for controlling mouse selection in the MOE Window.
  • Trackball. Trackball for rotating, translating, and zooming the view; View popup menu.
  • Dial. Zoom dial.
  • Fog. Defocus slider.
  • ZClip. Molecule and surface clipping planes sliders.
  • Contacts. Non-bonded contacts controls.
  • Atoms. Atom render panel.
  • Ribbon. Backbone ribbon render panel.
  • Forcefield. Opens the Potential Setup panel for selecting forcefields and solvation models.
  • Crystal Parameters. Opens the Crystal Parameters panel for enabling periodic boundary conditions and installing crystal parameters.
  • Footer Editing. Selection-sensitive area for reporting information about the system and for:
    • Editing atom name.
    • Measuring/editing interatomic distances.
    • Measuring/editing atom angles.
    • Measuring/editing atom dihedral angles.

Trackball

The trackball changes the view onto the molecular system. Dragging left over the trackball results in rotational motion. If the small box in the lower left corner of the trackball widget is pressed, then X-Y translational motion results. The same effect is achieved by holding down the Shift key while dragging (without pressing the box). Successive presses of the box toggle between rotational and translational modes; the box label switches between R and T accordingly.

Rotation can be restricted to one of the X or Y axes by clicking close to the edge of the trackball widget. Clicking in the corner of the widget restricts rotation to the Z axis. Clicking anywhere else in the widget will restore unrestricted rotational motion.

If there are selected atoms, then holding down the Alt key changes the positions of only those atoms, rotating and translating them just as if key modifiers had been used.

There is a popup menu on the trackball widget. Pressing the right mouse button brings up the View menu.

Dial

The dial beneath the trackball is used for visual zoom (translation of the view in Z). Holding down the Ctrl while dragging the mouse gives finer control (slower zoom). Zooming the view can also be achieved by holding down the Ctrl key while dragging over the trackball. The zoom operation affects only the view — holding down the Alt key has no effect.

ZClip
Fog

The ZClip sliders control the position of the front clipping plane (F) and, via the setting of the width (W) of the clipping area, the position of the back clipping plane. All atoms and objects closer than the front and farther than the back clipping planes are hidden from view. A second set of ZClip sliders (SF and SW) control the positions of the surface-only clipping plane, which only clips graphics objects such as molecular surfaces. The Fog sliders control where fading of the image begins (F), and the strength (S) of the fading beyond that point. The purpose of fogging is to give depth cues: objects farther away will appear dimmer. More details are given in the section on Clipping Planes and Depth Cueing.

The footer editing area provides information about the current system and is also sensitive to current atom selection, providing controls for:

  • Measuring/editing interatomic distances.
  • Measuring/editing atom angles.
  • Measuring/editing atom dihedral angles.

Two atoms selected. When two atoms are selected, the footer provides a textfield that reports the distance between them. If the atoms are bonded and free to move (not in a ring, not fixed), the interatomic distance can be modified by entering a new number (in angstroms) in the textfield and pressing Apply. The side that moves is specified using the star button, which toggles between the two atoms. In the MOE Window, the respective atom will be rendered with a starry halo. Three atoms selected. When three chained atoms are selected, the footer provides a textfield that reports the angle between them. If the atoms are free to move, the textfield is active, and the angle can be modified by entering a new number (in degrees) and pressing Apply. Four atoms selected. When four chained atoms are selected, the footer provides a textfield that reports the dihedral angle between them. If atoms are free to move, the textfield is active, and the angle can be modified by entering a new number (in degrees) and pressing Apply or by using the wheel. The side that moves is chosen using the star button, which toggles between the two sides. In the MOE Window, the respective atom will be rendered with a starry halo. Similarly to when two bonded atoms are selected, the Profile button pops open a plot of energy (kcal) vs. dihedral angle (degrees). Clicking in the plot sets the dihedral angle accordingly.

In all other cases, when atoms are selected, the reported information may include the number of selected atoms, whether they belong to the ligand or receptor, and the total number of atoms in the system.

Additional information is provided in the footer bubble help.

MOE Window Popup Menu

The MOE Window popup menu is opened by clicking the right mouse button in the 3D rendering area. It contains the same commands as found in the Rendering menu, (MOE | Render). Rendering options will be applied to selected atoms only or to all atoms if none are selected.

Selecting Molecular Objects

In many instances, operations are performed on only part of the molecular system. MOE lets you select the specific molecular objects that you wish to manipulate in the rendering area. Selected atoms are identified by a change in color.

There are three distinct selection sets in MOE: an atom selection set, a residue selection set, and a chain selection set. Atom selection can be accomplished in the MOE Window, Atom Manager, or Sequence Editor. Residue and chain selection is primarily performed in the Sequence Editor.

There are several basic methods used for selecting atoms. You can select atoms directly in the 3D rendering area using the mouse, you can select atoms using the selection menu in the menu bar, you can use the Atom Selector or you can type your own SVL commands at the command line to select atoms based on some criteria.

The following sections explain each of these atom selection methods. Note that there are many other ways of selecting atoms in MOE.

A residue remains selected as long as any of its child atoms are selected. Deselecting a residue deselects its atoms. Deselecting all atoms of a residue deselects the residue.

Tip   If you accidentally click in the 3D rendering area and lose your current selection, you can use the MOE | Selection | Restore command to restore the last selection set.

Selecting Atoms Using the Atom Selector

For more complex selection criterion, MOE provides a separate panel called the Atom Selector, which is opened with MOE | Selection | Atom Selector. In this panel, you can perform logical operations on selection sets, as well as select atoms based on specific properties, substructure or proximity. The Atom Selector is fully explained in Selection Panels.

Rotating, Translating and Zooming the View

You can use the mouse, menu View commands, and the trackball and zoom dial in the Footer to rotate, translate or zoom the molecular system in real time in the MOE Window. The SVL View function can also be used. Note that changing the view of the molecular system does not alter the positions of the atoms. To change atom coordinates, select the atoms to move and drag mouse middle (in conjunction with Shift for translation) while holding down the Alt key.

The most common ways of changing the view are:

  • Dragging the mouse while holding down the middle mouse button. This produces in rotational motion. When combined with Shift, translation results. When combined with Ctrl, zooming results. More details can be found in the chapter on Using the Mouse.

  • Using the MOE | Footer | Trackball and dial.

  • Using MOE | Render | Center or MOE | RHS | Center. These commands operate on selected atoms only and adjust the view so that the atoms are centered in the rendering area. If no atoms are selected then the commands will apply to all visible atoms, or, if no atoms are visible then to all atoms.

  • Using the MOE | Footer | Trackball Popup | View menu. The Reset command returns the view to the default setting (view look-at point [0,0,0], view orientation vectors [0,1,0], and [0,0,1]).

    The Save and Load commands allow you to save and load up to 8 view settings. Views saved into the viewing registers are preserved only for the duration of the current session. A view saved in one session cannot be retrieved in a subsequent session of MOE.

    Views can also be saved as part of molecule files; see Saving the System.

The view may also be modified through other panels or SVL functions. For instance, the Sequence Editor will perform a View operation after isolating the atoms of a specified chain. The view is specified by its look-at point as well as its y- and z-direction vectors. (For more information on the view look-at point, orientation and scale, see the View functions.)

Note: It is also possible to rotate atoms about a bond.

Changing the Center of Rotation

When the view is rotated, the atoms in the 3D rendering area appear to rotate. The center of rotation is, by default, the center of mass of the molecular system. To display x, y and z axes, select Coordinate Axes from MOE | Render | Draw. If the center of rotation is changed, the axes will be moved to the new center position.

It is possible to rotate the view with respect to a particular atom, i.e. the center of rotation can be made to coincide with any atom.

To change the center of rotation, click the middle mouse button on the atom that will be the new center of rotation. Then, rotate the system by dragging the middle mouse button.

To reset the center of rotation to the default (center of mass of the system), click the middle mouse button away from any atoms.

Rendering Molecular Objects

The Render menu is used to control molecule rendering. Different rendering modes are available for displaying molecular objects. Atomic nuclei can be rendered as points or spheres, and bonds as lines or cylinders. Backbone ribbons for proteins and nucleic acids can be rendered with smooth splined surfaces in flat or cylindrical styles, and hydrogen bonds can be displayed dynamically as dashed lines.

MOE does not support a "current rendering mode" that is applied to all subsequent additions to the molecular system. Rendering options apply only to atoms already in the system. The same is true for atom coloring, labeling and hiding. One exception is additions to the molecular system that are made using the Builder. Groups that are substituted onto a molecule inherit the atom nucleus and bond rendering modes of the atom being substituted.

The complementary Show menus are located next to the Hide menus. Hiding and showing individual molecular subsystems is most easily done using MOE | RHS | System. The right button bar also contains a Hydrogens button dedicated to the setting the visibility of hydrogens. By default, MOE colors atoms by element. Other coloring styles are offered in the MOE | Render | Atoms panel.

Atom coloring for molecular systems as a whole or in part can be controlled using the coloring style buttons in the system list (MOE | RHS | System).

Labeling Atoms

Atoms in the 3D rendering area may be labeled with a variety of components. The display of these components is controlled using the MOE | Render | Atoms panel (also available from MOE | Footer | Atoms).

Labels are applied to the selected atoms only. If no atoms are selected then the labeling is done on all atoms. The buttons in the panel act as toggles: pressing once turns on the display of the associated label component; pressing a second time turns off the display. If multiple labels are displayed on an atom, the labels are separated by commas and slash characters. The X button clears all labels from the applicable atoms.

Rendering Ribbons and Backbones

Rendering Nonbonded Contacts

Hydrogen bond contacts and van der Waals contacts (clashes) can be rendered in the MOE Window in a variety of styles. The MOE | Render | Contacts popup panel, which is also accessible from MOE | Footer | Contacts, provides fine control over which interactions are drawn as well the rendering mode. Separate controls are provided for hydrogen contacts and clashes.

At the top of the popup panel, the H-Bond and Clash checkboxes enable or disable rendering. Next to each checkbox is a button which cycles through the contact rendering modes: dashed, dashed with a strength bar, and dashed with contact strength annotation. A text field with arrows provides the threshold (kcal/mol) below which contacts will not be displayed.



The lower part of the panel provides an interaction matrix for specifying which receptor (Rec) / solvent (Sol) / ligand (Lig) interactions are displayed. Buttons within the matrix cycle between displaying the contacts using the rendering style specified in the top of the panel, displaying the contacts weakly as thin dashed lines, and not displaying the contacts at all.

For hydrogen contacts, additional checkboxes enable/disable the display of backbone-backbone interactions (BB), CH..X contacts involving non-standard H-bond donors and acceptors (CH), and proton-π interactions (H-pi). Proton-π interactions – interactions between planar rings and protons – are calculated using a geometric model and drawn in a similar fashion to hydrogen bonds. These interactions are intended to represent non-standard hydrophobic interactions. Standard hydrogen bond donor and acceptor definitions are controlled by the $MOE/lib/hbond.svl configuration file which contains SMARTS pattern tables that determine candidates for hydrogen bonds.

For clashes, additional checkboxes enable/disable the display of backbone-backbone interactions (BB), van der Waals clashes (vdW), and lone pair repulsions (LP).

The Contacts panel popup menu, opened by right-clicking in the Contacts panel, is used to save the current panel settings and use them as defaults for future MOE sessions. The menu also has a command to restore the default settings.

Clipping Planes and Depth Cueing

The Footer contains controls for the Z-axis clipping region and the Z-axis depth cue shading parameters. Atoms hidden by Z-clipping participate normally in computations.

ZClip

Parts of molecules can be clipped away using 3D front and back clipping planes. The global clipping planes affect the complete molecular system, including atoms, ribbons, surfaces, graphics objects, etc. The placement of these planes is controlled by the ZClip F (front) and W (width) sliders. The positioning of the clipping planes is relative to the bounding box that delimits the current molecular system. The bounding box updates dynamically with changes to the system.

F moves the front clipping plane away from you. The default position is at the front of the molecular system bounding box.

W sets the distance between the front and the back clipping planes in terms of percentage of the bounding box. The default position of the back clipping plane is at the back of the bounding box (i.e. width is 100%).

Note: Hidden atoms cannot be selected by clicking on them with the mouse (they can, however, be unselected by clicking with the mouse). Selected hidden atoms are otherwise treated normally, and will be affected by any editing or movement commands that apply to selected atoms.

A secondary clipping region just for graphics objects such as molecular surfaces is controlled by the ZClip SF (front) and SW (width) sliders.

SF moves the front surface clipping plane away from you. The default position is at the front of the molecular system bounding box.

SW sets the distance between the front and the back surface clipping planes in terms of percentage of the bounding box. The default position of the back clipping plane is at the back of the bounding box (i.e. width is 100%).

Fog

The three-dimensional appearance of molecules can be enhanced using distance-dependent shading. Fog (depth cue) parameters are controlled by sliders:

F (front) sets the distance from the viewer at which shading begins to take effect. The distance is given as a percentage of the bounding box delimiting the current molecular system.

S (strength) sets the rate at which intensity decreases as a function of distance.

Rotating About a Bond

To rotate atoms about a bond, select the first atom, press Shift and select the second atom in the bond. Rotating about the bond can be achieved using the controls in the MOE Footer or by pressing the Alt key and dragging the left mouse button.

Only atoms on one side of the bond will move. When using the MOE Footer wheel, the side that moves can be specified using the star button. When using the Alt key, the side that moves is defined to be the one with fewer atoms or, if there are fixed atoms in the system, the one with no fixed atoms. No rotation will occur if:

  • More than two atoms are selected
  • Selected atoms are not bonded
  • One of the selected atoms has no other atoms bonded to it
  • The bond is part of a ring
  • Fixed atoms are preventing the rotation

Editing Atoms Using the Edit Menu

The Edit menu contains a variety of commands for editing atom properties. Some editing commands open a prompt in the CLI of the MOE Window, allowing you to make changes. Modifications are done immediately and are reflected in the entire MOE system. Note that, when atom prompting is active, normal atom selection is not available.

Until explicitly canceled, prompts remain active and stack on top of each other in the prompting area. Press Esc to remove a prompt, or cancel the editing command by choosing it from the Cancel menu.

Copy and Paste

MOE provides clipboard copy and paste commands to facilitate the transfer of data between MOE and third party software. For example, data from 2D sketchers can be imported into MOE via the clipboard, and high quality images can be placed on the clipboard.

The MOE | Edit | Copy As menu provides commands for copying data from the MOE Window onto the clipboard. Each item in the menu determines a data format that will be placed onto the clipboard: molecule formats place molecular data onto the clipboard, the Picture option places image data onto the clipboard. If the MOE molecule format is chosen and no atoms are selected, graphics, view settings,and crystal parameters (if periodic systems are enabled) are also written to the clipboard.

MOE | Edit | Copy As | Picture places a picture onto the clipboard after setting size and related options in the Copy Picture to Clipboard panel.

  • Format. Specifies the image format in which the picture will be put onto the clipboard. One can select between Windows Bitmap (.bmp), Enhanced Windows Metafile (.emf), and Encapsulated Postscript (.eps). The first two formats are only available under Microsoft Windows.

  • Dimension. Specifies the picture size in pixels, inches, centimeters or percent. Pictures of any size can be created. Units of percent are calculated relative to the current MOE window size.

  • Resolution. Specifies the picture printing resolution. The resolution determines the size of the picture when printed. For example, a 300 dpi picture of 300x300 pixels will occupy a 1 inch square when printed. The resolution is not saved with the picture on the clipboard; it is used here to calculate the picture size in pixels of the picture to be created.

  • Anti-Alias Picture. This will apply high quality anti-aliasing to the output picture.

  • High Quality. This will apply maximum ribbon and focal blur quality to the output picture, otherwise the quality settings from the Visualization Setup panel are used.

  • White Background. Linux and Mac. If on, the picture background color will be white. If off, the background color is retained in the image file.

  • Transparent Background. Windows. If on, the picture pixels matching the color of the MOE Window background are marked as transparent. If off, the background color is retained in the image file. This option is only available for the EMF format.

The MOE | Edit | Paste menu command allows the loading of clipboard data into MOE. If the clipboard contains a SMILES string, a molecule file, or an ISIS/Draw or ChemDraw sketch of a molecule, the data will be loaded directly into MOE. If it contains an ISIS/Draw or ChemDraw sketch of a query molecule (for example with ambiguous bond orders) a sketch panel will appear instead (see below). In all other cases the Paste Special panel will be raised allowing loading of other formats.

Sketch View

When a query mol file (SKC file, .skc) or sketch is pasted from the clipboard or loaded into MOE using the Open panel, the Sketch View panel will be displayed.

Some features will be indicated by a letter (a, b, c, etc.) with a legend indicating what the letter represents (a bond type, element list, and so on). Several buttons appear below the sketch. If the query is a 3D query molecule, a message to this effect will appear. Use the Create button in this case to display the query in the MOE Window; constraints, lines, normals and planes will be drawn as graphics objects. The representation in the MOE Window is strictly for purposes of visualizing the query in 3D.

Export... allows the sketch to be saved in various graphical formats and Print... outputs the sketch to a printer. Close will exit the panel.

Creating and Deleting Meters

Meters are used to measure bond distances, bond angles and dihedrals of atoms. To create meters, use the MOE | Edit | Measure menu (or, equivalently, MOE | RHS | Measure) and choose the type of measurement to perform. In all cases, the Measure CLI prompt appears and prompts for the selection of the appropriate number of atoms:

  • Distances. Measure and display the distance between 2 atoms.
  • Angles. Measure and display the angle between 3 atoms.
  • Dihedrals. Measure and display the dihedral angle between 4 atoms.

Distance, Angle and Dihedral Meters

If a mistake occurs and the wrong atom is selected, it can be deselected by simply clicking on it a second time. The prompt in the CLI will update automatically to reflect this change. As soon as the required number of atoms has been selected, a meter will appear. You can continue selecting atoms and creating meters as long as the prompt is up. Press <Esc> to cancel the prompt.

Meters are displayed in the MOE rendering area as a colored line with a measured value. Meters are automatically updated when atom positions are altered.

To delete meters, use the Remove commands from the MOE | Edit | Measure (or, equivalently, MOE | RHS | Measure) menu. This will allow you to remove all meters or only distance, angle, or dihedral meters.

Interactive Superposition

MOE | Edit | Superpose superposes any number of molecules. Start by specifying atom "associations," or sets, each of which comprises one atom from each molecule in the superposition. Atoms in each set are to be superposed exactly (or as nearly as possible) in the final optimal solid body rotations and translations of the molecules, and thus specify how the superposition is done.

A minimum of 3 and a maximum of 10 atom sets may be defined. The sets should all be different from one another, and their atoms must all originate from the same molecules. Atom sets need not be filled in order, and one set need not be complete before beginning another: any set may be added to or otherwise modified at any time by first clicking on the appropriate number in the Set button bank to activate that set. Atoms already in the currently active set are shown as selected. To remove an atom from the set, click the left mouse button on it to toggle its selection state. When a set has its full complement of atoms (up to a maximum of the number of separate molecules present in the system), no more atoms can be added.

When all sets are of the same size, and at least three have been filled, superposition is possible. Click on the Superpose button in the prompt area to perform the superposition. If you want the prompter to stay up once the superposition is done, first turn off Close on Superpose.

Protein Superposition is also available, and can be invoked from the Sequence Editor Homology menu.

System Priority

A process priority level is a value used by the operating system to determine how CPU time should be allocated. Using the MOE | Compute | Priority menu options you can choose to change the system priority. The priorities range from Highest to Lowest. A lower value results in less favorable scheduling and hence slower execution. Conversely, a higher value results in more favorable scheduling and, as a consequence, faster execution. The number of priority levels available and the ability of the user to change the priority levels depend on the operating system being used. On some machines, some or even all of the buttons found in this menu may be grayed out.

MOE can be completely suspended using MOE | File | Suspend.

Doing so will render the current MOE session inactive and return its tokens to the license server. MOE may be reactivated by pressing Resume, in which case the session once again will consume license tokens.

  • For more information on licenses, please refer to the chapter on Installation and Running MOE.

Closing the System

Deleting atoms can be done using the Delete button in the Edit menu. To delete all objects in the system (i.e. to close the system), use the MOE | File | Close command. This command will delete all molecular objects in the current system as well as all graphic objects. A confirmation dialog is displayed.

Plotting Dihedral Energy and Dihedral Contour

Dihedral Energy

The MOE | Compute | Conformations | Dihedral Energy Plot command plots potential energy as a function of a dihedral angle. MOE calculates the potential energy of the entire molecular system and makes a succession of calculations as the atoms attached to one side of a user-specified rotatable bond are rotated through 360 degrees. The differences in potential energies are then plotted against the dihedral angle, with the current angle and energy indicated by a vertical line.

MOE prompts you to select four atoms defining the bond about which the dihedral energy is to be calculated at different angles of rotation. MOE then displays a plot of energy vs. angle. (Note that if the chosen atoms belong to a ring system, or if there are fixed atoms, rotation may be impossible.)

The dihedral energy can also be plotted using the Profile button in the MOE Footer. This button is available when two bonded or four chained atoms are selected and which define a rotatable dihedral angle. Clicking in the profile plot will set the dihedral angle accordingly.

Dihedral Contour

 The MOE | Compute | Conformations | Dihedral Contour Plot command plots potential energy as a function of two dihedral angles. This command is similar to the Dihedral Energy Plot command (above) except that the potential energy is calculated as a function of two dihedral angles, thus yielding an energy surface. The surface is plotted in 2D as a contour plot.

Canceling Tasks

The Cancel menu shows all currently running tasks. Tasks are SVL processes, such as commands issued from menus or the CLI as well as MOE windows and panels. More formally, a task is a single SVL instruction execution stream, together with its associated resources, such as open files or windows. A new task is started each time you issue a command, press a button that triggers an activity or select an option from a MOE menu.

In MOE, more than one task can run at a time. For instance, when building a small molecule, you can have both the Builder and the GizMOE Minimizer active concurrently. Running tasks are listed in two places:

  • Cancel menu in the menu bar (as well as in the Cancel menu of the Sequence Editor and Database Viewer windows).

    Active tasks are colored black, while those that are waiting or in a state of sleeping are red. You can terminate the execution of any of the tasks by selecting the task in the menu. This will halt its operation, leaving MOE in whatever state it was in at the time of cancellation.

  • Tasks page of the Modules & Tasks panel available from the Window menu.

Note: The non-graphical version of MOE (moebatch) is not multi-tasking. For more information, see Installing and Running MOE/batch.
