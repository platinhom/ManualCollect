Using the Atom Manager

The Atom Manager is used to display and modify the properties of molecular objects currently loaded in the MOE Window. The Atom Manager is for viewing and editing purposes only, and cannot be used to add or delete molecular objects.

In the MOE Window, open the Atom Manager by choosing Window | Atom Manager. Double-clicking on an atom in the MOE Window also opens the Atom Manager. The clicked-on atom is highlighted in the Atom Manager.

The list in the upper section of the panel shows the hierarchy of currently loaded molecular objects (chains, residues and atoms), along with their names and properties.

The Selection Only and Visible Only toggle buttons confine the list to atoms and residues that are currently selected or currently visible in the MOE Window. Chains in the system are always listed, regardless of their selection or visibility states.

To edit an object's properties, select the object in the Atom Manager. Only one object may be selected at a time. Fields pertaining to the object's properties will appear below the list. Editing the relevant field, followed by pressing Apply at the bottom of the panel, will modify the properties of the object.

Important! Modifications made to a molecular object will only be recorded when Apply is pressed. If another molecular object is selected before Apply is pressed, you will lose any changes made. No warning is given.

Each chain and residue can be expanded to show its component objects or compressed to hide its component objects. Double-clicking on a residue or chain toggles its expanded/compressed state. All the chains and residues in the system can be expanded or compressed simultaneously using the Expand All and Compress All buttons at the bottom of the panel.

Double-clicking on an atom in the list toggles its selection state in the MOE Window.

Chain Properties

The properties of a chain are its selection state (shown as a fletch in the leftmost column), number, number of associated residues [r:], number of associated atoms [a:], and name (usually the compound name). These properties are displayed on a single line in the Atom Manager. The example below shows a chain named 1MCT.A which contains 223 child residues and 1642 component atoms.

Selecting a chain in the list displays the properties name, tag and header which may be edited. These properties have meaning in certain file formats, but otherwise they may be filled according to preference; for instance, in PDB files, tag is the 4-letter PDB code (e.g. 9INS), name is the 4-letter code followed by a period and a letter (used if there are multiple chains with the same code, e.g. 9INS.A), and header is the molecule name (e.g. INSULIN).

Press Apply after completing the modifications for a given chain.

Residue Properties

The properties of a residue are its selection state (shown as a fletch in the leftmost column), number, name, tag, UID and INS codes, and number of associated atoms [a:]. These properties are displayed on a single line in the Atom Manager.

Selecting a residue in the list displays relevant residue information such as residue name, residue type (e.g. amino acid), and the PDB UID and INS codes. The residue name has a maximum of four characters. The residue type may be either one of the six residue types defined in MOE (amino, l-amino, d-amino, DNA, RNA, heme), or "none" if the residue type is undefined in MOE. The UID and INS codes are indicators of a residue's position with respect to a reference protein, and are only significant in the context of proteins. UID is the residue sequence number, and INS is a letter that indicates that the residue is an insertion at the position given by UID (the default for INS is a blank space).

Press Apply after completing the modifications for a given residue.

Atom Properties

Atom properties include selection state (>), fixed-position state (F), atom name, backbone state (|), element, lone-pair hint state (:), hybridization, partial charge (q), atomic mass (m), molecular mechanics forcefield atom type (MM) and the temperature factor (B, as provided in PDB files, for example).

Selecting an atom in the list displays its relevant information in the editable area below. You can enter or select new values for the highlighted atom and press Apply to update information in the list.

Atom names, for instance, can be any combination of up to four characters or digits. To display them in the MOE Window, you would choose MOE | Render | Atoms | Name. The position of the atom can also be modified by editing the xyz coordinates (in angstroms). An arbitrary scalar can be assigned to the atom using the Scalar field. The fixed-position, backbone, and lone-pair hint status flags can also be altered.

Note: Although the ionization state, scalar and xyz coordinates of the atom do not appear in the Atom Manager list, these values are editable in the panel.

Toggle buttons at the bottom of the panel let you turn on or off the following properties (see picture above):

Property Appears as: Description
Selected fletch > in leftmost column Atom is selected in the MOE Window.
Fixed character F in the second column Fixed atoms are anchored to their current positions and cannot be moved until they are unfixed.
Backbone vertical bar | before the element Backbone atoms are considered to be part of the mainchain of the molecule.
HintLP colon : before the hybridization HintLP indicates the presence of a lone pair which is not delocalized into a pi electron system.

Note: When editing atom properties using the Atom Manager, automatic hydrogen adjustment is not performed.

SVL Commands

Although atom, residue, and chain properties are easily accessible from the graphical interface, SVL functions offer greater control and flexibility for modifying the attributes of a molecular object. Atom, residue, and chain properties can be accessed and edited both at the SVL command line and from within SVL programs using the following SVL functions:

  • Atom Functions
  • Residue Functions
  • Chain Functions

The hierarchical relationship between atoms, residues, and chains can be managed using the SVL Molecule Object functions.
