MOE 2010.10

Residue Functions

Syntax

    [restraint_keys, residue_sets, vis_flag] = rConstraints [] 
    restraint_keys = rConstraintSet [ constraint_key, residue_keys ]
    rConstraintDisplay [ constraint_key, vis_flag ]
    AlignmentConstraintsManager []

Description

[restraint_keys, residue_sets, vis_flag] = rConstraints []

Returns information on all the alignment constraints in the system. Alignment constraints are sets of residue object keys.

The following information is returned:

restraint_keys
Vector containing the identifying keys of all the alignment constraints.

residue_sets
Vector containing the corresponding residue sets.

vis_flag
Vector of ones and zeros, indicating whether or not the respective constraints are to be drawn in the Sequence Editor. 

restraint_keys = rConstraintSet [ constraint_key, residue_keys ]

Used to create or modify an alignment constraint. To create a new constraint, pass rConstraintSet a key value of 0. The second argument is a vector of residue object keys. It is an error to pass invalid residue keys to rConstraintSet. To delete a constraint, pass the constraints key along with an empty residue set.

For example: 

    svl> c1 = pro_SeqCreate ["AGCE"];
    svl> c2 = pro_SeqCreate ["YWPQ"];
    svl> res = cResidues [c1,c2]
    svl> key = rConstraintSet [0, [first res(1), last res(2)]]
    svl> [keys, sets, flags] = rConstraints[]
    svl> rName sets(1)
    ['GLN','ALA']
To delete this constraint: 
    svl> rConstraintSet [key, []]


rConstraintDisplay [ constraint_key, vis_flag ]

Used to control the visibility status of an alignment constraint. Invalid constraint keys are ignored. 

AlignmentConstraintsManager []

Opens a panel which allows the user to create, delete, show or hide alignment constraints.
