Introduction

The purpose of the Pharmacophore Search is to use a pharmacophore query to filter databases of 3D conformations based on the positions of annotation points derived from each of the conformations. The main characteristics of the Pharmacophore Search are:

  • Boolean Expressions. Query features can contain boolean expressions to encode concepts such as "Donor and not Cationic".

  • Substructure Matching. Pharmacophore query features can be augmented with SMARTS patterns to search for particular functional groups. The substructure tests are efficiently performed during the search itself (not as a post-filter).

  • Volume and Shape Filtering. The query can be focused by adding restrictions on the shape of the matched ligands in the form of a union of spheres. Alternatively, ligand hits can be filtered to enforce shape occupancy. SMARTS patterns can be applied to individual volume and shape constraints.

  • Partial Matching. Only those features of the query that are marked as "essential" must be matched to a corresponding ligand annotation point. Other query features may be left unmatched, resulting in a "partial match". The search outputs all partial matches of the query that contain at least a given minimum number of matched features.

  • Group Constraints. Match constraints can be applied to, for example, match "exactly one of" a subset of features, or to impose that a hydrogen bond heavy atom matches with its projected feature simultaneously.

  • Existing Conformations. External conformation databases can be searched efficiently since no indexing or prior preparation steps are required.

  • Systematic Matching. All possible matches of a ligand conformation and the query are systematically examined (and are reproducible).

The Pharmacophore Search searches databases of 3D conformations. Existing conformation databases may be searched (such as those distributed with MOE or generated by other programs) without pre-processing. A pharmacophore query is required for use by the Pharmacophore Search. A query can be built manually using the Pharmacophore Query Editor or automatically using the Pharmacophore Elucidator.

A general introduction to pharmacophores can be found in Introduction to Pharmacophores in MOE. The chapter on Overview of Pharmacophore Applications provides terminology and definitions.

Methodology

The Pharmacophore Search applies a pharmacophore query to each conformation of each molecule in a 3D conformation database. This database can be in one of the following formats:

  1. Unpacked MOE Molecular Database - 1 Conformation per Entry. An unpacked molecular database must contain the following fields:
    • a molecule field with a single conformation per entry.
    • an optional molecule sequence field with any numerical field such that the molecule sequence number. The conformations of the same molecules are stored in consecutive entries that share the same molecule sequence number. Use of this field is optional: if the molecule sequence field is not used, the molecule sequence is calculated from the molecule field.
    The unpacked database is the default way of storing molecular conformations in MOE.

  2. Packed MOE Molecular Database - 1 Molecule per Entry. A packed molecular database must contain the following fields:
    • a molecule field containing a (possibly 2D) molecule.
    • a conformation field containing multiple conformation coordinates of the atoms in the molecule field, compressed into a single binary vector. The name of the conformation field must be formed by appending the suffix ':conf' to the name of the molecule field, e.g. 'mol:conf'. Optionally, the conformation field may also store the energies of the compressed conformations. The presence or absence of the conformation field indicates that the database is packed or unpacked, respectively.
    The packed database must be generated from an unpacked version using the db_ConfPack utility function.

  3. OMEGA OEB - 1 Molecule per SD Entry.
    • The OEB file must have been previously generated by the OMEGA program and given extension .oeb or .oeb.gz.
    • A special extractor is used to convert (at run-time) the OEB data into SD format with a special convention on conformation and energy encoding.
    • The default name of the extractor is 'oeb2sdconf'. The user can specify a different name by setting the RC variable 'oe_omega', e.g.
          SetRC ['oe_omega', 'my_new_oeb2sdconf']
    • The extractor is expected to be an executable file located either in the $OE_DIR/bin/ directory or one of the directories listed in the $PATH environment variable.
    Searching OEB files is only possible on machines that have access to the OMEGA OEB extractor. The OEB extractor is not a part of MOE distribution files.

The database type is automatically detected from the specified input file.

Given a pharmacophore query and a collection of conformation files, the Pharmacophore Search will produce a MOE Molecular Database with the following fields (all of which are optional):

Molecule: mol (molecule)
Contains the conformations found in the input databases possibly rotated and translated to match the pharmacophore query.

RMSD: rmsd (float)
The root of the mean square distance between the query features and their matching ligand annotation points.

Sequence: mseq (int)
An integer value that has the properties of a molecule sequence number. The value of the field starts at 1 and is incremented with every new hit molecule.

Row: row (int)
The entry number of the matched conformation in the searched input database.

Map: hitmap (int)
A numerical representation of the mapping between the query features and their matching ligand annotation points. All annotation points are distinguished, regardless of their symmetry. For each feature of the query, the field stores the index of the matched ligand annotation point or 0 if the feature is not matched.

The search compares each ligand conformation in the database to the query. Instead of formal output, the algorithm can select hits in an opened Molecular Database Viewer. The algorithm for generating matches is as follows.

Input: A single molecular conformation and a query.
Output: New entries in a database of hits created as the output of Pharmacophore Search.

For each possible mapping between the query features and the ligand annotation points:

Step 1a. If the set of labels present on the ligand annotation points does not satisfy the expressions associated with the matched query features, reject the mapping.

Step 1b. If the distances between the ligand annotation points do not agree with the distances between the matched query features (after discounting the radial tolerances), reject the mapping.

Step 2. If the matched query features do not satisfy the additional group constraints, reject the mapping.

Step 3. Superpose the ligand annotation points and the matched query features using rigid-body superposition. If the ligand features fail to fall within the specified tolerance radii of the matched query features, reject the mapping. During the superposition, the relative weights of the matched query features are inversely proportional to their radii.

Step 4. Read the atoms of the ligand from the database and superpose the ligand atoms onto the query using the superposition transformation calculated in Step 3. If any of the ligand atoms violate any of the volume restrictions in the query, reject the mapping.

Step 5. If the mapping has not been rejected, there is a "hit". Report the result by increasing the hit count.

Step 6. If there is a hit and there is an output database, generate a new entry in the output database. The amount and nature of the information written to the output database for each match found is customizable.

Step 7. If there is a hit, and only one hit per conformation or one hit per molecule is requested or there is no output database, skip all remaining mappings.

Note! Steps 1a and 1b are accomplished using a special low-level state machine that iteratively generates all subgraph isomorphisms of clique graphs with labeled edges and vertices that satisfy given constraints. The state machine is accessed with the SVL built-in functions dmatch_XXX.

Running Pharmacophore Search

MOE's Pharmacophore Search is used to apply a query to a database of ligand conformations and report those conformations that satisfy the query. The query must have been created with MOE's Pharmacophore Query Editor.

The Pharmacophore Search panel can be opened using one of the following methods:

  • From the Database Viewer, choose DBV | Compute | Pharmacophore | Search....
  • From the Pharmacophore Query Editor press the Search button at the bottom of the panel. Use the Open panel that appears to choose the database of conformations to be searched and press OK.
The following panel appears:

If the input database is in the packed format, the Sequence option will be replaced by a Conf textfield showing the name of the conformation field.

Input Settings


The Input database shows the name of the database(s) to be searched. The Open button (to its right) is provided as a convenience to open the specified database in a Database Viewer (if possible). If the Entries or Hits options are to be used in the search, the Database Viewer must be opened.

The Input field is not directly editable but can be changed using the Browse button to the right whereupon a panel (depicted to the right) will appear. It is possible to specify more than one database on which to operate the search. Locate as many databases as desired, and double click or select then press Add. The Remove button correspondingly removes databases from the list. After pressing OK, the Pharmacophore Search window will display the path of the database in the Input field if there was only one, or (...multiple files...) if more than one.

The Entries option menu determines which entries to search (subject to the Subrange discussed below). The choices are:

  • All. All the input entries are searched.
  • Selected. If the input database is open in the Database Viewer, only selected entries are searched.
  • Visible. If the input database is open in the Database Viewer, only visible entries are searched.

If Subrange is on, only the input entries within the specified range (to the right) are searched. Note that the entries searched are actually the intersection of this subrange with the Entries setting above.

The Molecule menu specifies the name of the molecule field containing conformations (or topology, as the case may be) to be searched. Conformations of the same molecule must be stored in consecutive entries or in a packed conformations field (see Methodology section).

The Sequence field specifies a data field used to differentiate molecules in the input database when one conformation per entry format is used. In such a case, conformations of a single molecule must have the same molecule sequence number and must be stored in consecutive entries. Different molecules must use different sequence numbers. If the pre-calculated field is not used, new molecules in the list of conformations will be detected during the search by comparing the molecular graphs. If the Molecular Sequence field is not used, the Sequence option must be set to '(none)'.

Specifying the Query


To perform a search, a query needs to be specified. There are two methods of specifying the query:
  • Specifying the query from a file. The Browse button next to the Query field allows you to use a predefined query from a file. The name of the query file will be displayed in the Query text field and the Title field will show the title of the query if one has been saved with the query. The query file must have been created using the Pharmacophore Query Editor.

  • Using the Pharmacophore Query Editor. If the Pharmacophore Search panel was opened from the Pharmacophore Query Editor, the query is already specified. Press Edit to open the Pharmacophore Query Editor to create or modify an existing query. If this is the case then the Browse button is removed and the Query text field will show the text 'Pharmacophore Query Editor'.

The Use Absolute Positions toggle allows for two methods of determining the output molecules. If absolute positions are used, the molecules are assumed to be pre-aligned and fixed in space. The atoms of each hit molecule will not be repositioned to minimize the RMSD between the query points and the matching ligand annotation points.

Output Configuration

Each ligand typically has multiple conformations. During the search only some may match the query. Moreover, a single conformation may match the query in more than one way. Pharmacophore Search offers several alternatives for viewing the results of the search:

  1. A report can be generated showing the number of hits and other statistics. The report is displayed in an SVL Text Editor.

  2. The conformations that matched the query can be indicated in the searched database, either by selecting the matching entries or by showing the matching entries and hiding all non-matches. This functionality is controlled by the Hit Entries option.

  3. A new, output database can be created that contains only the molecules that match the pharmacophore query. The output database is controlled by the Results option.

The output of the Pharmacophore Search window is controlled by the following options:

Hit Entries. Determines whether the search will highlight entries in the input database that match the query, either by selecting the matching entries or by showing the matching entries and hiding all non-matches. The number of "hits" (matches) and other statistics can be displayed as a textual report in an SVL Text Editor.


Results. Determines the contents of the output database:
  • No Output. No output database is generated. The number of hit molecules is still written to the CLI and displayed in the search status area of the window. The hit conformations can still be highlighted in the Database Viewer of the input database using the Hit Entries option. Choosing "No Output" desensitizes the rest of this section and generates no output database.

  • Conformations. Generates an output database that contains alignments of the input conformations that match the query. The Hits option further determines whether all or only some of the hit conformations are to be output. The fields to be generated are determined by the toggles in the Setup Page (press the Setup button) of this panel. The fields to be copied from the search database are also chosen from the Setup Page via the list of field names.

    If Use Absolute Positions is set, the conformations are not aligned to match the query. Note: multiple hits of one conformation can still be reported in the output, each corresponding to a different matching of query features to the ligand annotation points .

  • Molecules. Generates an output database that is a subset of the input database, which contains only hit molecules. If any conformation of a molecule matches the query, then all conformations of that molecule will be copied to the output, as-is. All other fields of the input database are also copied to the output database.

Output. Name of the output database to contain the matching conformations. The Mode determines whether the new result replaces any previous data stored in the output database or appends to the existing data. The three options are:

  • Create. Creates a new output database or rewrites an existing database. Before a non-empty output database is to be rewritten, the user is prompted for confirmation.

  • Append. Creates a new output database or appends to an existing database.

  • Rewrite. Similarly to Create, this case also creates a new output database or rewrites an existing database. In this case, however, there no warning given when a non-empty database is about to be rewritten.

Hits. Restrict the matches that are reported when Results is set to "Conformations".

  • All. Reports all matches found.

  • First Per Conformation. Reports only the first match found for any conformation.

  • First Per Molecule. Reports only the first match found for any molecule.

  • Best Per Conformation. Reports only the N lowest RMSD matches found for any conformation. The number of best conformations written to the output (N) can be specified using the N: field.

  • Best Per Molecule. Reports only the N lowest RMSD matches found for any Molecule. The number of best molecules written to the output (N) can be specified using the N: field.

Setup. The output database can be specified to contain a copy of any or all of the fields in the input database as well as the results of the search. The choice of fields to write to the output database is defined in the Setup Page of the Pharmacophore Search panel. To see this page press the Setup button. The following is displayed:

The Generated Fields toggles on the left side of the page are possible output fields as described in the methodology section Output Fields.

The list of Copied Fields shows the fields of the input database. Fields selected in this list will be copied from the input database to the output database. Fields with names identical to names of generated fields will not be copied.

Press Back to return to the page containing the search controls.

Executing the Search

Once the input database, the query, and the output settings have been specified, the search can be executed. Press the Search button at the bottom of the Pharmacophore Search panel. Progress of the search is shown in the status line at the bottom of the panel. When the search is complete, the word 'Finished' appears in the status line and the number of hits (matches) appears next to it.

To see the results of the search, the Open button can be used to open the output database. Each entry in the output database shows the conformation aligned with the query. Each alignment corresponds to one possible mapping of ligand annotation points and the query features.

A report can be produced that summarizes various statistics regarding the results of the last pharmacophore search of the query against the conformation database. To create the report, press the Report button at the bottom of the Pharmacophore Search panel. The report is displayed in an SVL Text Editor and is divided into sections.

The command section records the parameters used in the search. It does so in the form of an SVL command that can be executed to repeat the search. This code fragment can be copied and pasted directly into an SVL program or the CLI. Note, however, that if the query was taken from the Pharmacophore Query Editor instead of a file then the second parameter will say: [... Pharmacophore Query Editor Data ...] instead of a filename. The following is an example of the command section: 

    COMMAND:
    ph4_Search [
	'linker.mdb',
	'Query.ph4',
	[
	    abspos             : 0,
	    action             : 0,
	    esel               : 0,
	    maxconfhits        : 0,
	    maxmolhits         : 0,
	    molfield           : 'mol',
	    mseqfield          : 'mseq',
	    o_molfield         : 'mol',
	    o_mseqfield        : 'mseq',
	    o_rmsdfield        : 'rmsd',
	    out_append         : 0,
	    out_dbfile         : 'ph4out.mdb',
	    out_dbv            : 0,
	    out_type_molecules : 0,
	    sortby             : 'RMSD',
	    use_mseqfield      : 1,
	    use_o_hitmapfield  : 0,
	    use_o_molfield     : 1,
	    use_o_mseqfield    : 1,
	    use_o_rmsdfield    : 1,
	    use_o_rowfield     : 0,
	    use_out_dbfile     : 1
    ]];

The query section summarizes the query used in the search. It includes the title of the query, the number of features, group constraints, and volumes comprising it, and the minimum number of features required to be matched. The following is an example of the query section: 

    QUERY:
        Title: Test query
        Features: 4,  Constraints: 1,  Volumes: 1
        Minimum Match Size: 4

The timing section gives measurements of the speed of the search. The information shown consists of the duration of the entire search in seconds, and then data for individual parts of the search. For each item we give the total number of instances of that item that were found, the average time in seconds spent processing each such item, and the same speed expressed alternatively as the number of instances of that item processed per second.

The following information is given:

molecules. Molecules in the input database. Each molecule can be represented by several conformations.
conformations. Conformations in the input database.
hit molecules. Molecules for which at least one mapping has been found that satisfies the query.
hit conformations. Conformations for which at least one mapping has been found that satisfies the query.
all hits. Mappings of query features to ligand annotation points that satisfy the query. There may be multiple mappings for the same conformation.

The following is an example of a time section: 

TIME: 30.299 sec
  molecules: 21372              1.418 msec/mol        705.370 mol/sec       
  conformations: 43436          0.698 msec/conf       1433.579 conf/sec     
  hit molecules: 1199           25.270 msec/hit       39.572 hit/sec        
  hit conformations: 4228       7.166 msec/hit        139.543 hit/sec       
  all hits: 6288                4.819 msec/hit        207.532 hit/sec
