Introduction

Crystal cell parameters may be defined in MOE to specify the shape and dimensions of the crystallographic unit cell, and symmetry-generated siblings as implied by the space group.

The space group, dimensions and angles are specified in a single panel, which is illustrated in the above example. When the periodic system is enabled, the MOE Window displays the unit cell itself, as well as symmetry-generated cousins (whose positions are dynamically adjusted), and optionally adjacent unit cells to make up a portion of the lattice.

Methodology

Unit Cell

A crystal is an array of atoms packed together in a regular pattern. A unit cell is a piece of the pattern which, when repeated through space without rotation and without gaps or overlaps, reconstructs the pattern to infinity. To fill 3D space without holes, a unit cell must be a parallelepiped.

The symmetries of a pattern constrain the shape of a unit cell. There is an infinite number of possible unit cells for any pattern (e. g. a given unit cell can generate a family of unit cells by repeated doublings in size). By custom, the unit cell is chosen to be the smallest one that reveals the special geometry characteristic of the symmetry. Thus, although an oblique parallelepiped can be used for a pattern with 4-fold symmetry, a cube is preferred.

When a unit cell is repeatedly translated to fill all of 3D space, the vertices of all the unit cells in the filled space constitute a lattice. A lattice is an infinite array of regularly spaced points. All points in the lattice have identical "environments" - the view from every point in the lattice is identical to that from any other point in the lattice. The absolute positions of the points of a lattice, and hence the unit cell, are arbitrary with respect to a pattern.

Not all lattice points need coincide with unit cell vertices. Primitive unit cells use every lattice point as a unit cell vertex. Non-primitive unit cells, however, contain extra lattice points not at the corners.

A primitive unit cell contains exactly one lattice point. In a lattice of primitive cells, each lattice point is a vertex shared among eight cells. A vertex can thus be thought of as contributing 1/8 of a lattice point to each adjacent cell. Since each primitive unit cell touches eight vertices, each contains a total of exactly one lattice point.

A non-primitive unit cell has additional lattice points:

  • end-centered : an extra lattice point is centered in each of two opposing faces of the cell (with none in the two other pairs of opposing faces)
  • face-centered : an extra lattice point is centered in each face of the cell
  • body-centered : an extra lattice point is centered in the exact middle of the cell

Although primitive unit cells are simpler than non-primitive unit cells, the non-primitive unit cell is preferred when its geometry is simpler. For instance, a rectangular non-primitive cell would be chosen over a rhomboid primitive cell. In general, the unit cell used is the smallest one with the most regular geometry.

Lattice and Unit Cell Parameters

A lattice may be specified by three non-coplanar vectors in 3D. The vectors lie along the edges of the unit cell and are labeled a, b, and c. The magnitude of the vectors is given by the dimensions of the unit cell in the real crystal under study.

The faces of the unit cell are labeled as follows:

  • A : edges defined by lattice vectors b and c
  • B : edges defined by lattice vectors a and c
  • C : edges defined by lattice vectors a and b

Similarly, the inter-facial angles of the unit cell are defined to be:

  • alpha : angle between edges b and c
  • beta : angle between edges a and c
  • gamma : angle between edges a and b

Crystal Lattice Systems: the 14 Bravais Lattices

Lattices can be classified into "systems," each system being characterized by the shape of its associated unit cell. In three dimensions, the lattices are categorized into seven such systems. Within several of these, lattices supporting non-primitive unit cells can be defined. The classification scheme yields a total of 14 possible lattices (called Bravais lattices).

The lattice symbols used for classification are as follows:

  • P : primitive
  • B : end-centered on B-face (convention for Monoclinic systems)
  • C : end-centered on C-face (convention for Orthorhombic systems)
  • I : body-centered (from German innenzentriertes)
  • F : face-centered
  • R : rhombohedral primitive

These symbols also appear in the Hermann-Mauguin International space group symbols, where they have the same meanings. For example, group Fmmm is face-centered orthorhombic, and R3 is rhombohedral primitive.

Crystal Parameters

To begin working with crystal cells (sometimes referred to as periodic systems), choose MOE | Window | Crystal Parameters or use Footer | Space Group, which is a button labeled by the space group of the current system (default P1). The following panel appears:

Space Group All 230 of the 3D space groups are available, and may be selected from the option menu. The default is the simplest of all, the triclinic P1 group which has no symmetry elements and no cell shape constraints. More elaborate space groups tend to have unit cell angles which are fixed, and symmetry operations which generate multiple copies of the molecular species within the unit cell. The label description underneath the option menu describes the system in slightly more detail. The last part, which reads "1X" in the default case shown above, indicates how many symmetry-related species are in the unit cell. In this case it is just one; if the space group were P-1, the number would be 2 due to the center of symmetry.
Enable Periodic System By default this option is off, which means that the periodic box information entered is not presently displayed or enforced. To see the unit cell and the symmetry-generated images, enable this option (and press Apply). Notice that adding, deleting or changing atoms via the usual methods in MOE cause the unit cell to be updated appropriately.
Cell Size The dimensions of the unit cell (A, B, C) can be specified, and these correspond to the X, Y and Z axes in MOE. Only the extents of the axes may be specified, not the origins. For most space groups the dimensions may all be chosen independently; the obvious exceptions are the cubic groups.
Cell Shape The values of alpha, beta and gamma (angles between the axes) may be specified for certain space groups, though the degree flexibility varies. In triclinic systems all axes may be chosen arbitrarily; monoclinic systems allow just beta to be modified, whereas orthorhombic and cubic systems are defined to have all angles equal to 90°.
Coordinates There are several ways that coordinates can be mapped into the unit cell. The default (Normal) is that no mapping is performed; the original species is not necessarily within the unit cell at all, though its symmetry-generated images will still be generated. Selecting Packed ensures that all atoms are packed into the unit cell, and Molecule takes this one step further by locating the center of gravity within the unit cell and ensuring that all bond lengths are reasonable, which is usually the most aesthetically pleasing form.
Overlay Packed Coordinates Ensures that the unit cell will be populated with the appropriate number of symmetry images, even if the original molecule has coordinates outside of the unit cell.
Image Style The original molecular species is the main point of reference, but its symmetry-generated sibling images are equally valid representations of the same entity. By modifying the image style, it can be made clear which is the original. Style has options Same, Line and Hidden, which are fairly self-explanatory - they apply to the drawing mode of all copies of the molecule, except the original. Similarly the Label option allows labels of images to be forcibly hidden. Color defaults to the same scheme as the original, but can be set to a single color by selecting from an option menu.
Lattice Style Just as presentation can be overridden for images generated within the unit cell, the same can be done for duplicate lattice cells. The original lattice retains its appearance, but the duplicated neighbors can be made to take on particular Style/Label/Color representations.
Lattice By default only the unit cell is represented visually. The number of unit cells drawn in each of the three dimensions can be specified by changing the numbers in this section. Each number refers to how many unit cells are stacked along that dimension, hence just the unit cell itself is [1,1,1].
Inset By default when there is a lattice drawn composed from multiple unit cells, the original cell is considered to be on the far edge/vertex. Particularly when there are three or more cells drawn in one of the dimensions it may be desirable to draw the unit cell further inset. Pressing the 1 or 2 options moves the original cell closer to the other end of the lattice (in all of the dimensions).
OK, Apply The parameters entered within this panel are not updated dynamically as they are modified; use the Apply button to update the MOE window, or OK to update and close.
Build As unit cell information is entered, the symmetry-generated species and/or the lattice are actually "ghosts", though they can be visualized as if real. They will disappear if the Enable Periodic System option is deactivated. The Build feature finalizes what is currently seen, and makes each symmetry image into a new chain. Note! It is recommended that Build be used when the Coordinates mode is Molecule; in this way, individual molecules will have their atoms close together and not be split across cell boundaries.

Crystal Parameters are stored along with other molecular information in MOE, so if the panel is closed and reopened, the parameters may be modified.
