MOE 2010.10

MOE 2010.10 Release Notes

The 2010.10 release of Chemical Computing Group's Molecular Operating Environment (MOE) software includes a variety of new features, enhancements and changes that are summarized here.

Core System

NEW! MOE System Popup. The System button on the MOE Button Bar now raises a pop-up panel that provides streamlined control over the objects in MOE. All chains are grouped by chain tag into a collapsible listing. Within each tag group, the atoms are partitioned into Receptor, Solvent and Ligand classes and a control line presented. Each control line can affect the display of the atoms it represents:

  1. Select/deselect.
  2. Active/Inert without affecting visibility
  3. Hide/Show and simultaneously activate/deactivate
  4. Color Atoms (carbon atoms only or all atoms)
  5. Render Mode (point, line, stick, ball and stick, etc.)
  6. Hide/Show Surfaces associated with the atoms
  7. Delete atoms or surfaces or both

The Ctrl and Shift modifiers affect the behavior; for example, Ctrl+Click will on the Hide/Show will show the clicked line and hide all other lines within the same tag/receptor/solvent/ligand class; i.e., "isolate this line". Consult the bubble help in the popup for more information about the keyboard modifiers. In previous versions, the System button raised the Atom Manager window. This window must now be raised with MOE | Window | Atom Manager.

NEW! MOE Footer. The MOE Footer now adapts to the current selection and provides information and controls for editing objects. Atom names, bond lengths, bond angles and dihedral angles can modified directly from the footer. In the case of dihedral angles, a wheel can be used to modify a torsion, and the Profile button displays a graph of energy vs. dihedral angle. Clicking in the plot will modify the structure. These footer controls supersede the Builder controls which have been eliminated. The footer bubble help contains additional information about the selected atoms.

NEW! Atom Selection Masking. The R, S, and L buttons on the MOE Button Bar (above the trackball) facilitate atom selection by mouse in the MOE Window. Mouse selection of receptor, solvent and ligand atoms can be individually enabled or disabled. Double-click to enable selection of one class of atoms and simultaneously disable selection of other classes.

NEW! Nonbonded Contacts Visualization. Hydrogen bonds and van der Waals contact control has been enhanced and are now controlled in the MOE | Footer | Contacts popup.

  1. Rendering. Contacts can be displayed in a number of styles: dashed line with or without an indicator of contact energy (in kcal/mol) - either a strength bar or a text label. Contacts can be enabled, disabled or drawn faintly by interaction class combinations (Receptor, Ligand and Solvent). Backbone-backbone interactions can be filtered and energy thresholds are adjustable. The settings can saved by right-clicking in the Contacts panel.

  2. Definition. Hydrogen bond donor and acceptor definitions are controlled by the $MOE/lib/hbond.svl configuration file which contains SMARTS pattern tables that determine candidates for hydrogen bonds. Non-standard CH..X hydrogen bonds are permitted.

  3. Proton-π Interactions. A geometric model for proton-π interactions has been added. Interactions between planar rings and protons are calculated and drawn in a similar fashion to hydrogen bonds. These interactions are intended to represent non-standard hydrophobic interactions.

NEW! Integration of LigX and the Right Button Bar. The MOE Right Button Bar and the specialized LigX Button Bar have been integrated into the new default button bar. Common functions such as Open, Close, Delete, Select, Minimize, etc. have been retained and other functions have been streamlined and/or added. The View button has been renamed to the Center button.

  1. SiteView. The SiteView button renders all objects in the system in a manner similar to the Isolate function of several applications. Pocket atoms are rendered in line mode and ligands (or the selected atoms) in stick. Atoms far from ligands are hidden.

  2. Hydrogens. The Hydrogens button cycles the display of hydrogens between three states: no hydrogens, polar hydrogens only and all hydrogens. The display is restricted to the atom selection (all atoms if nothing is selected).

  3. Surface. The Surface menu provides streamlined access to Surfaces and Maps. Molecular surfaces colored by the new Atom Coloring can be created around active receptors and ligands. Van der Waals surfaces (as in LigX) can also be created. In addition, the Surfaces and Maps application can be raised (to edit the created surfaces).

  4. Constrain. The Constrain menu allows for Fixing, Unfixing and Tethering atoms as in LigX. LigX rendering has been replaced by an on-demand menu item in the Constrain menu.

  5. LigX. The LigX button raises an application for preparing systems for LigX. The Protonate3D parameters and tether settings are adjustable in the LigX prepare application. Multiple systems can be prepared simultaneously.

NEW! Ligand Menu on the Button Bar. The Ligand menu on the MOE Button Bar contains functions related to structure-based design focusing on the ligand. Ligand Interactions and the 2D Molecules applications can be raised. In addition:

  1. Choose Ligand. The Choose Ligand function provides a means to focus on a particular ligand (and classify others as Receptor) similar to the LigX application.

  2. Ligand Properties displays the LigX ligand properties: logP, logS, toxicity, rsynth, donor and acceptor counts. If a ligand is minimized in a pocket with the Minimize button then solvation energy of complexation and the docking score are displayed. Also, the local ligand strain is calculated (by minimizing the ligand without the receptor) and the uncomplexed RMSD is displayed. Energy values are also displayed in "efficiency" form (energy divided by heavy atom count).

  3. Ligand R-Vectors enables the display of "exit vectors" from visible hydrogens on ligand atoms. Each exit vector represents the potential for a methyl substitution without steric clash (either to the receptor or the ligand). These vectors contain much of the information of the LigX Interaction (VDW) surface without the need to display the surface.

  4. Duplicate. The Duplicate function duplicates a single selected ligand and hides the original. Fixed status, tethers and set membership are replicated.

  5. Minimize (Vac). The Minimize (Vac) function duplicates a single selected ligand and performs an isolated energy minimize (in the background) for more detailed investigation of local strain. The results are superposed onto the original and the newly created ligand is set to visible but inert.

Enhanced Graphics. The GPU-accelerated graphics in MOE have been enhanced for further speed, especially when displaying backbone ribbons and non-bonded contacts. In addition

  1. NEW! Surface Z-Clipping. An additional pair of clipping planes applying exclusively to surfaces is now available in MOE and can be accessed through the new SF and SW sliders under ZClip in the MOE Footer.

  2. NEW! Anaglyph Stereo. MOE now provides Green/Magenta and Red/Cyan anaglyph stereo graphics that can be perceived using the stereo glasses that have provided as part of the MOE distribution. Anaglyph stereo rendering is accessed from the MOE | Render | Stereo menu.

  3. NEW! Anaglyph Stereo Theme. To enhance color perception when in anaglyph stereo mode, the Anaglyph Stereo visualization theme can be used. This can be selected from the Visualization Setup panel (MOE | Render | Setup).

Bond Selection. Bonds are now selectable in the MOE Window. Clicking towards the middle of a bond will select the atoms on either end of the bond.

NEW! Mouse Reference Panel. A panel describing mouse commands in the MOE Window, Sequence Editor, and Database Viewer can now be opened from the Help menu in each of these windows.

SKC Sketch Files. 2D and 3D queries can now be pasted into MOE. The new Sketch View panel offers a 2D diagram view of the molecular structure of a query, and provides export and print options. The fully annotated query can be visualized in the MOE Window.

Sequence Editor.

  1. Chain Tag Display. Chain tags can now be displayed in the Sequence Editor residue area using the SE | Display | Chain Tag command.

  2. Menus. New menu commands for displaying chain tags, for manipulating chains by tag, and for chain coloring have been added to the Sequence Editor menus. The Edit and Display menus and the Chain and Selected Chain popups are affected. The new commands allow for setting chain tags, selecting chains with the same tag, sorting chains by name or tag, and coloring chains by number or by tag.

Applications

Fragment-Based Drug Design. Scaffold Replacement, Fragment Linking and Growing, BREED and the new MedChem Transformations applications have been consolidated into a single panel opened with from the MOE | Compute | Fragments menu.

  • NEW! MedChem Transformations. A new application to perform common medicinal transformations in 3D has been added. A database transformations is systematically applied to a given molecule in an active site. The resulting molecules are refined and scored. Transformation rules are specified using MDL format RXN (reaction) files and stored in a database ($MOE/lib/medchem_rxn.mdb by default). ChemDraw or SymyxDraw can be used to create or edit the rules.

  • Scaffold Replacement. A new Settings panel provides access to some specialized parameters for Scaffold Replacement, such as Link Feature Radius or Intra-Ligand and Protein-Ligand clash energy thresholds.

Molecular Dynamics. Molecular dynamics simulations in MOE have been enhanced in the following ways:

  1. NEW! MOE now provides an interface to the NAMD program through the Dynamics panel. The molecular system is prepared in the same way as for MOE dynamics. Once the simulation is done, the resulting trajectory file(s) are imported into MOE using the NAMD_Import SVL function.

  2. The stages of the dynamics simulation can now be specified using a new protocol language. The protocol language describes a sequence of stages, each with a name (e.g. heat or equil) and a set of parameters enclosed in braces (e.g. { ps=100 T=(0,300) }). The sampling intervals and splitting of the output trajectory files can also now be specified.

  3. NEW! DCD Trajectory Files. MOE can now read dynamics coordinate data (DCD) trajectory files produced by molecular dynamics and crystallography programs such as NAMD, CHARMM, and XPLOR.

NEW! Solvate. A new application for solvation replaces the previous Water Soak application. MOE | Edit | Build | Solvate adds solvent molecules to a molecular system, typically prior to a molecular dynamics simulation, and offers the following features:

  1. Selection of solvent: water, DMSO, methanol, chloroform, carbon tetrachloride.
  2. Support for periodic (cell) or aperiodic (droplet) modes.
  3. Configuration of Crystal Parameters or Wall Restraint.
  4. Centering and axis aligning the solute.
  5. Optional addition of salt counterions to neutralize the system.

The solvent is read from precalculated periodic solvent files, stored in SD file format, in the $MOE/lib/solvent directory.

NEW! Kinase Database. New applications have been added for managing and exploring kinases. An application is now available for creating an aligned kinase database and a specialized search application can be used to locate and browse entries.

  1. Kinase Identification. Special SVL functions have been created to identify kinase domains and important structural regions. These regions are used to align the domains into an absolute frame of reference.

  2. Kinase Database Update. Kinase Database Update is a new application for preparing and adding structures to kinase structure databases. This application was run on the PDB to produce $MOE/lib/kinome.mdb.

  3. Kinase Search. The Kinase Search application searches the kinase database based on specific criteria:

    1. Text. Partial, exact, and phrase searches, with include all, include any, and include none conditions.
    2. Date. Search for records after a given date; useful for updates.
    3. Species, Family. Restrict search to specific species and families.
    4. DFG, alphaC, HRD. Search for specific activation loop conformations or arginine within the HRD motif.
    5. Sequence/Structure. Sequence only or sequence and structure search, subject to identity and RMSD cutoffs.
    6. Ligand Descriptor Filter. Filter for molecular descriptor criteria, expressed as thresholds or binary conditions.
    7. Ligand Fingerprint Filter. Filter for ligands that are similar to a specified molecule in MOE.
    8. Ligand Class, Binding. Restrict ligands to specified classes and binding modes.

    Results from successive searches can be combined using logical operators. The panel is opened with MOE | Compute | Biopolymer | Kinase Search.

  4. Kinase Statistics. Statistics on the kinome database or on just the search results can be viewed as a series of pie charts. The charts display such information as the distribution of groups, families, species, DFG and alphaC loop activation conformations, and date, among others. The images can be exported for inclusion in presentations.

  5. Kinase 3D Browser. The search results (or the entire database) can be browsed using the Kinome Database Browser, which can be started from either the Kinase Search application or a Database Viewer. The Browser can be used to view different fields of the kinome database. When the kinome complex field is chosen for display, the complex is shown in 3D in the MOE Window, with the pocket region isolated and important regions color coded.

The Kinase Database and related applications are intended for use in homology modeling and in structure-based design of kinase inhibitors.

LowModeMD. The Conformational Search application has been enhanced to provide for explicit water conformational search using LowModeMD. Two new options are available for excluding water molecules and counter ions from the RMSD calculation used for duplicate testing. In this way, the conformations of a solvated peptide, for example, can be generated with duplicate testing applying only to the peptide.

Surfaces and Maps. The Surfaces and Maps application has been rewritten to include integrated Graphics Object Manager capabilities. The panel now features a list of the surfaces in the system, providing controls for showing, hiding, deleting, and renaming them. An existing surface can be edited by selecting it in the list and then changing the surface parameters in the panel. The new layout is intended to improve ease of use and streamline workflow. In addition, there are new options for Atom Coloring: an option to force coloring by element and an option to add hydrogen bond geometry information onto the surface. This coloring option is the new default.

PSILO/RCSB Browse. The RCSB Browse panel has been enhanced and is now renamed to PSILO/RCSB Browse. The panel now functions as a direct link between MOE and a PSILO server in addition to the RCSB Protein Data Bank internet site as before. Access to PSILO requires PSILO connectivity and a PSILO license. The panel is opened using MOE | File | PSILO/RCSB. The

NEW! Molecular Descriptors. There is a new 'mutagenic' molecular descriptor that identifies potential toxic substructures in ligands. The mutagenic descriptor is a rule-based method [Kazius et al; J. Med. Chem., 48 (2005) 312–320] which identifies (bio)reactive groups including polycyclic aromatic motifs.

Builder. The small molecule Builder has been updated in several respects. A new narrow layout reduces screen space while still providing most of the functionality of the old builder.

  1. The behavior of many of the editing operations (including Undo) has been made more consistent and more clearly defined.
  2. Addition of functional groups and rings includes a 3D torsion sampler to generate better initial placements.
  3. A Minimize button (integrated with Ligand Properties) has been added to streamline 3D building in the style of LigX.
  4. The bond length, angle and dihedral angle controls have been moved to the MOE Window footer.

In addition, the MOE | Window | Options | Builder contains configuration settings for the Builder. The SMILES and Atom Name and Hybridization (HintLP functionality of the "old" builder) controls can be enabled or disabled.

NEW! Tutorials. The following new tutorials have been added to the manual. These are available from MOE | Help.

  • Visualization and Making Pictures
  • LowModeMD
  • MedChem Transformations In-Cleft
  • Protein Alignment
  • SD Pipelining Tools: How to Create a Fragment Database
  • Prepare a Small Molecule
  • Prepare a Protein or a Protein-Ligand Complex
  • Prepare a Small Molecule Dataset
  • Prepare a Conformation Database

Residue Selector. The residue selector now has an option to extend the current residue selection to neighboring residues, with or without taking into account gaps.

SAReport. A new Grow Scaffolds option provides control over whether to grow scaffolds whose implied R-group substituents are completely uniform; such scaffolds can be grown by absorbing the implied substituents or used as is.

Docking. Receptor sidechains can now be tethered using tether weights derived from atom temperature (B) factors. This option can be accessed through the Configure Forcefield Refinement panel. In addition, the GOLD interface has been updated to provide for more control over Genetic Algorithm options.

Pre-defined Material Styles for POV-Ray Images. New pre-defined material styles for rendering atoms, bonds, and ribbons are now available when generating ray-traced images of the MOE Window. The themes are available in the Ray Tracing Settings panel, which is opened from the Save panel.

Protein Structure and Family Databases. The PDB databases included with MOE have been updated. They are derived from all entries in the PDB up to September 21, 2010. The full database pdb_2010_09_21.mdb is located on the DVD. The non-redundant set is located in $MOE/lib. The structural family database is located in $MOE/lib/pdb.fam. The Antibody Structure database is located in $MOE/lib/fab*.mdb. The Kinase Database is in $MOE/lib/kinome.mdb.

SVL Programming

NEW! Material Widget. The SVL widget toolkit now includes a pull-down material chooser widget for selecting material properties (shading, gloss, brightness and transparency). A preview area shows the current settings and preset values are easily accessed.

NEW! Special Shell Widgets. The SVL widget toolkit now offers two new specialized shells:

  • Popup Shell. The shell manager widget now supports a popup mode. In this mode, a panel is created as a popup shell that will open at the location specified by the location attribute. The location can be with respect to an SVL panel, or to a particular widget in an SVL panel. Clicking away from the Popup or pressing Esc dismisses the panel.

  • Sidepanel Widget. Left or right sidepanels can be attached to the MOE Window by specifying a shell widget with the location attribute set to MOE_Left or MOE_Right, respectively.

Text Widget. The text widget now has the ability to display text when the field is empty. This text disappears when characters are entered in the text field, and reappears when the field is cleared. The text to be displayed is assigned to the emptyText attribute, with the display color controlled using the emptyForeground attribute.
