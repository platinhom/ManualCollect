Summary

A run in Prime is an execution of the Prime Structure Prediction workflow. A Maestro project can contain more than one Prime run. This is useful for comparing the results of runs made with different selections and options chosen. When you find that you would like to explore what would happen if you chose a different template in Find Homologs, or produced a different alignment in Edit Alignment, but don't want to overwrite your existing results, you can start a new run. The commands for saving the old run, starting a new one, and switching between Prime runs are found under the Prime File menu.

Run names must be unique.

Run Commands

The commands for using runs under the Prime Structure Prediction File menu are:

New

Asks for a new run name (all run names must be unique), and starts a new run at Input Sequence.

Save As

Asks for a new run name, then copies the current workflow to the new run and switches to the new run. The original workflow is saved under the original run name. If you opened a new project and began a workflow without explicitly giving the run a name, it is given the default name of run1.

Rename

Asks for a new name for the current run. All run names must be unique.

Delete

Deletes the current run (after asking for confirmation).

Open

Displays a list of options for switching to another run. The list includes the most recently used runs, up to four of them. If there are more than four runs in the current project, click the More option to select a run by name.

Copying a Workflow to a New Run

To backtrack and try a different set of choices without overwriting your current workflow:

  1. Select Save As from the Prime File menu. This will save the current run under the old name (run1 is the default for the first run in a new project) and create a copy under the new name.

  2. You are now in a new workflow identical to the first except for name. Navigate back to the step where you would like to make a different choice. If you are going back more than one step, it is convenient to click the step name in the Guide diagram.

Two Maestro projects may be merged into one (see under Related Topics.) If you merge two Maestro projects containing Prime runs, all the runs are included in the merged Project. If two runs have the same name, Maestro automatically makes the names unique by appending -mrg1 to one of them.
