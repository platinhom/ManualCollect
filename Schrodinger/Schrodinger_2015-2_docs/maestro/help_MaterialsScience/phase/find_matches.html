Summary

In this panel you can search a prepared database for matches to a selected hypothesis. The panel provides options for matching and for processing the hits.

Opening the Advanced Pharmacophore Screening Panel

To open the Advanced Pharmacophore Screening panel, you can:

  • Choose Applications → More Applications → Phase → Advanced Pharmacophore Screening in the main window.

  • Click Search for Matches in the Score Hypotheses step of the Build Common Pharmacophore Hypotheses panel.

  • Click Search for Matches in the Build QSAR Model step of the Build Common Pharmacophore Hypotheses panel.

Using the Advanced Pharmacophore Screening Panel

To perform a search, you must select a source of structures and a hypothesis, set any options for conformer generation matching and for filtering and treatment of the hits, then click Run to start the job. You might want to make job settings before starting the job, to choose a host for the job. See the Job toolbar section for more information.

The process of finding matches involves searching the structure source for matches to the hypothesis, which generates a list of matches; then applying any filtering to the matches, such as limits on the number of hits and excluded volumes; then computing a score for the hits and a predicted activity, if the hypothesis has a QSAR model.

Selecting a Structure Source

The structures that are searched can come from a 3D database that is prepared beforehand, from a file, or from the selected entries in the Project Table. You can choose the structure source from the Search in option menu.

To specify which 3D database or file to use, you can either enter a file name in the File name text box, or click Browse and navigate to the desired database or file. If the database is not available locally, you will have to enter the name, and you should select Remote database.

If you selected a 3D database, you can use the full set of structures or a subset (if any are defined). To specify the subset, enter the name in the Subset text box, or click Browse and select the subset in the dialog box that opens. You can enter the file name for a subset that does not exist in the database, by specifying the path to the file, which must end in _phase.inp, and can be either a local file or a remote file.

The structure source must consist of reasonable, all-atom 3D structures. The search for matches also requires a set of conformers for each molecule, and site points for each conformer. If the structure source does not have conformer sets, or if you wish to regenerate them with a different conformational search, you can generate them during the search by selecting the appropriate option in the Searching tab.

Selecting a Hypothesis

The hypothesis can be chosen from any of the hypotheses in the Project Table. If you open this panel from the Score Hypotheses step or the Build QSAR Model step of the Build Pharmacophore Model panel, the selected hypotheses are added to the Project Table, and the first added hypotheses is chosen as the default hypothesis. If you want to choose a different hypothesis from the Project Table, click Choose and choose the hypothesis in the dialog box that is displayed.

If the hypothesis that you want to use is not part of the current project, you can import it, provided that it was exported from the project in which it was generated. To do so, click Import to navigate to the hypothesis and import it.

Setting Matching Criteria

When you search for matches to the hypothesis, you can set the tolerance for matching intersite distances, apply tolerances for matching individual features, set the number of site points that must match, select individual features that must match, permit or prohibit matches to other features, geometrically constrain matches, and apply excluded volumes. These options are available in or from the Matching tab.

Filtering the Matches

The matches are filtered to generate a reduced list of hits, and properties are calculated for these hits. There are several ways of filtering the matches to obtain a list of hits:

  • Limiting the number of hits—You can limit the number of hits to a given maximum, or limit them to a maximum number of hits per molecule in the database. The matches that have the highest fitness score are returned as hits.

  • Applying scoring cutoffs—You can exclude matches that do not meet cutoffs for the alignment score, the vector score, or the volume scores. This feature can be used to ensure a minimum level of overlap and alignment with the reference ligand.

If the hypothesis has a QSAR model, you can calculate the activity from this model by selecting Apply QSAR model. You can also customize the fitness score by adjusting the weights of the alignment, vector, and volume scores. These options are available in the Hit Treatment tab.

Starting the Search

To submit the search job, click Run. If you want to make host settings first, choose Job Settings from the Settings button on the Job toolbar. You can select the host and the number of CPUs to use in the dialog box that opens, and then either just save the settings or save the settings and start the job.

If you are searching a database and the feature definitions of the hypothesis do not match those of the database, a dialog box is displayed to allow you to resolve the discrepancy. You can choose to rebuild the database with the new feature definitions, which changes the database permanently, or you can recompute the sites for the ligands in the database "as needed", which creates new site definitions for the duration of the job. If the database is not accessible to Maestro, however, the feature definitions cannot be checked, and the job fails if the feature definitions differ.

Each time you search the database, the hits are added to the Project Table as an entry group, where you can apply the full range of applications and facilities available from Maestro to the hits. The fitness score and the activity predicted by the QSAR model (if any) are added as properties, along with a property that indicates which hypothesis was matched.

If you want to run the search from the command line, using the options you set up in the panel, choose Write from the Settings button on the Job toolbar. The input files and a shell script to run the job are written to the current directory. You can then use the files that are written out with phase_find_matches. The shell script contains the command to be used to run the job, but the command does not have Job Control options.

Advanced Pharmacophore Screening Panel Features

The Advanced Pharmacophore Screening panel consists of a toolbar, a section to specify the structure source and the hypothesis, and a set of three tabs for setting options for the search and the output.

  • Toolbar
  • Structure and hypothesis selection section
  • Searching tab
  • Matching tab
  • Hit treatment tab
  • Job toolbar
  • Status bar

Toolbar

The toolbar contains four buttons, three of which are the same as in the Develop Common Pharmacophore Hypotheses panel, and allow you to view the selected hypothesis and its excluded volumes, intersite distances, and angles. The fourth allows you to view site-matching tolerances.

Structure and hypothesis selection section

At the top of the panel are controls for selecting the source of structures to search and the hypothesis to match.

  • Search in option menu
  • Remote database option
  • File name text box and Browse button
  • Subset text box and Browse button
  • Hypothesis text box and buttons
Search in option menu

Choose the source of structures from this menu, from 3D database, File, or Project Table (selected entries).

Remote database option

Select this option to use a database on a file system that is not accessible to the local host. The full path to the database on the remote file system must be entered into the File name text box. Checking of the database is not done until the job starts. You can specify a subset with this option enabled, and the subset can be either local or remote.

File name text box and Browse button

Enter the name of the database or the file name in the text box, or click Browse and navigate to the file or database. These controls are not available if Project Table (selected entries) was chosen from the Search in option menu.

When searching in a file, the file can be in Maestro or SD format, and can be compressed with gzip (.maegz, .mae.gz, or .sdf.gz) or uncompressed (.mae, .sdf)

Subset text box and Browse button

Enter the name of the subset of the 3D database to use, or click Browse and select one of the available subsets in the dialog box that opens. These controls are only available if 3D Database was chosen from the Search in option menu. If the Subset text box is blank, the entire database is searched.

You can enter the name of a subset file that is not included in the database. The name must end in _phase.inp. The file need not exist on the local file system: it can be on a remote host, such as the host where the database is stored.

Hypothesis text box and buttons

The hypothesis text box displays the entry ID and the hypothesis ID of the hypotheses to which matches are sought.

Choose button—Choose a hypothesis from the Project Table. Opens a dialog box that displays a list of hypotheses.

Import button— Import a hypothesis from a file. The feature definitions for the hypothesis should match those in the database, if you selected a database for searching. If they do not match, you can apply the new feature definitions to the database when you run the search, either temporarily or permanently.

Searching Tab

In this tab, you set options that affect the way the search is performed. You can specify whether to use existing conformations or to generate them during the search, and set options for each choice.

  • Use existing conformers option
  • Score in place option
  • Refine matches option
  • Generate sites during search option
  • Perform pre-screen using database keys option
  • Generate conformers during search option
  • Conformer generation and refinement section
  • Existing conformers options
Use existing conformers option

With this option, the search is performed with existing conformers. When searching a file or Project Table entries, consecutive structures with identical titles and connectivities are treated as conformers of a single molecule.

Score in place option

Score structures without doing any alignment to the hypothesis.

Refine matches option

Generate extra conformers for the top-ranked match from each molecule matched. These conformers are subjected to the hit filters and are returned in the hit file. This procedure improves the fitness score, and can return matches when excluded volumes eliminate every match for some molecules. When you select this option, the Conformer generation and refinement section is activated, so you can set parameters for the extra conformers.

Generate sites during search option

Generate the sites during the search, rather than use preexisting sites. You should select this option if the feature definitions for the hypothesis differ from those in the database. This option is only available if you are searching a database, where the sites are already present. For other input sources, sites are generated automatically. These sites are not saved, but only used during the search.

Perform pre-screen using database keys option

Perform a pre-screening of the database using database keys, to speed up the search. Use of these 3D keys rapidly filters out the majority of molecules that cannot possibly match the hypothesis. This option is recommended except when searching a small subset of the database or when the search is split across a large number of CPUs.

Generate conformers during search option

With this option, the conformers are generated as each structure is read, and site points are generated automatically along with the conformers. When you select this option, the Conformer generation and refinement section is activated, so you can set parameters for the conformational search.

Conformer generation and refinement section

In this section you can specify options for the conformational search when generating conformers during the search or when refining matches by generating extra conformers.

Existing conformers options

These options allow you to discard the current conformer set (Discard) or keep it (Keep, when generating new conformers. If you keep the existing set, the new conformers are appended to the set. The set might therefore contain redundant conformers, but it is also guaranteed to contain the original structure.

Number of conformers per rotatable bond text box

This text box specifies the maximum number of conformers to generate in the conformational search for each rotatable bond in the structure. The number given in the text box is multiplied by the number of rotatable bonds in the molecule to arrive at the maximum number of conformers. The default for this value is 10.

Maximum number of conformers per structure text box

This value limits the number of conformers returned from the generation process. If the number of conformers generated is higher than this value, a sample of all the conformers generated is returned. The default is 100.

Sampling options

The extent of the search depends on the sampling method as follows:

  • Rapid— All the core conformations are generated, and the conformations of the peripheral (rotamer) groups are sampled one by one.

  • Thorough— A complete set of conformations is generated for both the core and the peripheral groups.

Amide bonds option menu

There are three options for treatment of amide bonds in the search:

  • Vary conformation—This option allows the amide dihedral angle to be varied. It can take any value, not just cis or trans.
  • Retain original conformation—Retain the conformation that was in the input structure.
  • Set to trans conformation—Set the conformation to trans for all amide bonds.
Relative energy window text box

In this text box you can set the an energy cutoff for eliminating conformers that are too high in energy. Any conformer whose energy relative to the lowest conformer exceeds this value is discarded.

Skip conformer generation for structures with > N rotatable bonds text box

If a structure has more than the specified number of rotatable bonds, do not generate conformers for this structure. Conformer generation time increases rapidly with the number of rotatable bonds, particularly for thorough sampling.

Matching Tab

In this tab you can set options and parameters for matching the ligands to the hypothesis.

  • Limit CPU time for matching to N seconds/molecule option and text box
  • Intersite distance matching tolerance option and text box
  • Apply matching conditions section
  • Must match on at least N site points option
  • Prefer partial matches involving more sites option
Limit CPU time for matching to N seconds/molecule option and text box

Select this option and enter a time in the text box to restrict the amount of CPU time used per molecule in the search. Limiting the CPU time is useful when doing partial matching for hypotheses with many sites, as the time spent finding matches to a given molecule may become very large due to the number of possible combinations.

Intersite distance matching tolerance option and text box

If this option is selected, apply the tolerance for matching intersite distances specified in the text box. If the difference in distance is less than this value, the distance is considered to match.

Apply matching conditions section

In this section you can select from a variety of conditions for matching the hypothesis. These conditions are set up in the Matching Conditions dialog box, which you can open by clicking Edit. A fuller description of each type of condition is given in the help topic for this dialog box.

  • Tolerances—Apply site-specific tolerances ("positional tolerances") when matching sites.

  • Required matches—Require matches to specific sites. By default, all sites must match. The value in the Must match on at least N site points text box is set to the number of required matches.

  • Permitted / Prohibited matches—Allow or prohibit matches of sites of a particular type to sites of another type, e.g. aromatic rings and hydrophobes.

  • Constraints—Apply distance, angle, and dihedral constraints on selected combinations of site points.

  • Excluded volumes—Exclude matches that have atoms inside any of the defined excluded volumes.

Must match on at least N site points text box

Number of site points required for a match. The number of site points required to match is affected by settings in the Matching Conditions dialog box, in which you can require specific sites to match. The number actually required is the maximum of the number of specific sites required and the number set in this text box.

You can request matches on only 2 site points, but in this case, a full alignment cannot be done, and the alignment score is set to zero. In addition, you cannot use excluded volumes with only 2 site points unless you have Score in place selected in the Searching tab.

Prefer partial matches involving more sites option

If this option is selected, matches involving fewer than n sites will not be sought if there are any matches with n sites. This option is only available if you are matching less than the maximum number of sites.

Hit Treatment Tab

This tab provides options for filtering the matches to define a set of hits, and calculating properties for the hits.

  • Sort hits by decreasing fitness option
  • Return at most N hits total option and text box
  • Return at most N hits per molecule option and text box
  • Use QSAR model option
  • Consider atom types when computing volume scores option
  • Compute included volume scores option
  • Fitness text boxes
  • Reject hits with options and text boxes
Sort hits by decreasing fitness option

Sort the hits by decreasing values of the fitness score. If this option is not selected, hits are written out in the order that they are encountered in the input structure source.

Return at most N hits total option and text box

When sorting the hits, keep only the N top hits. This option is not available of you do not sort hits by decreasing fitness.

Return at most N hits per molecule option and text box

Limit the number of hits returned per molecule the the specified value. Some molecules can return more than one hit because different alignments or different conformers might match the hypothesis.

Use QSAR model option

Select this option to calculate the activities for the hits using the QSAR model for the hypothesis. This option is selected by default if the hypothesis has a QSAR model, and is unselected by default if the hypothesis does not have a QSAR model.

Consider atom types when computing volume scores option

Option for computing volume scores using overlap only between atoms of the same MacroModel atom type. This option favors alignments that superimpose chemically similar atoms.

Compute included volume scores option

Compute the score for the overlap of the structure with the included volumes specified in the hypoID.ivol file. The score is the ratio of the volume overlap to the total included volume.

Fitness text boxes

In the text boxes you can specify the weights of the alignment score, the vector score and the volume scores, and set the threshold for distance matching. See the topic Phase Score Actives Dialog Box for more information on the scoring function.

Reject hits with options and text boxes

These four options allow you to reject hits that do not meet the specified cutoff for the given scores.

Output option

Below the tabs is an option for control of the amount of output. Selecting the Verbose output option provides details of each match in the output file. By default, only one line is written per match. You should only select this option if you are interested in the details of the matching, as it could generate a large amount of output if there are many matches.

Job toolbar

The Job toolbar is used to make settings for a job and to start it.

Job name text box

Enter a name for the job in this text box.

Settings button menu

This button opens the Job Settings dialog box. The arrow to the right opens a menu, from which you can make settings or perform actions that are related to the job.

The menu items are described below.

  • Job Settings—Make and save settings for the incorporation of the job into the Maestro project, the job name, the host, the distribution of subjobs, and any other application-specific job settings. Opens the Job Settings dialog box.
  • Preferences—Set general preferences for running jobs. Opens the Preferences panel at the Jobs - Starting section.
  • Write—Write the input files for the job but do not start it. The file names are based on the current job name, and the files are written to a subdirectory of the current working directory that is named with the job name.
Job status button

This icon indicates when there are jobs running for the application that belong to the current project. It starts spinning when the first job starts, and stops when there are no more jobs running. If a job fails to start, it changes to an exclamation point.

Clicking the button shows a small job status window that lists the job name and status for all active jobs submitted from the current panel (for Jaguar and MacroModel this means all jobs from any of the application panels). You can double-click on a row to open the Monitor panel and monitor the job, or click the Monitor button to open the Monitor panel and close the job status window. The job status is updated while the window is open. If a job finishes while the window is open, the job remains displayed but with the new status. The rows are colored according to the status: yellow for submitted, green for launched, running, or finished, red for incorporated, died, or killed. Click anywhere outside the window to close it.

Run button

Run the job with the current job settings.

Status bar

The status bar displays information about the current job settings for the panel. This includes the job name, task name and task settings (if any), number of subjobs (if any) and the host name and job incorporation setting. It also displays a message for a short time when you start a new job.

The status bar also contains the Help button, which opens the help topic for the panel.
