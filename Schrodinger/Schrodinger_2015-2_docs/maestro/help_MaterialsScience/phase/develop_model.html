Summary

This panel provides the tools for developing a pharmacophore model from a set of ligands. There are five steps in the process used to develop a pharmacophore model: preparing the ligands, identifying features and creating pharmacophore sites from them, finding common pharmacophores, scoring the hypotheses, and building a QSAR model.

Opening the Develop Common Pharmacophore Hypotheses Panel

To open the Develop Common Pharmacophore Hypotheses panel, you can:

  • Choose Applications → More Applications → Phase → Develop Common Pharmacophore Hypotheses in the main window.

Using the Develop Common Pharmacophore Hypotheses Panel

The Develop Common Pharmacophore Hypotheses panel is designed like a wizard, with five steps. To develop a model from scratch, you proceed through each step in turn. However, the design allows you to pick up the process at any step for which you have the required data. So if you have a set of ligands and you want to change the features to develop a different model, you do not need to start at the first step, which is preparing the ligands. Instead, you can go straight to the second step, which is defining features and creating sites. The design also allows you to exit at an intermediate stage and pick up the process again later.

To navigate the steps, you can click the Back and Next buttons, click the desired step in the Guide, or choose the desired step from the Step menu.

The process of developing a pharmacophore model is called a run. The data for each run is stored as a separate entity, which you can open from the File menu.

The tasks involved in each step are summarized below.

The Prepare Ligands Step

In this step, you add to the run the ligands that you want to use as the basis for the pharmacophore model and any other ligands that you want to use to build or test the QSAR model, and you select ligands for the active set that is used for the pharmacophore model. If you include activity data with the ligands when you add them, you can use a cutoff to select the most active ligands. You can also select the active set manually.

To develop a pharmacophore model, you should ensure that you have all-atom 3D structures, and generate different conformations for each structure. If the structures need to be converted from 2D to 3D or otherwise need cleaning up, you can do so in this step. You can also convert the structures to the most probable ionization (protonation) state at a given pH, and generate generate different chiralities for the ligands in this step. The ligands that you add are automatically grouped into conformer sets. You can choose to group stereoisomers in the same set or different sets. If you add only one conformer for a given ligand, you can generate the rest in this step.

Once you have added the ligands and classified them as active or inactive, you can proceed to the next step.

The Create Sites Step

In this step, you use a set of chemical structure patterns to identify pharmacophore features in each ligand. Once a feature has been mapped to a specific location in a conformation, it is referred to as a pharmacophore site. The number of occurrences of each feature in each ligand is tabulated, and you can display the locations of the pharmacophore sites in the Workspace.

While the built-in set of features is adequate for many purposes, you might want to add new patterns to the built-in features, ignore patterns in the built-in features, or add custom features. You can add functional groups, defined as SMARTS patterns, to the definition of a feature. You can also designate functional groups that should be excluded from consideration as part of a feature, and you can choose to ignore functional groups.

The Find Common Pharmacophores Step

In this step, you perform a search for common pharmacophores among the set of high-affinity (active) ligands that you chose in the first step. The search spans one or more families of pharmacophores, known as variants. You can choose the number of site points in the pharmacophore, filter out variants that have too many or too few of a particular kind of feature, and select a set of variants from the filtered list. You can also set a lower limit on the number of ligands that must match a pharmacophore before it can be considered to be a hypothesis.

The search proceeds by enumerating all pharmacophores of a given variant and partitioning them into successively smaller high-dimensional boxes according to their intersite distances. Each n-point pharmacophore contains n(n−1)/2 unique intersite distances, so each box contains n(n−1)/2 dimensions. Pharmacophores that are clustered into the same box are considered to be equivalent and therefore common to the ligands from which they arise. The size of the box defines the tolerance on each intersite distance, and therefore how similar common pharmacophores must be. You can set parameters to control the minimum box size and you can exclude pharmacophores for which any intersite distance is below a certain threshold.

Boxes that contain pharmacophores from the minimum required number of ligands are said to survive the partitioning process. Each surviving box contains a set of common pharmacophores, one of which is ultimately singled out as a hypothesis.

Once all desired variants have been processed in this manner, you can continue to the scoring step.

The Score Hypotheses Step

In this step you apply a scoring function that identifies the best candidate hypothesis from each surviving box, and provides an overall ranking of all the hypotheses. You can finish at this point, or select a hypothesis for the generation of a QSAR model and continue to the next step. You can also add to the hypothesis volumes that may not be occupied by atoms in any active molecule ("excluded volumes").

The scoring algorithm includes contributions from the alignment of site points and vectors, volume overlap, selectivity, and number of ligands matched. You can adjust some of these in the survival score, and you can create a custom score. The scores for each hypothesis are displayed in a table. You can select a hypothesis and view scores for the ligands that match the hypothesis, and the energy of the ligand relative to the lowest conformation.

If you have both active and inactive molecules that match a hypothesis, you can use their structures to define excluded volumes. Any part of an inactive molecule that is not occupied by the active molecules is a good candidate for an excluded volume. The excluded volumes are used to filter out molecules in the database search that are likely to be inactive.

When you have selected a hypothesis, you can proceed to the next step.

The Build QSAR Model Step

In this step, you build a QSAR model for a selected hypothesis using the activity data for molecules that match at least three points in the hypothesis. You can use molecules with any level of activity, including those that may be inactive due to steric clashes with the target receptor. The QSAR model partitions space into a grid of uniformly sized cubes, and characterizes each molecule by a set of binary-valued independent variables that encode the occupancy of the cubes by each of six atom classes. Partial least-squares regression is applied to these variables to build a series of models with successively greater numbers of factors.

You can visualize the QSAR model in the Workspace, and analyze it by atom class and ligand. This can be used to identify ligand features that contribute positively or negatively to the predicted activity.

When you have developed a QSAR model, you can continue to the database search and use the QSAR model to predict activities for matches.

Develop Common Pharmacophore Hypotheses Panel Features

The Develop Common Pharmacophore Hypotheses panel is wizard-like in design, with five steps. Each step occupies the center of the panel, and consists of a title with a brief description of the step at the top, a set of controls and tables for results, and a Back and a Next button at the bottom. The features of each step are described in detail in separate help topics - see the Related Topics section below.

In addition to the step features, the panel contains a menu bar, a toolbar, and an job status button at the top, and a step guide at the bottom above the Close and Help buttons. These features are described below.

  • The File menu
  • The Display menu
  • The Step menu
  • The toolbar
  • The job status button
  • The Guide

The File menu

The File menu allows you to work with the runs that are available in the project.

New Create a new run
Open Open an existing run from the submenu. If there are more than 4 runs, choose More to open a dialog box and select a run.
Save As     Save the current run with a new name
Rename Rename the current run
Delete Delete the current run

The Display menu

The display menu provides options for viewing hypotheses and related attributes in the Workspace. These items are only available in Step 4 (Score Hypothesis) and 5 (Build QSAR Model). It also has an item for displaying ligand properties as Workspace feedback. This item is available in all steps.

  • Hypothesis
  • Excluded Volumes
  • QSAR Model
  • Site Measurements
  • Workspace Feedback
View Hypothesis

Displays the selected hypothesis in the Workspace, as a spatial arrangement of feature symbols. For a description of these symbols, see the Phase — Create Sites Step topic.

View Excluded Volumes

Displays excluded volumes for the selected hypothesis in the Workspace.

View Excluded Volumes

Displays the selected QSAR model for the hypothesis. Only available in Step 5.

Site Measurements

Opens the View Site Measurements panel, in which you can select the distance and angle measurements between sites that are to be displayed in the Workspace.

Workspace Feedback

Display ligand properties from the current Ligands or Alignments table in the Workspace. Opens the Phase Workspace Feedback dialog box, in which you can turn feedback on and off, choose the properties to display, and whether to display property names. The settings apply to the current project, but can be stored as preferences by clicking Save Preferences in the dialog box.

The Step menu

The Step menu contains an item to display or hide the Guide, and items for each of the steps. The current step is marked with a red diamond. If the Guide is displayed, it is marked with a red square. The items for the steps that are not available are dimmed. You can go to any available step by choosing the corresponding menu item.

The toolbar

The toolbar contains buttons for displaying the items on the Display menu. The buttons are only active in steps 4 and 5.

View Hypothesis
Displays the selected hypothesis in the Workspace, as a spatial arrangement of feature symbols. For a description of these symbols, see the Phase — Create Sites Step topic.
View Hypothesis Labels
Displays feature labels for the selected hypothesis in the Workspace.
View Excluded Volumes
Displays excluded volumes for the selected hypothesis in the Workspace.
View QSAR model
Displays the selected QSAR model for the hypothesis. Only available in Step 5.
View Site Measurements
Display markers for the distances and angles in the selected hypothesis. Opens the View Site Measurements panel, in which you can select the distances and angles to be displayed.

The job status button

The icon on the job status button at the upper right of the panel spins when a job is running. When the job has finished, the icon turns to an exclamation point while the job is incorporating, then returns to its original state when the incorporation has finished. You can click the button to open the Monitor panel and monitor the job.

The Guide

The Guide displays the steps in the model as a set of buttons linked by lines. The buttons for the steps that are not available are dimmed. The current step is highlighted with a white background. You can go to any available step by clicking its button in the Guide. The Guide can be displayed or hidden from the Step menu.
