Summary

In this panel you can use the results of a Glide XP docking run with XP descriptors to generate a Phase hypothesis, in which features are selected based on the XP scores. This hypotheis is termed an e-pharmacophore hypotheses.

Opening the E-Pharmacophores Panel

To open the E-Pharmacophores panel, you can:

  • Choose Tools → Scripts → Docking Post-processing → E-Pharmacophores in the main window.

Using the E-Pharmacophores Panel

The basic approach taken to develop e-pharmacophores is to use the results of Glide XP docking to assign energy contributions to individual atoms in a ligand molecule or fragment molecules, then assign pharmacophore features to the atoms using Phase, and select the features to include in a pharmacophore hypothesis based on the energy contributions of the atoms in the features. The methodology is described in detail in J. Chem. Inf. Model. 2009, 49, 2356–2368 and J. Comput. Aided Mol. Des. 2009, 23, 541–554.

If you are using docked fragments to build up a hypothesis, you can cluster the fragments spatially before generating the hypothesis. Clusters are created with a hierarchical agglomerative algorithm, using full linkage. You can specify the number of clusters to use for developing the hypothesis. Each cluster represents fragments that are docked in the same location in the active site. The best scoring fragment is taken from each cluster to develop the hypothesis. This approach reduces the number of fragments to consider for the hypothesis, as it is assumed that each such fragment is contributing the same set of pharmacophore features to the hypothesis.

Input PoseViewer file text box and Browse button

Specify the pose viewer file that contains the docking results and the XP descriptors. You can enter the file name in the text box, or click Browse and navigate to the file.

Mode options

Select an option for the mode of operation.

  • Single ligand—Use a single ligand for the Phase hypothesis. The ligand used is the first in the pose viewer file. Features with a negative XP score are included in the hypothesis. If the maximum number of features has not been reached with these features, rings that do not receive an XP score are included, but only if their ChemScore lipophilic interaction is favorable by at least 20 kJ/mol. Distance constraints are not applied.
  • Fragments—Use multiple docked molecules, which can be either full ligands or fragments. When this option is selected, the clustering option becomes available. Scoring is based on the XP descriptor terms (and the ChemScore lipophilic term for rings, if it is more favorable than the XP term). Phase features are ranked by this score and the best-scoring features are chosen.
Perform clustering option

Cluster the fragments by volume. No alignment is performed, so that the clustering collects fragments that occupy the same region of space. The fragment with the best docking score in each cluster is then used when creating the hypothesis. In this way, large numbers of fragments can be reduced to a few key fragments.

Create N clusters text box

Specify the number of clusters to use for hypothesis generation. The default is 15.

Create receptor-based excluded volumes option

Create excluded volumes for the hypothesis from the receptor. Atoms that lie between 1.5Å and 5Å of the ligand are used for excluded volumes.

vdW scaling text box

Enter the scaling factor for the van der Waals radii of the receptor atoms. The scaled radius defines the sphere sizes of the excluded volume.

Maximum number of features text box

Specify the maximum number of pharmacophore features to use for the hypothesis.

Advanced Options button

Make settings for the creation of the hypothesis. Opens the E-Pharmacophores - Advanced Options dialog box.

Create Hypothesis button

Create a hypothesis from the ligand or the fragments. Opens a dialog box in which you can specify a job name. This name is used for the hypothesis. When you click OK, the job is started. The E-Pharmacophores panel is unavailable until the job finishes. When it finishes, the hypothesis table is populated with the features of the pharmacophore model.

Hypothesis table

Lists the pharmacophore features with their rank and score. The rank is the order in which the features were selected for inclusion. You can select rows to use for creating a new hypothesis by checking or unchecking the boxes in the Use column.

Write Hypothesis with Selected Features button

Write a hypothesis with the features that are selected in the hypothesis table. Opens a dialog box in which you can specify a job name, which is used to name the hypothesis. When you click OK, the job is started, and the Monitor panel opens so that you can monitor the job.

Manage Hypothesis button

Open the Phase Manage Pharmacophore Hypotheses panel, in which you can perform editing operations on the hypothesis.
