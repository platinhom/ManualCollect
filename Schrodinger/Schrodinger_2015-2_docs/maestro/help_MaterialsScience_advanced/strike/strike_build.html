Summary

The Build QSAR Model panel is used to generate QSAR/QSPR models using multiple linear regression (MLR), partial least squares (PLS), or principal component analysis (PCA).

Opening the Build QSAR Model Panel

To open the Build QSAR Model panel, you can:

  • Choose Tasks → Model Building and Statistics → Build QSAR Model in the main window.

  • Choose Applications → Strike → Build QSAR Model in the main window.

Using the Build QSAR Model Panel

Use this panel to generate a QSAR model given a training set of molecules selected from the Project Table, a set of independent descriptors, and a dependent variable for which a prediction will be made. Three regression techniques are available: multiple linear regression (MLR), partial least squares regression (PLS), and principal component analysis (PCA). Certain options are available only for the appropriate regression method.

When you have finished selecting options, click Run to run the job.

To view the log for the job while it is running, open the Monitor panel by choosing Applications → Monitor Jobs. Most model-building jobs take a few seconds to finish, but outlier detection can add significantly to the time requirement.

Once a model has been built, 2D plots of predicted properties versus the dependent descriptor data can be created. Clicking points in a plot brings the molecule or molecules selected into the Maestro 3D Workspace for viewing and manipulation. (For more information about these and other Maestro plots, see the Scatter Plot Panel topic.)

From Build QSAR Model you can proceed directly to the Predict panel or save the generated models in the project for later use.

Input is N entries currently selected in the Project Table message

The entries that are selected (highlighted) in the Project Table will be used to build the QSAR model. The number of entries selected is displayed in the upper portion of the panel.

Optionally, you can use the Random option in the Select menu of the Project Table to randomly select a specified percentage of molecules from the total entries or from only the currently selected entries. By default, this option chooses only from the selected entries, and the default quantity is 50%.

Property selection tools for model

Choose an appropriate set of independent descriptors that is likely to correlate with the dependent descriptor, by selecting them from the Available properties list, and clicking Selected to transfer them to the Select descriptors to be included in the model list.

These tools are standard property selection tools. For more information, see the Property Selection topic.

Regression method option menu

The options for the regression method are:

Partial Least Squares (PLS)

The PLS model-building job generates a set of f equations or predictors, each using a different number of factors from the underlying data, from 1 to f, the maximum number of factors. The number of molecules selected to be used in building the model must be greater than or equal to the value specified in the Maximum number of factors text box, and for good statistics, should be several times this number.

Principal Component Analysis (PCA)

The PCA model-building job generates a set of f equations or predictors, each using a different number of factors from the underlying data, from 1 to f. The number of molecules selected must be greater than or equal to the number of descriptors chosen.

Multiple Linear Regression (MLR)

For this method, the number of molecules selected must be greater than or equal to the number of initial descriptors. It is recommended that the number of molecules be at least five times greater than the number of descriptors. If the y-intercept through origin option is selected, the regression line is forced to pass through the origin.

Automatically remove outliers option

Select this option to remove outlying molecules before the model is built, using the LOCI algorithm for outlier detection (see Chapter 5 of the Strike User Manual for a description). This option does not remove outliers based on the model. Selecting this option can significantly increase the time required to build the model. The default is not to remove outliers.

y-intercept through origin option

Force the regression line to pass through the origin. When this method is selected, the y-intercept through origin option and the Descriptors options become available.

Descriptors options

When the chosen regression method is Multiple Linear Regression, you can choose to use all selected descriptors or to automatically select the optimal subset of selected descriptors. When Automatically select optimal subset is selected, the number of descriptors to keep is selected by entering a value in the Size of optimal subset text box.

Size of optimal subset text box

When the regression method selected is Multiple Linear Regression, this text box is available. Enter the size of the optimal subset to be chosen from the set of descriptors.

Maximum number of factors text box

When the regression method selected is Partial Least Squares or Principal Component Analysis, this text box is available. Enter the maximum number of factors or components to be used in the regression. The allowed range for the number of factors is from 1 to the number of selected descriptors.

Activity property text box and Choose button

Click Choose to open a list of all the descriptors in the data, from which you can select the property you want the model to predict. The property name is displayed in the text box after you make your choice.

Results table

The Results table lists the models that have been calculated in the current Maestro project.

A Partial Least Squares or Principal Component Analysis model-building job simultaneously creates a set of f predictors, where f is the Maximum number of factors, which cannot exceed the number of independent descriptors included in the model. Each predictor extracts a different number of factors from the underlying data, from 1 to f, and is represented as a single row in the table, with a name using the format model.predictor. A Multiple Linear Regression model-building job generates a model that is represented by a single row in the table.

Along with the name of each predictor or model, the table includes the regression method used, the number of PLS/PCA factors or MLR descriptors, the eigenvalue for PCA models, and standard diagnostic statistic values from the regression. Click on a single row to select an MLR model or a particular predictor generated as part of a PLS or PCA model. (Any predictor or model that consists of a single row in the table can be selected and used in similar ways, and will be referred to as a "model" in the following discussion.) You can export, view, or delete the selected model, use the model for making further predictions, and plot the results from fitting the model.

Export button

Export the currently selected model to an external file.

View button

Click View to review the output file of the model-building job for the selected model, which contains all the data needed to completely describe the model.

Plot button

Generate a plot of the predicted property versus the dependent data for the currently selected model. Opens the Scatter Plot panel to display the plot. The regression line and the parallel lines at one sigma distance are displayed in the plot, and the R-squared value is displayed below the horizontal axis label.

Delete button

Delete the currently selected model from the table.

Predict button

Open the Predict Based on QSAR Model panel with the currently selected model chosen.

Job toolbar

The Job toolbar is used to make settings for a job and to start it.

Job name text box

Enter a name for the job in this text box.

Settings button menu

This button opens the Job Settings dialog box. The arrow to the right opens a menu, from which you can make settings or perform actions that are related to the job.

The menu items are described below.

  • Job Settings—Make and save settings for the incorporation of the job into the Maestro project, the job name, the host, the distribution of subjobs, and any other application-specific job settings. Opens the Job Settings dialog box.
  • Preferences—Set general preferences for running jobs. Opens the Preferences panel at the Jobs - Starting section.
Job status button

This icon indicates when there are jobs running for the application that belong to the current project. It starts spinning when the first job starts, and stops when there are no more jobs running. If a job fails to start, it changes to an exclamation point.

Clicking the button shows a small job status window that lists the job name and status for all active jobs submitted from the current panel (for Jaguar and MacroModel this means all jobs from any of the application panels). You can double-click on a row to open the Monitor panel and monitor the job, or click the Monitor button to open the Monitor panel and close the job status window. The job status is updated while the window is open. If a job finishes while the window is open, the job remains displayed but with the new status. The rows are colored according to the status: yellow for submitted, green for launched, running, or finished, red for incorporated, died, or killed. Click anywhere outside the window to close it.

Run button

Run the job with the current job settings.

Status bar

The status bar displays information about the current job settings for the panel. This includes the job name, task name and task settings (if any), number of subjobs (if any) and the host name and job incorporation setting. It also displays a message for a short time when you start a new job.

The status bar also contains the Help button, which opens the help topic for the panel.
