Summary

This panel allows you to set up a sequence of jobs in which ligands are docked with Glide, then charges on the ligand induced by the protein are calculated quantum-mechanically with QSite (QM/MM program), and the ligands are redocked with these charges.

Opening the QM-Polarized Ligand Docking Panel

To open the QM-Polarized Ligand Docking panel, you can:

  • Choose Applications → Glide → QM-Polarized Ligand Docking in the main window.

Using the QM-Polarized Ligand Docking Panel

The QM-Polarized Ligand Docking protocol aims to improve the partial charges on the ligand atoms in a Glide docking run by replacing them with charges derived from quantum mechanical calculations on the ligand in the field of the receptor. In this way the polarization of the charges on the ligand by the receptor is accounted for, and redocking of the ligands with these new charges can result in improved docking accuracy. The protocol works by taking a small set of the best-scoring poses for each ligand, calculating charges in a QM/MM calculation, redocking each of these poses, and selecting the best poses from the set.

The QM-Polarized Ligand Docking panel collects the relevant controls for each part of the protocol, with judicious selection of defaults for options that are not presented in the panel (these are usually the defaults for the application).

To set up a QM-polarized ligand docking calculation:

  1. Select the source of the ligands to be docked from the Dock ligands from option menu.

    If the source is the Project Table, you must select the entries that you want to use. If the source is a file, you can enter the file name in the text box or click Browse and navigate to the file, which can be a Maestro or SD file. The ligands must be properly prepared (for example, by using LigPrep).

  2. Set up the receptor grid, or select an existing grid in the Receptor tab.

    To set up the grid, the receptor must be displayed in the Workspace, with or without the ligand. If a ligand is not displayed, you must define the center of the enclosing box in terms of receptor residues. The protein must be properly prepared for a Glide calculation.

  3. Set the initial docking options in the Initial docking tab.

    The main tasks in this tab are to set parameters for the elimination of duplicate poses, scale the van der Waals radii if necessary, select the docking mode, and decide how many poses to keep per ligand. The defaults represent reasonable choices.

  4. Select the source of the quantum mechanical charges in the QM Charges tab. For full QM charges, the selection is a trade-off between speed and accuracy. The charges are calculated from the electrostatic potential energy surface of the ligand, which is generated from a single-point calculation using density functional theory for the QM region, which consists of just the ligand.

  5. Set the redocking options in the Redocking tab, including the precision, scaling, duplicate elimination, and number of poses to keep.

  6. Select the energy parameter by which the final poses are scored in the Final Selection tab.

  7. Choose Job Settings from the Settings button menu,

    and set job options in the Job Settings dialog box.

  8. Click Run to run the job.

When the job finishes, a Maestro file is generated with the final selections. In addition, a jobname.log file is generated.

If you want to change any of the options for which controls are not provided in this panel, you can click Write, and the files for the calculation are written. You can then run the calculations from the command line.

If you want to calculate the RMSD of the docked poses from a corresponding set of reference ligands, select Calculate RMSD from reference ligand file and specify a Maestro or SD file that contains the reference ligands, which must have the same structures and titles as the input ligands. The poses are matched to the reference ligands using the title. The results are written to a jobname_rmsd.out file.

Ligand input

In this section, you choose the ligands to be docked. Select the source from the Dock ligands from option menu, from Project Table (selected entries) or File. If you choose File, enter a file name in the Input ligand file text box, or click Browse and navigate to the file containing the ligands.

Receptor tab

In the receptor tab, you specify the source of the receptor. You can either use an existing grid, or generate a grid from the Workspace.

  • Use existing grid option and text box
  • Ligand molecule number text box and Pick option
  • Box center options
  • Box size options
  • Receptor van der Waals scaling text box
Use existing grid option and text box

Select this option to use an existing grid. You must specify the path to the grid file in the text box, or click Browse and navigate to the grid file.

Ligand molecule number text box and Pick option

Specify the ligand molecule number. You can eneter the molecule number in the text box, or select Pick and pick a ligand atom in the Workspace.

Box center options

Specify the location of the center of the grid box as the centroid of a specified set of atoms. There are two options:

  • Centroid of the Workspace ligand—Select this option to center the grid box on the ligand centroid. The centroid is the mean position of the non-hydrogen atoms in the ligand.
  • Centroid of the residues—Select this option to center the grid box on a group of receptor residues. To select residues for the centroid, click the Select button. The Atom Selection dialog box opens, in which you can select the residues to define the centroid. The ASL expression for the residues is displayed in the text box to the left of the Select button.
Box size options

Specify the size of the grid box. There are two options:

  • Auto—Select this option to automatically determine the size of the grid box. If the Box center option is Centroid of the ligand, the box size is calculated automatically from the size of the ligand. If the Center option is Centroid of the residues, the box size is set to 26 Å on each side.
  • Dock ligands with length <=—Select this option to set the size of the grid box. Enter the desired side length in the text box, in angstroms, or use the slider. The grid box has sides of equal length given by the value in the text box.
Receptor van der Waals scaling text box

Enter the scaling factor for the van der Waals radii of the nonpolar part of the receptor. See the Van der Waals Radii Scaling section of the Glide Grid Generation Receptor Tab topic for details.

Initial docking tab

In this section you specify the source of the initial poses. They can come from an existing pose-viewer file, or from an initial docking run on the ligands.

  • Use existing pose-viewer file text box and Browse button
  • Generate initial charges with semiempirical method option
  • Precision options
  • Ligand van der Waals scaling text box
  • Discard pose as duplicate if text boxes
  • Maximum poses per ligand text box
Use existing pose-viewer file text box and Browse button

Select this option if you want to use poses from a file as input to the QM/MM step. When you select this option, the text box and Browse button become available, and you can enter the name of the pose file in the text box, or click Browse and navigate to the file. At the same time, the remaining controls in the panel become unavailable.

If you select this option, the structure titles in the pose-viewer file must be consistent with the input ligand file, and structures with the same title must be conformers, and have the same atom numbering. This file is only used as input to charge generation. The input file is the file used for the redocking stage.

Generate initial charges with semiempirical method option

Select this option to generate the initial charges on the ligand with the semiempirical NDDO method, rather than using the force field.

Precision options

Select SP or XP docking precision. For more information on the docking precision, see the Ligand Docking — Settings Tab topic.

Ligand van der Waals scaling text box

Specify a scaling factor for the van der Waals radii of the nonpolar parts of the ligand in this text box. For more information on the scaling, see the Scaling of Van der Waals radii section of the Ligand Docking — Ligands Tab topic.

Discard pose as duplicate if text boxes

These two text boxes specify the thresholds for discarding a pose as a duplicate. Both thresholds must be met for a pose to be discarded. Specify the RMS deviation and the maximum atomic displacement thresholds in angstroms.

Maximum poses per ligand text box

Enter the number of poses to use for the QM ESP calculations. The ESP calculation is time-consuming, so only a small number of poses should be used. The default value is 5.

QM Charges tab

This tab provides some control over the accuracy of the QM calculation on the ligand and of the environment of the ligand.

  • Calculate charges for free ligand options
  • Use semiempirical method option
  • Charge type option menu
  • Use Jaguar option
  • QM level option menu
Calculate charges for free ligand options

Select this option to perform the QM calculation on the free ligand rather than on the ligand in the field of the receptor. If you select this option, the two options In gas phase and In water become available. You can then choose whether to treat the ligand in the gas phase or with a continuum solvation method appropriate for water.

Use semiempirical method option

Use the semiempirical NDDO module to calculate the charges.

Charge type option menu

Choose the type of charges to calculate with the semiempirical method, from Coulson, Mulliken, and ESP.

Use Jaguar option

Use the full quantum mechanical treatment in Jaguar to calculate the charges.

QM level option menu

This option menu allows you some control over the speed of the full QM calculation, by selecting combinations of basis set, grids, and density functional.

  • Fast— Uses the 3-21G basis sets with the BLYP density functional and Quick SCF accuracy level.
  • Accurate— Uses the 6-31G*/LACVP* basis sets with the B3LYP density functional and Ultrafine SCF accuracy level .

Redocking tab

  • Precision options
  • Ligand van der Waals scaling text box
  • Discard pose as duplicate if both text boxes
  • Maximum poses per ligand text box
Precision options

Select SP or XP docking precision. For more information on the docking precision, see the Ligand Docking — Settings Tab topic.

Ligand van der Waals scaling text box

Specify a scaling factor for the van der Waals radii of the nonpolar parts of the ligand in this text box. For more information on the scaling, see the Scaling of Van der Waals radii section of the Ligand Docking — Ligands Tab topic.

Discard pose as duplicate if both text boxes

These two text boxes specify the thresholds for discarding a pose as a duplicate. Both thresholds must be met for a pose to be discarded. Specify the RMS deviation and the maximum atomic displacement thresholds in angstroms.

Maximum poses per ligand text box

Enter the number of poses to use in the redocking calculations. The ESP calculation is time-consuming, so only a small number of poses should be used. The default value is 5.

Final selection tab

  • Final selection by option menu
  • Calculate RMSD from reference ligand file option, text box and Browse button
Final selection by option menu

This option menu offers the choice of three quantities by which the poses are ranked: the Coulomb-van der Waals energy, the GlideScore, and Emodel. For a description of these quantities, see the Glide Overview and Ligand Docking — Output Tab topics. The poses are written in rank order in the output structure file jobname-SORT_OUT.mae.

Calculate RMSD from reference ligand file option, text box and Browse button

This option allows you to calculate the RMSD of the various poses of the redocked ligand relative to a set of reference ligands, which is read from file. Specify the path to the file, or click Browse to navigate to the file. The file must be a Maestro file or an SD file. The reference ligands must have the same structure (atom types and order) as the input ligands, and are matched to the poses by title (by default). The poses are listed in rank order with their RMSD values in the file jobname-RMSD.log, and the RMSD is returned as a property in the output structure file, jobname-RMSD_OUT.mae.

Job toolbar

The Job toolbar is used to make settings for a job and to start it.

Job name text box

Enter a name for the job in this text box.

Settings button menu

This button opens the Job Settings dialog box. The arrow to the right opens a menu, from which you can make settings or perform actions that are related to the job.

The menu items are described below.

  • Job Settings—Make and save settings for the incorporation of the job into the Maestro project, the job name, the host, the distribution of subjobs, and any other application-specific job settings. Opens the Job Settings dialog box.
  • Preferences—Set general preferences for running jobs. Opens the Preferences panel at the Jobs - Starting section.
  • Write—Write the input files for the job but do not start it. The file names are based on the current job name, and the files are written to a subdirectory of the current working directory that is named with the job name.
  • Reset Panel—Reset the values in the panel to their defaults.
Job status button

This icon indicates when there are jobs running for the application that belong to the current project. It starts spinning when the first job starts, and stops when there are no more jobs running. If a job fails to start, it changes to an exclamation point.

Clicking the button shows a small job status window that lists the job name and status for all active jobs submitted from the current panel (for Jaguar and MacroModel this means all jobs from any of the application panels). You can double-click on a row to open the Monitor panel and monitor the job, or click the Monitor button to open the Monitor panel and close the job status window. The job status is updated while the window is open. If a job finishes while the window is open, the job remains displayed but with the new status. The rows are colored according to the status: yellow for submitted, green for launched, running, or finished, red for incorporated, died, or killed. Click anywhere outside the window to close it.

Run button

Run the job with the current job settings.

Status bar

The status bar displays information about the current job settings for the panel. This includes the job name, task name and task settings (if any), number of subjobs (if any) and the host name and job incorporation setting. It also displays a message for a short time when you start a new job.

The status bar also contains the Help button, which opens the help topic for the panel.
