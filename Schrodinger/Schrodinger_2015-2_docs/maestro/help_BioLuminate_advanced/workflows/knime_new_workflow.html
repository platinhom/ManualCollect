Summary

This panel allows you to edit a new KNIME workflow in KNIME, install the workflow, set up parameters and test the workflow. It is a generic panel that is opened with the details of the new workflow after setting it up in KNIME.

Opening the KNIME Workflow (New) Panel

To open the KNIME Workflow (New) panel, you can:

  • Choose Tasks → KNIME Workflows → New in the main window.

This action first opens KNIME with an example and instructions, and the panel opens when you close KNIME.

Using the KNIME Workflow (New) Panel

To edit the new workflow:

  1. Click Edit. The KNIME interface starts and loads the workflow.

  2. Make your changes in the KNIME interface. (See Editing KNIME Workflows for instructions.)

  3. Close the KNIME interface.

To test the new workflow:

  1. Select entries in the Project Table or include them in the Workspace, according to what is listed under Input.

  2. Make choices with the controls under Parameters.

  3. If you want to store the results of testing the workflow (including input, output, and parameter settings) in the workflow, select Save after execution.

  4. Click Test.

To install the workflow under Workflows → KNIME:

  • Click Install.

You can also install workflows by using the Install Workflows panel.

KNIME Workflow (New) Panel Features

  • Workflow name and description
  • Edit button
  • Install button
  • Input description
  • Ouptut description
  • Parameters controls
  • Save after execution option
  • Test button
Workflow name and description

The name of the workflow and its description (if any) are displayed at the top of the panel, along with a list of required programs.

Edit button

Edit the workflow in the KNIME interface. Starts the KNIME interface and loads the workflow.

Install button

Install the workflow on the KNIME submenu of the Workflows menu. You are notified that the installation was successful. When you click OK, the Knime Workflow panel closes also.

Input description

The source of the input structures is described under this heading. The two possibilities are Selected entries and Included entries. You must ensure that the appropriate selection or inclusion is made before testing the workflow.

Ouptut description

The types of output are listed below this heading. They are:

  • New group—Incorporate the output structures into the project as a new entry group.
  • Maestro message message—Display an alert box with the specified message, and any associated numerical values.
  • Maestro commands commands—Run the specified Maestro commands at the end of the workflow (after incorporating any output structures).
Parameters controls

These controls allow you to set parameters requested by Quick form nodes in the workflow.

Save after execution option

Save the workflow after testing it. The workflow will then include all the results of running it, and any settings you changed under Parameters are also changed.

Test button

Test the workflow by running it in batch mode.
