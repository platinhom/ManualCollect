Summary

In this step, you build a model structure for the query based on the template chosen in the Find Homologs step and aligned in the Edit Alignment step.

Working in the Build Structure Step

The process of building a structure from one or more templates involves the following steps.

  • Copying of backbone atoms for aligned regions and side chains of conserved residues
  • Optimization of side chains
  • Minimization of non-template residues
  • Building of insertions and closing of deletions in the alignment

By default, all of these steps are performed, except that insertions of more than 20 residues are omitted. Also by default, only the protein structure is built: ligands, cofactors and waters are deleted before building. However, you can choose to leave out one or more steps (except the first), and you can choose to build in ligands and cofactors. (Water molecules must be added afterwards.) These choices must be made before you build the structure. If you are using multiple templates, you must also choose the type of model you are building: composite, consensus, homo-multimer or hetero-multimer. For composite models, you must select the regions from each template that are used to build the structure. The following sections provide more detail on the choices to be made.

For details and technical information about the Build Structure step, see the Build Structure chapter in the Prime User Manual.

Building a Composite Model

If you want to build a composite model from the templates you selected in the Find Homologs step, select Composite/Chimera in the Multiple-template modle type section. You must specify which template should be used for each region of the query, otherwise, the first template in the table is used for the entire query.

To specify the regions to be used for each template, first select Set template regions (selected by default), then select the desired region on each template in the Sequence Viewer. When you begin to select the template regions, only the first template is in use. As you select regions in the second template, the corresponding residues in the first template are automatically deselected. If you select regions for use from a third template, the corresponding regions of the first and second templates are deselected, and so on. Thus, the selection process ensures that only one template residue is assigned to a query residue (the most recent one selected). Regions of a template that are not used to model the query are colored dark gray (regardless of color schemes.)

Building a Consensus Model

If you want to build a consensus model from the templates, select Consensus in the Model type when using multiple templates section. The model is then built from the consensus between the templates at each residue position: for example, if two templates have a loop in the same spatial region but in the third template, the loop occupies a different region, the model is built from the two that agree. The alignment is done as part of the model-building job, so there is no need to align the structures for the templates.

Building Multimers

In addition to building models for a single chain, you can build models for multimers that are composed of copies of the same chain (homomultimers) or of different chains (heteromultimers).

To build homomultimers, you must have an alignment for each chain in the templatea. Often the template is another homomultimer, but it can also be a heteromultimer. The main requirement is that structures for each template chain must be in the correct spatial relation to product the multimer.

To build heteromultimers, you must first model the individual units of the multimer (which can be homomultimers) in separate runs, up to this step. The templates that you choose to build the individual units must be properly aligned to each other, so that the resulting built structure has the desired configuration of the units. To build the model, click Hetero-multimers, choose the runs in the Hetero-multimers Modeling dialog box, and click Start.

Setting Constraints

If you know that certain residues must be in close contact in the final structure, such as cysteines forming a disulfide bond, you can constrain their positions in the final model by setting proximity constraints on residue pairs. Proximity constraints can only be applied if at least one of the residues in the pair is in a loop, as helices and strands can't be adjusted to satisfy proximity constraints. A disulfide bond is formally constructed between pairs of CYS residues. To apply these constraints, select Pick residue pairs in query sequence, and pick the pairs of residues that you want to be in contact in the final model. You can also apply proximity constraints between a ligand and the query, by picking a residue and a ligand.

Proximity constraints are only available with the energy-based model building method.

Including Ligands and Cofactors

Cofactors and ligands that are present in the template you have chosen (except water) are listed in the Include ligands and cofactors list. If you want to include any of these cofactors and ligands in the build process, select them in the list. You can use control-click and shift-click to select multiple rows. The selected ligands and cofactors are highlighted in yellow in the Workspace. By default, only the ligands and cofactors within 5 Å of the chain used for the template are included in the list. To display the full list, select Show All.

Choosing Options for Building

If you want to select the steps that are performed in the building process, you can do so in the Build Structure - Options dialog box, which you open by clicking Options.

Results of Model Building

When the model is built, the structure is automatically incorporated into the project, and the model is displayed in the Workspace. For composite and multimer models, the residues are colored according to how the template was used:

  • blue—all coordinates were taken from the template
  • cyan—the backbone coordinates were taken from the template
  • red—the residue was rebuilt.

This information is stored in the Homology Status property (i_psp_Homology_Status), with values of 0, 1, and 2 respectively.

The coloring of ALA and GLY residues in the model that came from other residues in the template depends on whether you clicked Preview Model or Build Model. These residues are colored cyan in the preview, which only indicates that they are different from the template: the coloring is based on the alignment. In the built model, these residues are colored blue, because they have no side chains and therefore all the coordinates in the built model were taken from the template.

The sequence in the Structure Prediction panel remains as it was before building: it reflects only the state of the sequences brought into the Build Structure step. The Workspace sequence viewer displays the full sequence of the built model.

Build Structure Step Features

  • Model-building method options
  • Multiple-template model type options
  • Include ligands and cofactors list
  • Proximity constraints section
  • Build Model buttons
  • Write button
  • Options button
  • Hetero-multimers button
Model-building method options

Choose a method for building the structural model.

  • Knowledge-based—Construct insertions and close gaps using segments from known structures. With this option, you can choose to return multiple models of the structure. By default, only one is returned.

  • Energy-based—Construct and refine residues not based on the template based on the energy.

Multiple-template model type options

These options select the kind of model that is build when you have multiple templates.

  • Composite/Chimera—build a composite model, in which each template contributes to a different region of the query. If you select this option, you must select Select template regions to define the regions of each template that are used to build the model, by selecting the residues in the sequence viewer. This option is selected by default.

  • Homo-multimer—Build a model for a multimer that is made up of identical units. The query sequence is duplicated for each unit of the multimer, and all units are built in the same job.

  • Consensus model—Build a model as "majority rule" selection from all the templates. The structure is built from scratch, with the templates as constraints (making each Cα a " potential well").

Include ligands and cofactors list

Lists the ligands and cofactors that are part of the template structures. You can select multiple list items with the usual shift-click and control-click. The selected items are added to the structure when the protein is built. DNA and RNA can be included as cofactors. By default, only the ligands and cofactors within 5 Å of the chain used for the template are included in the list. To display the full list, select Show All.

Proximity constraints section

In this section, you specify pairs of residues that you require to be in close contact in the final model, or pairs that involve a residue and a ligand. At least one of the residues must be in a loop, since helices and strands do not have the flexibility to adjst to the constraint. A harmonic constraint is added for the terminal heavy atoms of the side chains. If you specify a pair of CYS residues, a disulfide bond is formed.

Pick pairs in query sequence and ligand table option

Select this option to pick pairs of residues in the query sequence. At least one residue must be in a loop. After picking two residues, the pair is added to the list.

Residue pair list

This list contains the pairs of residues that have been picked, represented by the residue type and residue number, e.g. A41. You can select multiple pairs in the list; the only available action for the selection is to delete them.

Delete and Delete All buttons

Delete the pairs that are selected in the list, or delete all the pairs in the list.

Build Model button

Build the structure, with the options that are selected in the Build Structure - Options dialog box.

Write button

Write out the input file for building the structure. A dialog box is displayed that shows the command to use for running the job, which you can copy and paste into a terminal window (Linux) or SchrodingerShell (Windows).

Options button

Opens the Build Structure - Options dialog box, in which you can select options for retention of rotamers, optimization of side chains, minimzation of non-template residues, and handling of structural discontinuities. These options apply to both model-building methods.

Hetero-multimers button

Set up the building of a hetero-multimer. Opens the Hetero-multimer Modeling dialog box, in which you can select the runs that you want to use to build the hetero-multimer.
