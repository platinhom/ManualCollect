Summary

In the Find Common Pharmacophore - Options dialog box, you can specify the parameters that govern the search for common pharmacophores. Choosing options is a balance between the size of the features in the hypotheses, the size of the ligands, and the time taken and storage requirements for the search.

Opening the Find Common Pharmacophore - Options Dialog Box

To open the Find Common Pharmacophore - Options dialog box, you can:

  • Click Options in the Find Common Pharmacophore step of the Build Common Pharmacophore Hypotheses panel.

Find Common Pharmacophore - Options Dialog Box Features

Minimum intersite distance

The minimum distance allowed between two features. If the features in the ligand are closer than this distance, the hypothesis is rejected.

Maximum tree depth

The number of binary partitioning steps used to sort the pharmacophores into similar groups. This is the maximum recursion level in the partitioning, and the depth of the resulting binary partitioning tree.

Initial box size

The size of the initial box in the partitioning algorithm, computed from the final box size and the maximum tree depth, n:

Initial size = Final size * 2n

Noneditable text box. This size should be approximately as big as the binding site, or the smallest ligand. You should therefore choose the final box size and the maximum tree depth to ensure that the initial box size is big enough. The default size is 32 Å.

Final box size

The size of the boxes that contain intersite distances that are considered to be equivalent. This option governs the tolerance on matching: the smaller the box size, the more closely pharmacophores must match. However, the smaller the box size, the longer the search takes. If you choose a smaller final box size, you might have to increase the maximum tree depth so that the initial box size is large enough to accommodate the ligands. If the final box size is too small, the tolerance on matching may be too strict to produce any common pharmacophores.
