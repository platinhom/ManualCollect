Running KNIME Workflows from Maestro

KNIME workflows can be created, installed, and run from Maestro. With some special configuration of the workflow, structures and data can be transferred between Maestro and KNIME, so that you can use Maestro for structure manipulation and input, and store the results of the workflow in the Maestro project.

Maestro does not run the workflows directly, however: it is done via a Python panel, which is created from a specially configured workflow. To configure an existing workflow or build a new workflow that is configured for running from Maestro, choose Workflows → KNIME → New. This action starts KNIME, where you can import or build a workflow and make the appropriate settings, and then creates the Python script for the workflow when you close KNIME. You can then test the script and install it onto the KNIME Workflows submenu of the Workflows menu.

Note: You can build the workflow and make settings in a regular KNIME session, but the Python panel for running the workflow from Maestro is only created when you run KNIME with Workflows → KNIME Workflows → New.

Configuration of the input and output to communicate with Maestro is done by editing the node names of the Molecule Reader and Writer nodes, the CSV Writer node, and by adding Quick Form nodes that are used to set options, with an edited node name.

For structure input, the Molecule Reader node name must contain either "Selected entries" or "Included entries". For structure output, the Molecule Writer node name must contain "New group". To display an information box with results, use a CSV Writer node with a name that includes "Maestro message". To run Maestro commands, use a CSV Writer node with a name that includes "Maestro commands". See Editing KNIME Workflows for Use with Maestro for details.

To specify options or parameters, use a Quick Form node, and include GUIsetting in the node name. For the String Radio Button nodes, the node name must contain GUIsettingRADIO to be shown in the panel as a radio button group, or GUIsettingCOMBO to be exposed as a combo box (option menu). Configure these nodes with the labels you want to use on the radio buttons or the option menu, the default values, and the values displayed for each menu item or radio button. The GUI components for these nodes are then added to the Python panel that is created to run the workflow.

Knime workflows are run from Maestro via Python panels, which are installed on the KNIME submenu of the Workflows menu. These panels have the same structure, with controls that depend on the workflow. See the KNIME Workflow Panel topic for details on how to run an installed workflow.

The KNIME workflows can be created, managed, and installed with the following items on the KNIME Workflows submenu of the Workflows menu:

  • New—Open a new KNIME session to build or import a workflow for installation into Maestro.

  • Manage—Reorder workflows or group them into categories, which appear as submenus under Workflows → KNIME Workflows.

  • Install—Install a workflow that has already been set up to run from Maestro.

  • Update—Update the Schrödinger workflows with the latest versions from the web site.
