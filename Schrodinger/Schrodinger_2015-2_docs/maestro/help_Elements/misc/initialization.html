Initialization of Maestro

The following UNIX environment variables affect the way that Maestro runs. Generally only the first, SCHRODINGER, needs to be set since most of the others are automatically set based on SCHRODINGER if they are not defined, or have defined defaults. SCHRODINGER is normally set to the top-level location of the installation. You can, however, explicitly define any of these before running Maestro. In that case, Maestro uses the value you define.

SCHRODINGER Top-level directory of the installation.
SCHRODINGER_DOCS Top-level directory of the documentation. The default is $SCHRODINGER/docs.
SCHRODINGER_HELP_DIR Directory that contains the online help. The default is $SCHRODINGER/docs/maestro/help_Elements.
DISPLAY Specifies the name of the display machine.
SCHRODINGER_TEMP_PROJECT Location that Maestro uses for temporary projects. The default is $HOME/.schrodinger/tmp. If you have limited space in your home file system, you might want to set this variable to a directory on a larger file system. You might also want to set this variable to a directory on a local disk, to improve the speed of project operations.
SCHRODINGER_TEMP_LOCATION Location of some temporary files, including files used for communication with other programs and files used by the help system. The default location is $SCHRODINGER_TEMP_PROJECT/pid, where pid is the Maestro process ID.
SCHRODINGER_GL Specifies use of the Schrödinger-supplied OpenGL library (instead of using the default system OpenGL library). Equivalent to using the -sgl command line option. Normally this should not be needed unless you are experiencing problems using the system OpenGL library.
MMSITEDIR Typically set to $SCHRODINGER/mmshare-vversion/data by the startup script. This is a backup location for system-wide aliases, and should not be set by the user.
MMRESDIR Directory containing system-wide Maestro startup and resource files. One of the locations in which Maestro looks for a startup command file, maestro.cmd. This is a backup location, and should not be set by the user.
MMXRESDIR Directory containing Maestro X Windows resource files.

Maestro also sets a range of other environment variables on startup. These are listed in the terminal window where Maestro was started. You should not set any of these environment variables yourself.

When Maestro starts, it attempts to find a file called maestro.cmd. If it does, it loads the contents and executes the Maestro commands in this file. The order in which it looks and executes is: 

$MMRESDIR/maestro.cmd
~/.schrodinger/maestrovv/maestro.cmd

where vv is the 2-digit Maestro version. If it finds both of these files, they are executed in the above order. In this way, the user command file overrides the standard, system-wide command file.

Once this is done Maestro attempts to read in the aliases (see related topics for more information) that are located in a file called alias.cmd. It searches the following locations and the first instance of alias.cmd it finds is the one it executes. No subsequent searching is done after that. 

$PWD
~/.schrodinger/maestrovv
$MMSITEDIR
$MMRESDIR
$MMXRESDIR

Note that MMXRESDIR is for X resources so it is ill-advised to place an alias.cmd file in that directory. The recommended locations are the current directory or ~/.schrodinger/maestrovv.
