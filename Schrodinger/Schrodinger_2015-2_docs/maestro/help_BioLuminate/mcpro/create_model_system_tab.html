Summary

In this tab, you specify the model system for the MCPRO+ calculation. This task is required for all MCPRO+ calculations, and involves selecting the receptor and ligands as appropriate, truncating (chopping) and capping the protein, neutralizing the system, and defining constraints. The resulting model system is written out in a series of files that include a Z-matrix file and a Maestro file.

Model System Creation Tab Features

The Model System Creation tab provides a wizard-like interface that takes you through the stages of setting up a model system. For new systems that contain proteins, there are five steps. For ligand-only systems, two of these steps are not present.

Each step contains the main controls for the step, then a Display Options section, below which is a Guide for navigating between the steps. The Guide consists of a button for each step, connected by arrows. The buttons for steps that are not available are dimmed. The Display Options section contains options for displaying the various structures belonging to the model in the Workspace.

The main controls for each step, and the common Display Options section, are described below.

  • Import step
  • Chop/Cap step
  • Neutralize step
  • Constrain step
  • Create Model step
  • Display Options section

Import step

In this step, you define the type of system and import the structures for the calculation. There are two sections in the main part of the step.

  • Model system section
  • Import section
Model system section

In this section, you choose the basic types of structures to be included in the system, and set a base name for the model system.

Model system basename text box

Set the base name for files created to store information for the model system.

System options

These options define the system type. The options vary according to the type of calculation, and can include one or more of the following:

  • Protein-ligand complex
  • Ligands only
  • Protein only

When you choose a system option that does not contain a protein, the Chop/Cap and Neutralize steps are removed from the Guide, since they are no longer relevant.

Import section

In this section, you import the structures or the Z-matrix for the system. The controls available depend on the choice of the system type. The structures must be valid structures in Maestro format.

Ligand structures controls

Enter the name of the ligand structure file in the text box, or click Browse and navigate to the file. The file name is displayed in the text box. The number of ligands to be imported depends on the type of job. The ligand is imported and displayed in the Workspace when you have entered the file name or clicked OK in the file chooser.

You can also select Use charges from input structures to make use of the partial charges stored with the ligands, rather than generate them from the force field.

Protein structure controls

Enter the name of the protein structure file in the text box, or click Browse and navigate to the file. The file name is displayed in the text box. The protein structure must not include a ligand, but can include cofactors, metal ions, and so on. The protein is imported and displayed in the Workspace when you have entered the file name or clicked OK in the file chooser.

If the protein was used previously for a model system, you can select Use associated protein zmatrix instead of regenerating to skip over the protein preparation steps. Selecting this option removes the Chop/Cap and Neutralize steps from the Guide.

For protein-ligand complexes, you can choose to import a reference ligand that will be used in the next two steps, for protein truncation and neutralization. To do so, select Use imported reference ligand in model system creation, then enter the name of the ligand structure file in the text box, or click Browse and navigate to the file. The file name is displayed in the text box.

For protein-only calculations, you must import a reference ligand to be used in the next two steps, for protein truncation and neutralization. To do so, enter the name of the ligand structure file in the Reference ligand structure text box, or click Browse and navigate to the file. The file name is displayed in the text box.

When you have made your choices, the button for the next step becomes available, and you can click it to proceed.

Chop/Cap step

This step is only available when a protein is being used. The purpose of the step is to truncate ("chop") the protein, and cap the truncated parts with NMA or ACE. There is only one section, Chop, with the following features:

Keep protein residues within N Å of reference ligand option and text box

Select this option if you want to chop the protein at a specified distance from the ligand, and enter the distance in the text box. The reference ligand is used for this purpose if one has been provided. The residues that are within this distance are colored yellow, and residues outside this distance are colored blue.

The truncation procedure keeps all residues in the protein that are within the specified distance of any atom in the ligand, and the residues at the cut are capped with NMA or ACE as appropriate. A smaller protein model also means that the number of solvent water molecules added to the system will be smaller.

Do not chop protein system option

Select this option if you do not want the protein to be truncated. The time taken for the simulations and the number of solvent waters added to the system is related to the protein size. If you select this option, the Neutralize button becomes available, and you can proceed to the next step.

Chop/Cap Protein button

Click this button to perform the truncation. A message is displayed in the lower part of the panel while the truncation job is running. If you select Keep protein residues within N Å of reference ligand, you must click this button to chop the protein. After the protein is truncated, the Neutralize button becomes available, and you can proceed to the next step. The capping groups are now displayed in blue.

Reset Protein button

Click this button to undo the truncation.

Neutralize Step

This step is only available when a protein is being used. The purpose of the step is to ensure overall electrical neutrality of the protein or protein-ligand complex. This is important to obtain good results. Neutralization is performed by adding or removing protons from ionizable residues that are distant from the ligand.

There is one section for this step, Neutralize system, which has two options for neutralizing the protein, and action buttons to perform or undo the neutralization task. Residues that are neutralized are highlighted in a unique color in the Workspace.

Neutralize to first ligand charge option

Select this option to neutralize the complex by adding or removing protons from the protein so that the protein charge balances the ligand charge, and the charge on the complex is zero.

Target protein charge option and text box

Select this option to add or remove protons from the protein so that the protein charge is equal to the value given in the text box.

Neutralize Protein button

Click this button to perform the neutralization. A message is displayed in the lower part of the panel while the neutralization job is running. When the job finishes, the residues that were deprotonated are colored red, those that were protonated are colored yellow, and the remaining residues are colored gray. You must click this button to neutralize the protein and proceed to the next step. After the protein is neutralized, the Constrain button becomes available, and you can proceed to the next step.

Reset Protein button

Reset the protein to its original state, before addition or removal of protons.

Constrain Step

In this step, you choose which parts of the protein are treated flexibly, and whether the ligands or cofactors are kept rigid or allowed to be flexible. If you are doing a minimization, the choices you make in this section affect the choice of optimization method (see the Minimization Tab topic).

There is one section for this step, Constrain molecules.

Protein options

There are three options for constraining the protein:

  • Residue side chains within N Å are flexible—Atoms in the side chains of residues that are within the specified distance of the ligand are allowed to move; all other atoms are held fixed, including the backbone atoms for residues whose side chains are flexible. Side chains within this distance are colored yellow; all other atoms are colored blue.
  • Fully flexible—All atoms in the protein are allowed to move.
  • Fully rigid—No atoms in the protein are allowed to move.
Ligands options

There are only two options, to make the ligands flexible, i.e. allow all atoms in the ligands to move, or make the ligands rigid, in which case all ligand atoms are kept fixed.

Cofactors options

There are only two options, to make the cofactors flexible, i.e. allow all atoms in the cofactors to move, or make the ligands rigid, in which case all cofactor atoms are kept fixed. Cofactors include metal atoms, counter ions, and so on.

This section completes the selection of options. When you click Create Model, a job is run to set up the files for the model system, and the final step is displayed when the job finishes.

Model Step

This "step" presents the results of the creation of the model system files. No action is required. You can examine the log file for the job that writes the files to ensure that the system is set up correctly and that there were no errors.

Display Options Section

This section contains options for the display of each of the structures in the model system. The controls depend on the model system and the calculation type. For most models, there is a list of options for each structure. For FEP calculations, there is an Animate Ligand Perturbation button, which animates the perturbation in the Workspace.
