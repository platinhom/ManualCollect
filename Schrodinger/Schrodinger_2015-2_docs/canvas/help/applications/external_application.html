Summary

In this panel you can set up a job to run an external application and incorporate the data into the Canvas project.

Opening the External Application Dialog Box

To open the External Application dialog box:

  • Choose Applications → External Application in the main window.

External Application Dialog Box Features

  • Data set option menu
  • Save results as text box
  • Previous jobs option menu
  • Load Settings button
  • Application text box and Browse button
  • Application input section
  • Application output section
  • Command text area
  • Contents of text area
  • Compute button
  • Stop button
Data set option menu

Select the source of the data. The data source can involve both row and column selection. The choices are:

  • Selected rows—Use the rows that are selected in the master view. You must select the rows before opening the dialog box.

  • Visible rows—Use the rows that are displayed (visible) in the master view. You must ensure that the required rows are visible before opening the dialog box.

  • Saved view—Use the rows and columns in a chosen saved view. When you select this item, an option menu that lists the saved views in the project is displayed to the right. You can choose the desired view from this option menu. A second option menu, labeled Domain, allows you to choose whether to restrict the rows or the columns, or both, to those of the saved view. Otherwise the visible rows or columns of the Master View are used.

  • Original data set for jobname—Use the rows and columns that were used for the specified job. This item is only available when you are cloning a job or viewing the results of a job.

Save results as text box

Assign a name to the results. This name is listed under External Applications in the Project View window.

Previous jobs option menu

Choose a previous run of an external application, so that the settings from this run can be loaded.

Load Settings button

Load the settings from a previous run of an external application, which you can choose from the Previous jobs option menu.

Application text box and Browse button

Specify the full path to the external application to run in the text box, or click Browse and navigate to the application.

Application input section

Choose the format and content of the application input.

Format option menu

Select a format for the structure or fingerpront input to the application, from Maestro, SD, SMILES+Properties, SMILES only, or Canvas FP file.

Filename text box

Specify the name of the input file. The extension should match one of the options on the Format option menu. The default is derived from the name given in the Save results as text box. When you change the file name, the command in the Command text area is updated to reflect the new name.

Properties lists

Select the properties that you want to include in the application input file from the Visible properties list and transfer them to the Available properties list for use in the application. The lists are available for all file types except SMILES+CanvasUID.

Application output section

In this section you can choose how to treat the results, and which file to display in the panel when the job is done.

Schedule for incorporation option and text box

Select this option if you want the results to be incorporated into the Canvas project, and specify the file name in the text box. It must be a CSV file. When you change the file name, the command in the Command text area is updated to reflect the new name.

The incorporation is done automatically if you select this option: no action in the Project View is needed. The file that is used for incorporation must be in CSV format and must include a heading row, the SMILES string for the structure in the first column, and the structure name (title) in the second column. The application must generate this file, which must also be included in the command to run the application. If the application is unable to process a structure, it must still produce a row of output for that structure with the correct number of commas but no values for the properties that cannot be computed.

Options button

Set options for the CSV file, including column headers and column number of SMILES strings for the structure and strucure names, if present.

Display contents of text box

Display the contents of the specified file in the Contents of text area when the job finishes.

Command text area

Specify the command to use to run the external application. The path to the application and the path to the specified input and output files can be omitted because these paths are already determined. A sample command is given in the text area, which includes the input and output files. You can modify this command to include everything that is needed to run the application. If you specify a file to display, it must be one of the output files of the command and must be specified as part of the command. If you include other files for input or output, the absolute path to these files must be given.

The command is run directly from Canvas, and does not use a shell (Linux or DOS) for running the command. The application must therefore process all the arguments you give for the command, and not use shell redirection for input or output. This means that the input and output files must be specified as arguments to the application.

Contents of text area

Display the contents of the file specified in the Application output section, when the job finishes. Contents are not monitored while the application is running.

Compute button

Start the job. The panel must remain open until the job finishes. If the name given for the results has been used, you are prompted to overwrite the existing results or cancel.

Stop button

Stop (kill) the job. When the job is running this is the only button that is active.
