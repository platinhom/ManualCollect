Summary

In this dialog box you can set up and run a job to create hashed binary fingerprints or structural keys for the input structures.

Opening the Binary Fingerprints from Structures Dialog Box

To open the Binary Fingerprints from Structures dialog box:

  • Choose Applications → Binary Fingerprints from Structures in the main window.

Binary Fingerprints from Structures Dialog Box Features

  • Data set option menu
  • New fingerprint column name text box
  • Fingerprint type options
  • Hashed fingerprint list
  • Advanced button
  • Structural key type options
  • Create button
Data set option menu

Select the source of the data. The data source can involve both row and column selection. The choices are:

  • Selected rows—Use the rows that are selected in the master view. You must select the rows before opening the dialog box.

  • Visible rows—Use the rows that are displayed (visible) in the master view. You must ensure that the required rows are visible before opening the dialog box.

  • Saved view—Use the rows and columns in a chosen saved view. When you select this item, an option menu that lists the saved views in the project is displayed to the right. You can choose the desired view from this option menu. A second option menu, labeled Domain, allows you to choose whether to restrict the rows or the columns, or both, to those of the saved view. Otherwise the visible rows or columns of the Master View are used.

  • Original data set for jobname—Use the rows and columns that were used for the specified job. This item is only available when you are cloning a job or viewing the results of a job.

New fingerprint column name text box

Enter a name for the new fingerprint column. This name is listed under Binary Fingerprints in the Project View window, and is used for the column header when a column is added to the spreadsheet with the bit count for the fingerprint. You must provide a name before you can run the job.

Fingerprint type options

Select the option for the type of fingerprint you want to generate. The two types available are Hashed and Structural keys.

Hashed fingerprint list

Select the type of hashed fingerprint you want to generate from this list. The available types are:

  • Linear—Linear fragments + ring closures.
  • Radial (ECFP)—Fragments that grow radially from each atom. Also known as extended connectivity fingerprints.
  • Dendritic—Linear and branched fragments.
  • MOLPRINT2D—A radial-like fingerprint that encodes atom environments using lists of atom types located at different toplogical distances.
  • Atom pairs—Pairs of atoms, differentiated by type, and the distance separating them.
  • Atom triplets—Triplets of atoms differentiated by type, and the three distances separating them.
  • quartet—Quartets of atoms differentiated by type, and the six distances separating them.
  • Topological torsions—Linear paths of length three.
Advanced button

Opens the Hashed Fingerprints - Advanced Options dialog box, in which you can set parameters for generation of hashed fingerprints.

Structural key type options

Select the type of structural key you want to generate, from MACCS or Custom. The MACCS keys are a SMARTS-based implementation. If you choose Custom, you must provide a file containing the SMARTS patterns that define the structural keys, by clicking Import SMARTS and navigating to the file. The file name is displayed in the Imported SMARTS file text box. Each line in the file must contain one SMARTS pattern, then the name of the key, separated by one or more spaces.

Create button

Run the job to create the fingreprints. Opens the Start Application dialog box, in which you can choose a host for the job, and select the number of CPUs to use to distribute the job tasks.
