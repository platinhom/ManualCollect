Summary

Search the structures in the current view for matches to a substructure.

Opening the Substructure Query Dialog Box

To open the Substructure Query dialog box:

  • Choose Query > Substructure Query in the current view.

Screen options

Select an option for the structures to search for the substructure. The options are:

  • Selected rows—search the rows that are selected in the current view. You must select the rows before opening this dialog box.
  • All rows—search all rows in the current view.
  • External—search all rows in the specified project. Click Select to choose the project. The project name is shown in the text box. The matching structures are imported from the external project into the current project.

Store results in options

Choose the view in which to store the results of the sort. If you choose Current View, the current view is updated with the new ordering of the rows. If you choose choose New View, the Save Custom View dialog box opens after the sort is done, and you can name, describe, and save the sort results as a new view.

Require exact match option

Require the query to match structures in the project exactly.

Invert (return non-matches) option

Apply the inverse of the query: return the structures that do not match rather than the structures that match.

Use 2D Index option

Use the 2D index for the structure to screen out structures that cannot match the query, before checking for a match. This feature speeds up the search. The 2D index is a special 512-bit fingerprint created for each structure on import. It can be used for pre-screening if the query is an exact SMARTS pattern; it cannot be used if the query contains SMARTS primitives that disqualify it as an exact substructure, such as [C,N], or [*].

Query source option menu

Choose the source of the query. The controls below this option menu depend on the choice made.

  • Builder—Use the 2D builder to create or edit a query structure.
  • SMARTS—Enter a SMARTS pattern to use as the query.

Enter SMARTS query list box

Enter the SMARTS query into the text box, or choose a previously used pattern from the menu (click the arrow button). If Require exact match is selected, the query must be a SMILES string for a particular structure. This control is displayed if you choose SMARTS from the Query source option menu.

Query file list box

Choose one of the recently used files from the menu (click the arrow button), or choose New Query to create a new query file. This control is displayed if you choose Builder from the Query source option menu.

Import button

Import a structure for the query from an external file, which must be in SD or MOL format. After the file is selected, you must provide a name, which is displayed in the Query File text box. This control is displayed if you choose Builder from the Query source option menu.

New or Edit button

Click this button to open the 2D structure-editing application. If the button is labeled Edit, the structure that you selected is loaded for editing; if it is labeled New, the editor is empty when it opens. When you close the editor, you are prompted to name the query file, and it is saved in the Canvas project. This control is displayed if you choose Builder from the Query source option menu.

Display area

A 2D representation of the query is shown in the display area when you import a structure or finish editing a structure.

Export button

Export the structure for a query to an external file.

Find Matches button

Click this button to find the structures that match the query. The view is updated to display only the rows for the matching structures.
