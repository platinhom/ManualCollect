Chemistry External Tool 1:2

This node allows users to run a user-specified external program that inputs one column and outputs two columns. Both the input and outputs can be of chemistry types, such as Maestro, Sequence, Alignment, Smiles, SD, Mol2, or PDB, or they can also be primitive types, such as a column of Strings, Integers, or Doubles. Return code for the external process needs to be zero or else the node will fail.

Note: for the Hypothesis and Glide Grid types, since each cell represents a group of files, the input and output variables (i.e., %input_1%, %output_1%, ...) represent text files that have a file prefix on each line. Therefore, the underlying script will parse these files to access the inputs and generate a text file with a list of prefixes on each line to send the output back into Knime.

Dialog Options

Include Input 1 in Output 1
All columns in the input table are included in the first output
Include Input 1 in Output 2
All columns in the input table are included in the second output
Column containing input
Choose the input column for the external program
Run command line for each input row
If this is checked, the command line is run for each input row, and for each row, there is one output row. Note: for Smiles, String, Double, and Integer output types, the Include Input option cannot be set. This is because there is no clear way to group the output of these types if there is more than one of output rows for each input.
Command line(s)
The command line(s) specified for the external tool. Environment variables can be specified by using a dollar sign ('$') before it.

A '%flow_n%' string, where n is 1,2, or 3, can be used to specify the flow variables flow_n.
Note: If the flow variable has a newline character in it, the node writes it to a file and the string gets converted to the file name. Otherwise, it gets converted to the value of the flow variable.

To specify the input file, use the string '%input_1%', and to specify the output files, use the strings '%output_1%' or '%output_2%'. For each row, the row id is available by using the string '%row_id%'. Note: Both output files, which are specified in the command line(s), need to be generated by the external tool or else this node will fail.
Output 1 Type
The first output type for this node. This is dependent on what type of file the external program generates.
Output 2 Type
The second output type for this node. This is dependent on what type of file the external program generates.
Output 1 Column Name
If selected, the text field allows the user to specify the first output column name
Output 2 Column Name
If selected, the text field allows the user to specify the second output column name

Ports

Input Ports
0 One column from this input data is used as an argument for the external process
Output Ports
0 The first output data from the external process (can also include the input, if specified)
1 The second output data from the external process (can also include the input, if specified)
