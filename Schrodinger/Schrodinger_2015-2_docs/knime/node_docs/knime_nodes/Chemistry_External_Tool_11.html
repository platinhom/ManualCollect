Chemistry External Tool 1:1

This node allows users to run a user-specified external program that inputs one column and outputs one column. Both the input and output can be of types: Maestro, Sequence, Alignment, SD, Mol2, Smiles, String, Double, Integer, Text, FingerPrint, Canvas Matrix, Hypothesis or Glide Grid. Return code for the external process needs to be zero or else the node will fail.

Note: for the Hypothesis and Glide Grid types, since each cell represents a group of files, the input and output variables (i.e., %input_1%, %output_1%) represent text files that have a file prefix on each line. Therefore, the underlying script will parse these files to access the inputs and generate a text file with a list of prefixes on each line to send the output back into Knime.

Dialog Options

Include Input
All columns in the input table are included in the output
Column containing input
Choose the input column for the external program
Run command line for each input row
If this is checked, the command line is run for each input row, and for each row, there is one output row. Note: for Smiles, String, Double, and Integer output types, the Include Input option cannot be set. This is because there is no clear way to group the output of these types if there is more than one of output rows for each input.
Command line(s)
The command line(s) specified for the external tool. Environment variables can be specified by using a dollar sign ('$') before it.

A '%flow_n%' string, where n is 1,2, or 3, can be used to specify the flow variables flow_n.
Note: If the flow variable has a newline character in it, the node writes it to a file and the string gets converted to the file name. Otherwise, it gets converted to the value of the flow variable.

To specify the input file, use the string '%input_1%', and to specify the output file, use the string '%output_1%'. For each row, the row id is available by using the string '%row_id%'. Note: The output file, which is specified in the command line(s), needs to be generated by the external tool or else this node will fail.
Output Type
The type of output for this node. This is dependent on what type of file the external program generates.
Output Column Name
If selected, the text field allows the user to specify the output column name

Ports

Input Ports
0 One column from this input data is used as an argument for the external process
Output Ports
0 Output data from the external process (can also include the input, if specified)
