Setting Job Options and Starting the Property Optimization Job

The optimization involves running a set of Jaguar jobs on each structure in each generation. You can set keywords for the Jaguar calculations in the Jaguar keywords box, if you want to use something other than the defaults. A geometry optimization is performed on each structure, so you may want to consider including optimization-related keywords if you encounter failures.

When you have made all the settings you want, enter a job name in the Job name text box, and click Run to start the job immediately, or click the Settings button to make job settings first.

If you want to run jobs simultaneously or use multiple threads for each Jaguar job, you should click the Settings button to choose a multiprocessor host for your job, and set the number of subjobs and threads. You should not set the number of jobs to more than the number of individuals (the population size). If you have more processors available than the population size, setting the number of threads per individual to a value greater than 1 will give faster turnaround. If not, it is more efficient to set the number of simultaneous jobs as high as possible.
