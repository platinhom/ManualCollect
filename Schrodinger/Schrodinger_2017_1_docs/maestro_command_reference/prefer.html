The Maestro input/output directory is used for starting jobs and for other file input and output. The i/o directory is always displayed in the title bar of the Main Application Window. The initial cwd is the directory in which Maestro was started. Changing projects does not affect the cwd. If "currentdir" is in effect than the i/o directory will be changed every time the cwd changes or when initworkdir is set to "currentdir" (same as "startdir"). When the i/o directory is set to currentdir Maestro will attempt to read/write files from the cwd and will place job files in the cwd. When the i/o directory is set to "project", "projectparent", or "projectjobs", the i/o directory is changed when the "initworkdir" option is set, and again whenever the current project directory is changed. In these cases Maestro sets the i/o directory to the directory of the project, the parent directory of the project's directory, or the "jobs" subdirectory of the project, respectively.

workspacemultiplepickingpass

If true, workspace rendering for picking is performed using multiple pass, otherwise with single pass. This option actually defines the object rendering pass for picking. Some graphics card does not do accurate picking if workspace objects rendering is performed in single pass, so Maestro supports both single pass and multiple pass picking. Single pass picking is fast, but may not be accurate on some graphics cards. Multiple pass is accurate, but it is slower. For 24-bit color-depth, in multiple pass, there will be 2 pass to represent objects in the range of 1 - 2^24-1. For 16-bit color-depth, it is always multiple pass as all objects can not be rendered in single pass. If single pass selected, then there will be 2 pass of rendering for picking. If multiple pass selected, then there will be 3 pass of rendering for picking.

Valid values: boolean (true|false; yes|no; y|n; on|off)
