Using LigPrep

Although LigPrep contains a variety of options, you will probably use only a subset of them regularly. The options you choose will depend on your goals, the types of input structures you are processing, and the types of subsequent processing that will be carried out. For a complete list of LigPrep options, see The ligprep Command. A general description of how LigPrep processes structures is provided in The LigPrep Process.

In general, we recommend limiting the size of the input structure file to roughly one million structures. When running jobs with many input structures it can be helpful to automatically split up and run many smaller LigPrep jobs across a number of processors, which you can do by distributing the job. The results are recombined into one large output file for the whole job. See Submitting the LigPrep Job for more information.

Some LigPrep stages do not adjust structures with more than 200 atoms, to avoid spending excessive processor time on molecules that are atypical for ligands. This limit is adjustable using the -ma option from the command line. However, increasing it significantly will result in the processing of ligands larger than LigPrep is tested for, and you can encounter problems, including much slower processing of structures. For information on file size issues, see LigPrep Limitations.
