Prime Structure Prediction — Input Sequence Step

This step is used to read in protein (amino acid) sequences and designate the query sequence.

For general information on the panel, see the topic Structure Prediction Panel.

Using the Input Sequence Step

The purpose of this step is to define a query sequence. You can read the sequence from a sequence file, add it as text, obtain it from the PDB, or obtain it from the Workspace. Several common sequence file formats are supported, including FASTA, EMBL, GENBANK, and PIR. Sequences can be read from PBB files, from the SEQRES records, if present, or from the ATOM records if SEQRES records are missing. You can add more than one sequence, but only one can be used as a query. If you want to build a multimer, you can do so by combining information from separate runs for each chain in the Build Structure step.

To read a sequence from a file:

  • Click File and specify a file containing the desired sequence.

    Several file formats are supported, including FASTA, EMBL, GENBANK, PDB, and PIR. For a complete list, see Appendix C of the Prime User Manual. File formats are auto-detected by the program. Sequences must be in standard uppercase one-letter code format.

To read a sequence from the PDB:

  • Click PDB ID and specify the PDB ID and optionally the chain name.

    The PDB file is downloaded and the sequences are read in.

To enter a sequence as text:

  • Click Text and type in the sequence or paste it from the clipboard.

    If there are multiple chains with the same sequence, the sequence will be read in only once.

To read a sequence from the Workspace:

  • Click Workspace.

    All sequences represented in the Workspace are added to the Sequences table. If there are multiple chains with the same sequence, the sequence is read in only once.

Once you have obtained one or more sequences, you can select the sequence you want to use for the query from the Sequences list. By default, the first sequence read in is selected and displayed in the sequence viewer in the upper section of the panel.

If you want to modify the sequence you have selected as a query, click Edit Sequence and make any changes you want in the dialog box that opens. You might, for example, want to remove extra terminal residues or insertions, or mutate the sequence.

Sequences longer than 1,000 residues generally include more than one domain. You should consider dividing these sequences into probable domains and treat each domain as an individual query.

When you are satisfied with the query sequence, click Next to go to the Find Homologs step.

Error Messages

If one of these messages appears when you click Next, you will not be able to proceed until the problem is resolved.

Invalid File Message

This message is shown when a valid sequence cannot be extracted from the file.

  • Check that the sequence is in one of the supported sequence file formats.
  • If it is in FASTA format, check that there are no extra spaces between the ">" and the sequence name and that there are no extra spaces at the end of the line.
No Sequences Found

This message is shown when no valid sequences can be extracted from the Maestro Workspace.

  • Make sure that the correct project is open.
  • Check the Project Table to make sure the correct entries are included in the Workspace.
  • Display the Workspace sequence viewer to check that the structure has a sequence.
Invalid Sequence Characters

The sequence contains nonstandard one-letter codes.

  • Check that the sequence is in one-letter code and is all in upper case.
  • Change or edit out any nonstandard characters.

Input Sequence Step Features

  • Get sequence from buttons
  • Sequences table
  • Edit Sequence button
Get sequence from buttons

Use these buttons to get sequences from various sources.

  • File—Read the sequence from a file. Several file formats are supported, including PDB, FASTA, EMBL, GENBANK, and PIR. File formats are auto-detected by the program. Sequences from PDB files are read from the SEQRES records, if they exist; otherwise they are read from the ATOM records. If SEQRES records do not exist and residues are missing from the ATOM records, the query sequence will contain gaps. For other file formats, sequences must be in standard uppercase one-letter code.

  • Text button—Enter the sequence as text. Opens the Get Sequence from Text dialog box, in which you can type in the sequence, or paste it from the clipboard with Ctrl+V (⌘V).

  • PDB ID—Obtain sequences from the Protein Data Bank (PDB). Opens the Get Sequence from PDB ID dialog box, in which you can enter the PDB ID and optional chain name. The PDB file is downloaded and the sequence read from the file, as described above.

  • Workspace—Read sequences from proteins in the Workspace. If there are duplicates, the sequence is read in only once.

Sequences table

This table lists the sequences that have been added to the run, in the order they were read in, with their chain lengths. You can select a single sequence as the query. This sequence is displayed in the sequence viewer.

Edit Sequence button

Edit the sequence that is selected in the Sequences table. Opens a panel in which you can edit the sequence as a text string.
