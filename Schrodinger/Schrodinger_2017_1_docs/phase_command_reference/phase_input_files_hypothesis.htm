Phase Hypothesis File

This file contains all the information that belongs to a single hypothesis, and has a .phypo extension. The file is in gzipped Maestro format, so it can be easily imported to and exported from Maestro. The file contains a hypothesis entry (CT), and may contain a reference ligand, and a set of additional structures that are aligned to the hypothesis. All the information that defines the hypothesis—feature definitions, radii, and rules; excluded and included volumes; matching and positional tolerances; sites, projected sites, site masks, site types; constraints; QSAR models—is stored as Maestro string properties, encoded as Base64 strings. This means that the hypothesis file is not human-readable, and it is not intended to be edited directly.

This file format supersedes the use of multiple plain-text files that define a hypothesis. You can convert between the single-file and the multi-file format with the utility phase_hypo_util. This also allows you to unpack the hypothesis, edit it, and repack it into the single-file format.
