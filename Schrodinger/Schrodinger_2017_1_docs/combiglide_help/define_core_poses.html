CombiGlide — Define Core Poses Step

In this step you define the core poses that will be used for the library docking. These poses can be generated from a Glide XP docking run with molecules that include the core, or from entries in the Project Table that are already docked to the receptor.

Using the Define Core Poses Step

The core poses are defined by docking a small set of molecules to the receptor. Because Glide constraints are not applied, this step also provides some means of constraining the motion of the core molecule during docking.

If you want to dock a set of core-containing molecules to the receptor without constraints, select Unconstrained within bounding box in the Pose definition mode section. Once the poses are docked they are clustered, and the best pose from each cluster is selected. This mode generally provides the best sampling of the possible binding modes.

The structures used to define the core poses can be smaller than the original core molecule, and therefore they might bind to locations in the active site that are not accessible to the original. Many of these poses could be unreasonable, and would be eliminated during the library docking. Although you can't use Glide constraints for the core docking, you can constrain the center-of-mass of the core to a sphere, by selecting Spatially constrained center of mass. The center can be set by clicking Define, and using the picking tools in the Define Core Constraint dialog box to pick atoms whose centroid is used to define the sphere center. The radius of the sphere is 1 Å by default, and can be set in the Define Core Constraint dialog box.

Another way of defining the poses of the core structure is to import structures that are already docked to the receptor into the CombiGlide run, and select Constrained by initial core positions in the CombiGlide docking mode section. These structures must contain the core (the part that is left when you break the grow bonds and remove the side chains). To import the structures, you add them to the Project Table, then click From Project and select the relevant entries. The positions of the cores defined by these structures will be varied during the library docking step, but the RMS deviation of the core atoms from the defined pose is limited by the value specified in the Maximum RMSD of core text box.

Define Core Poses Step Features

  • CombiGlide docking mode options
  • Display Receptor button
  • Maximum RMSD of core text box
  • Core structures table
  • Delete button
  • Add buttons

CombiGlide Docking Mode Options

These options allow you to choose how the core poses are defined.

  • Unconstrained within bounding box option
  • Spatially constrained center-of-mass option
  • Filter by initial core positions option
  • Apply Glide core constraints option

Unconstrained within bounding box option

Dock the core structures without any constraints other than that the structure must remain within the bounding box.

Spatially constrained center-of-mass option.

Dock the core structures with the center-of-mass of the core constrained to a sphere. To define the sphere, click Define. This button opens the Define Core Constraint dialog box, in which you can pick atoms to define the center of the sphere and set the sphere radius. The sphere radius must not extend beyond the enclosing box of the grid.

Filter by initial core positions option

Use the positions of the supplied core structures to constrain the core poses. The structures must be correctly aligned to the receptor.

Apply Glide core constraints option

Use Glide core constraints to constrain motion of core.

Display Receptor Option

Displays the receptor used to define the grid that was selected in the previous step. Select or deselect to display or undisplay the receptor.

Maximum RMSD of Core Text Box

The Maximum RMSD of core text box specifies the maximum RMS deviation of the core atoms from their initial location during docking. Core poses with higher RMSD are discarded.

Core Structures Table

Lists the structures that are used to define the core poses. The In column is used to include the molecule in the Workspace, in the same way as in the Project Table. The Title column is the title of the molecule, and is editable. You can select multiple rows in the table.

Delete button

Deletes the selected core molecules from the table.

Add Buttons

These buttons allow you to add structures to the Core Structures table.

From Project button

Opens an entry chooser so that you can import molecules for the core poses from the Project Table. The entries must satisfy the following criteria:

  • They must contain only a single molecule
  • The molecule must be inside the grid enclosing box
  • The molecule must contain the core structure

No checking is done for duplicates, so you must ensure that you don't add the same structure more than once. The structures are copied into the run, so that any changes in the Project Table do not affect these structures.

User-capped Core button

Adds the original molecule used to define the core (selected in the Define Combinations step) to the table. Unavailable if the original core molecule is already in the table.

Minimally Capped Core button

Adds the minimally capped core to the table. Unavailable if the minimally capped core molecule is already in the table.
