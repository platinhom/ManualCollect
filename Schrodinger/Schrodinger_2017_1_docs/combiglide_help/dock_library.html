CombiGlide — Dock Library Step

In this step, you run the jobs that perform the library docking. You can also export the best reagents and the top structures to the Project Table or to a file.

Using the Dock Library Step

This step is the heart of the combinatorial screening process, where the docking jobs are performed to screen out reagents and molecules that do not dock well. The screening is performed in two stages. In the first stage, the attachments defined by the reagents are added, one at a time, to the minimally capped core. The resultant molecules, each of which has an attachment at a single position, are docked, and molecules that do not dock well are eliminated. The attachments added are taken from the expanded set of reagents, in which different ionization states, conformers, or tautomers can be represented.

The reduced set of attachments for each position is then combined and scored with an intermediate scoring function, and the top scoring molecules with all attachments in place are then docked. This second stage can result in the elimination of further reagents.

The two stages can be run in the same job, or run separately. The choice of how to run the stages is made in the Dock dialog box, which opens when you click Dock.

As the processing of each set of reagents finishes, the results are posted in the Single position results table. If QikProp is available, it is run for the completed structures, and the QikProp properties are added to the run.

Once the results are available, you can add them to the Project Table or export them to a file. The structures from both stages are sorted by their GlideScores. Singly-substituted structures or completed structures are added to the Project Table as entry groups, with QikProp properties if these are available. For the singly-substituted structures, all poses from the selected number of reagents are exported.

To add or export a set of singly-substituted structures:

  1. Select the table row for the reagent type.
  2. Enter the number of reagents in the Top n reagents text box.
  3. (Optional) If you want to include the receptor, select Include receptor.
  4. Click Add to Project Table or Export.
  5. If you are exporting to a file, navigate to the directory, enter the file name, and click Export.

To add or export a set of completed molecules:

  1. Enter the number of structures in the Top n structures text box.
  2. (Optional) If you want to include the receptor, select Include receptor.
  3. Click Add to Project Table or Export.
  4. If you are exporting to a file, navigate to the directory, enter the file name, and click Export.

If you have only one attachment, the normal workflow is not relevant because there are no combinations to be made. Instead, you can run an "enumerate and dock" job, in which the entire library is enumerated and docked.

Dock Library Step Features

  • Dock and Cancel buttons
  • Reagent retention table
  • Single position results section
  • Combinatorial results section

Dock and Cancel Buttons

These buttons run or cancel the docking jobs that result in the combinatorial library. The Dock button opens the Dock dialog box, in which you can set up the type of docking job and select options for that job. Clicking Start in the Dock dialog box starts the job.

The Cancel button cancels the current docking job, on clicking OK in a confirmation dialog box. If you are running the combinatorial docking job, and the single-positions job has already finished, the results of the single-positions job are retained.

Reagent retention table

This table presents statistics on the reagents that survive the various docking stages. The table is noneditable. You can select a single row for exporting reagents. The table columns are as follows:

Name   Attachment position label
Reagent File   Name (minus extension) of the file that contains the reagents for the specified position.
Reagents   Number of reagents in the original reagent file.
Single   Number of reagents from this file that survived the single-side-chain analysis. This column is updated as the single-position docking jobs finish.
Comb   Number of reagents from this file that survived the combinatorial screening. This number includes all reagents that contributed to any structure that docked well.

Single Position Results Section

This section presents results of the single-position docking stage and provides tools to export the docked structures. The name and reagent file for the position selected in the Reagent retention table is displayed at the top of the section.

  • Top n structures text box
  • Add to Project Table button
  • Export button
  • Include receptor option

Top n structures text box

Specify the number of structures to export. The structures are ordered by GlideScore. Because there may be variations of each reagent (such as different ionization states, tautomers, conformers), it is the top-scoring variation for each reagent that is exported.

Add to Project Table button

Add structures from the selected table row to the Project Table as an entry group. The number of structures to export is specified in the Top n structures text box. If the receptor is included, it is the first entry in the entry group. All surviving poses for the given reagent or reagent combination are added to the Project Table.

Export button

Export reagents from the selected table row to a Maestro file. The number of reagents is specified in the Top n reagents text box. If the receptor is included, it is the first structure in the file. Opens a file selector, in which you can navigate to a directory and specify a file name.

Include receptor option

When selected, the receptor is included as the first structure exported, followed by the reagents.

Combinatorial Results Section

This section reports the number of structures actually generated in the combinatorial library, and provides tools to export the structures. The source of the combinatorial results is displayed at the top of the section.

  • Number of final structures text box
  • Top n structures text box
  • Export to Project button
  • Export to File button
  • Include receptor option

Number of final structures text box

Reports the number of structures actually generated. This number can be less than the product of the numbers in the # Comb column of the Reagent retention table, because not all combinations of reagents necessarily produce structures that dock well.

Top n structures text box

Specify the number of structures to export. The structures are ordered by GlideScore.

Export to Project button

Export the number of structures specified in the Top n structures text box to the Project Table as an entry group. If the receptor is included, it is the first entry in the entry group.

Export to File button

Export the number of structures specified in the Top n structures text box to a Maestro file. If the receptor is included, it is the first structure in the file. Opens a file selector, in which you can navigate to a directory and specify a file name.

Include receptor option

When selected, the receptor is included as the first structure exported, followed by the reagents.
