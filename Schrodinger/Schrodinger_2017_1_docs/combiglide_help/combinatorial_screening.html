Combinatorial Screening Panel

Create a small combinatorial library of compounds that should have high activity towards to a selected target. The library is screened, first by docking to the target receptor and eliminating compounds that do not dock well, then by analyzing the docked compounds to define a small reagent set that is likely to have a large number of actives among the compounds that are generated.

To open this panel, choose Tasks → Ligand Preparation and Library Design → Library Design → Combinatorial Screening in the Task Tool.

Using the Combinatorial Screening Panel

Each set of data defined in this panel is called a run. The data for each run is stored as a separate entity, which you can open from the File menu. You can also import runs from this menu.

The tasks involved in each step are summarized below.

The Define Combinations Step

In this step, you select the molecule that you want to use for generation of a library and screening the library by docking it to a receptor. The library is generated by replacing selected side chains in the molecule with other side chains. These side chains are called attachments. For each attachment, you identify the bond that defines where the substitution will take place, and specify a reagent library that contains the side chains you want to substitute. The bond is called the grow bond. The core of the molecule is the part of the molecule that is left when you remove the attachments. The library must be prepared beforehand (using the Reagent Preparation panel.

Once you have selected a molecule and the attachment positions for the reagents, you can proceed to the next step.

The Configure Docking Step

The screening process uses Glide XP docking to filter out reagents that produce molecules that do not score well. This step assumes that you have already generated a Glide grid, using the Receptor Grid Generation panel. In this step, you set options for the docking of the libraries to the receptor. As well as selecting a grid and making basic settings as for a Glide run, you can specify the full range of Glide constraints for the docked molecules.

After selecting a grid and setting any desired options, you can proceed to the next step.

The Define Core Poses Step

In this step, you select the core structures that you want to use in the docking step to define the core poses. These poses are generated by docking the molecules that contain the core. The molecule can be the core molecule, the minimally capped core, or some other molecule that you choose. The minimally capped core is the core of the molecule with a minimal attachment at each attachment position. In most cases, the minimal attachment is a methyl group. You can constrain the core position to prevent it from moving to regions of the receptor that would generate unacceptable binding modes.

When you have selected or generated the core poses, you can proceed to the next step.

The Dock Library Step

In this step you run the jobs that generate the combinatorial libraries and run the docking jobs. The docking is performed in two passes. In the first pass, each attachment from each reagent library is added in turn to the core, to produce molecules with a single attachment and minimal capping groups at the remaining attachment points. These molecules are docked using Glide XP docking. Reagents that generate bad poses are eliminated. This pass can be run independently.

In the second pass, the poses with a single attachment are combined intelligently to produce molecules that are likely to dock well. These molecules have attachments from the reagent libraries at all attachment points. This pass is the most time-consuming part.

Once you have docked the combinatorial library, you can proceed to the next step. You can also export the results of either the single-position docking or the combinatorial docking for further examination.

The Analyze Library Step

In this step, the number of reagents used at each position is reduced to a small set by application of filtering and selection strategies. Filtering is based on structural properties such as molecular weight and counts of structural features, and on properties generated by QikProp. The selection strategies locate the "best" reagents at each position, based on the GlideScore of each of the docked structures.

Combinatorial Screening Panel Features

The Combinatorial Screening panel is wizard-like in design, with five steps. Each step occupies the center of the panel, and consists of a title with a brief description of the step at the top, a set of controls and tables for results, and a Back and a Next button at the bottom. The features of each step are described in detail in separate help topics - see the Related Topics section below.

In addition to the step features, the panel contains a menu bar and an job status button at the top, and a step guide at the bottom above the Close and Help buttons. These features are described below.

  • The File menu
  • The Step menu
  • The job status button
  • The Guide

The File menu

The File menu allows you to work with the runs that are available in the project.

New Create a new run
Open Open an existing run from the submenu. If there are more than 4 runs, choose More to open a dialog box and select a run.
Save As     Save the current run with a new name
Rename Rename the current run
Delete Delete the current run
Import Import a combinatorial screening run that was initiated from the command line. Prompts for a run name, then opens a directory selector, in which you can navigate to the directory that contains the output files from the run, and select the run. The panel is then positioned at the Dock Library step, and you cannot go back to previous steps.

The Step menu

The Step menu contains an item to display or hide the Guide, and items for each of the steps. The current step is marked with a red diamond. If the Guide is displayed, it is marked with a red square. The items for the steps that are not available are dimmed. You can go to any available step by choosing the corresponding menu item.

The job status button

The icon on the job status button at the upper right of the panel spins when a job is running. When the job has finished, the icon turns to an exclamation point while the job is incorporating, then returns to its original state when the incorporation has finished. You can click the button to open the Monitor panel and monitor the job.

The Guide

The Guide displays the steps in the model as a set of buttons linked by lines. The buttons for the steps that are not available are dimmed. The current step is highlighted with a white background. You can go to any available step by clicking its button in the Guide. The Guide can be displayed or hidden from the Step menu.
