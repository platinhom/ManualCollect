Rerunning or Recovering Jaguar Jobs from the Command Line

The jaguar run command lets you rerun a Jaguar job using the Jaguar recovery file (jobname.recover) that you specify. The recovery file contains all the information needed to rerun the job exactly as it was originally run, so no other command options are needed. This information includes the installation, host, and other command-line options, as well as the input file and any other information needed to reproduce the job. The recovery file is written automatically and copied back to the directory the job was launched from.

The main use for this command is to rerun a failed job with exactly the same command as it was run with originally.

If a job fails, for example due to network issues or problems with the local or remote host, you can rerun it with this command. For simple jobs, the job is rerun from the beginning. For complex jobs that consist of multiple subjobs, Jaguar checks which subjobs finished successfully, and reruns only the failed subjobs. These subjobs are run from the beginning: intermediate results are not used (nor are the Jaguar restart files). The results of any successful jobs are used in the workflow, however. This allows you to recover a partially completed job and bring it to completion. Recovering a failed complex job from where it failed is available for jobs that ran the following scripts:

  • canonical.py
  • pka.py
  • deltah.py
  • distributed_scan.py
  • h_abstraction.py
  • jreactions.py

These scripts are described in later sections—see Running Complex Jaguar Jobs with Python Scripts.

Before a job is rerun, the job record and the Jaguar output files are checked. If the job is reported as still running or queued, or if the output indicates that it finished successfully, the job is not rerun. You can, however, rerun a successful job by copying the recovery file to a new location, or remove the output files from the successful job.
