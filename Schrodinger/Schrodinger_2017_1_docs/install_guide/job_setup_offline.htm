Configuring Offline Job Management

When a user submits a job from a host such as a laptop and the host is taken offline, a staging area for the job output is needed until the host is online again. This is done by defining a job server host, which is simply a host that has a designated nonvolatile file system where the user's job output can be stored until the user requires it. The job server host must be defined in the host entry for each execution host for which this service is needed. Some configuration is needed on the job server host and also on the job submission host (the client). These tasks are described in the topics listed below.

Note: Job submission that makes use of the job server is currently only available from Maestro.
