Screening Ligand Files using Feature Definition Presets

In this exercise we will screen a collection of compounds without conformers against two pharmacophore hypotheses and analyze the results. We will employ feature definition presets to broaden chemistry that will match sites in the hypothesis without having to change feature definitions.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file at1_dcph_phypo.mae.gz.

    This file contains multiple hypotheses from a develop common pharmacophore hypotheses experiment.

  6. Click Open.

    The Import dialog box closes. The first imported hypothesis is displayed in the Workspace.

  7. Click the Tasks button, type “phase” and press Enter.

  8. Select Ligand and Database Screening from the results list.

    The Phase Ligand Screening panel opens.

  9. Select the DHNRRR_1 and ADHNRR_3 groups by control-clicking (command-clicking) on their group entries.

  10. In the Phase Ligand Screening panel, click  Add Hypothesis and choose Selected entries(Project Table).

    The two selected hypotheses are added as rows in the Phase Ligand Screening panel.

  11. Change Ligands to screen to File and browse to the file at1_test.maegz.

    The panel indicates that 24 ligands are in the file to be screened.

  12. Change Hits must match at least N features to 5 and ensure that Prefer partial matching of more features is selected.

  13. Click Screening Settings to open the Screening Settings panel.

  14. In the Conformers tab choose Generate conformers during search, since the input ligand structures do not have conformers. Leave the target number of conformers at 50.

  15. In the Features tab select all three presets.

    This will treat hydrophobes and rings as equivalent, acceptor and negative features as equivalent and donor and positive features as equivalent. This makes the hypotheses less selective. Leave options for other tabs at their default values.

  16. Click the Save button to close the Screening Settings panel.

  17. Change the Job name to "at1_screen" and click Run to start the job.

    The job will take a few minutes on a single CPU.

    Matches to the two hypotheses are imported into the Project table as separate groups. In the at1_screen group there are two subgroups, ADHNRR_3 with results from screening against that hypothesis and DHNRRR_1 with results from screening against that hypothesis.

  18. Double click the In column for the ADHNRR_3 hypothesis that was screened against to include and fix it in the Workspace.

  19. Click the In column for the top-ranked screening hit, test09, to include it in the Workspace.

    Note the Negative ionic N8 feature matches a ligand aldehyde which is the case since we used the feature definition preset to treat negative features and acceptor features as equivalent.

  20. Ensure all ligands in the at1_screen-hits group are selected by clicking the at1_screen-hits group entry.

  21. Use the right arrow key to scroll through ligands quickly to view how well ligand groups match the pharmacophore hypothesis.

    Also note that ligand van der Waals radii do not overlap with the excluded volumes that were included in the screen. As in the previous exercise you can open the Project Table and view the matched ligand sites and PhaseScreenScore properties to get an at-a-glance view of matching features and the quality of the matches, where larger PhaseScreenScore indicates a better match to the hypothesis.
