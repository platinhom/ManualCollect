Managing Jobs

The Job Control facility provides tools for monitoring and controlling the jobs that it runs. These tools have both a graphical user interface in the Maestro Monitor panel and a command-line interface in the jobcontrol command. The common tasks can be performed from the Monitor panel, but the full range of job control tasks is only available from the command line.

The job information is loaded from a job database that is set up for each user, and uses this information to perform various tasks, like providing job status and killing jobs.

You can monitor jobs that are run locally, run remotely, or run in a batch queue. Certain kinds of subjobs cannot be monitored: if they are launched from a master job that runs on a remote host, and the submission directory on the remote host is not mounted on the local host, the information is unavailable.

The topics listed below describe the job database, the jobcontrol command, and some utilities. For information on the Monitor panel, see the help topic for the panel, Monitor Panel.
