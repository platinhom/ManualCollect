Analyzing a QSAR Model for Binding Affinity

In this exercise we will analyze the QSAR models created by AutoQSAR for binding affinity of a FactorXa congeneric series.

  1. Open the Task Tool, type autoqsar in the Search box, and press Enter.

  2. Choose Other Applications > AutoQSAR to open the AutoQSAR panel.

  3. Select View model and make prediction.

  4. From the option menu choose factorXa_activity_model.qzip.

    If the previous exercise to create the QSAR model was not performed in the current Maestro session that option will not be available. You can instead import factorXa_activity_model.qzip into the AutoQSAR panel by clicking Browse and importing it manually.

  5. In the Model Report section click the + button to display more information about the QSAR models that were created. Click Show More to show all key information about the models.

    Note that the 10 top-ranked QSAR models have good R2 and Q2 and small Null Hypothesis Q2 values. This indicates that the Binding Affinity property doesn’t correlate well with ligand molecular weight.

  6. Select the first row with Model Code value of kpls_molprint2D_38.

    The title indicates that the QSAR model was generated by KPLS fitting with molprint2D fingerprints, using the 38th split of the learning set into a test and training set.

  7. Click the Report Details button to open a report containing details of the KPLS model.

  8. Click Scatter Plot to show a scatter plot of this QSAR model’s performance in predicting experimental binding affinity for the learning set.
