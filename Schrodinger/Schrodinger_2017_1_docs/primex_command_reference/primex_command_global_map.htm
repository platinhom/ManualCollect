PrimeX Keywords for Map Settings

These parameters describe both the extent of the map itself and some aspects of the model and method used to create it. For jobs that produce maps, multiple maps can be specified and the keywords MAP_COEFF_0, MAP_COEFF_1, and so on, can be used, each with a single value, such as 1_1 for an Fo-Fc (difference) map.

Keywords for map settings are given in Table 1.

Table 1. Keywords for map settings

Keyword

Description

BOX_COORD

Set the coordinate type as either a fraction of the unit cell, or Cartesian. Required only if MAP_EXTENT is set to box. Allowed values: frac, cart.

GRID_SIZE

Fraction of the resolution to use for the grid spacing.

KICK_DIST

Maximum displacement to use in a kicked omit map. Values over 1 Å are not recommended. Used only if OMIT_TYPE is set to kicked. Default: 0.1 Å.

KICK_NUM

Number of kicked omit maps to average. Used only if OMIT_TYPE is set to kicked. Default: 10.

MAP_BOX_i_j

Minimum and maximum coordinate values of map box. i can take values X, Y, and Z, and j can take values MIN and MAX, e.g. MAP_BOX_X_MIN. Required only if MAP_EXTENT is set to box.

MAP_BUFFER

Size of buffer around selected molecule, in angstroms. Required only if MAP_EXTENT is set to mol. Default: 5.0.

MAP_COEFF_n

Map coefficients for map n, where n is a serial number starting from 0. The value of the keyword takes the form i_j, where i is the coefficient of Fo and j is the coefficient of Fc: for example, Fo-Fc is 1_1.

MAP_COEFFA_n

Name of column for amplitudes for map n. Only valid when MAP_FORM is set to mtz-refl. Commonly, when MAP_COEFF_n is 1_0 the value is FOWT, for 1_1 it is DELFWT, for 2_1 it is FWT, and for 3_2 it is 3FO2FCWT.

MAP_COEFFP_n

Name of column for phases for map n. Only valid when MAP_FORM is set to mtz-refl. Commonly, when MAP_COEFF_n is 1_0 the value is PHFOWT, for 1_1 it is PHDELWT, for 2_1 it is PHFWT, and for 3_2 it is PH3FO2FCWT.

MAP_EXTENT

Determines how the map extent is defined. Allowed values:

asym

Asymmetric unit

unit

Unit cell

box

Box of given dimensions, either in (a,b,c) or (x,y,z).

mol

Model structure plus buffer zone, specified by MAP_BUFFER.

MAP_FORM

Map output format. Allowed values:

cns-form

CNS formatted (default)

cns-bin

CNS unformatted

cns-refl

CNS phased reflections (not valid with omit maps)

ccp4

CCP4 unformatted

mtz-refl

MTZ phased refelctions

MAP_TYPE

Type of map—regular map, omit map, or composite omit map. Allowed values: reg, omit, comp. Atom selection data keywords (see Table 1) are required if MAP_TYPE is set to omit. If omitted, no map is generated.

MAP_WEIGHT

Weighting model. Allowed values: sigma, unweighted.

OMIT_DATA

Percentage of data to omit. Used only if MAP_TYPE is set to comp.

OMIT_TYPE

Type of omit map to generate. Allowed values: bhat-cohen, kicked. Used only if MAP_TYPE is set to omit or comp. Default: bhat-cohen.

SCALE_MAP

Scale map to sigma units. Allowed values: yes, no. Default: no.
