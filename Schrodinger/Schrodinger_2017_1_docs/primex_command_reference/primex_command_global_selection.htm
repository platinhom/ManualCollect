PrimeX Atom Selection Keywords

These keywords can be used to specify an atom selection, where the interface uses picking tools. The keywords are listed in Table 1.

Table 1. Keywords for specifying atom selections

Keyword

Values

ATOM_FILE

Name of file containing a list of atoms. Each line in the file must be of the format chainID:resnum:atomID, as described for the ATOM_i keyword.

ATOM_i

Atom specification. The format of this specification is chainID:resnum:atomID, where chainID is the single-letter chain identifier, resnum is the residue number, and atomID is the PDB atom name, with underscores replacing blanks—for example, A:162:_C__. i is an index that starts from zero.

RESIDUE_FILE

Name of file containing a list of residues. Each line in the file must be of the format chainID:resnum, as described for the RESIDUE_i keyword.

RESIDUE_i

Residue specification. The format of this specification is chainID:resnum, where chainID is the single-letter chain identifier, and resnum is the residue number—for example, A:132. i is an index that starts from zero.

SELECT

Atom selection type. If an atom selection is required by another keyword, this keyword is required. Allowed values: all(select all atoms), pick (pick atoms to use). If set to pick, exactly one of the other types of ASL data keywords should be used.
