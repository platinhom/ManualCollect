Quit Maestro Panel

The Quit Maestro panel allows you to save the commands generated in the current Maestro session to a log file and to save the panel layout.

To open this panel, choose File → Quit

Using the Quit Maestro Panel

The Quit Maestro panel serves three functions. The first is to confirm that you really intend to quit the program. If you do, want to quit Maestro, click Quit at the bottom of the panel. Otherwise, click Cancel.

The second function is to present you with the option of saving a log of all the commands that you have issued during the current Maestro session. If you want to save the log file, select Write command log file. To specify the name of the file, enter it in the text box. The default name is maestrolog.cmd, and it is written to the current directory.

The third function is to save the panel layout. If you want to save the layout of all panels that you have had open in the Maestro session, select Save panel layout. The size and position of all the panels is saved for the next Maestro session.

Both of the above options can be set in the Preferences panel, under General → Quitting Maestro.

Log Files

Saved log files can be executed in subsequent Maestro sessions, or they can be sent to Schrödinger when a problem is being reported. To execute a saved log file in Maestro, enter the command run log_name, where log_name is the name of the saved log file, in the Command Input Area at the foot of the main window. To read or edit a log file, open it in the Command Script Editor.
