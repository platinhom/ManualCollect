FEP Mapper Input and Output Files and Directories

The input files and directories are listed below:

jobname_pv.maegz input pose-viewer file containing the structures of the protein and all the ligands.
jobname.pkl input pkl file containing information on how the ligands are connected into a graph, where each edge of the graph represents a perturbation.
jobname.sh shell script to run the FEP mapper job.
restart_jobname.sh shell script to restart the FEP mapper job if the calculations for some edges fail.
extend_jobname.sh shell script to extend the calculation for certain edges of the FEP mapper jobs for another 5ns after the initial 5 ns finish.This script adds time to the final state calculation only.
$FF_DIR subdirectory (folder) containing the custom FF parameters specified in the Preferences panel, under Jobs − Force field.

The output files and directories are listed below.:

jobname_1-out.tgz gzipped tar file containing the input _pv.maegz file.
jobname_2-out.tgz gzipped tar file containing the input Maestro file for each edge of the perturbation graph.
jobname_3 folder containing the trajectories for all the perturbations.
jobname_4-out.tgz gzipped tar file containing the _ddg.mae file for each edge of the perturbation graph where the calculated bennett ddG is given as a entry level property. These _ddg.mae files are from the last frame of the production simulation for the last lambda window. 
jobname_5-out.tgz gzipped tar file containing the csv file, -out.pkl file and result file.
jobname_6-out.tgz gzipped tar file containing the cleaned trajectory files for the two end points of each perturbation, and the _ddg.mae files. The tar file includes the following two folders:
  • ddg_mae_files—includes the _ddg.mae files with the file names replaced by the title of of the ligands
  • trajectories_files—includes the gzipped tar file of trajectory files for the two end points of each perturbation (in complex and in solvent)
