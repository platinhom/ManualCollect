Similarity normalization option menu

Choose a scheme for normalizing the similarity. The similarity is defined by Sim(A,B) = O(A,B)/f[O(A,A),O(B,B)], where O is the overlap between the query A and a molecule B that is being screened. The function f can be one of four choices, which are given on the option menu:

  • Maximum—Normalize to the maximum of O(A,A) and O(B,B)
  • Minimum—Normalize to the minimum of O(A,A) and O(B,B)
  • Query structure—Normalize to O(A,A)
  • Screen structure—Normalize to O(B,B)
Skip structures with similarity below option and text box

Structures whose similarity to the query are less than the value specified in the text box are not returned in the output if this option is selected. By default all structures are returned, with the calculated shape similarity property.

Shape treatment options

Select the kind of shape treatment to use:

  • Pharmacophoric sites—Treat each structure as a collection of pharmacophore sites, whose locations are assigned by applying Phase pharmacophore feature definitions. Overlapping volumes are computed only between sites of the same feature type. Each site is represented by a sphere of radius 2 Å.
  • Atoms—Treat each structure as a collection of atoms whose volumes are defined by their van der Waals radii. Overlapping volumes are computed only between atoms of the same type, which can be selected from the Atom types option menu.
Atom types option menu

Choose the types of atoms between which volume overlaps will be calculated for determining the shape similarity. Volume overlaps are computed between atoms that have the same atom type as defined by the choice of atom typing scheme below.

  • None—Don't distinguish different types of atoms when calculating volume overlaps: all atoms are treated the same.
  • MacroModel—Calculate volume overlaps only between atoms that have the same MacroModel atom type.
  • Elements—Calculate volume overlaps only between atoms of the same element.
  • Pharmacophore—Calculate volume overlaps between atoms that have the same pharmacophore type (Acceptor, Donor, etc.) as defined for Phase QSAR models.
Include polar hydrogens option

Select this option if you want polar hydrogens (which are almost always donors) to be considered in the shape search.

Report dual similarities option

Report the similarities with and without atom typing. An additional property is returned, Shape Sim Pure, which contains the similarities without atom typing.

Existing conformers options

Select an option for handling existing conformers, from Discard or Keep.

Search method option

Choose an option for the conformational search. Rapid is faster but generates fewer conformers; Thorough is slower but generates more conformers.

Maximum number of conformers box

Enter the maximum number of conformers per input molecule to be generated in this text box. Default: 100.

Retain up to N conformers per rotatable bond box

Enter the number of conformers to keep per rotatable bond for each input molecule. Default: 10.

Amide torsions option menu

Choose the way that torsions around amide bonds are treated in the conformational search, from the following:

  • Vary—allow the conformation around the amide bond to vary freely.
  • Retain original—do not vary the conformation around the amide bond, but keep the original conformation.
  • Set to trans—Set the conformation around the amide bond to trans (180°).
