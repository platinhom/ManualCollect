Hierarchical Clustering Dendrogram Panel

Display a dendrogram from the results of a hierarchical clustering job.

Hierarchical Clustering Dendrogram Panel Features

  • Update button

  • Color by property option and menu

Number of clusters option and box

Select this option to choose the number of clusters, and enter the number in the box or use the arrow buttons to change the number. The dendrogram is updated with information for the selected number of clusters.

Use Kelley criterion option

Use the Kelley criterion to select the number of clusters. The dendrogram is updated with information for the selected number of clusters.

Merging distance option and text box

Select this option to specify the number of clusters by the merging distance, and enter the distance in the text box. The dendrogram is not updated until you click Update.

Update button

Click this button to update the dendrogram after changing the merging distance. Only available when Merging distance is selected.

Dendrogram

Displays the dendrogram for the hierarchical clustering. The cluster indices for each structure are shown in the boxes at the bottom of the dendrogram, and the lines are colored according to the cluster index. The merging distance is displayed in a tool tip when you pause the pointer over a split point. You can zoom in and out with the mouse wheel.

Color by cluster option

Select this option to color the dendrogram and the leaf nodes by cluster membership. A color cycle of 10 colors is used.

Color by property option and menu

Select this option to color the dendrogram and the leaf nodes by the value of a property, which you can choose from the option menu after you have set up the color ramps in the Heat Map dialog box.

Edit Heat Map button

Open the Heat Map dialog box to choose properties whose values are used to color the dendrogram, and set the color range. The properties chosen in this dialog box are added to the Color by property option menu.

Export button

Export the clusters at the selected clustering level. Opens a dialog box in which you can choose to export the clusters or their centroids to a file or to the spreadsheet. Three properties are exported: Cluster, which is the cluster index, Centroid, which is 1 for the centroid of each cluster and 0 otherwise, and Total, which is the number of structures in the cluster.

If you choose To file, click Browse to navigate to the location for the file, select the file format from Maestro, SD, or CSV, and name the file. You can export the structures in each cluster to a separate file, labeled with the cluster index, by selecting Place results into separate files. CSV files include the structure name, a SMILES string for the structure, and the three properties.

If you choose To spreadsheet, the column names for the properties are derived from the property name by adding a prefix, which you can specify in the Name prefix text box. The resulting column names are prefix:Cluster, prefix:Centroid, and prefix:Total. The default prefix is HClust::jobname.

If you select Place clusters into separate views, a saved view is created for each cluster, named with the same prefix as for the properties as prefix::cluster N, where N is the cluster number. If you export only the centroids, a single view is created with the name prefix::centroids.

Save Image button

Save an image of the dendrogram, in PNG, JPEG, or TIFF format. Opens a dialog in which you can name the file, and set the dimensions of the image, in pixels.
