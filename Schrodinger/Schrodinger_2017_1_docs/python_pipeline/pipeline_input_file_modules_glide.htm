Pipeline Module glide

This module runs Glide grid generation and ligand docking calculations. It is used by VSW. Many of the keywords are the same as in the Glide input file.

Stage GridgenStage

This stage generates the receptor grid for Glide docking calculations. The keywords are described in Table 1. Correspondences between the Glide input file keywords (given in The glide Command) and the GridgenStage keywords are listed in the table. The default values are the Glide defaults unless otherwise specified.

Table 1. Keywords for the GridgenStage stage of the glide module

Description

General van der Waals radius scaling factor. Same as Glide input file.

RECEP_CCUT

General van der Waals radius scaling partial charge cutoff. Same as Glide input file.

GRID_CENTER

x, y, and z coordinate for grid center.

XCENT

x coordinate for grid center. Obsolete - use GRID_CENTER instead.

YCENT

y coordinate for grid center. Obsolete - use GRID_CENTER instead.

y dimension for inner box.

INNERBOXZ

Stage DockingStage

This stage runs a docking calculation with predefined grids. The INPUTS keyword must specify the ligand file first, followed by the grid file. The stage keywords are described in Table 2. Most of the keywords are the same as in the Glide input file, but not all Glide input file keywords are supported.

Table 2. Glide input file keywords for ligand docking jobs Keywords for the DockingStage stage of the glide module

Keyword

 

GRIDFILE filename

Filename of the grid .zip file, if compressed, or the .grd file.

Docking precision mode. Set in the Settings tab.

confgen—Dock flexibly.

penal—penalize nonplanar conformation

GLIDE_CONS_FINALONLY    boolean

Specify the name of the constraints feature file. Default: jobname.feat.

[ CONSTRAINT_GROUP:n ]

Define constraint group n. The square brackets are mandatory, because this is a section heading that must be followed by other keywords. For pipeline jobs, a second pair of square brackets must be added.

USE_CONS list

List of labels of the constraints that are to be used in docking. Used in a constraint group definition; can also be used independently for defining constraints. Default: no constraints.

Number of constraints that are required to match, or ALL. Used in a constraint group definition; can also be used independently for defining constraints. Default: require all constraints in the group.

[ FEATURE:n ]

PATTERNn smarts atoms

Pattern to be used for a custom feature, given as a SMARTS pattern (smarts) followed by a list of atom indices in the SMARTS pattern that must match (atoms). By default, atoms matching the pattern are included in the feature. You can specify whether to include atoms in the feature or exclude them from the feature by adding include or exclude.

Stage GlideSortStage

This stage is used for sorting Glide pose files. The keywords are described in Table 3.

Table 3. Keywords for the GlideSortStage stage of the glide module

Keyword

Description

BLOCK_SORT_BY_TITLE

Sort the poses by title. All poses with the same title are grouped and then sorted by the INTRATITLE_SORT_KEYS keys. The blocks themselves are sorted by applying the SORT_KEYS to the top pose in each block. Default: True.

INTRATITLE_SORT_KEYS

List of properties to use as sort keys within a title block. Multiple sort keys must be separated by white space.

Default: r_i_glide_emodel.

SORT_KEYS

List of properties to use as sort keys for sorting the poses. If BLOCK_SORT_BY_TITLE is True, the blocks are sorted by the values for the top pose. Multiple sort keys must be separated by white space.

Default: r_i_docking_score.

OUTPUT_POSES_PER_TITLE

The number of top poses to keep for each title block. Applies only if BLOCK_SORT_BY_TITLE is True. 0 means retain all poses.

Default: 0.

INPUT_TYPE

Specify whether the receptor is (pv) or is not (lib) in the file. Default: determine the type from the file suffix, _pv.mae or _lib.mae.

Stage MergeStage
