Using the Convert Phase Database Panel

The Convert Phase Database panel allows you to convert a database from the earlier format, in which the database was marked by a file whose name ends in _phasedb, to the current format, in which the database is a directory with the extension .phdb. The conversion is done by the phase_database application, which is run as a job under Job Control. A new database is created without removing the old database.

To convert a database:

  1. If the database to convert is on a remote host, select Remote databases.

  2. Enter the full name of the database to convert in the Source database text box, or click Browse and navigate to the database. You must enter the name of a file that ends in _phasedb.

  3. Enter the full name of the new database in the Destination database text box, or click Browse and navigate to the location.

  4. Select an option for the feature definitions to use.

  5. Click Run to run the job locally, or click the Settings button to specify the host and number of processors and start the job. If the database is remote, you will have to select a host on which the database is accessible to run the job.

  6. Choose whether to open the database or not.

    When the conversion finishes, you are prompted to open the database. If you click Open, the Manage Phase Database panel opens with the database loaded. If you click Don't Open, the database is not opened.

    After dismissing this dialog box, the status is reported as "Status: Conversion is complete" in the panel.

If the database conversion fails, you will need to determine why it failed, which you can do by examining the log file in the Monitor panel. If the source database was incomplete for some reason, the database cannot be converted, and you should consider removing the newly created database. If the conversion failed because of resource issues, you can restart the database conversion from the command line. Failures of subjobs or subtasks are retried up to three times before giving up.

Convert Phase Database Panel Features

  • Remote databases option
  • Source database text box and Browse button
  • Destination database text box and Browse button
  • Feature definitions options
  • Job toolbar
  • Status bar
Remote databases option

Select this option if the source database is on a remote host. The destination database will be created on the same host.

Source database text box and Browse button

Enter the full file name of the Phase database in the old format, or click Browse and navigate to the database. You must enter the name of a file that ends in _phasedb. If Remote databases is selected, the path you choose must exist on the remote host.

Destination database text box and Browse button

Enter the full name of the new Phase database, or click Browse, navigate to the desired location and enter the database name. You must enter a name that ends in .phdb. By default, the same location and name is used as for the source database: path/name_phasedb is converted to path/name.phdb Once the name is entered, it is used to define the default job name, which is entered into the Job name text box. If Remote databases is selected, the path you choose must exist on the remote host.

Feature definitions options

Select an option for the feature definitions to use when converting the database. The options are:

  • Use definitions from source database—Use the definitions from the old database. This option allows you to continue to use the old feature definitions with the new database format. This may be important for reproducing the results of previous runs with newer software.

  • Use current default definitions—Use the latest default feature definitions. The database is converted, and the conversion may result in a change in the sites identified on each molecule. Newer definitions are in general to be preferred, as they usually represent improvements in the science.

Job toolbar

Manage job submission. See Job Toolbar for a description of this toolbar.

Status bar

The status bar displays information about the current job settings for the panel. This includes the job name, task name and task settings (if any), number of subjobs (if any) and the host name and job incorporation setting. It also displays a message for a short time when you start a new job.

The status bar also contains the Help button, which opens the help topic for the panel.
