Build Structure - Options Dialog Box

In this dialog box, you can select a number of options for building the structure from the templates in the Build Structure step.

To open this dialog box, click Options in the Input Sequence step of the Structure Prediction panel.

Build Structure - Options Dialog Box Features

  • Preserve the residue numbers of the template option
  • Retain rotamers for conserved residues option
  • Optimize side chains option
  • Minimize residues not derived from templates option
  • Omit structural discontinuities options
Preserve the residue numbers of the template option

Number the residues in the built structure the same as in the template, as far as possible. Does no apply to consensus modeling. When using multiple templates for a single chain, the residue numbers are taken from the first template. Sequential numbering from 1 is used if the attempt to use template numbering fails. This option is off by default.

Retain rotamers for conserved residues option

Retain the the rotamers for the side chains of conserved residues. These side chains will not be optimized. This option is on by default.

Optimize side chains option

Optimize side chains, except for those attached to conserved residues if Retain rotamers for conserved residues is selected. This option is on by default.

Minimize residues not derived from templates option

If side chains are optimized, also optimize the residues whose structure is not derived from any of the templates. This option is selected by default.

Omit structural discontinuities options

Disable the building of structural discontinuities arising from insertions, template junctions, and deletions. The discontinuities are capped with NMA and ACE, so the final structure is discontinuous. The breaks in the chain do not result in a new chain name. If you choose any of these options, you should check that your final structure is reasonable. The options are:

  • Insertions—If there is an insertion in the query that requires the building of a long loop, you can select this option to cut the query, cap it with NMA and ACE, and not build the loop. You can also specify the maximum length of loops that will be built in the text box, so that only longer loops are cut rather than built. When the template that spans the insertion is cut to build the query, the template residue on the high side (higher residue number) of the cut is also deleted, to avoid introducing steric clashes. Cutting could be useful where there are long insertions that are not in the region of interest. This option is selected by default, with a length of 20. If it is not selected, all loops are built.

  • Deletions and Template Junctions—If there are deletions in the query (a query gap), or if there is a junction between two templates, you can choose to cap the ends of the two pieces and not try to join them in building the structure, by selecting one or both of these two options. These options are not selected by default.
