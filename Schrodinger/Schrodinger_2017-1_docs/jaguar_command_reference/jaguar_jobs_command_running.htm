Running Jaguar Jobs from the Command Line

The jaguar run command lets you run a Jaguar job using the Jaguar input file you specify and any of the jaguar run command options shown in Table 1 and described below. You can also use the standard Job Control options—see Running Jobs From the Command Line for more information. Note that running OpenMP parallel jobs is not supported on Windows.

Table 1. Options for the jaguar run command

Option

Effect

Default Behavior

-WAIT

Wait for the Jaguar job to finish before returning control to the shell.

Return control to the shell immediately.

-TPP nthreads

Use nthreads OpenMP threads per process. These threads can only run on the same node. As the jobs are compute-intensive, you should only allocate one thread per core, so nthreads is effectively limited by the number of cores per node on the execution host.

Do not use threads.

-PARALLEL npr

Use npr processors, and distribute them between subjobs and OpenMP threads to optimize resource use.

Run a serial job.

-t

Write time stamps to the log file after each executable has run.

Write time stamps to the log file at the start and the end of a job.

-subdir

Create a subdirectory for the job, named with the job name, and store all input and output files in the subdirectory.

Do not create a subdirectory for the job.

To run a Jaguar job, you first need a Jaguar input file. The file should be named in the form jobname.in.You can create an input file using the GUI (see The Jaguar Edit Job Dialog Box for more information). If you create or edit an input file using a text editor, make sure its format agrees with that described in The Jaguar Input File.

You can run a single Jaguar job from the command line with the command

jaguar run jobname.in

where jobname is the stem of your input file name, jobname.in. With this command, the job runs on the machine upon which you have submitted the command.

To run a Jaguar job on another machine, use a command in this form:

jaguar run -HOST hostname jobname.in

where your input file is named jobname.in and hostname is one of the hosts in the file schrodinger.hosts. For instance, if you were logged into a machine named alpha and wanted to run a job with the input file ch4.in on a machine named beta, you would enter

jaguar run -HOST beta ch4.in

The jaguar run command has several other command line options. For example,

jaguar run ‑NICE ‑SAVE jobname.in

causes executables to be run with a lower CPU scheduling priority (see the man page on nice) and leaves all temporary files generated during the job in the temporary directory.

To submit a series of independent jobs, you can replace jobname.in with a list of input file names. If you do not specify a host, or specify a single host, the jobs run sequentially. If you specify multiple hosts with the -HOST option, the jobs are distributed over the hosts specified. When a host finishes running a job, it starts the next job, until there are no more jobs to be run. The list of hosts must be separated by spaces and enclosed in quotes. For hosts that have more than one processor, you can append the number of jobs to run on that host to the host name, separated by a colon, as in the following example.

jaguar run -HOST beta:2 ch4.in nh3.in
