Running Jaguar Jobs

Running, monitoring and controlling jobs is done by the Schrödinger job control facility. This facility has both a graphical user interface in the Maestro Monitor panel and a command-line interface in the jobcontrol command. The job control facility handles scratch directory creation and cleanup, and ensures that each job has a unique scratch directory. Output files are copied to the working directory while the job is running. Detailed information on Job Control can be found in the Job Control Guide. Some of this information is repeated here. Basic information on running jobs is given in Starting Jobs from Maestro and Running Schrödinger Software.

If you intend to run jobs on various hosts, you must provide information on the hosts to the job control facility through a file named schrodinger.hosts. How to provide this information is described in The Hosts File.

Jaguar can use large amounts of resources, so you should ensure that you have plenty of memory and scratch disk space on the host you use to run Jaguar. Jobs such as frequency and LMP2 calculations can use several gigabytes of scratch disk space, for example. You should also use local disks for scratch space wherever possible, as performance is significantly reduced if an NFS-mounted scratch disk is used.

In addition to using the job control facility, you can use the jaguar command to perform a number of job submission tasks. The jaguar command is described in the following section, and creating batch scripts to submit multiple Jaguar jobs is described in the subsequent section.
