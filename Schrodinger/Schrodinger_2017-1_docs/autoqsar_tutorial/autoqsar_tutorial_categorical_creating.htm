Creating a QSAR Model for Categorical Mutagenicity Data

In this exercise a global categorical QSAR model will be created to predict mutagenicity using 2D molecules.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file mutagenicity_2D_train_small.maegz, and click Open.

    The Import dialog box closes. Only one of the imported structures is displayed in the Workspace. The structures are all 2D representations.

  6. Ensure all ligand entries in the mutagenicity_2D_train_small group are selected.

  7. Open the Task Tool, type autoqsar in the Search box, and press Enter.

  8. Choose Other Applications → AutoQSAR to open the AutoQSAR panel.

  9. For Choose task, select Build model.

  10. Ensure that Use structures from option is set to its default value of Project Table (selected entries).

  11. Set the Property type to Categorical.

  12. Set the Prediction property to r_sm_Ames_category.

    Generally integer or real numbered categories are prefered over string categories when creating categorical models, ie. 1 and 0 as opposed to Yes and No.

  13. Set the Number of categories to 2.

  14. Click Advanced Options.

    The Advanced Options panel opens, where you will see a graphical representation of the compounds per category. 150 compounds are non-mutagens (0) and 187 are mutagens  (1). While it is unnecessary in this case, fine control over classification categories can be made in this panel.

  15. Click Save to close the Advanced Options panel.

  16. Change the Job name to mutagen_model and click Run.

When the job starts, the job status icon starts spinning.  You can click this icon to display information about the job status.  The job can also be monitored in the Monitor panel, which is not displayed by default.  To open it, click the job status icon, then click Monitor.  While the job is in progress, the Status column displays running.  When the job finishes, the status changes to completed:finished.
