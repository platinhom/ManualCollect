Creating a QSAR Model for Binding Affinity

In this exercise we will use the AutoQSAR panel to create a QSAR model to predict binding affinity for a congeneric series of FactorXa binding ligands.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file fxa_1z6e_train.maegz, and click Open.

    The Import dialog box closes. Only one of the imported structures is displayed in the Workspace.

  6. Ensure all ligand entries in the fxa_1z6e_train group are selected.

  7. Open the Task Tool, type autoqsar in the Search box, and press Enter.

  8. Choose Other Applications → AutoQSAR to open the AutoQSAR panel.

  9. Ensure that Use structures from is set to its default value of Project Table (selected entries).

  10. Ensure that Property type is set to Numerical.

  11. Set the Prediction property to r_user_Binding_Affinity.

  12. In the Job name text box, change the name to factorXa_affinity_model.

  13. Click Run.

When the job starts, the job status icon starts spinning.  You can click this icon to display information about the job status.  The job can also be monitored in the Monitor panel, which is not displayed by default.  To open it, click the job status icon, then click Monitor.  While the job is in progress, the Status column displays running.  When the job finishes, the status changes to completed:finished. 

This job takes a few minutes on a single CPU, but the time may vary depending on your particular system configuration and workload.
