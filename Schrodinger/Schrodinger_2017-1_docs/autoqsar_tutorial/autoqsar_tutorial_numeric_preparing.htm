Preparing Data for QSAR Modeling

Prior to using data in QSAR modeling some curation is recommended. In this exercise we will use the Maestro Project Table to prepare binding affinity data in uM for QSAR modeling. This step is not required to perform further steps in this exercise.

Data curation is critical for the following reasons:

  • Ensure data doesn’t span multiple orders of magnitude. This is most often the case examining binding affinity data such as IC50, KI, and EC50 which are commonly expressed in units of uM. Prior to use in QSAR modeling such data should be converted to a free energy scale by taking -log ( experimental binding data in Molar ).
  • We strongly recommend splitting the dataset into a learning set and an external validation set. The model will be trained (R2) and validated (Q2) against compounds in the learning set. Performance of the QSAR model in predicting the endpoint for compounds in the external validation set is the ultimate test of a model. We recommend splitting datasets randomly with 75% selected for the learning set and 25% selected for the external validation set.
  • Ensure dataset is of reasonable size.
    • For continuous response endpoints we recommend use of at least 20 compounds. For very large datasets AutoQSAR selects up to 5000 compounds to be used as the learning set and ignores the rest.
    • For categorical response endpoints at least 5 compounds should be present for each class and the distribution of compounds per class should be roughly equal. Having significantly more compounds in one class than others (as is often observed in HTS datasets) makes it difficult to create a predictive model with reasonable performance in avoiding false positives and false negatives.
  • Ligands can be provided as 2D or 3D structures with a single ionization and tautomeric state per ligand. Generation of the state should be consistent between learning set compounds, external set validation compounds and future predictions.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file fxa_1z6e_train.maegz, and click Open.

    The Import dialog box closes. Only one of the imported structures is displayed in the Workspace.

  6. Open the Project Table panel by choosing Window → Project Table.

  7. Ensure all ligand entries in the fxa_1z6e_train group are selected.

  8. Click the Calculator button to open the Calculator panel

  9. Ensure the Selected Rows Only option is active.

  10. Enter Binding Affinity in the Create text box.

    We will use the equation pIC50 = -log10(IC50) where IC50 is in Molar.

  11. Enter -1 in the Constant text box.

  12. Click the * button.

  13. Click Functions to open the functions list

  14. Pick log10 from the list and click OK.

  15. Enter 0.000001 in theConstant text box.

  16. Click the * button.

  17. Click Properties and select IC50(uM) from the list.

  18. Click the ) button to close the log10 parenthesis.

  19. Click Execute to create or update the Binding Affinity property with the pIC50 value suitable for QSAR modeling

  20. Click Close to close the Calculator panel.
