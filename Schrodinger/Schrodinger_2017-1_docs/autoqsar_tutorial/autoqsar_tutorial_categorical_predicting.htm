Making Predictions from a Categorical QSAR Model for Mutagenicity

In this exercise an existing QSAR model will be used to make predictions for a collection of ligands. These ligands may be provided in 2D or 3D and should have the same preparation as learning set compounds for consistency.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file mutagen_predict.maegz, and click Open.

    The Import dialog box closes. Only one of the imported structures is displayed in the Workspace.

  6. Ensure all ligand entries in the mutagen_predict group are selected.

  7. Open the Task Tool, type autoqsar in the Search box, and press Enter.

  8. Choose Other Applications > AutoQSAR to open the AutoQSAR panel.

  9. Select View model and make prediction.

  10. From the option menu choosemutagen_model.qzip.

    If the previous exercise to create the QSAR model was not performed in the current Maestro session that option will not be available. You can instead import mutagen_model.qzip into the AutoQSAR panel by clicking Browse and importing it manually.

  11. Set Use structures from to Project Table (selected entries).

  12. Set Model to test to Best model (bayes_dendritic_15).

    While we strongly recommend using a consensus prediction that includes all ten top-ranked models, in this exercise we demonstrate how to apply a single QSAR model. The choice above generates predictions using only the bayes_dendritic_15 QSAR model that was created using Naive Bayes fitting with dendritic fingerprints.

  13. Set AutoQSAR Prediction to mutagen.

    AutoQSAR will return binding affinity predictions in a property with this name.

  14. Change the Job name to mutagen_prediction and click Run to start the job.

When the job starts, the job status icon starts spinning.  You can click this icon to display information about the job status.  The job can also be monitored in the Monitor panel, which is not displayed by default.  To open it, click the job status icon, then click Monitor.  While the job is in progress, the Status column displays running.  When the job finishes, the status changes to completed:finished.
