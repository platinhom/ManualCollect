Incorporation of Job Output

When Maestro project entries are used as input for jobs, the structure and property output can be incorporated into the project. The output files are always copied back to the submission directory: the incorporation options affect what is done in the Maestro project. You can specify how you want the results to be incorporated in the Job Settings dialog box (see Starting Jobs from Maestro). You can also specify a project and incorporation mode when you run jobs from the command line, by using the -PROJ and -DISP options—see Table 1. The default for -DISP is ignore for jobs run from the command line, since the default for -PROJ is not to specify a project. The allowed values of the -DISP option are given in Table 1.

Table 1. Allowed values of the -DISP option

Option

Description

append

New entries are added to the end of the entry list in the project as an entry group. The group name is the name of the file from which the entries originated, minus the extension. If there is only one entry, it is appended as an individual entry, not as a group.

appendinplace

Each new entry is added to the project immediately below its source entry, i.e. the entry that was used as input for the job. If there is no source entry, the new entry is added to the end of the entry list in the project as an individual entry.

appendungrouped

New entries are added to the end of the entry list in the project as individual entries.

replace

Incoming entries replace the existing entries from which they originated. Matching of the incoming and original entries is done by entry ID.

workspace

The Workspace is replaced with the first structure from the output file, but no project entries are created or modified.

ignore

Structure and property data are not incorporated into the project. You can import the results later using the Import panel. See Import Dialog Box for more information on importing structures.
