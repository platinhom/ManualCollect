Identifying Common Pharmacophore Hypotheses for Lead Optimization from Ligands in a Congeneric Series

In this experiment common pharmacophore hypotheses will be identified from known active and inactive ligands in a congeneric series for use in a lead optimization project. Excluded volumes are derived from active and inactive alignments. The resulting hypotheses are recommended for use in lead optimization projects.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file at1.maegz.

  6. Click Open.

    The Import dialog box closes. The imported ligand is displayed in the Workspace.

  7. View all newly imported ligands in the group at1 by including the first ligand in the group in the Workspace and then shift clicking to include the last ligand in the group.

    All ligands between the first and last ligand in the group are included in the Workspace.

  8. Click the Tasks button, type “phase” and press Enter.

  9. Select Develop Pharmacophore Hypothesis from the results list.

    The Develop Pharmacophore Model panel opens.

  10. Set Create pharmacophore model using to Multiple ligands (selected entries)

  11. Ensure that all entries in the at1 group are selected.

  12. Ensure that Generate conformers is selected.

  13. Click Options to open the Generate Conformer Options dialog box.

  14. Set the target number of conformers to 50.

  15. Select Minimize output conformers.

  16. Click Save to close the dialog box and save these settings.

The next task is to define the active and inactive ligand set.

  1. Open the Define Actives and Inactives panel by clicking Define.

  2. Select pIC50-Exp in the Available properties list.

    This property contains experimental binding affinities converted to a free energy scale by pIC50=-log(IC50).

  3. Ensure that Define units is set to dG (kcal/mol).

    We will use only active ligands with <50nM affinity to create common pharmacophore hypotheses in this exercise. IC50 ≤ 50nM translates to pIC50 ≥ 7.3. Similarly, the threshold for recognizing an inactive will be 10 uM or pIC50 ≤ 5.0.

  4. Set the value for Active if activity is [above] to 7.3.

  5. Set the value for Inactive if activity is [below] to 5.0.

    From this panel you can also specify ligands that must match a hypothesis for the hypothesis to be returned by clicking the checkbox in the Required Matches column for the desired ligand. Here we will not require any particular ligands to match.

  6. Click Apply to set Actives and Inactives in the Pharm set column of the Selected Project Table entries table.

  7. Click OK to save these settings.

  8. ClickHypothesis Settings to open the Hypothesis Settings dialog box.

  9. In the Features tab, change the Number of features in the hypothesis to range from 5 to 6.

  10. Select Preferred minimum number of features and set the number to 5.

  11. In the table, set the Minimum number of Donor sites to 1 by editing the table cell.

    This ensures that only hypothesis with at least one donor should be returned.

  12. Set the Minimum number of Negative sites to 1.

  13. For Feature presets, choose Make acceptor and negative equivalent option.

    This option forces acceptors and negatives to be recognized as the same feature. You can keep other options at their default settings.

  14. In the Scoring tab ensure that PhaseHypoScore is selected as the scoring function.

  15. In the Excluded Volumes tab select Create excluded volume shell.

  16. For Create shell from, select Actives and Inactives. Keep other options at their default values.

  17. Click Save to close the Hypothesis Settings dialog box and save the settings.

  18. Change the Job name to "at1_dcph".

    The job can be distributed over multiple CPUs, up to the number of active ligands to be investigated, which in this case is 5. If you want to distribute the job, click the Settings (gear) button and set the number of CPUs. The calculation  takes a few minutes on a single CPU.

  19. Click the Run button.

  20. If you prefer not to run the calculation, import the results from the tutorial file set manually by choosing File → Import Structures and importing the file at1_dcph_phypo.mae.gz.

After the calculation finishes, the 20 hypotheses found and their ligand alignments are imported into the project and shown in the Workspace Navigator and the Project Table under the at1_dcph group. The top-ranked hypothesis is DHNRRR_1 and includes one donor, one hydrophobe, one negative and three ring features.

You can view a hypothesis in the Workspace by double clicking on the blue H button or by opening a hypothesis group entry and including the hypothesis entry.

You can right click the green H button of the at1_dcph entry group to detect binding multiple binding modes across the hypotheses (see Identifying Multiple Binding Modes from a Set of Hypotheses) and view enrichment data in detail to understand the likely performance of these hypotheses in virtual screening (see Viewing Enrichment for a Set of Hypotheses).
