Screening Ligands From Phase Databases and Analyzing Hits

In this exercise we will screen a collection of compounds stored in multiple Phase databases against a pharmacophore hypotheses with constraints and analyze the results.

  1. Choose File → Import Structures.

    The Import dialog box opens.

  2. From the Files of type option menu, ensure that Maestro is selected.

  3. If the options are not displayed, click Options.

  4. Ensure that Import all structures and Replace Workspace are selected.

  5. Select the file erb_common_phypo.mae.gz.

    This file contains multiple hypotheses from a develop common pharmacophore hypotheses experiment.

  6. Click Open.

    The Import dialog box closes. The first imported hypothesis, APRR_2, is displayed in the Workspace.

  7. Right click on the blue H button of the APRR_2 hypothesis entry and select “Add to database screen”.

    The Phase Ligand Screening panel opens, with the hypothesis included to be screened against. Alternatively you can open this panel by clicking the Task button, typing “phase” and pressing enter. Then select Ligand and Database Screening from the results list.

    We will screen ligands from two Phase databases against the APRR_2 hypothesis.

  8. Ensure that Ligands to screen is set to Phase Database.

  9. Click the Add Database button and select Browse.

  10. Select the 50ligs.phdb database and click Open.

    This adds a row to the table, showing the database and enabling selection of only a subset of ligands if subsets are present.

  11. Import the erb_test.phdb database in the same way.

  12. Change Hits must match at least N features to 3 features.

  13. Ensure that Prefer partial matches of more features is selected.

    These choices require only 3 (or more) of the features in the hypothesis to match for the match to be recognized as a hit, with a preference for more features.

  14. Click Screening Settings to open the Screening Settings dialog box.

  15. In the Conformers tab ensure that Use existing conformers is selected, as our Phase databases include conformers.

    This tab controls conformer generation. If you are screening from a file or selected entries without conformers, Generate conformers during search is typically selected.

  16. In the Features tab, ensure that no feature definition presets are selected.

  17. In the Scoring tab ensure that PhaseScreenScore is selected.

    Here you can modify the default scoring function for rank ordering hits returned from screening.

  18. In the Constraints tab, set up a distance constraint between the A3 and R12 features:

    1. Select Pick features.

    2. Pick the A3 then the R12 features in the Workspace.

      You could change the distance criteria or the allowed tolerance but we will retain their default values.

      The Constraints tab allows you to create distance, angle and dihedral constraints, to have finer control over the geometry of matches returned.

  19. In the Criteria tab, ensure that Use default (recommended) is selected.

  20. This tab has options that are used to define what compounds are recognized as matching the hypothesis.

    In the Output tab are options that control information returned after the screen. Leave the values at their defaults.

  21. Click Save to close the Screening Settings panel and save changes.

  22. Change the Job name to "erb_screen".

  23. Click Run to start the job.

    The job takes a few minutes on a single modern CPU. Screening calculations can be run on more than one CPU by making settings in the Job Settings dialog box.

    To open this dialog box, you can click the Settings button in the Phase Ligand Screening panel.

    To run on more than one CPU, select a multiprocessor host, and change the Number of processors. Leave the Incorporate option at Append new entries, so that results are automatically incorporated into Maestro. Click Run in this dialog box to close it and start the job.

  24. Close the Phase Ligand Screening panel.

    Results are incorporated automatically into the project in a group named erb_screen-hits when the job finishes.

  25. Double click the In column for the APRR_2 hypothesis that was screened against to include and fix it in the Workspace.

    This is equivalent to fixing a protein to remain displayed in the Workspace while scrolling through docked poses.

  26. Click the In column for the top-ranked screening hit, CHEMBL360872 to include it in the Workspace.

  27. Ensure all ligands in the erb_screen-hits group are selected by clicking the erb_screen-hits group entry.

  28. Use the right arrow key to scroll through ligands quickly to view how well ligand groups match the pharmacophore hypothesis.

    For example the top-ranked hit, ligand CHEMBL360872 has the A3 feature matching a hydroxyl with good orientation of the lone pairs with the vectors, the R10 feature matching an aromatic ring that is part of a larger ring system, the R12 feature matching an aromatic ring and the P9 feature matching the positive nitrogen in a piperidine ring. Known estrogen receptor beta actives were taken from CHEMBL while other ligands came from the Glide decoy ligands set. The hypothesis shows good discrimination of known actives from presumed decoys as suggested by the PhaseHypoScore of 0.913 for the APRR_2 hypothesis.

Next we will look at another representation of features matched per ligand and the PhaseHypoScore used to rank order hits.

  1. Open the Project Table panel by clicking the Show Project Table buttonin the upper right corner of Maestro.

  2. Click the Show button in the Project Table and choose Select.

    The Show Properties dialog box opens.

  3. Click the None button to deselect all properties.

  4. Select the Matched Ligand Sites and PhaseScreenScore properties to view only those properties in the Project Table.

The colored circles in the Matched Ligand Sites property show features that are matched by a ligand. So for example ligand 550000 did not match the blue (P) feature but did match the two R and one A features.

Hits are by default sorted in descending order of PhaseScreenScore, indicating that compounds at the top of the list better align with the hypothesis than those further down the list.
