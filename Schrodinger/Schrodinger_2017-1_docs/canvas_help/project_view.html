Project View Panel

This panel displays a tree view of the project, which provides access to all the objects created in the project, such as views, jobs run, and so on.

Using the Project View Panel

The project tree lists the categories for the information that is stored in the project.

By default, the panel is docked into the main window. You can undock it and redock it by clicking the docking button.

Project View Panel Features

  • Notify when jobs complete option
  • Project tree
  • Shortcut menu
Notify when jobs complete option

Select this option if you want to be notified by an alert box when a job finishes. Some jobs require action to incorporate the results. When jobs finish, the item in the tree is highlighted and the parent is displayed in bold face, until you take action on the item.

Project tree

The project tree displays a list of applications and data structure types that can be run or created (categories). These are the "branches" of the tree. The "leaves" are the individual jobs or the data structures. When one of these categories has some data, a turner appears on the branch (a box with a &plus; sign or a − sign). You can click the turner to display the list of items (leaves) for this category.

For the applications, the leaves have a status field that shows the job status.

For views, the leaves have a tool tip that displays the description that was added to the view.

Shortcut menu

A shortcut menu is displayed when you right-click and hold on a branch or a leaf. It provides actions to be taken for the category defined by the branch or on the data for the leaf.

The items that appear on the branch shortcut menu depend on the branch, and are taken from the following:

  • Open—Open an application panel or a filter panel; open the dialog box for selecting a view or a partition.
  • New—Create a new partition from a property or a random assignment.
  • Import Queries—Import queries from another project. Opens a file selector so you can navigate to and select the project. Only on the Substructure Queries branch.
  • Sort—Sort views in the default order (order of creation), ascending order or descending order. Only on the Views branch.
  • Delete All—Delete all items in the category and the data associated with them.

The items that appear on the leaf shortcut menu depend on the kind of leaf. For all except queries, the following item is present:

  • Delete—Delete the item and the data associated with it.

For jobs, the following items are present:

  • View—View the results of the job. Opens the relevant panel with the results displayed.
  • Incorporate—Incorporate the results of the job into the project. Only present for jobs that require incorporation
  • Clone Job—Run a job with the same options as the selected job, with new data or the original data.
  • Kill Job—Kill a running job. Only available if the job is active.

Other items are taken from the following:

  • Open—Open a view or a partition. Only present for views and partitions
  • Rename View—change the name of the view and its description. Only present for views.
  • Apply to Master—Apply the view to the Master View. Only present for views.
  • Apply Filter—Apply the filter to the Master View. Only present for filters.
  • Run Query—Run the selected query. Only present for substructure queries.
  • Delete Query—Delete the selected query. Only present for substructure queries.
