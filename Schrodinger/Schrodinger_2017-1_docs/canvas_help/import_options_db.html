Import Options Dialog Box

The import options dialog box allows you to specify which structures to import and how they are to be imported into the Canvas project. It opens automatically after you select the structure files or project to import, and is labeled Import Structure File, Import SMILES and Properties, Import SMILES, or Import Canvas Database, depending on whether you are importing files or a project and what types of files you are importing from. An instance of this dialog box is opened for each type of file when you import multiple files.

Import Options Dialog Box Features

The features present in the dialog box depend on whether you are importing a file or a project and what type of file you are importing from.

  • Store each input file as a view option
  • Structure selection section
  • Record selection section
  • SMILES property section
  • Other options

Store each input file as a view option

Create a new custom view for each input file. Each new view contains the structures for the corresponding input file. This option is only available when importing from files.

Structure selection section

In this section you specify which structures to import from the file. This section is present in all versions of the dialog box.

Import all structures option

Select this option to import all structures in the file.

Start import at option and box

Select this option to import a subset of structures in the file, and specify the index of the first structure to import in the box.

Import at most option and box

Select this option to specify the maximum number of structures to import, and provide the limit in the box.

Import a random subset of option and box

Select this option to import a random subset of the structures in the input file, and specify the percentage of structures to import in the box.

Seed text box

Specify a seed for the random subset selection.

Generate Seed button

Generate a seed for the random subset selection. The seed is displayed in the Seed text box.

Record selection section

In this section you specify how the records are added to the project.

Append to existing records option

Append records to the project from the file or project. No checking is done for duplicates.

Replace existing records option

Delete all existing project records, then import the new records from the file or project.

Skip duplicate structures option

Skip structures that are already in the project. This option is only available when appending or replacing records.

Match by tautomer option

When determining if the structure to be imported is a duplicate of one in the project, consider tautomers of the structure as well as the original. This option is only available when skipping duplicate structures.

Update using Canvas UID option

Update the existing structures in the project by matching the Canvas UID in the structure file with the UID in the project. The structure file must have been exported from the same Canvas project with the Canvas UID included. This option is only intended for re-importing data from the same Canvas project. Any other use is likely to corrupt your project. An example of its use is to export binary fingerprints and reimport the bits as ordinary numeric properties with the value 0 or 1.

This option is not available if you selected SMILES Only as the file format or if you are importing from a project.

Update using property map option

Update the existing structures in the project. The correspondence between the structures in the file and the structures in the project is made by matching a property in the Canvas project (the project key) with a property in the structure file (the file key), as follows:

  • If the value of the file key property for a given structure matches a value of the project key property, the project record is updated with the properties from the file for that structure, and the structure is not changed.
  • If multiple project records are matched for a single file key property value, and Allow duplicate mappings is selected, the properties are updated for all matching project records.
  • If the file key property has multiple instances of the same value, then the last one in the file is used to update the project.
  • If no match is found for the file key property, a new record is added.
Allow duplicate mappings option

Import the structures even if there are duplicate values of the specified project property. If this option is not selected, the import fails if duplicate project property values are found. This option is only available if Update using property map is selected.

File key option menu

Choose the property in the structure file or project that you are importing that is used to map to the chosen property in the project. You might, for example, choose a corporate ID, a unique SMILES string, or a title. This option is only available if Update using property map is selected.

Project key option menu

Choose the property in the project that is used to map to the chosen property in the structure file. You might, for example, choose a corporate ID, a unique SMILES string, or a title. This option is only available if Update using property map is selected.

SMILES property section

In this section you can specify how the columns and rows in a .csv file containing SMILES strings and properties are treated, or whether there is a title in a .smi file.

Treat first row as column names option

Select this option if the first row in the file contains names of the columns (property names). If this option is deselected, the property names are set to Prop1, Prop2, and so on.

Text qualifier option menu

Choose a character that is used to delimit text strings. This value is removed from the string when the string is added as a project property. If you choose None, all quotes are preserved in the project. If you choose the single quote, then single quotes are removed but double quotes are retained. Likewise, if you choose the double quote, then double quotes are removed but single quotes are retained.

Smiles in column option and box

Select this option to specify the column that contains the SMILES pattern and enter the column number in the text box. The default is column 1. If you deselect this option, you can import properties without structures. Any records that are added to the file are then added with an empty structure field. If you use the Update using property map option, you can add properties from a CSV file to existing structures, provided you have a column in the CSV file to match a project property.

This option is not available if you are importing from a SMILES file.

Structure names in column option and box

Select this option to specify the column that contains the structure names (or titles), and set the column number in the box. The default is column 2.

If you are importing from a SMILES file, you can select or deselect this option to indicate that the file contains a title, separated from the SMILES string by a space or a tab. The column number is set to 2 and cannot be changed.

Other options

Ignore all errors option

Automatically skip faulty structures during import. By default, you are prompted for an action when faulty structures are encountered. Problematic structures are written in the same format as the input file to the directory failedimports.

Do not store atom coordinates option

When importing structures that have coordinates, do not store the atom coordinates. The structures then are represented by the connectivity information. This option is only present for Maestro and SD file import.

Retain largest fragment only option

If a structure in the input file contains disconnected fragments, import the largest fragment and discard the rest.
