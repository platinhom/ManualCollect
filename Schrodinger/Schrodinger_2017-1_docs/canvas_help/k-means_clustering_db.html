K-Means Clustering Dialog Box

In this dialog box you can set up and start a job to perform k-means clustering. This clustering method can handle much larger data sets than hierarchical clustering. It is useful when you know the number of clusters you want. Two properties are added by the clustering: the cluster number for each structure and an indicator of the centroid of each cluster.

Background

K-means clustering is performed using Lloyd's algorithm (Lloyd, S.P. IEEE Transactions on Information Theory, 1982, 28, 129). This consists of a set of runs, each of which starts with selection of k data points at random as the initial centroids of the clusters. Then, over a series of iterations, each of the points in the dataset is assigned to a cluster based on the centroid closest to that point, and each centroid is updated by setting it equal to the average position of the points assigned to that cluster. The cost function being minimized is the sum of intra-cluster variances. A run is terminated when the maximum number of steps is reached, or a convergence condition on the cost is reached. The set of clusters returned is taken from the run that has the smallest cost.

The clustering is order-independent, but it tends to create clusters of the same size. There is no guarantee of finding the global optimum clusters for the given number of clusters, and the clusters can depend on the initialization: hence the use of multiple runs.

This method requires the specification of the number of clusters (k) in advance. It does not determine the number of clusters.

Data set option menuSelect the source of the data. The data source can involve both row and column selection. The choices are: Selected rows—Use the rows that are selected in the master view. You must select the rows before opening the dialog box. Visible rows—Use the rows that are displayed (visible) in the master view. You must ensure that the required rows are visible before opening the dialog box. Saved view—Use the rows and columns in a chosen saved view. When you select this item, an option menu that lists the saved views in the project is displayed to the right. You can choose the desired view from this option menu. A second option menu, labeled Domain, allows you to choose whether to restrict the rows or the columns, or both, to those of the saved view. Otherwise the visible rows or columns of the Master View are used. Original data set for jobname—Use the rows and columns that were used for the specified job. This item is only available when you are cloning a job or viewing the results of a job.
Save results as text box

Assign a name to the results. This name is listed under K-Means Clustering in the Project View window.

From fingerprints options

Select this option to generate a similarity or distance matrix from pregenerated fingerprints.

From properties option

Select this option to generate a similarity or distance matrix from properties.

Autoscale options

When generating a matrix from properties, select this option to place all the properties on the same scale before clustering.

Fingerprint or property list

Displays the available fingerprints or properties, depending on the option selected for the cluster creation. Select the fingerprints or properties to use in the clustering. By default, all are selected.

Number of clusters text box

Specify the desired number of clusters in this text box.

Number of runs text box

Maximum number of runs to use in clustering.

Number of steps/run text box

Maximum number of steps to use per run. Each step involves updating of the centroids of each cluster and reassignment of the data points to the clusters.

Maximum total steps text box

Maximum total steps.

Cost convergence text box

Specify the threshold for cost convergence in this text box.

Create Clusters button

Run the job to perform the clustering. Opens the Start Application dialog box, in which you can choose a host for the job.
