Recursive Partitioning Trees Panel

In this panel you can display the decision trees from a recursive partitioning model, one at a time.

Recursive Partitioning Trees Panel Features

  • Tree number box
  • View classifications for options
  • Categories Key button
  • Tree display area
Tree number box

Specify the index number of the tree that you want to display in this box.

View classifications for options

Specify the data set that you want to view the classifications for. If you built a single tree, there are only two options: Training set and Test set. If you built an ensemble of trees, there are three options: Sampled training set (the random sample used to build the chosen tree), Full training set, and Test set.

Categories Key button

Show the correspondence between the classes and the colors. Opens a panel that lists the classes with a color patch.

Tree display area

This area displays the tree. You can zoom in and out on the tree with the mouse wheel.

Each node is represented by a set of boxes that correspond to the classes. The boxes contain the number of observations (data points) that belong to each class. The splitting criterion is shown on the line below each node that connects the node with the next nodes in the tree. The leaf nodes have a diamond below them that is colored according to the majority class, and below it, boxes that contain the probabilities for each class (from the training set).

If the test set is being shown, and the class values are not known, the nodes are represented by a white box that contains the number of observations for that node.

If you click on any of the boxes in the node, a custom view containing just the structures rempresented in that box is displayed.
