Ligand FEP - Ligand Functional Group Mutation Panel

Calculate the change in binding free energy for mutation of a ligand that is docked to a receptor, by FEP.

To open this panel, choose Tasks → Free Energy Calculations → Ligand FEP in the Task Tool.

Using the Ligand FEP - Ligand Functional Group Mutation Panel

Note: The Ligand FEP panel is only licensed to academic users. If you are a commercial user, please contact your sales representative about our commercial FEP+ capabilities.

Note: A tutorial is available from the Schrödinger web site. Click here to download the zip file containing the tutorial. The tutorial instructions are in a PDF document in this zip file.

This panel is designed for calculating the change in binding free energy when functional groups on a ligand that is docked to a receptor is replaced with other functional groups.

The first step is to select the receptor and the ligands. If you have previously run a Glide docking experiment with a series of ligands that are related by substitution of functional groups on a common core, you can use the Glide output directly.

Otherwise, you will need to prepare the two ligands and the receptor. You could, for example, choose a ligand-receptor complex with the native ligand, extract the ligand from the complex, then copy and modify the ligand. To do this in Maestro:

  1. Import the complex.

  2. Make sure it is properly prepared (with the Protein Preparation Wizard).

  3. Split the complex into ligand and receptor: right click on the project entry and choose Split → Into Ligands, Water, Other.

  4. If there are more than 2 entries, recombine all but the ligand: select the entries other than the ligand, right-click on them and choose Merge. You can change the title of the merged entry if you want.

  5. Duplicate the ligand entry: right-click on the entry and choose Duplicate → In Place.

  6. Use the tools on the Build and Fragments toolbar to modify the duplicate (which is already in the Workspace).

  7. Minimize the modified ligand groups: select the atoms in the Workspace and press CTRL+M.

  8. Ensure that the receptor and the two ligands are selected, ready for importing into the workflow.

Copying and modifying the ligand ensures that its position with respect to the binding site does not change, and minimizing only the changed ligand group does so too. You can now use these three entries, the receptor and the two ligands, in the panel.

  1. Open the Ligand FEP panel.

  2. Select the source of the receptor and ligands from the Import complex from option menu.

  3. Click Browse, if using a file, or Load, if using project entries, to load the structures into the workflow.

  4. If you have more than two ligands, choose the ligands from the option menus above the ligand display areas.

  5. Check the mutation by using the options under Show atom property.

  6. If you have not already selected a host for the FEP calculations, click the settings button to open the Job Settings dialog box, and make settings for running the job. The calculation is done using Desmond, and as FEP calculations are resource-intensive, you will need access to suitable computational resources. Our recommendation is to run the job across 4 GPU cards.

  7. Click Run to submit the job to the queue for execution.

When the job finishes, you can import the output .cms file, which contains the ΔΔG value as a property.

The force field used in the FEP simulations is the OPLS_2005 force field.

Import complex from option menu and button

Choose the source of the ligand-receptor complex and ligands and import them into the workflow. The source must contain the receptor and two or more ligands.

  • Maestro Pose Viewer File—Import the receptor and ligands from a Maestro file in pose-viewer format. Click Browse to locate the file and import the structures.

  • Project Table—Import the receptor and ligands from the Project Table. The entries you want to import must be selected in the Project Table before you click Import.

After the import, the receptor name is shown below these controls, along with the number of ligands imported. The receptor is checked for potential issues, and if found, an indicator is displayed to the right of the receptor name. Clicking the button opens the Protein Reliability Report panel, which provides information on the potential issues. You should check that there are no major problems, such as missing residues or atoms near the active site.

Ligand menus

If you imported more than two ligands, you can select the ligands you want to mutate from these option menus. The ligands are displayed in 2D in the display area below. The ligands you select for the perturbation must have the same net charge.

Ligand display areas

These areas display the selected ligands, in 2D. The ligands are marked with the atom property that is selected under Show atom property. The property gives information on the mutation. The RMSD of the cores of the two ligands is shown below the display areas, on the right.

Show atom property options

Select an atom property to annotate the ligand displays.

  • Mutation—highlight the mutation. The mutated atoms are marked with red circles and the changed bonds are colored red.
  • Atom mapping—Show the atom numbering on the structures.
  • Common core—Mark the common core with green bonds.
  • Hot atoms—Atoms for which the potential is scaled down during the process (effectively heating them up) so that they are sampled better.
  • None—Show the structures without any annotation.
Job toolbar

Manage job submission. See Job Toolbar for a description of this toolbar.

Status bar

The status bar displays information about the current job settings for the panel. This includes the job name, task name and task settings (if any), number of subjobs (if any) and the host name and job incorporation setting. It also displays a message for a short time when you start a new job.

The status bar also contains the Help button, which opens the help topic for the panel.
