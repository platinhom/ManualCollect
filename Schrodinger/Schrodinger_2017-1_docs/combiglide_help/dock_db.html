Dock Dialog Box

In this dialog box, you select the docking task and select options for the selected task, then submit the docking job.

Dock Dialog Box Features

  • Docking task options
  • Single-position docking section
  • Combinatorial docking section
  • Enumerated docking section
  • Post-docking section

Docking task options

Choose the docking task to be performed.

Both single-position and combinatorial

Docks the molecules in which a single substitution has been made on the user-capped core, and uses these molecules to construct fully substituted structures, which are then docked.

Single-position only

Docks the molecules in which a single substitution has been made on the selected core.

Combinatorial only (from existing single-position data)

Dock the fully substituted structures derived from already available singly substituted structures.

Fully enumerated

Enumerate and dock the entire library. This is the only option available when there is only a single attachment defined.

Single-position docking section

In this section you can choose which core to use for the docking of the singly substituted structures. The choices are:

  • User-capped core—Use the original core-containing molecule. The capping groups are the original side chains.

  • Minimally capped core—Use the core molecule with the minimal capping groups.

Combinatorial docking section

In this section you choose options for the docking of the fully substituted structures.

Maximum number of structures to dock text box

This text box sets the limit on the number of molecules to dock in the combinatorial docking job. The molecules are selected on the basis of their intermediate score. The limit is estimated on entry to this step. You can change it to any value you like. Larger values will produce more molecules but will also take longer. However, if you plan to use Glide constraints, you should significantly increase the limit to ensure sufficient output structures.

Docking mode options

Choose one of two options: SP docking or CombiGlide XP docking. SP docking performs standard Glide SP docking, and is faster than XP docking. CombiGlide XP docking is similar to Glide XP docking in that it is designed to penalize poor poses and thereby eliminate false positives.

Enumerated docking section

Choose the method for docking all possible molecules that arise from the reagent set, i.e. the entire enumerated library. No filtering is done to reduce the number of reagents or structures. If you only have one attachment defined, this is the only docking task you can perform.

In addition to using CombiGlide XP docking, you can select any of the Glide docking modes: XP, SP, or HTVS. You must have a separate Glide license for the Glide docking modes. For more information on these modes, see the Ligand Docking — Settings Tab topic.

Post-docking section

Select Perform post-docking minimization to minimize the docked poses. This option improves the geometry of the poses, since it adjusts bond lengths and angles as well as dihedral angles. If you select this option, you can also select Apply strain correction terms to add strain correction estimates to the final scoring. More detail is availaible in the Glide Ligand Docking — Output Tab topic.
