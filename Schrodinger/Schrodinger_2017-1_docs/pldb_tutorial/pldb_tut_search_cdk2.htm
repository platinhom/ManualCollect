Searching for Entries for the CDK2 Gene

In this exercise, we'll retrieve results for the CDK2 Gene.

  1. Click the New Search button in the history panel to open the PLDB search form.

    On the left side of the form, you will find search fields for filtering query results. The sketcher and associated toolbars on the right side of the form are used for substructure and geometric queries.

  2. Scroll down slightly until the Gene Name field appears.

    Click the button to open the PLDB documentation for this search field. You can do this for any search field for which help is available. You can also open the documentation window by clicking the Help button in the bottom-left corner of the search form.

    If you start typing in any PLDB search form field containing the words "Type to search", you will be provided with a list of suggestions based on your input.

  3. Type CDK2 into the Gene Name field.

  4. Enter the title Gene: CDK2 for your query in the Search Name field on the bottom-right corner of the form.

  5. Click the Start Search button to run your query.

    The query should take a few minutes to finish running.  Once your query has run, you will see it in the Previous searches section of the history panel.


    You may have a different number of results than what is shown in the example if new entries have been added to the PDB.

  6. Click the View Results button to open the results panel.

    You can see a high-level view of all of your query's results in the results panel. Each row in the results table represents one PLDB entry. Search criteria appear as columns in the results table. In this case, we see the values for the Gene Name that matched our query for CDK2.

    More in-depth information on a per structure basis is available in the Complex, Ligand, Protein, and Files tabs in the panel to the right of the results table. For example, you can view a Ligand Interaction Diagram for each structure in the Complex tab (see Ligand Interaction Diagram Panel for details on the diagram).

    Entries flagged as problematic appear in the results table with a red exclamation mark next to their value in the Code column.

    You can see a list of the problems flagged by PLDB in the Complex tab on the right hand side of the panel.
