Diagnostics Panel

Run diagnostic tests to check the status and availability of your licenses and hosts, and report the results, with descriptions of problems and ways to address them.

To open this panel from Maestro or Canvas, choose Help → Diagnostics.

This application can be opened outside of Maestro or Canvas:

  • Linux: Enter $SCHRODINGER/diagnostics in a terminal window.

  • Windows: Choose Start → All Programs → Schrodingerrelease → Diagnostics, or enter diagnostics in a Schrodinger Command Prompt window.

  • Mac: Choose Applications → Diagnostics.app, or enter $SCHRODINGER/diagnostics in a terminal window.

Using the Diagnostics Panel

After starting the application, a set of tests is run. Their progress is reported in the Running Tests information box, which is displayed along with the Diagnostics panel. The results of these tests are then displayed in the panel.

If you run this application on the license server host, you can use the Stop License Server and Start License Server buttons to stop and start the license server. These buttons are not available otherwise.

The licenses are located using the same mechanism as when starting a job. A breakdown of the number of jobs you can run for each product is displayed in the Job Capacity tab.

The hosts file is identified using the same mechanism for locating the hosts file that is used when you start a job. If you have a custom hosts file in your Schrödinger user resources directory, it will use that file. If you have a custom hosts file in some other location, you can test it from the command line by changing to the directory where that file is located and running this application.

To change the reporting level, you can start the application from the command line with the -l option, which takes the values critical, error, warning, info, debug. These values represent increasing levels of reporting, and decreasing levels of severity of problems encountered. The default level is error.

SCHRODINGER display

The path to the current installation (defined by the SCHRODINGER environment variable is displayed at the top of the panel.

License tab

This tab reports on various aspects of the license tests.

License display

This text display shows the actual location of the license file that is in use.

Defined by display

This text display shows how the license file was located.

Server Status tab

This tab shows the status of the license server, if a license server is being used. If you are not using server-based licenses, the text area shows a message indicating this fact.

Below the text area are three buttons for actions that you can take.

  • Recheck—Run the license checks again and refresh the information in the Server Status, License Test, and Job Capacity tabs.
  • Stop License Server—Stop the license server, if you are running this application on the license server host.
  • Start License Server—Start the license server, if you are running this application on the license server host.
License Test tab

This tab displays the text output of the license test. The Recheck button runs the license checks again and refreshes the license information.

Job Capacity tab

In this tab, the number of jobs you can run for each product and job type is reported, based on the licenses that you have. The Recheck button runs the license checks again and refreshes the license information.

Hosts tab

This tab reports on the hosts available and the current access to those hosts.

Hosts file display

This text display shows the full path to the hosts file that is currently in use.

Reload button

Click this button to reload the hosts file and update the information in the Hosts tab.

Hosts table

This table shows a summary of the tests performed on each of the hosts defined in the hosts file. Each row corresponds to a test performed on a host. The table columns are described below. You can select a single row in the table to perform a test.

Host The name of the host entry in the hosts file.
Queue The type of queue represented by this entry, if applicable.
CPUs The number of processors associated with this entry.
Check The kind of check that was performed on this host. There are two kinds: hosts file, and testapp.
Result Status of the check that was performed. The messages produced by the check are listed in the Results text area below.
Run test job on buttons

These buttons run a test job, either on the host that is selected in the table (Selected Host) or on all hosts in the table (All Hosts).

Result text area

Reports on the outcome of the check for the row that is selected in the table.

Tech Support button

Create an archive containing the files produced by the diagnostic tests, and display information on how to upload the file and contact technical support.
