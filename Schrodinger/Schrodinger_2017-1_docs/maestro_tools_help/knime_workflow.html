KNIME Workflow Panel

This panel allows you to edit the chosen KNIME workflow in KNIME, copy the workflow, set up parameters and run the workflow. It is a generic panel that is opened with the details of the specific chosen workflow.

Using the KNIME Workflow Panel

To edit the chosen workflow:

  • Click Open in KNIME. The KNIME interface starts and loads the workflow.

To run the chosen workflow interactively in KNIME:

  • Click Open in KNIME. The KNIME interface starts and loads the workflow.

To run the chosen workflow as a job:

  1. Select entries in the Project Table or include them in the Workspace, according to what is listed under Input.

  2. Make choices with the controls under Parameters.

  3. If you want to store the results of running the workflow in the workflow, select Save after execution.

  4. Click Run.

KNIME Workflow Panel Features

  • Workflow name and description
  • Open in KNIME button
  • Create a Copy button
  • Input description
  • Ouptut description
  • Parameters controls
  • Save after execution option
  • Run button
Workflow name and description

The name of the workflow and its description (if any) are displayed at the top of the panel, along with a list of required programs.

Open in KNIME button

Open the Workflow in KNIME. Starts the KNIME interface and loads the workflow. You can then edit it, examine Report nodes for data that cannot be easily displayed in Maestro, use the Report Designer mode, and perform other tasks.

Create a Copy button

Create a copy of the workflow and place it on the KNIME Workflows submenu of the Workflows menu. The copy has "copy" appended to the name. When you select this new workflow from the menu, a prompt to rename it is posted and the KNIME interface is started. You should make any changes that you want, and then close the workflow. You must then rename the workflow in KNIME and close KNIME before it will be available for execution from Maestro, under the new name. (Right-click on the workflow in the Workflow Projects pane, and choose Rename.)

Input description

The source of the input structures is described under this heading. The two possibilities are Selected entries and Included entries. You must ensure that the appropriate Project Table selection or Workspace inclusion is made before running the workflow.

Ouptut description

The types of output are listed below this heading. They are:

  • New group—Incorporate the output structures into the project as a new entry group.
  • Maestro messagemessage—Display an alert box with the specified message, and any associated numerical values.
  • Maestro commandscommands—Run the specified Maestro commands at the end of the workflow (after incorporating any output structures).
Parameters controls

These controls allow you to set parameters requested by Quick form nodes in the Workflow. The controls vary according to the worfklow.

Save after execution option

Save the workflow after running it. The workflow will then include all the results of running it, and any settings you changed under Parameters are also changed. You can open the workflow in KNIME to inspect Report nodes. Saving the workflow can also be useful for diagnosing execution problems. Saving the workflow when running on a large data set is not recommended.

If you run the workflow again after saving it, only the nodes that take input from Maestro are reset.

Run button

Run the workflow as a job. The workflow itself is run under Job Control on the local host, but it uses any individual node settings for execution on a remote host.
