Parameter Flow Variables

If you want to set an option that is not available in the node configuration panel, you may be able to use parameter flow variables to add the option to the command that is executed by the node.

As not all nodes allow the use of parameter flow variables, you will have to check if the node you are interested in supports them. Not all of these nodes run the Schrödinger command directly, but some execute a Python script that runs the command instead. To check whether the node supports parameter flow variables and to obtain the necessary information to set them, follow the instructions below.

1. Look at the node description to see if it has a Parameter Flow Variables section. If it does, then you can add command options.
2. Make a note of the Flow variable prefix keyword in the Parameter Flow Variables section, under Usage. This keyword is used to construct the name of the flow variable.
3. Make a note of the command that is executed by the node, which is shown in the Backend implementation section of the node description. It is the options to this command that you can set with parameter flow variables.
4. Run the command with the -h (or --help) option to get a usage statement (or look in the relevant manual). Make a note of the options that you want to set, and what the allowed values are. The option is also used in the name of the flow variable.

With this information, you can add upstream nodes to your workflow that are attached to the variable input port. Any node that can set the parameter flow variable can be used.

  • The variable name must be constructed by concatenating the keyword and the option. For example, the keyword for the Prime MM-GBSA node is MMGBSA, and if you want to set the -rflexdist option, the flow variable name is MMGBSA-rflexdist.

  • The value of the variable must be of type string, as it is added as a string to the command. In this example, the value might be 6.0, but it is passed to the node as a string, not a real.

The simplest way to add a parameter flow variable is to use a String Input Quick Form node. In the Control tab of the configuration dialog box for this node, enter the flow variable name in the Variable Name text box, and the value in the String Value text box. Connect the Flow Variable Output port to the Variable Inport on the node. You can chain together a set of String Input nodes to set multiple options, and you can bundle these nodes as a metanode.
