Opening a Project

To open any existing project, choose File → Open Project or press Ctrl+O (⌘O) . The Open Project project selector opens, in which you can navigate to and select the project. When you click Open, the current project is closed and the selected project is opened. This button is not active until you have provided a valid project name.

To open a recently used project, choose File → Open Recent Project → project. The number of projects displayed can be set in the General – History tab of the Preferences panel.

If the project version of the project you select does not match the current project version, a dialog box opens, allowing you to convert the project with a new name to the current project version. If the conversion fails for any reason, another dialog box informs you of what was tried and suggests an action to take.

If the project you want to open is already open in another Maestro session, you can request it to be closed so that you can open it, provided the other Maestro session was started using a release no earlier than 2015-3. A dialog box opens asking if you want to close the project in the other session and open it in the current session. You can click the Show Details button to find out which session on which host is using the project. If you click Yes, the project is closed by the other session and opened in the current session.

If the Maestro session was started using an earlier release than 2015-3, you cannot remove the lock on the project and open it without damaging the project. However if the lock is present due to a failure and the Maestro session is no longer running, it is safe to remove the lock and open the project.
