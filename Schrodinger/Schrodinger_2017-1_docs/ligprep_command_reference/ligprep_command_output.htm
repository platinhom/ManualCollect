LigPrep Output Files

The main LigPrep output files are an output structure file in the chosen format, and a log file (jobname.log), which records information about the processing of the structures.

If a structure cannot be processed by a particular stage, several other output files are generated:

  • The original structure from the input file is written to a file named jobname-dropped.ext, where ext is the extension for the chosen input file format.
  • The index in the input file of the original structure is recorded in the file jobname-dropped-indices.txt.
  • The actual structure that triggered a failure in one of the stages is written to the file jobname-dropped-intermediates.maegz file. The Maestro property s_lp_failure_stage is added to the structure properties to indicate the stage that failed.

A report is given at the end of the log file, listing the failures by the structure index in the input file, and specifying the stage in which the failure occurred.

If a distributed job runs and some subjobs fail to run after they have been retried, the input files for the subjobs are returned, and the failures are reported in the log file. They are named in_jobname_subjob.ext, where subjob is the subjob number, and the extension is the same as for the input file.
