Jaguar pKa Shell Model

A functional group that is not identified by Jaguar as one of the functional groups listed in Table 1 in Training Set Results for Jaguar pKa Calculations is treated differently. Because there is no parameterization available for that specific group, Jaguar treats the group more generally, and comes up with the best possible parameterization for it, using the training data from Table 1 in Training Set Results for Jaguar pKa Calculations. This is the essence of the “shell model” which identifies each functional group as a series of matched nested patterns of increasing specificity. The least specific shell is the generic shell. It is matched by any functional group. The most specific shells are those listed in Table 1 in Training Set Results for Jaguar pKa Calculations. Typically, the more specific the shell the more accurate its parameterization (the smaller the RMSD of its linear fit). The RMSDs and maximum deviations for these less specific shells are given in Table 1. The Jaguar pKa output file prints all the shells that match along with the pKa values predicted by the corresponding models. The list of results is arranged in the order of increasing RMSD. This means that the top shell’s result are the most trustworthy. Maestro only incorporates the pKa coming from the shell with the smallest RMSD (the top shell in the output file).

The shell model guarantees that there is a pKa parameterization model associated with any functional group. Thus, it provides an essentially 100% coverage of chemical space. The shell model is activated by the input file setting ipkashell=1, which is the default. In order to disable the shell model and use an earlier Jaguar pKa parameterization, set ipkashell=0.

In some cases it might be advantageous to develop your own parameterization for functional groups of interest—see Developing Your Own Jaguar pKa Correction Parameters.
