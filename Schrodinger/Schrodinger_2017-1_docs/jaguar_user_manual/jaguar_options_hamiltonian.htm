Hamiltonian Settings

For heavy elements, it is necessary to include relativistic effects in the Hamiltonian. In Jaguar there are two ways of doing this: using ECPs, and with an explicit relativistic Hamiltonian. ECPs are handled via the basis set selection. The Hamiltonian option menu, at the bottom of the Theory tab, allows you to choose the Hamiltonian. The default is Nonrelativistic, which is the usual Schrödinger Hamiltonian. The other choices are for relativistic Hamiltonians, and include (relativistic) in the menu item, to indicate that they include relativistic effects.

  • Nonrelativistic—Use the (nonrelativistic) Schrödinger Hamiltonian. This is the default, and should be used with ECPs. If you choose LMP2 for the level of theory, this Hamiltonian is selected, as LMP2 cannot currently be used with ZORA.

  • Scalar ZORA—Use the scalar ZORA Hamiltonian1Keyword relham = zora_scalar or zora-1c in the gen section., with local evaluation of the integrals2Keyword potential = local in the relativity section. (see The ZORA Hamiltonian for details). This option can be used for HF and DFT calculations, but not for LMP2 calculations. If you choose this Hamiltonian for LMP2, the Level of theory is reset to DFT. You can calculate energies and properties and perform geometry optimizations. The ZORA integrals are only evaluated for elements with Z > 18; lighter atoms are treated nonrelativistically.

  • Spin-orbit ZORA—Use the spin-orbit ZORA Hamiltonian3Keyword relham = zora_so or zora-2c in the gen section., with local evaluation of the integrals4Keyword potential = local in the relativity section. (see The ZORA Hamiltonian for details). This option can be used for HF and DFT calculations, but not for LMP2 calculations. If you choose this Hamiltonian for LMP2, the Level of theory is reset to DFT. You can calculate energies and properties and perform geometry optimizations. The ZORA integrals are only evaluated for elements with Z > 18; lighter atoms are treated nonrelativistically. Spin is not conserved, as the spin-orbit operator mixes states of different spin symmetry. Time-reversal or Kramers symmetry replaces spin symmetry, and is applied similarly: spin restriction is replaced with Kramers restriction.

It is recommended that you use one of the basis sets that is contracted for ZORA calculations for the heavy atoms (see Table 1). These basis sets have “zora” in the name. The dyall-v2z_zora-j-pt-gen basis set is a double-zeta polarized basis set similar to cc-pvdz; the dyall-2zcvp_zora-j-pt-gen basis set adds outer core polarization. Both basis sets are generally contracted, and cover elements up to Rn. There is a corresponding semi-segmented contraction of the Dyall basis sets (ending in -seg instead of -gen) which run 2-3 times faster than the -gen basis sets, without loss of accuracy. Semi-segmented dyall_3zvp and dyall_3zcvp basis sets, which are of triple-zeta quality, are available for the 5d elements. The sarc-zora basis set is of double-zeta quality, and is a segmented contraction, covering elements from La to Rn.
