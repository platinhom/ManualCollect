Summary

In this tab you select the transition state search method and the structures that define reactant and product, and set parameters for the search.

For information about the controls at the top and bottom of the panel, see the Jaguar Panel topic.

For recommendations on conducting a transition state search, see the topic Optimizing to Transition States with Jaguar.

Transition State Tab Features

  • Search method options
  • Structures section
  • Initial LST guess text box
  • Search controls
  • Hessian refinement controls

Search method options

Select the search method, from Standard, LST (linear synchronous transit), or QST (quadratic synchronous transit).

Standard

Standard search. Requires a guess of the transition state structure, which is the structure selected in the Use structures from option menu at the top of the panel. Sets iqst=0 in the gen section.

LST

Linear synchronous transit search. Requires reactant and product geometries, which you select in the Structures section. The transition state guess is obtained from the linear path between the two; the distance from the reactant is given in the Initial LST guess text box, which becomes available when you select this search option. Sets iqst=1 and itrvec=-5 in the gen section.

QST

Quadratic synchronous transit search. Requires reactant, transition state guess, and product geometries, which you select in the Structures section. Sets iqst=1 and itrvec=-5 in the gen section.

Structures Section

Select the transition state, reactant, and product geometries in this section for the LST and QST search methods. This section is only available when you select one of these two methods. The selections in this section add zmat2 and zmat3 sections to the input file, as appropriate for the search method.

You must choose entries from the Project Table for these structures. If the atom numbering or labeling scheme is incompatible, a warning is displayed.

  • Structure type
  • Inclusion state check box
  • Entry name text box
  • Choose button
Structure type

The structure type (Transition state, Reactant, Product) is dimmed for structures that cannot be set for the selected search method.

Inclusion state button

When you have chosen a structure for a given structure type, this button becomes available. You can click it to display the structure in the Workspace, and click again to remove the structure from the Workspace. These buttons do not behave like the In column of the Project Table: they only affect the inclusion state of the corresponding entry. The inclusion state is reflected in the Project Table.

Entry name text box

Displays the entry name for the structure type. You can enter an entry name in this text box to select it for the structure type.

Choose button

Click this button to open an entry chooser, in which you can choose an entry for the structure type.

Initial LST Guess Text Box

Enter the distance along the linear synchronous transit path between reactant and product that defines the transition state guess. The distance is measured from the reactant structure.

Search Controls

Specify the direction to search in standard and LST searches.

  • Search along option menu
  • Eigenvector text box
  • Follow same eigenvector option
Search along option menu

Choose the direction to search from the initial structure. Sets itrvec in the gen section. The options are:

Lowest Hessian eigenvector

Follow the lowest Hessian eigenvector, excluding the trivial (rotational and translational) modes. Not available for QST searches. Sets itrvec=0.

Lowest non-torsional mode

Follow the lowest non-torsional mode derived from the initial Hessian. Not available for QST searches. Sets itrvec=-1.

Lowest bond-stretch mode

Follow the lowest mode derived from the initial Hessian that corresponds to a bond stretch. Not available for QST searches. Sets itrvec=-2.

Reactant-product path

Follow the path between the supplied reactant and product structures. Not available for standard searches. Sets itrvec=-5.

User-selected eigenvalue

Follow the eigenvalue of the initial Hessian specified in the Eigenvector text box. Not available for QST searches. Sets itrvec to the value in the text box.

Eigenvector text box

Specify the eigenvector of the initial Hessian to follow in the search. Only available if you select User-selected eigenvalue from the Search along option menu.

Follow same eigenvector option

Follow the eigenvector of the Hessian that has the largest overlap with the eigenvector followed in the previous step. Sets ifollow=1 in the gen section. This option is set and cannot be changed if you select User-selected eigenvalue from the Search along option menu.

Hessian Refinement Controls

Select the Hessian refinement option to turn on Hessian refinement, and enter the number of eigenvectors of the Hessian to use in the refinement in the Low-frequency modes text box. The eigenvectors are counted from the lowest nontrivial mode.
