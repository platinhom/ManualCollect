Summary

In this panel, you can perform all the tasks necessary to set up, enumerate, and dock a combinatorial library. The panel provides the means to define the attachment points on the core-containing structure and to set up the libraries for each attachment point. You can select structures from a predefined fragment library or create a new fragment library from a set of 3D structures. Unlike the Combinatorial screening workflow, the entire combinatorial library is enumerated and, if desired, docked. Thus, this facility is not intended for use with very large combinatorial libraries.

Opening the Interactive Enumeration and Docking Panel

To open the Interactive Enumeration and Docking panel, do one of the following:

    To open the Interactive Enumeration and Docking panel, do one of the following:

  • Choose Tasks → Docking → Interactive in the main window.

  • At a Linux/Unix command prompt, enter

    $SCHRODINGER/utilities/cg_interactive_enum_dock

Interactive Enumeration and Docking Panel Features

  • Combinatorial Library tab
  • Fragment Collection tab
  • Structure count
  • Combinatorial definition table
  • Action buttons

Combinatorial Library tab

In this tab you can set up the core-containing molecule, either by importing an existing combinatorial library definition, or importing a new core-containing molecule and defining its attachment bonds.

  • Import from option menu and action button
  • Export Library button
  • Export Core button
  • Revert button
  • Clear button
  • Display area
Import from option menu and action button Browse button

Import a core-containing molecule that is used to define the combinatorial library, from one of the sources on the option menu. The sources available depend on whether you opened this panel outside Maestro or within Maestro. The action button depends on the choice made from the option menu. The choices are:

  • Maestro file—Import the core-containing molecule from a Maestro file. Click Browse to navigate to the file and select it. The first structure in the file is imported.

  • Combinatorial definition—Import a combinatorial library definition from file, including all the attachments. Click Browse to navigate to and select a combinatorial library definition (-comdef.tar.gz) file. In addition to using the current panel, a library definition can be generated in the Define Combinations step of the Combinatorial Screening panel or in the Combinatorial Library Enumeration panel.

  • Sketcher—Sketch the core-containing molecule using the 2D Sketcher. Click Sketch to open the 2D Sketcher panel. When you close this panel, the structure is converted to 3D with LigPrep and imported.

  • Workspace—Import the core-containing molecule from the Workspace. The Workspace must contain only the core-containing molecule as a single entry. Click Import to import the molecule.

  • Project Table—Import the core-containing molecule from the Project Table. The project entry for the core-containing molecule must be the only entry selected. Click Import to import the molecule.

Export Library button

Export the current combinatorial definition. Opens a file selector, in which you can navigate to the desired location and name and save the definition.

Export Core button

Export the core-containing molecule, with its core positions (attachment points) defined. You can export the molecule to a file, to the Workspace as a scratch entry, or to the Project Table as a named entry. A menu is displayed so you can choose the location for the molecule. If you choose to export to a file, a file selector opens so that you can navigate to a location and name the file.

If you opened this panel outside Maestro, the choices are not available and the file selector opens directly.

Revert button

Discard all changes made since the combinatorial library definition was loaded, and reload the original definition.

Clear button

Clears the current combinatorial library definition.

Define core area

Displays the 2D structure of the core-containing molecule, with the attachment points marked by colored arrows. The arrows are color-coded and match the rows in the Combinatorial definition table. The head of the arrow indicates the fragment or atom that will be removed when an R group is attached in this position.

You can select bonds in the structure to create attachment points (core positions). If you right-click on an existing core position, the shortcut menu allows you to switch the direction of the arrow or delete the core position.

Fragment Collection tab

In this tab you can set up a fragment collection to add at an attachment point on the core.

  • Append and Replace options
  • Import button
  • Create button
  • Columns box
  • Export button
  • Revert button
  • Clear button
  • Fragments display area
  • Apply collection to attachment point buttons
Append and Replace options

Select Append to add fragments to the existing set; select Replace to replace the existing set.

Import button

Import a fragment collection from the Workspace, the Project table, or a file. Displays a menu from which you can choose the source. If you choose to import from a file, a file selector opens in which you can locate and load the fragment collection, which must have a .bld extension . By default, the file selector opens in a directory containing a set of Schrödinger-supplied fragments that you can load and select from. The file name is displayed in the text box below the buttons. If you opened the panel from the command line, importing from file is the only choice, and the file selector opens immediately. When importing an entry group from the Project Table, the collection is named after the entry group.

The structures that are imported must have been prepared either with the CombiGlide reagent preparation procedure or by using the Create Fragment Collection panel, which you open with the Create button.

The fragments that are imported are checked for uniqueness, and any duplicates are discarded.

Create button

Create fragments from a set of structures in a Maestro file. Opens the Create Fragment Collection panel.

Columns box

Set the number of columns in the display area.

Export button

Export the current fragment collection to the Project Table or to a file. Displays a menu from which you can choose the destination. If you export to a file, a file selector opens, in which you can navigate to the location and name the file. If you opened the panel from the command line, importing from file is the only choice, and the file selector opens immediately. The default file location is My Collections. If you export to the Project Table, the fragments are exported as an entry group that is named after the collection file.

Revert button

If the fragments were imported from a fragment file or from a combinatorial library definition file, revert the selection to the contents of that file. If changes were made, you are prompted to save or discard them.

Clear button

Clear all the fragments from the current collection. If you loaded the fragments from a file and made changes, you are prompted to save or discard the changes.

Fragments display area

Displays the 2D structures of the fragments, with a filled circle indicating the connection point to the core. You can select a single structure by clicking on it; you can select multiple structures by dragging or with shift-click and control-click.

When you right-click on a structure, the shortcut menu offers the choice of deleting this structure or the selected structures, or retaining only this structure or only the selected structures.

Apply collection to attachment point buttons

Apply the fragments in the collection to the selected attachment point.

  • Apply All Fragments—apply the entire collection to the attachment point
  • Apply Selected Fragments—Apply the fragments that are selected in the Fragments display area to the selected attachment point.

Structure count

The number of structures in the combinatorial library is displayed in a noneditable text box above the Combinatorial definition table.

Combinatorial definition table

This table lists the attachments that are defined in the current combinatorial definition. The first column is an index and is color-coded to match the position in the display area of the Combinatorial Library tab. If no definition is present, the Fragment Collection tab and the Create panel can still be used to build and save fragment collections for later use. The row for the current position is highlighted in olive-green. If the associated collection is modified, the highlighting changes to red.

Attachment name Name given to the attachment on the core-containing structure. The name can be edited.
Number of structures Number of structures in the collection for this attachment. Noneditable. When the collection changes, both the number in the Fragment Collection tab and the number in the collection file are reported.
Minimum linkers Minimum number of methylene linkers to add between the core and the fragment. This value can be edited. Default is zero.
Maximum linkers Maximum number of methylene linkers to add between the core and the fragment. This value can be edited. Default is zero.
Collection file Name of the collection file used for this attachment point. To set or change the name of the file, right-click on the table row and choose Browse for new collection file. A file selector opens so that you can navigate to and select the file.

The table has a shortcut menu, which is displayed when you right-click and hold on a table row. The actions are applied to the position defined in the table row. The menu has the following items:

  • Replace Collection—Open a file selector to choose a new collection file to use at this position. The collection from the file replaces the existing collection.

  • Append to Collection—Open a file selector to choose a collection file to add to the collection at this position.

  • Apply Fragment Collection—Update the position with the entire collection as defined in the Fragment Collection tab, which may have been altered from the original.

  • Apply Selected Fragments—Reduce the fragment collection associated with this position to the fragments that are selected in the Fragment Collection tab.

  • Rename—rename the collection file.

  • Delete—remove the selected attachment point from the core.

Action buttons

There are three action buttons:

  • Enumerate and Dock—Enumerate the library and then dock all the library members. Opens a dialog box in which you can select the Glide precision for docking, specify the grid file, and if the grid has constraints, select the constraints. Only hydrogen-bond constraints are permitted. When you click OK, a Start dialog box opens, in which you can set job and output options.

  • Enumerate—Enumerate the library. A Start dialog box opens, in which you can set job and output options.
  • Dock Previously Enumerated—Dock a library that has previously been enumerated. Opens the Glide Ligand Docking panel, where you can set up the docking job. This includes selecting the library as the set of ligands to be docked: the library is not selected automatically.
