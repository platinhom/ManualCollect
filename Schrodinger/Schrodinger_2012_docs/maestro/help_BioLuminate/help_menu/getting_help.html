Getting Help

Welcome to the Maestro Online Help.

Maestro has automatic, context-sensitive help (Auto-Help and Balloon Help, or tooltips), and an online help facility:

  • The Auto-Help text box automatically displays a single line of information about the task you are performing, if this information is available. This text box is located at the foot of the main window.

  • For many GUI elements, such as buttons and options, tooltips (balloon help) is available. To display the tooltip, pause the pointer over the GUI element. The tooltip should appear within a few seconds near the pointer location. If it doesn't appear, check the Appearance tab of the Preferences panel to ensure that Show tooltips is selected.

  • The online help facility contains more detailed information. For information about a panel or the tab that is displayed in a panel, click the Help button in the panel, or press F1. The relevant help topic is displayed in your browser.

    The browser window is divided into two frames, the navigation frame and the content frame. The content frame is used to display help topics. The navigation frame has two tabs:

    • Topics—A list of topic categories is displayed by default. This list has links to a list of topics in each category, which replaces the topic list.
    • Search—The search facility allows you to search for words in various combinations. The search results have links to the topics found, and the search words are highlighted in these topics when you click these links to open them.

  • The Help menu provides access to the online help and a range of other types of documentation.

If you don't find the information you need in any of these places, try the following sources:

  • The manuals. To open a manual, choose Help > Manuals Index and follow the links in the PDF manuals index that is displayed. To search the manuals, choose Help > Search Manuals.

  • The Knowledge Base. The Help menu has a link to the Knowledge Base.

  • The Known Issues pages on the Support Center.

All the documentation, both PDF and HTML, is available from the HTML manuals index

Generally, e-mail correspondence is best because you can send machine output, if necessary. When sending e-mail messages, please include the following information:

  • A description of the problem and the operation being performed when it occurred
  • All relevant user input and machine output
  • Product purchaser (company, research institution, or individual)
  • Primary Product user
  • Computer platform type
  • Operating system with version number
  • Product version number
  • Maestro version number
  • mmshare version number

Most of this information can be obtained under UNIX by entering $SCHRODINGER/machid in a terminal window. If you are sending information about a calculation, you can obtain more information with the postmortem utility, which is described in Appendix A of the Job Control Guide.

On Windows, this information is stored in the file schrodinger_machid.txt on your desktop. If you have installed versions for more than one release, there will be multiple copies of this file, named schrodinger_machid-N.txt, where N is a number. In this case you should check that you send the correct version of the file (which will usually be the latest version).

If Maestro fails, an error report named maestro_error.txt that contains the relevant information is written to the current working directory on UNIX, and to %USERPROFILE%\Local Settings\Application Data\Schrodinger\appcrash on Windows. You should send this file to help@schrodinger.com along with a description of the circumstances of the failure. On Windows, the file maestro.EXE.dmp is also written if Maestro fails after startup. This file should also be attached to the email message to Schrödinger.
