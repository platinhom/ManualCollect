Summary

In this tab, you choose a source of structures for new cores, choose options for finding new cores, and run the job.

Find New Cores Tab Features

  • Use structures from option menu
  • Input file text box and Browse button
  • Use Phase excluded-volume file option and Browse button
  • Flexibly align new potential cores option
  • Restrict to defined CombiGlide attachment points option
  • Minimum shape similarity score box
  • Database Filters button
  • Advanced Options button
  • Start button
  • Write button
Use structures from option menu

Choose the source of structures for the potential new cores from this option menu, from Project Table (selected entries) or File.

Input file text box and Browse button

Specify the file containing the structures for the potential new cores. Click Browse to navigate to the file, which can be a Maestro file (.mae*) or an SQLite database (.sqlite). The file name is displayed in the Input file text box.

In the file selector, under Look in, there is an item for the standard core libraries in the installation, labeled Schrodinger Cores—. The actual directory is combiglide-vversion/data/core_library. The libraries are SQLite databases.

Use Phase excluded-volume file option and Browse button

Select this option to use excluded volumes from Phase to filter out new core molecules that occupy these excluded volumes. Click Browse to navigate to and select the file, which must have the extension .xvol or .ev.

There are several ways of creating excluded volume files, all of which require a Phase hypothesis. You can create the hypothesis from the template molecule by choosing Phase → Create Hypothesis, choose the template molecule, and choose at keast 3 sites for the hypothesis (which sites is not important for this purpose). The hypothesis is then available in the Manage Pharmacophore Hypotheses panel. You can then click Excluded Volumes and add excluded volumes to the hypothesis.

If you use a receptor for this purpose, it is useful to consider receptor rearrangement and flexibility when setting up excluded volumes.

Flexibly align new potential cores option

When aligning the new potential cores, allow conformational changes to produce the best alignment.

Restrict to defined CombiGlide attachment points option

Restrict the attachments to the atoms that are marked as attachment points in the protocore structure. By default any bond to hydrogen can be used for an attachment.

Minimum shape similarity score box

Set the minimum shape similarity between the protocore and the template cores for a protocore to be accepted as a match.

Database Filters button

Open the Database Filters dialog box, in which you can filter the protocores from the SQLite database that is specified as input. Only available when you choose an .sqlite file for input. The dialog box allows you to set the minimum and maximum numbers of heavy atoms, the minimum number of hydrogen-bond acceptors, hydrogen-bond donors, nitrogen and oxygen atoms, and chiral centers.

Advanced button

Open the Isosteric Matching Advanced Options dialog box, to control the parameters of the search.

Start button

Start the job to find new cores. Opens a Start dialog box, in which you can name the job and choose a host, then click Start to submit the job to the host.

Write button
Write button

Write out the input files and a shell script (.sh)to run the job. Opens a dialog box in which you can specify the job name, which is used for the script and associated files; and the host information, which is included in the commands that are written to the script. A directory named with the job name is created in the current working directory, and the files are written to this directory. The absolute paths to the files are written in the script, so you would have to edit the script if you moved the files. On Windows, you can run this script in a Schrodinger Command Prompt window after entering the sh command to start a Unix shell.
