Summary

In the Find Common Pharmacophores step, you define and select a set of variants for which you want to find common pharmacophore and perform the search for common pharmacophores. The result is a set of hypotheses for each selected variant.

Working in the Find Common Pharmacophores Step

When you enter this step for the first time, a list of all available variants in the active ligand set is computed, from the number of available sites of each type for each ligand. This list is usually less than the theoretical maximum, because the ligands don't necessarily include all possible variants. The list is then filtered with the default settings for the number of sites before it is displayed in the Variant list table. The frequencies of occurrence of the various features are used to determine how many occurrences of each feature could be found in a valid hypothesis, given the number of ligands that must be matched. These values are listed in the Available column of the Feature frequencies table.

The first task is to decide how many pharmacophore points to search for. The default for both the maximum and the minimum is 5, but you can choose any number between 3 and 7, inclusive, from the Maximum number of sites and Minimum number of sites option menus. The search then starts with the maximum number of sites, and if it does not find any common pharmacophores, it decreases the number of sites and runs the search again, until it either finds common pharmacophores or passes the minimum number of sites.

When you set the maximum number of sites, the variant list is updated automatically. Because this step is quick, you can experiment with different numbers of pharmacophore points before proceeding, or you can create runs with different numbers of pharmacophore points if you want to examine hypotheses with different numbers of points.

By default, all of the active ligands (or active ligand groups) must contain the variant for the variant to be listed. If you want to widen the search, you can set the number of active groups that must contain the variant to a number less than the total, in the Must match at least text box. When you have entered the number of ligands, the variant list is updated automatically.

Not all the variants are likely to be useful: for example, a variant with five acceptors might be physically unreasonable, and should be excluded from the search. You can limit the number of occurrences of any of the features by entering a minimum and maximum number in the Minimum and Maximum columns of the Feature frequencies table. For example, you might want variants that have between 1 and 3 acceptors. In this case you would enter 1 in the Minimum column of the A row, and 3 in the Maximum column. The changes are automatically reflected in the variant list.

When you have defined the list of variants, you can proceed to the search for common pharmacophores. At this stage, you can manually select variants from the Variant list for the search. By default, all variants are selected. You can use control-click to select or deselect a single variant, and you can use shift-click to select a range of variants. You must have at least one variant selected to run the search. If the search needs to decrease the number of sites, it forms a set of variants for the smaller number of sites based on your choices, and consistent with the constraints you applied to the feature frequencies.

The common pharmacophores are identified using a binary partitioning algorithm, in which the pharmacophores are split into progressively smaller and more similar groups based on intersite distances. If you want to change the minimum intersite distance, the final box size (the range of intersite distances for which pharmacophore sites are considered to be identical), or the number of recursions performed (maximum tree depth), you can do so by clicking Options and setting the values in the Options dialog box that is displayed.

To start the search, click Find. A dialog box is displayed, in which you can select the host, the number of CPUs to use, and the user name on the host.

The results of the search can take a large amount of disk space, depending on the number of ligands and their size and flexibility. The search results are kept inside the run, which is stored in the Maestro project. You should make sure that you have adequate disk space: in the temporary storage on the host or hosts on which you run the job, in the Maestro I/O directory, and in the project. Because of the disk space requirements, it is not advisable to run from a scratch project, which is kept by default in /home/username/.schrodinger. Instead, you should save the project to a disk that has plenty of free space.

The results of the run are displayed in the Results table, for all the variants in the list. Variants for which no hypotheses were found are not passed to the scoring step.

Define Variant List Section

In this section of the panel, you can make settings that define the variants and restrict the list of possible variants. The variant list is updated automatically whenever you make a change in this section.

  • Maximum number of sites option menu
  • Minimum number of sites option menu
  • Must match at least text box
  • Feature frequencies table
Maximum number of sites option menu
Minimum number of sites option menu

These option menus allows you to select the maximum and minimum number of sites to match (the number of pharmacophore points). The possible values range from 3 to 7, with a default value of 5. A search is conducted for each number of sites, starting from the maximum and decreasing the number until common pharmacophores are found.

Must match at least text box

This text box specifies how many of the active ligands or ligand groups ("actives") must contain the variant in order for the variant to be included in the list. The default is to use all actives. You can use less than the total by entering the number to match in the text box. The number must be between 1 and the maximum, inclusive. The maximum number of actives is displayed to the right of the text box. The fewer actives you require a match to, the more variants will be listed.

Feature frequencies table

This table lists the number of times a feature can occur in a variant, and can be used to set limits on the number of occurrences of a feature that are allowed in a variant. The columns and their use is described below.

Type   Lists the features by code letter. Noneditable.
Available   Number of sites of this type that are available, defined as the largest number of occurrences of this feature for which a match can be found for the number of ligands to be matched. For example, with ten ligands, of which three ligands have four acceptors, six ligands have five acceptors, and one ligand has six acceptors, the number of acceptors available is four if all ten ligands are matched, but is five if seven ligands are matched. Noneditable, but updated if the number of ligands to match changes.
Minimum   Minimum number of features of the given type allowed in any variant. Editable: you can set the value to restrict the possible variants. The default is zero.
Maximum   Maximum number of features of the given type allowed in any variant. Editable: you can set the value to restrict the possible variants. The default is the maximum of the number available and the number of sites.

Variant list

This list displays the possible variants, subject to the constraints specified in the Define variant list section. Before you run the job to find the common pharmacophores, you must select the desired variants from this list. You can use shift-click and control-click to select multiple variants in the list. By default, and when you make any changes in the Define variant list section, all variants are selected.

Find button

This button starts the job that finds the common pharmacophores for the selected variants. The button is unavailable if no variants are selected. When you click this button, a dialog box is displayed in which you can make job settings, such as user name, host, and number of processors.

Options button

This button opens an Options dialog box, in which you can specify parameters for the search: minimum intersite distance, maximum tree depth and final box size.

Results table

The results table shows the maximum number of hypotheses that could be produced by each variant. You can sort the table by clicking the column headers.
