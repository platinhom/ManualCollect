Summary

In this dialog box you can select options to use site-matching tolerances, required site matches ("site mask"), and permitted and prohibited matches (matching rules); and make selections or set values for each of these features. If the hypothesis already has any of these features defined, they are read when the dialog box opens and used to set options.

Opening the Matching Conditions Dialog Box

To open the Matching Conditions dialog box:

  • Click Edit in the Matching tab of the Advanced Pharmacophore Screening panel.

Matching Conditions Dialog Box Features

  • Sites table
  • Site grouping buttons
  • Constraints button
  • Apply options
  • Save To Hypothesis button

Sites table

The table lists all the sites in the hypothesis. You can make settings in the various columns of the table, which are described below.

Row

Table row index. Noneditable.

Site

Site label from the hypothesis. Noneditable.

Tolerance

Site-matching tolerance. You can enter the desired tolerance for each site in this column. The tolerances define spheres of the specified radius around the site points for each site in the hypothesis; the corresponding site point on the aligned ligand must lie within this sphere for the ligand to match the hypothesis.

Group

Index of the group that the site belongs to.

Must match

Specify the number of sites that must match the hypothesis. For ungrouped sites, a check box is displayed and a match to this site is required if the check box is selected. For grouped sites, the number of sites that must match is displayed, along with the number of sites in the group. You can cycle through the possible number of sites by clicking in the cell for one of the sites.

You must also set the number of sites required to match to a value less than the maximum in the Matching tab of the Advanced Pharmacophore Screening panel. However if you set this number to a value less than the number of sites selected here, the number used is the number of sites selected.

Permitted matches

Columns for each feature type (A, D, H, N, P, R, X, Y, Z) that indicate which type of feature can match at each site. The native feature type is always selected and cannot be deselected. For each site, you can check the boxes for the features for which you want to permit matches. For example, you might want to allow a hydrophobic site to also match an aromatic ring.

Prohibited matches

Columns for each feature type (A, D, H, N, P, R, X, Y, Z) that indicate which type of feature must not match at each site. The native feature type is always unselected and cannot be selected. For each site, you can check the boxes for the features for which you want to prohibit matches. For example, you might want to prohibit inadvertent matching of an ionic site in any molecule in the search to a hydrophobic site for which a match is not required.

Site grouping buttons

Partial matching can be applied to multiple sites by grouping them, and requiring a number of matches that is less than the minimum. Grouping is done with the site grouping buttons.

Group Sites button

Place the sites that are selected in the table into a group. Opens a dialog box that displays the number of sites in the group, and allows you to specify the minimum number of sites in the group that must match. This value must be less than the number of sites in the group. It can be changed later by clicking in the Must match column. The group is given an index, which is displayed in the Group column for each of the sites, and the table rows are given a distinguishing background color.

Ungroup Sites button

Remove the selected sites from the group that they are in. The Group column is cleared, and the Must match column is populated with check boxes for these sites. You must have all sites in a single group selected before this button becomes available.

Constraints button

Constraints can be applied to the distances, angles, or dihedral angles between sites. These constraints can be set up in the Advanced Matching - Constraints dialog box, which you open by clicking this button.

Apply options

These options turn on or off the use of the corresponding feature.

Tolerances

Apply site-matching tolerances.

Must match

Require matches to the features that are checked in the Must match column. This feature is also known as a "site mask".

Permitted / prohibited matches

Apply the feature-matching rules defined in the Permitted matches and Prohibited matches section of the table.

Constraints

Apply distance, angle, and dihedral constraints.

Save To Hypothesis button

Click this button to save the settings made in the table with the hypothesis. The settings are saved in the files hypoID.dxyz for site-matching tolerances, hypoID.mask for the site mask, and hypoID.rules for the permitted and prohibited matches. The files are saved to the project if the hypothesis was taken from the project, or to external files if the hypothesis was imported.
