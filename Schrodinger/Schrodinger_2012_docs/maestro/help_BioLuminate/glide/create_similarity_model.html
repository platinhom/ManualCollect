Summary

Use this panel to create a trained atom pair similarity model. In the standard AP similarity algorithm, all atom pair matches are weighted equally. A trained model applies different weights to different AP matches in a way that maximizes similarity between actives and minimizes similarity between actives and inactives.

Opening the Create Similarity Model Panel

To open the Create Similarity Model panel, ensure that Use trained similarity model is selected in the Similarity tab of the Ligand Docking panel, then click Create.

Using the Create Similarity Model Panel

This panel allows you the option of biasing the similarity algorithm so that not all AP matches are counted with equal weight. You will need a file of active ligands and a file of presumed inactive structures. Glide will use these sets of active and inactive ligands to determine which AP matches should count more heavily in the trained model.

For example, if a group of actives have certain atom pairs in common, those AP matches will be weighted higher than average. Atom pair matches that occur between actives and inactives will be weighted lower. By maximizing similarity between actives while minimizing similarity between actives and inactives, the trained model focuses on ligands that are more similar to actives and less similar to inactives than in the standard (untrained) similarity model.

When you have specified the actives and inactives to use, click Start to open the Start dialog box. The default job name is glide_similarity. When you have specified the job options, click Start to launch the job.

When the job finishes, the name of the model file that it generates appears in the Model file text box in the Similarity tab of the Ligand Docking panel.

Create Similarity Model Panel Features

  • File of actives options
  • File of inactives options
  • Use random set of inactives check box
  • Approx. number to use text box
File of actives options

You need a file of known active ligands to use any form of similarity in a docking job. The same file you specified in the Similarity tab can also be used in this panel. The file must contain at least two known active ligand structures. Best results are obtained with at least 10, but more than 20 are unnecessary and may increase calculation time. To take full advantage of the weighting algorithm, include actives with a wide variety of structures.

Specify the actives file by typing the filename in the File of actives text box or by clicking the Browse button to open a file selector and navigate to the file. The file must be in Maestro, SD, or MOL2 format.

File of inactives options

To create a trained similarity model, you also need a file of structures presumed to be inactive. This may include untested compounds which are very unlikely to be active.

The inactives file must contain at least one ligand-like structure, but more will improve results. Using 5 to 10 times the number of actives is optimal. Typical numbers might be 15 structures in the actives file and 150 structures in the inactives file. If your inactives file is much larger than about 200 structures, or 10 times the number of actives, using all the structures in the inactives file can lead to convergence problems in the training algorithm. To avoid this difficulty, use the remaining options in the panel to specify that a random subset of inactives should be used.

Specify the inactives file by typing the filename in the File of inactives text box or by clicking the Browse button to open a file selector and navigate to the file. The file must be in Maestro, SD, or MOL2 format.

Use random set of inactives check box

Select this option when the number of structures in your inactives file is much greater than 10 times the number in the actives file. Glide will choose a random subset from the structures in the inactives file.

Approx. number to use text box

When you instruct Glide to choose a random subset of the structures in your inactives file, you must specify approximately how many structures to include. Accept the default setting, 100 inactives, or type another number. Optimally, the number of inactives should be 5 to 10 times the number of actives.
