Creating and Formatting Atom Labels Using Maestro Commands

Atoms can be labeled by entering Maestro commands in the Command Input Area or the Command Script Editor panel. When entering an atom label command, you have two options. First, you may choose to define a label prior to the time you need it. This has the same effect as setting a default. The command syntax for defining a label is:

labelatom  options

where options is a combination of commands for adding or deleting fields the label and formatting the label. For example, to add the partial charge to the label, you would use the command

labelatom 1charge=yes

and to change the format of the partial charge in the label to use one decimal digit and left-justify the value you would use the command 

labelatom 1chargeformatstring=%C1-3.1

The format of the descriptor (in this example, the descriptor is %C1-3.1) is given in the next section, along with a table that provides information on Maestro properties, the names of the commands for including the property, and information for constructing the descriptor.

When defining labels in this way, atom labels can be applied at any subsequent time by simply entering an ASL expression with the labelatom command:

labelatom  ASL-expression

If you choose not to label atoms ahead of time, you will need to define the label options when actually applying the labels. The command syntax for this is:

labelatom  options ASL-expression

For a list of available commands, see the description of the labelatom command in the Maestro Command Reference Manual.

You might want to have more control over label composition than is available from the Atom Labels panel or with the labelatom command. In particular, you may want to change the order of the fields in the label as well as the format of the numerical fields. The labelformat command can be used to give greater control over the atom labels. The syntax is:

labelformat format

When the labelformat command is issued, the Atom Labels panel is updated to reflect the fields that are included in the format string. Note that these fields cannot be removed from the label by using the Atom Labels panel, though fields can be added to the label.

The format string can be made up of any combination of the following descriptors. Note, however, that each descriptor appears only once and cannot be repeated. Descriptors have the form:

%field[-]width[.precision]

where the field name is represented by a mnemonic two-letter code, the width and precision are user-defined, and the precision can only be assigned for floating point fields (fields that may have non-integer values). The "-" in front of the width indicates that the field is to be left-justified. Any width that is entered will be considered to be the minimum width. If an item in that field is larger than the specified field width, all characters will still be displayed in the label. In other words, fields will never be truncated to fit inside width number of characters. If a field item is less than width number of characters, spaces will be added so that the item fills the entire specified width. (Note that the nomenclature follows that used for the C language printf command.)
