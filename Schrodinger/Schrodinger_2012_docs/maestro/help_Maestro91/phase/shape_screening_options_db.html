Summary

In this dialog box you can select a variety of options to control the process and results of a shape screen.

Opening the Shape Screening - Options Dialog Box

To open the Shape Screening - Options dialog box:

  • Click Options in the Shape Screening panel.

Write out at most N alignments per ligand text box

Enter the number of alignments per ligand to write out for each query in this text box. Redundant alignments are eliminated on a maximum atom deviation criterion of 0.5 Å. The default is to return only one alignment. Alignments are kept together in the output, and the alignments for a given query and molecule are ordered by similarity to the query.
(Sets -report for phase_shape.)

Use excluded volumes option, text box, and Browse button

Select this option if you want to use excluded volumes in the search. Enter the full path to an excluded volumes file in the text box, or click Browse and navigate to the excluded volumes file.

Include polar hydrogens option

Select this option if you want polar hydrogens (which are almost always donors) to be considered in the shape search. This option is not available if you chose Pharmacophore types for volume scoring in the Shape Screening panel.
(Sets -hydrogens for phase_shape.)

Filter out conformers with similarity below option and text box

Filter out conformers whose similarity to the query is lower than the value given in the text box.
(Sets -filter for phase_shape.)

Create Phase subset file from hits option

If you are searching a Phase database, select this option to create a Phase database subset from the matches to the query.
(Sets -osub for phase_shape.)

Score in place option

Select this option to compute similarities without aligning the molecules to the queries.
(Sets -scoreInPlace for phase_shape.)

Screen a subset only option, text box, and Browse button

If screening a Phase database, screen only the specified subset. Enter the full path to the Phase database subset file in the text box, or click Browse and navigate to the database subset file. The file must end in _phase.inp.
(Sets -isub for phase_shape.)

Sort output structures by decreasing similarity option

Select this option to sort the output. By default the output is not sorted. The sort takes about 2K of memory per output structure. The limit on the number of output structures is given in the Maximum number of structures to save text box; by default it is 10000, which corresponds to about 20MB memory used for the sort.
(Sets -sort for phase_shape.)

Maximum number of structures to save text box

Limit the number of structures to save when sorting by decreasing similarity to the value given in this text box. The default is 10000.
(Sets -keep for phase_shape.)

Similarity normalization option menu

Choose a scheme for normalizing the similarity. The similarity is defined by Sim(A,B) = O(A,B)/f[O(A,A),O(B,B)], where O is the overlap between the query A and a molecule B that is being screened. The function f can be one of four choices, which are given on the option menu:

  • Maximum—Normalize to the maximum of O(A,A) and O(B,B)
  • Minimum—Normalize to the minimum of O(A,A) and O(B,B)
  • Query structure—Normalize to O(A,A)
  • Screen structure—Normalize to O(B,B)

(Sets -norm for phase_shape.)

Use custom weights for atoms option and section

Select this option to supply weights for the atomic volumes used to compute overlaps. This option and section is not available if you chose Pharmacophore types for volume scoring in the Shape Screening panel. The weights are specified by choosing an atom-level property for the queries, and must be in the range 0–1. The controls for specifying the weights are:

Property for weights option menu

Update the values in the table from the chosen property. The property is reported in the table for each atom, for which various labels are also given. The only properties listed on this menu are real-valued properties whose values are in the range 0–1.

Update button

Update the list of properties from the query input, as displayed in the Workspace.

Set the weight of selected rows to text box and Apply button

Set the weight for the rows that are selected in the weights table to the value given in this text box, when Apply is clicked.

Weights table

Table listing the values of the selected atom property for each atom in the Workspace. The columns give the atom number, the element, the PDB atom name, and the value of the weight. You can change the weights by selecting the desired rows, then using the Set the weight of selected rows to text box and Apply button.

(Sets -atomWeights for phase_shape.)

Create checkpoint option, text box, and Browse button

Create checkpoint files for the job in the specified directory. Enter the path in the text box, or click Browse and navigate to the desired location.
(Sets -CHECKPOINT for phase_shape.)
