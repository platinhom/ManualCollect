Summary

Use the options in the Similarity tab to set up and incorporate atom pair similarity scoring in your docking job. Similarity algorithms provide a mechanism for quantifying how alike or unlike two molecules are. This similarity can be a criterion for selecting likely candidates— molecules similar or dissimilar to a collection of probe molecules (typically known actives)—from a large molecular database.

Similarity or dissimilarity scoring is combined with the results of Glide docking to produce a composite score. By default, no similarity scoring is performed.

Glide uses an atom-pair based similarity scoring algorithm which is dependent only on 2D molecular structures. In calculating an atom-pair similarity between two molecules, their structures are first processed to generate sets of atom pairs. Each non-hydrogen atom is represented by a similarity type (not the same as the atom typing used in Liaison or QSite) based on the connectivity, bond orders, and formal charges of the molecule. For each pair of similarity types, the shortest bonded path between the two atoms (the path with the smallest number of connections) is determined. The unique combination of:

type(atom A) + connectivity distance + type(atom B)

defines one atom pair. All atom pairs for a given molecule constitute the atom pair list for that molecule. The similarity between two molecules is a function of the number of atom pairs that appear on both lists. The similarity function is normalized so that the similarity score is a number between 0.0 (no atom pairs in common) and 1.0 (identical atom pair lists.)

Mode Options

The default is not to use similarity scoring. You can use similarity scoring to find ligands similar to known actives or to find ligands that are dissimilar to them:

  • Do not use similarity
  • Find similar ligands
  • Find dissimilar ligands

Do not use similarity

This option is selected by default. The docking job docks and scores ligands without calculating or using similarity scores.

Find similar ligands

Select this option to penalize ligands that are structurally different from known actives. In the results of the docking job, ligands that score well and are similar to known actives will be ranked high.

Find dissimilar ligands

Select this option to penalize ligands that are structurally similar to known actives. In the results of the docking job, ligands that score well and are different from known actives will be ranked high.

File of Known Actives

Specify a file with known active ligands, either by typing the file name in the File of known actives text box or by clicking Browse to use a file selector. The file must be in Maestro, SD, or MOL2 format.

Use Trained Similarity Model

The default, the untrained similarity model, assigns equal significance to all atom pair matches. A trained similarity model assigns greater significance to some atom pair matches than to others. The weighting of atom pair matches requires a file of active ligands and a file of presumed inactives. Glide uses the two files to determine the set of weights that maximizes active-active similarity scores and minimizes active-inactive similarity scores.

Use the options in this group to apply a particular trained similarity model by specifying the weights file to apply. If you have not yet created a weights file, but would like to use a trained model, you can generate a new weights file in the Create Similarity Model panel, which is opened from this tab.

  • Use trained similarity model option
  • Model file text box and Browse button
  • Create button

Use trained similarity model option

Select this option to employ a set of weighted atom pairs, creating a trained similarity model. If you do not select this option, the standard, unweighted similarity model will be used.

Model file text box and Browse button

A trained similarity model is the combination of the similarity algorithm with a set of weighted atom pairs. The weighted atom pairs are stored in a file named filename.wgt. For simplicity, this file is sometimes referred to as the trained similarity model file. If you already have a trained similarity model file, type the file name in the text box, or click the Browse button to open a file selector.

Create button

To generate a trained similarity model, click Create to open the Create Similarity Model panel, in which you can set up a job to generate a set of optimized weights for a trained model. See the Create Similarity Model topic for details. When the weight generation job finishes, the set of weights is written to the trained similarity model file, jobname.wgt. The file name and path appear in the Model file text box.

Similarity Scoring Function

Similarity information is incorporated into docking results according to a scoring function that assigns a GlideScore penalty based on a ligand's similarity score, which has been normalized to a value between 0.00 and 1.00. The general shape of the scoring function depends on your choice of similarity Mode. If you have selected Find similar ligands, penalties decrease with increasing similarity. If you have selected Find dissimilar ligands, penalties increase with increasing similarity.

The details of the scoring function are controlled by parameters that you can change if necessary. This section of the Similarity tab contains the four parameter settings for the Similarity scoring function. Beneath these options, a plot of the scoring function is displayed. The plot is updated each time you enter a new value for one of the parameters.

These are the features of this section of the Similarity tab. In describing features that differ according to similarity mode (Find similar ligands or Find dissimilar ligands), the default is assumed to be Find similar.

  • Maximum GlideScore penalty p kcal/mol
  • Reject below r
  • Partially penalize between m and n
  • Similarity scoring function plot

Maximum GlideScore penalty p

This is the maximum penalty for ligands with low similarity scores (or, if Find dissimilar ligands is selected, for ligands with high similarity scores.) The relative importance of the similarity score and the basic GlideScore to the combined result can be adjusted by changing this number. As the maximum penalty is increased, similarity becomes more important than docking; decreasing the maximum penalty has the reverse effect. The default value of p, 6.00 kcal/mol, gives roughly equal weighting to the similarity score and GlideScore for most systems, which is generally optimal.

Reject below r

Ligands with similarity scores below the threshold r will not be docked. The default value of r is 0.00. Since similarity scores cannot be negative, the default is to dock all ligands regardless of similarity.

If the mode is Find dissimilar ligands, the option is Reject above: ligands with similarity higher than r will not be docked. The default is 1.00, a score that cannot be exceeded, and therefore corresponds to docking all ligands.

Partially penalize between m and n

Ligands with similarity scores lower than m receive the maximum penalty. Those with similarity higher than n receive no penalty. Similarity scores between m and n receive a penalty that decreases linearly from the maximum value at m to zero at n, as shown in the scoring function plot. The default value of m is 0.30. The default value of n is 0.70.

In Find dissimilar ligands mode, the default values of m and n remain the same, but no penalty is applied to ligands with similarity scores of m or lower. Scores between m and n receive increasing penalties, reaching the maximum penalty at n and above.

Similarity scoring function plot

GlideScore penalties ranging from 0.0 to the the maximum, p, are plotted against similarity score, which ranges from 0.0 to 1.0. The plot is updated to reflect a change in the value of a parameter (p, r, m, or n) when you press ENTER or click in the next parameter text box.
