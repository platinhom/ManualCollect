Summary

Create groups according to the value of an entry property. All entries with the same value of the property or same range of property values are assigned to the same group. For boolean properties, two groups are created, one for TRUE values, and one for FALSE values. For string properties, a group is created for each unique string. For real and integer properties, groups are created that span a specified range of property values.

This panel is intended for the creation of multiple groups. To create a single group based on property values, use the Entry Selection dialog box to select the entries, then create a group.

Opening the Group By Entry Property Panel

To open the Group By Entry Property panel, choose Group By Entry Property from the Entry menu or shortcut menu of the Project Table panel.

Group By Entry Property Panel Features

  • Base name for group text box
  • Property list
  • Entries to group options
  • Group definition tools
  • Group button
Base name for group text box

Enter the base name for the group in this text box. The name and value of the property that defines each group is added to the base name to form the group name. For real and integer properties, the property range is used instead of the property value.

Property list and selection tools

Select the property whose values will be used to define the groups, using the property selection tools. When the property is selected, a set of tools in the Filter and Group by section is displayed that is appropriate for the type of property (real, integer, boolean, string).

Entries to group options

Select the class of entries to group. The options are all entries or the entries that are selected in the Project Table.

Group definition tools

This section displays text and tools to control the size of the groups, depending on the type of property.

For integer and real properties, the entries are sorted into contiguous bins of a specified size, based on the property value. The bin size, the alignment of the bins, and the range of properties considered can be specified in the four text boxes that are displayed, three of which are optional:

  • Step Size—Size of the bins used to divide the entries into groups. The bins include the lower end of the range of values but exclude the upper end: thus a bin of size 2.0 for a real property starting at 2.0 would cover the range 2.0 ≤ value < 4.0; for an integer property a bin of size 2 starting at 2 would include the values 2 and 3.

  • Align on value—Reference point for the lower end of each bin. The lower end of each bin is set to this value plus (or minus) a multiple of the step size. For example, if the reference point is 5 and the step size is 3, there will be bins covering the ranges 2–4, 5–7, 8–10, and so on. If this option is deselected, the reference point is the minimum of all values of the property in the entries that are to be grouped. This option is on by default, with the value zero.

  • Minimum—Lower limit for property values to be considered. Entries whose value of the selected property is less than this limit will not be grouped.

  • Maximum—Upper limit for property values to be considered. Entries whose value of the selected property is greater than this limit will not be grouped.

Group button

Group the entries according to the selected property.
