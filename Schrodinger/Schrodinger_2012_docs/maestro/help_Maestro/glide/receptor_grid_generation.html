Summary

The Receptor Grid Generation panel is used to specify a receptor structure and set up the grid generation job. This job creates the grid files, which represent the active site of the receptor for Glide ligand docking jobs.

Opening the Receptor Grid Generation Panel

To open the Receptor Grid Generation panel, do one of the following:

  • Choose Applications → Glide → Receptor Grid Generation.

  • Choose Tasks → Docking → Grid Generation.

Using the Receptor Grid Generation Panel

Glide (Grid-based Ligand Docking with Energetics) searches for favorable interactions between one or more typically small ligand molecules and a typically larger receptor molecule, usually a protein. The shape and properties of the receptor are represented on a grid by several different sets of fields that provide progressively more accurate scoring of the ligand poses. The options in each tab of the panel allow you to define the receptor structure by excluding any cocrystallized ligand that may be present, determine the position and size of the active site as it will be represented by receptor grids, and set up Glide constraints. Ligand docking jobs cannot be performed until receptor grids have been generated.

Receptor grid generation requires a "prepared" structure: an all-atom structure with appropriate bond orders and formal charges. In most cases, the preparation process can be performed automatically using the Protein Preparation Wizard.

With the prepared structure in the Workspace, use the Receptor Grid Generation panel to set up the receptor grid generation job. When you have completed your setup, you can click the Start button in the lower left corner of the panel. This opens the Receptor Grid Generation - Start dialog box, which has two sections:

  • Output
  • Job

The Output section contains the Directory for grid files text box and Browse button for specifying the directory to which output grid files are written. By default, grid files are written to the current working directory. It also contains a Compress option, which can be used to compress the grid-related files into a .zip archive.

The Job section contains the standard controls, Name, Host, and Username, which are described in the Start Dialog Box topic. In addition, it has a Standard Names option menu, which allows you to choose a name that is composed of various parameters from the calculation, and a Compose button, which allows you to select a file name or a job name from the current project, and edit it to form a job name. You can also set the title of the entry in the Entry Title text box.

Alternatively, you can click Write to write the input files without starting the job. The dialog box that opens has the same Standard Names option menu, Name text box, and Entry title text box, and Compress option that appear in the Start dialog box. It also has a Browse button that you can use to navigate to a location for the input file.

To restore the default settings in all tabs, click Reset.

Receptor Grid Generation Panel Features

The Receptor Grid Generation panel contains four tabs, which are described in full in separate topics:

  • Receptor tab— Use this tab to define the receptor (by identifying the Workspace ligand, if one is present) and optionally to scale the van der Waals radii of receptor atoms.

  • Site tab— Use this tab to determine the position and size of the volume for which grids will be generated, representing the active site of the receptor.

  • Constraints tab— Use this tab to designate certain receptor atoms (H-bond/metal constraints) or regions (hydrophobic constraints) for ligand-receptor interactions that you can then choose to require during docking jobs.

  • Rotatable Groups — Use this tab to specify groups that should be treated as rotatable in the grid generation. These groups are currently restricted to hydroxyls in Ser, Thr, and Tyr, and thiols in Cys.

  • Excluded Volumes— Use this tab to set up regions of space in which the ligand is penalized during docking.

In addition, there are three action buttons at the bottom of the panel:

  • Start—opens the Start dialog box, in which you make job-related settings and start the grid generation job. See the Start dialog box topic for more information.

  • Write—write a Glide input file to the current working directory, but do not run the job. You can then run the job from the command line with the glide command.

  • Reset—Resets the options in all the tabs to the default values.
