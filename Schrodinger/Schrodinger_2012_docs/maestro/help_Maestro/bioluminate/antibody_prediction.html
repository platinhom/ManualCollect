Summary

This panel provides a set of tools for prediction of the variable loop (CDR) region of an antibody. It includes:

  • Automated identification of the six CDR region loops based on sequence
  • Prediction of the loops using lookup tables and rules derived from an analysis of antibodies in the PDB (based on a new analysis from 2010)
  • Optional refinement of the loops where lookup is difficult, primarily for H3 and possibly H2.

Opening the Antibody Prediction Panel

To open the Antibody Prediction panel:

  • Choose Applications → BioLuminate → Antibody Modeling → Prediction in the main window.

Using the Antibody Prediction Panel

The basic procedure for running a prediction using a homology model is as follows:

  1. Import the sequences for the light and heavy chains: Click the part of the diagram for the region you want to import, and choose a source from the list that is displayed.

    A dialog box is displayed, in which you can provide the sequence, either directly or from a structure. The diagram is colored for the loop that you choose once you have imported a sequence.

  2. Set up the antibody databases that you want to use to search for homologs for the framework region and the loops. Click Configure Active Databases in the Framework Search tab, add databases and select them in the table.

  3. Select Homology and run the homology search by clicking Search.

    The results are listed in the table.

  4. Select the homolog you want to use for the framework region, and click Accept.

  5. (Optional) In the Basic Loop Model tab, select the loop cluster you want to use for each loop in the model by clicking View Clusters, and choosing a cluster in the Loop Clusters panel.

    By default a cluster is chosen automatically, based on similarity, then cluster size.

  6. (Optional) Set the number of models of the antibody that you want to generate.

  7. Click Generate Initial Models to generate the models.

  8. (Optional) If you think that the H3 loop needs further refinement, select it in the Advanced Loop Model tab, and click Run Prime. See the guidelines below.

The models are added to the Project Table.

You can also use coordinates from existing structures instead of homology models, for example if you want to vary just one of the loops. The procedure is similar to that given above.

  1. Import the sequences for the light and heavy chains: Click the part of the diagram for the region you want to import, and choose a source from the list that is displayed.

    A dialog box is displayed, in which you can provide the sequence, either directly or from a structure. The diagram is colored for the loop that you choose once you have imported a sequence.

  2. Set up the antibody databases that you want to use to search for homologs for the loops. Click Configure Active Databases in the Framework Search tab, add databases and select them in the table.

  3. Select Input coordinates and click Accept.

  4. In the Basic Loop Model tab, select the check box in the Keep Input column for the loops you want to take from the input structure.

  5. (Optional) In the Basic Loop Model tab, select the loop cluster you want to use for each loop that you model by clicking View Clusters, and choosing a cluster in the Loop Clusters panel.

    By default a cluster is chosen automatically, based on similarity, then cluster size.

  6. (Optional) Set the number of models of the antibody that you want to generate.

  7. Click Generate Initial Models to generate the models.

  8. (Optional) If you think that the H3 loop needs further refinement, select it in the Advanced Loop Model tab, and click Run Prime. See the guidelines below.

In many cases, the model based on homology is adequate, and no further prediction is needed. The H3 loop, however, is often difficult to predict, and may need more advanced refinement. Some general guidelines for when to refine a loop with Prime in the Advanced Loop Model tab are given below:

  • If the loop is long (9 residues or more) and the homology is good (similarity above about 80%) then refinement with Prime is not usually necessary.

  • If the loop sequence similarity is less than 40%, the basic model quality usually is very poor and a Prime refinement is recommended.

  • The quality of the Prime refinement is greater for shorter loops.

  • When building a new H3 loop with new sequences on the native (crystal) structure of an antibody, Prime refinement is recommended.

Antibody Prediction Panel Features

  • Input area
  • Framework Search tab
  • Basic Loop Model tab
  • Advanced Loop Model tab
  • View Sequences button

Input area

This area displays a diagram of the antibody. Clicking on the light variable or heavy variable region in the diagram displays a menu, from which you can choose the source of the sequence for this region. The choices are:

  • Browse for File—Open a file browser in which you can navigate to the desired location and select the file that contains the sequence.
  • From Workspace—Use the sequence for the structure that is displayed in the Workspace.
  • From Selected Entries in the Project Table—Use the sequence from the entries that are selected in the Project Table.
  • From PDB ID—Use the sequence from the specified PDB ID. Opens the Enter PDB ID dialog box, in which you can enter the PDB ID of the sequence. The sequence is retrieved from a local copy of the PDB if it is available, or from the RCSB web site, depending on the preference set for PDB retrieval.
  • Enter/Paste New Sequence—Type or paste in the sequence. Opens the Sequence Editor dialog box, in which you can name the sequence and type it or paste it in, as a string of single-letter codes.

Framework Search tab

In this tab you define the coordinates that will be used for the framework regions. You can either use input coordinates from an existing structure, or search for homologs for the heavy and light chains in your choice of antibody databases.

Configure Active Databases button

Select the databases that you want to use for homology modeling. These databases are the ones that are searched for templates, and are used for both the framework regions and the CDR regions of the variable domains. Opens the Database Configuration dialog box, in which you can add databases to the list, remove them from the list, and mark them as active. All databases that you select as active are searched when you run a search for homologs.

Source of coordinates options

Choose the source of the coordinates for the framework region. The options are:

  • Homology—Use homology modeling to determine the coordinates. When this option is selected, the Search and Define Search Limits buttons are active.
  • Input coordinates—Use the coordinates of the input structure for the framework region. The sequences that you imported for the light and heavy regions must be associated with a structure.

Search button

Run a homology search for the framework region. The results are listed in the search results table.

Define Search Limits button

Click this button to filter the search results by setting criteria on the available properties that must be met by the templates. Opens a dialog box in which you can set up the criteria to be met by the properties you select.

Search results table

This table lists the results of the homology search. You can select a single result and click Accept to accept it as the template. The table columns are described below.

Heavy PDB ID of the homolog for the heavy framework region
Light PDB ID of the homolog for the light framework region
Composite Score Average of the Heavy Sim. and Light Sim. scores. This score is used to order the homologs in the table.
Heavy Sim. Sequence similarity of the entire variable domain sequence (framework and CDR) for the heavy chain. The similarity is the number of matching residues divided by the total number of residues, where "matching" means that the two residues have a positive score in the BLOSUM62 matrix.
Light Sim. Sequence similarity of the entire variable domain sequence (framework and CDR) for the light chain. The similarity is the number of matching residues divided by the total number of residues, where "matching" means that the two residues have a positive score in the BLOSUM62 matrix.
Accept button

Accept the input coordinates or the row that is selected in the table as the template for the framework region. After clicking this button, the Basic Loop Model tab is displayed.

Basic Loop Model tab

In this tab you select initial input for predicting the six loops and run a basic loop prediction. The H3 loop should be predicted in this tab and then refined in the Advanced Loop Model tab.

Input selection table

This table lists the six CDR loops and provides checkboxes for selecting the source of input coordinates. The table columns are as follows:

CDR Loop Label of the CDR loop.
Antibody Loop Database Column of checkboxes for using the antibody loop databases to predict the loop.
Keep Input Use the input coordinates for the loop, if the antibody sequence was imported from a PDB structure with coordinates available.
Column containing View Clusters buttons, which open the Loop Clusters dialog box. This dialog box shows you the clusters for each loop, and allows you to select a cluster to use in the model. It also allows you to set a similarity cutoff for the loop to be used in the model.
Number of models box

Set the number of models that you want to build, by entering a number in the box or using the arrow buttons.

Generate Initial Models button

Generate the initial models of the antibody.

Model to view option menu

If you generated multiple models, choose the model to view, either in the Workspace or in the Structure Quality Viewer. This option menu is also used to select the model for Prime loop prediction in the Advanced Loop Model tab.

View in Workspace button

View the model selected from the Model to view option menu in the Workspace. The model is colored by residue with the following color scheme:

  • Blue—residues for which the full residue conformation was copied from the template.
  • Cyan—residues for which the residue backbone conformation was copied from the template, and the side chain was modeled (because there was a residue mutation in the template relative to the query).
  • Red—residues for which both the backbone and the side chain were modeled.
  • Maroon—residues in the CDR loops.

View in Structure Quality Viewer button

View the models in the Protein Structure Quality Viewer. All models are listed in the table at the top of the viewer, and the structures are colored according to the Ramachandran plot regions.

Advanced Loop Model tab

In this tab you can run a more accurate and detailed loop prediction. These predictions can take hours for each loop. It is not usually necessary to run an advanced loop prediction for any of the loops except the H3 loop. The H3 loop is selected by default for an advanced prediction, as it is the hardest loop to predict. The results are returned to the Project Table as new structures.

The input structure for the loop prediction is always the structure that came from the Basic Loop Model tab. This means that you cannot do sequential loop predictions in this panel, but you can use the Refinement panel to predict more than one loop. However, it is expected that the main use of this tab is to predict a single loop.

See Using the Antibody Prediction panel for general guidelines on refining a loop in this tab.

Loop selection table

This table lists the six CDR loops and provides checkboxes for selecting a loop to refine with Prime. The table columns are as follows:

CDR Loop Label of the CDR loop.
Prime Loop Prediction Use this column to select the antibody loop to predict with Prime. Only one loop can be predicted at a time.
Run Prime button

Run a Prime loop prediction job for the selected loop. Opens a dialog box, in which you can name the job, select a host, and specify the maximum number of processors to use. The single, best loop prediction is returned. The Prime loop prediction algorithm is automatically selected based on the length of the loop.

Model to view option menu

If you generated multiple basic models, choose the model to view and to use in the Prime loop prediction. Viewing can be done either in the Workspace or in the Structure Quality Viewer.

View in Workspace button

View the model selected from the Model to view option menu in the Workspace. The model is colored by residue with the following color scheme:

  • Blue—residues for which the full residue conformation was copied from the template.
  • Cyan—residues for which the residue backbone conformation was copied from the template, and the side chain was modeled (because there was a residue mutation in the template relative to the query).
  • Red—residues for which both the backbone and the side chain were modeled.
  • Maroon—residues in the CDR loops.

View in Structure Quality Viewer button

View the models in the Protein Structure Quality Viewer. All models are listed in the table at the top of the viewer, and the structures are colored according to the Ramachandran plot regions.

View Sequences button

Display the sequences of the variable heavy and variable light regions for both the query and the template, in the Antibody Prediction - Sequence Viewer panel. The sequence viewer has a facility for finding patterns in the sequence using PROSITE patterns, and a checkbox for each sequence to display it in the Workspace. The sequence is annotated with the secondary structure assignment (SSA), indication of S-S bonds, and identification of the CDR regions (in red).
