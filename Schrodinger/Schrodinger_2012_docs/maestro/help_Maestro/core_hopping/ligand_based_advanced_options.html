Summary

In this dialog box you can set options for the attachment-based core hopping method.

Opening the Ligand-Based Advanced Options Dialog Box

To open the Ligand-Based Advanced Options dialog box:

  • Click Advanced in the Find New Cores tab of the Ligand-Based Core Hopping panel.

Ligand-Based Advanced Options Dialog Box Features

  • General options
  • GPGPU options
  • Sampling options
  • Cutoff options
General options

  • Maximum linkers—Specify the maximum number of methylene liners to use to obtain a good alignment. The maximum you can specify is 3. Default: 2.
  • Verbosity—Specify the amount of output to write to the log file.
  • Allow structures containing Glide linker grownames—Allow the use of protocores with linkers that were generated by receptor-based core hopping, which involves Glide docking. The linker labeling in these protocores is ignored if you select this option, otherwise their existence is considered an error, and the job fails.
GPGPU options

The GPGPU options allow you to use CUDA-capable graphics processors to perform the work, which can speed up the calculation. You must run the job locally to take advantage of the GPGPU. The options are

  • Use all available CUDA devices—Use all the CUDA devices that are available.
  • Use single CUDA device—Use the CUDA device selected from the accompanying option menu.
  • Disable CUDA—Do not use the GPGPU.

Sampling options

The sampling options, listed below, should not need to be adjusted. If you have questions about these options, contact help@schrodinger.com for information.

  • Star frequency
  • Translations
  • Rotations
  • Maximum chord discrepancy
  • Maximum side-chain RMSD

Cutoff options

The cutoff options set criteria for hydrogen bonds to the receptor, if present, or for matching donor or acceptor atoms in the template.

  • Receptor H-bond cutoff—maximum length of a hydrogen bond to the receptor, if present.

  • Template acceptor/donor atom cutoff—Maximum distance from a designated template acceptor or donor atom for a corresponding protocore acceptor or donor atom to be considered a match, when requiring acceptor/donor matches.
