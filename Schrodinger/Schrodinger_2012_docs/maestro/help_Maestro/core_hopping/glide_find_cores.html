Summary

In this tab, you choose a source of structures for new cores, choose options for finding new cores, and run the job.

Find New Cores Tab Features

  • Use structures from option menu
  • Input file text box and Browse button
  • Align protocores option
  • Dock aligned protocores option
  • Add N linkers box
  • Glide grid text box and Browse button
  • Define Constraints button
  • Database Filters button
  • Start button
  • Write button
Use structures from option menu

Choose the source of structures for the potential new cores from this option menu, from Project Table (selected entries) or File.

Input file text box and Browse button

Specify the file containing the structures for the potential new cores. Click Browse to navigate to the file. The file name is displayed in the Input file text box.

If you are aligning a set of protocores (with or without docking), you can choose a Maestro file (.mae*) or an SQLite database (.sqlite) for the input. In the file selector, under Look in, there is an item for the directory containing the standard core libraries in the installation, labeled Schrodinger Cores—. These libraries are SQLite databases. (The actual directory is combiglide-vversion/data/core_library.)

If you already have some aligned protocores and now want to dock them (Align protocores is not selected), you must choose a file that contains the output from an alignment-only job, or, if the alignment-only job was incorporated into the project, you can choose the entries in the Project Table from this job.

Align protocores option

Align the protocores to the template molecule core. Glide constraints can be applied during the alignment—see Define Constraints below. The alignment is based on the orientation and position of the side chains of the protocores (with or without linkers), and any selected Glide constraints, which are applied to the protocore (without its side chains).

Dock aligned protocores option

After the side chains of the template have been added to the aligned protocores, dock these "dressed" structures to the receptor, applying any constraints that have been selected (see Define Constraints below).

Add N linkers box

Specify the maximum number of methylene linkers to add when aligning the protocores to the template.

Glide grid text box and Browse button

  • Specify the Glide grid to be used for constraints during alignment and for docking the aligned and dressed protocores.

    • Define Constraints button

    Open the Select Glide Constraints dialog box, to select the constraints that are defined with the Glide grid for use in each stage.

    Database Filters button

    Open the Database Filters dialog box, in which you can filter the protocores from the SQLite database that is specified as input. Only available when you choose an .sqlite file for input. The dialog box allows you to set the minimum and maximum numbers of heavy atoms, the minimum number of hydrogen-bond acceptors, hydrogen-bond donors, nitrogen and oxygen atoms, and chiral centers.

    Start button

    Start the job to find new cores. Opens a Start dialog box, in which you can name the job and choose a host, then click Start to submit the job to the host.

    Write button
    Write button

    Write out the input files and a shell script (.sh)to run the job. Opens a dialog box in which you can specify the job name, which is used for the script and associated files; and the host information, which is included in the commands that are written to the script. A directory named with the job name is created in the current working directory, and the files are written to this directory. The absolute paths to the files are written in the script, so you would have to edit the script if you moved the files. On Windows, you can run this script in a Schrodinger Command Prompt window after entering the sh command to start a Unix shell.
