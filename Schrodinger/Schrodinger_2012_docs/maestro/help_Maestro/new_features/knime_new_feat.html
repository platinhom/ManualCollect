New Features in KNIME Extensions 1.5

    Infrastructure

  • Based on KNIME version 2.5.1
  • Ported to Mac OSX
  • Start-up script options to easily use a stand-alone KNIME installation and specify the KNIME temporary directory.
  • On Windows the command shell window is suppressed so there is only the KNIME window displayed.
  • Reader, Writer, and KNIME-Maestro connector nodes do not require a Schrödinger KNIME license.
  • Column types
    • New surface column type that can be created with the Python script node and displayed with the Run Maestro node
    • Phase hypothesis and Glide grid type improvements
  • Usability:
    • Backend options reported in the console for all Schrödinger nodes
    • License checking mechanism improved to speed up the KNIME start-up
    • Improved console log

    • KNIME and Maestro

  • New KNIME Workflows menu under Workflows in Maestro for running prepared KNIME workflows.
    • Connect to KNIME mode to send structures to and from an open KNIME session
    • Workflows can be built or imported and run from Maestro. Specified input, output nodes and parameters are exposed and can be altered in an automatically generated start-up panel.
    • Workflows can be run on Project Table data.
    • Disconnected calculations and progress tracking with Job Control.
    • A license is not required to run the KNIME-Maestro connections.
    • Some workflow examples are available and others will be added to the workflow page

    • Functionality

  • Phase shape—ability search a database
  • Python script—access to the flow variables and new functions (for easier metanode development).
  • Many minor details implemented/fixed
  • Some nodes were renamed (eg Table Viewer, Run Spreadsheet Viewer)

    • Metanodes

    Metanodes are available on the Workflow page

  • Run Maestro 1:1
  • Run PyMOL
  • Jaguar pKa
  • Jaguar ESP surface
  • SiteMap
  • Prime loop sampling
  • Prime energy
