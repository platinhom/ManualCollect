New Features in Phase 3.4

    Pharmacophore Hypothesis Model Development

  • The Develop Pharmacophore Model panel has been renamed to Develop Common Pharmacophore Hypotheses.

  • You can split a set of conformers into two sets by title. This is useful when importing conformer sets that differ in their stereochemistry but this difference is not detected. In this case, you can give a different title to each set and then split the set by title.

  • Conformational searches started from the Prepare Ligands step can be distributed across multiple processors.

  • All ligands can be grouped, not just actives. Ligands in a group must have the same pharm set membership. Ligands can be grouped by title.

  • Excluded volumes can be created automatically from a receptor, from a set of reference structures, and from a set of inactives and actives. These capabilities are available from the Excluded Volumes dialog box. They provide interfaces to the create_xvolReceptor, create_xvolShell and create_xvolClash utilities. The first replaces the former Receptor-Based Excluded Volumes dialog box.

  • You can include the reference ligand automatically in the Workspace when you include a hypothesis, with a new option.

  • Ligands that are not aligned to the hypothesis can be hidden in the Alignments table.

  • Some control over the QSAR training and test set automatic assignments has been introduced. The pharm set actives and inactives can be included in the training set, and the selection of ligands can be uniformly distributed across ligand activities.

  • Hydrophobic site definitions have changed due to an improved solvent-accessible surface area calculation. To use the new definitions, you will have to re-run the pharmacophore model development.

    The Generate Phase Database and Manage Phase Database panels now use the new database program, phase_database. If you try to open a database in an old format in the Manage Phase Database panel, a conversion dialog box opens, in which you can choose to convert it to the new format.

  • The Convert Phase Database panel is available from the Phase submenu of the Applications menu. (Update 2)

  • You can import several files with phase_database by listing them in a text file, which is passed as an argument to the -i option.

  • You can replace existing records in the database with the -replace option. Structures in the input file that don't exist in the database are appended to the database.

  • When using the -unique option, duplicates that are rejected are reported in the file jobname_reject.out.

  • Maximum topological distances between feature types can be added to the database with the -paths option in revise mode. These properties can be used to create subsets or filter hits in which the topological distance is too large, for example (creating a U-shaped molecule).

  • Input structure files can be imported either from the job submission host (local files) or the job execution host (remote host). This allows you to create or update a remote database with remote files.

    • Hypotheses

  • You can create hypotheses from a receptor and a ligand based on complementary interactions, with the new utility phase_complex.

  • You can now run create_hypoFiles with an SD file input.

  • You can create a distance file (.dist) from a coordinate file (.xyz) with a new utility, convert_hypoXYZToDist.

  • The radius of an excluded volume can be specified in advance of picking, with a new Radius text box in the Excluded Volumes panel. (Update 2)

  • When creating a hypothesis manually, unused sites are displayed in a darker color than sites that have been added to the hypothesis.

    • Searching for Matches

  • A new panel, Simplified Pharmacophore Modeling and Screening has been added for basic screening of a set of compounds against a pharmacophore hypothesis, which you can import or create in the panel.

  • The Find Matches to Hypothesis panel is now Advanced Pharmacophore Screening.

  • The Advanced Pharmacophore Screening panel searches databases in the new format, with the new search program, phase_find_matches. It also allows you to search files. You can run searches on the old database format from the command line.

  • The detection of hydrophobic sites is now based on an improved calculation of the solvent-accessible surface area. To take advantage of the new definitions for searching, you will have to copy the new feature file into your database and regenerate the sites. You might find that the number of sites changes with the new code. Likewise, you will have to regenerate sites on any hypotheses for which you want to use the new definitions.

  • When generating conformers during a search or searching a file, conformer sets are separated by stereochemistry, by default.

  • You can use included volumes to score and filter hits. This requires an .ivol file, which has the same format as an excluded volumes file. You can turn this feature off, specify the weight in the fitness score, and use a cutoff to filter out hits with too small an included volume score, by using command options.

  • Subsets can be specified with or without the _phase.inp suffix, when passing arguments to database-related commands.

  • You can get more output in the log file from a search for matches by selecting an option. From the command line, use phase_find_matches with the -verbose option.

    • Other New Features and Changes

  • When running a shape search or a flexible alignment and aligning to SMARTS patterns, you can determine the desired atom numbers with the -atoms option to phase_shape or flex_align. With this argument, the program reports the possible atom mappings and exits without running the search or alignment.

  • In the options for phase_shape and flex_align, the pharm argument to the -atomTypes option has been changed to qsar, to avoid confusion with the -pharm option. The old pharm argument is still accepted, but is replaced with qsar.

  • You can delete some or all ligands from the set when building QSAR models. This makes it easier to change the model or create a new model.

  • You can specify the number of ligands to leave out in the leave-N-out cross validation when building QSAR models.

  • The contours of field-based QSAR models can be visualized as solid, mesh, or dot surfaces.

  • The error of prediction is shown in the results of building a QSAR model.

  • Partial charges from the input structures can be used in field-based QSAR models. (Update 2)
