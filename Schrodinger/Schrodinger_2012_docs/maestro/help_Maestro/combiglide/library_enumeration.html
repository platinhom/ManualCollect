Summary

This panel provides tools to develop a library of ligands from a core structure to which molecular fragments called attachments are added in user-specified positions. The attachments replace an existing group in the core structure.

Opening the Combinatorial Library Enumeration Panel

To open the Combinatorial Library Enumeration panel:

  • Choose Applications → CombiGlide → Combinatorial Library Enumeration in the main window.

  • Choose Tasks → Library Design in the main window.

Using the Combinatorial Library Enumeration Panel

The combinatorial library enumeration is carried out by substitution of attachments on the core structure with fragments from the reagent structures. For both the core and the reagents, the bonds to be broken must be designated. The fragments from the reagents are then attached to the core at the site of the broken bond. This process can be regarded as "growing" the new attachment onto the core in place of the old attachment, and the bond that is replaced (with its attached atoms) is called the grow bond. These bonds are not necessarily the bonds that are broken and formed in the real chemical reaction, but represent a means of varying the attachments to a core structure that might include part of the real reagent.

To generate a combinatorial library, you must have a core structure whose attachments you want to replace, and a set of reagent files that define the replacements. For information on preparing reagent files, see the Reagent Preparation Panel topic.

When you first open the Combinatorial Library Enumeration panel, Pick molecule is selected in the Core structure section, and most other controls are unavailable. After you have defined a core structure, the other controls become available.

To define the core structure:

  1. Ensure that the structure is displayed in the Workspace.

  2. Click on an atom in the core structure.

After you pick the core structure, Pick molecule is deselected, and the Pick option is selected in the Pick to define an attachment section. The core molecule is now defined, and you can define the positions of the attachments.

To define an attachment:

  1. Pick two atoms that define the position of one of the attachments, the atom that is kept first, then the atom that is deleted.

    A turquoise arrow is displayed over the bond, and the Select Reagent File dialog box opens.

  2. Select the appropriate reagent file, and click OK

    The reagent files have a .bld extension: they are Maestro-format files that have special information for the Maestro builder, which does the library enumeration. The dialog box closes, and the attachment is listed in the Attachments table. The size of the library with the current set of attachments is given in the lower portion of the panel. The arrowhead changes to a sphere and the color changes to gold.

If you choose the wrong location for an attachment position, you can delete it by selecting it in the table and clicking Delete Attachment. If you simply attached the wrong file, you can select the attachment in the table, and click the Browse button below the table to select the correct file.

To start the library enumeration job:

  1. Click Start.

    The Start dialog box is displayed.

  2. Change the output directory, if necessary, choose a host, set the job name, and click Start.

    The host you choose should have temporary storage of about 3 to 4 times the size of the library. This space is required for the untangling and minimization of the library members.

The job takes about 45-60 seconds per output structure, because each structure is minimized. The library is written to a Maestro file named jobname.mae.

Combinatorial Library Enumeration Panel Features

  • Core structure section
  • Library definition section

Core structure section

This section provides controls for selecting and viewing the core molecule.

  • Pick molecule option
  • Display Core Structure button
  • View as options

Pick molecule option

When selected, allows you to pick a molecule in the Workspace for the core structure. The molecule you pick should be a 3D, minimized structure. Picking a core molecule deletes any attachment data. This option is selected by default when you open the panel and no core molecule has been picked. After picking, the Workspace is cleared and the picked molecule is displayed. Any changes you make to the structure of the molecule are not saved in the run. If you make changes to the structure, you must pick the molecule again to make it the core structure.

Display Core Structure button

Redisplays the molecule used to define the core in the Workspace.

View as options

These options control the display of the core structure. The options are Minimally capped core and Original core, which have the obvious actions.

Library definition section

This section provides controls for the definition of attachment positions and selection of reagent files that will be used to enumerate the library.

  • Attachments table
  • Pick to define an attachment controls
  • Reagent file buttons
  • Delete Attachment button
  • Import Definition button
  • Export Definition button
  • Status area

Attachments table

Lists the attachment that have been defined. The table columns are defined below. You can select multiple rows in the table. Right-clicking in a table row displays a menu with four items: Select Reagent File, Clear Reagent File, Reagent File Statistics, and Delete Attachment. The actions are applied to all selected rows. The Reagent File Statistics item displays statistics on the selected reagent files. The other items perform the same actions as the Browse, Clear, and Delete Attachment buttons.

Name   Label for the attachment position. Default is 1, 2, 3, etc. Editable.
Reagent File   Name of the file containing the reagents. The tooltip for this table cell displays the full path to the file.
Functional Group   Name of the functional group that defines the reagents.
# Reagents   Number of reagents in the file

Pick to define an attachment controls

Pick option and menu: The option menu has two choices: Atoms and Bonds. When the Pick option is selected, you can pick either one bond or two atoms to define an attachment position, depending on the menu choice. If you pick a bond, you must click towards the attachment end so that the grow bond is oriented correctly. If you pick atoms, you must pick the atom that remains first, then the atom that is replaced. After picking, the Select Reagent File file selector opens, and you can browse to the desired reagent file. After selecting the file, the attachment is listed in the Attachments table.

Markers option: When selected, the grow bond for each attachment is displayed in the Workspace.

Reagent file buttons

Browse button: Opens the Select Reagent File file selector, in which you can browse to a reagent file. Properly prepared reagent files have the suffix .bld.

Clear button: Clears the table row of its reagent file and associated data. The reagent file is no longer associated with the attachment, and you must click Browse to select a new reagent file for the attachment.

Delete Attachments button

Deletes the attachment positions that are selected in the Attachments table.

Import Definition button

Import a previously-defined library definition, including the core structure and the attachments. Opens a file browser with a specified filter so that only valid library definitions are displayed by default.

Export definition button

Export the current library definition to a Maestro file, including the core structure and the attachments. Opens a file browser. The standard suffix is automatically added to the file name when the library is exported.

Status area

This area, in the lower right corner, displays the number of structures and the number of compounds in the library and the estimated file size. It is updated each time a reagent file is added (or removed).
