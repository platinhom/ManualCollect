Summary

In the Score Hypotheses step, you score the hypotheses for the variants you selected in the previous step, examine the hypotheses, add excluded volumes to hypotheses if desired, and select a hypothesis with which to build a QSAR model, if desired. When you have completed this step, you can proceed to building a QSAR model or to searching for matches to hypotheses.

Scoring Hypotheses

Each hypothesis contains a set of pharmacophores that are derived from conformations of a set of active ligands. The first stage of the scoring process is to find the optimum alignment of the pharmacophores, and eliminate hypotheses that do not have good alignments. The initial score is based on the RMS deviation of the site point positions and the average cosine of the angles formed by corresponding pairs of vector features (acceptors, donors, and aromatic rings). To the alignment contributions are added a score for the volume overlap of the aligned ligands, a selectivity score, a contribution for the number of matches, a contribution for the relative energy of the reference ligand and a contribution for the reference ligand activity. The "survival score" is a combination of these scores with user-adjustable weights. The scoring is performed on the ligands in the active set, which are the ligands used to develop the hypotheses, for all variants selected in the Variants table of the Find Common Pharmacophores step.

When you click Score Actives to score the hypotheses, the Score Actives dialog box opens. In this dialog box, you can examine and adjust the weights of the terms in the survival score, the alignment thresholds, and the filters on the number of hypotheses to keep. After you have made any changes, click Start. A dialog box is displayed, in which you can select the host, the number of CPUs to use, and the user name on the host.

When the job finishes, the surviving hypotheses and their scores are displayed in the Hypotheses table.

If you have assigned ligands to an inactive set, you can score the hypotheses for matches to the inactive ligands, and subtract a fraction of the survival score for the inactives from the survival score for the actives. Hypotheses that match inactives as well as actives are then scored lower than hypotheses that match actives but not inactives. The scoring function for the inactives is the same as for actives.

To calculate this score, click Score Inactives. The Score Inactives dialog box opens, in which you can specify the weight of the survival score of the inactives to subtract from the survival score of the actives. After you have made any changes, click Start. A dialog box is displayed, in which you can select the host, the number of CPUs to use, and the user name on the host. When the job is finished, the results are displayed in the Survival - inactive column of the Hypotheses table.

If you want to adjust the weights in the scoring function and compute a new score, you can do so by clicking Rescore. In Rescore Hypotheses dialog box, you can set the weights of the terms in the scoring function and the inactive score weight. After you have made any changes, click Start, make any job settings, and click Start again. The results of the rescoring are listed in the Post-hoc column of the Hypotheses table.

Examining Hypotheses and Ligand Alignments

Once you have generated scores for the hypotheses, you can examine the listed hypotheses, one at a time. To examine a hypothesis, select it in the Hypotheses table. The hypothesis is displayed in the Workspace, and data for the aligned ligands is displayed in the Alignments table: activities, pharm set membership, fitness score (which is the default survival score without the contribution from the number of matches), the number of sites matched, and the energy of the matching conformer relative to the lowest-energy conformer.

You can view the aligned ligands and information on their alignments in the Workspace by clicking the diamond in the In column of the Alignments table. This column is multiple-select: you can add ligands to the display with shift-click and control-click. You can display and undisplay the hypothesis by clicking the View Hypothesis toolbar button. From the toolbar (or the Display menu) you can also display the distances between the site points and the angles between all sets of three site points.

By default, only the ligands that match the hypothesis are aligned to the hypothesis. Although the Alignments table lists all ligands, the rows for the remaining ligands are colored dark gray, and do not display any data.

It can be useful to display the alignments and data for the remaining "non-model" ligands: those not in the active set or those that do not match all site points. You might want to examine a hypothesis more closely if it matches inactive ligands, for example, to determine the reason the inactive ligands match, and decide whether to keep or to remove the hypothesis.

To display data for these ligands, click Alignment Options below the Alignments table, and select Align non-model ligands in the Alignment Options dialog box. The table is updated with information for the entire ligand set used in the model development.

If in your examination of a hypothesis, you decide that the hypothesis is not a good one, you can delete it by clicking Delete. If you have found a hypothesis that you want to use with several databases, you can export the hypothesis to a file by clicking Export and providing the file name. You can later import this hypothesis into a project that contains a database you want to search.

Adding Excluded Volumes to Hypotheses

Excluded volumes are regions of space in which there should be no atoms from any active ligand. If you have included inactive ligands in the ligand set, you can use these ligands to define excluded volumes. This might be important if, for example, the inactive ligands otherwise match the hypothesis. The excluded volumes are then used when you use the hypothesis to search for matches.

To define one or more excluded volumes using the inactive ligands as a guide, first ensure that all ligands have been aligned, then display one or more of the active ligands along with one or more of the inactive ligands. Regions of space in which inactive ligands have atoms but active ligands do not are likely candidates for excluded volumes.

With the ligands displayed, click Excluded Volumes. The Excluded Volumes dialog box is displayed. To define a volume, pick a set of atoms in the Workspace that belong to the inactive ligand. A temporary sphere that defines the excluded volume is placed at the centroid of the atoms. When you have picked an appropriate set of atoms, click Add Volume to add the sphere to the hypothesis as an excluded volume. You can subsequently change the radius of the sphere and its location by editing the table in the dialog box.

Clustering Hypotheses

When there are several hypotheses of a given variant that look very much alike and have very similar scores, it is useful to cluster these hypotheses, and show only a single representative from each cluster.

To cluster the hypotheses, click Cluster, select a linkage method (Complete is usually fine), and click Start. When the job finishes, you can display the cluster representatives by clicking View Clusters and choosing a representative type in the View Clusters dialog box. You can also change the similarity level that defines the clusters.

Continuing from This Step

After scoring the hypotheses, you can continue directly to a search for matches, or continue to the next step and build a QSAR model.

To search for matches, select the hypotheses in the Hypotheses table that you want to use in the search, and click Search for Matches. The Advanced Pharmacophore Screening panel opens, with the selected hypotheses listed in the Available Hypotheses table, and the first of these hypotheses selected.

Note: To make hypotheses available in the Advanced Pharmacophore Screening panel, you must either export them, or select them and click Search for Matches.

To proceed to the next step Build QSAR Model, select the hypotheses you want to use to build QSAR models, then click Next. Building a QSAR model is not necessary for searching for matches, so you can also finish at this step.

Scoring and Clustering Buttons

Score Actives button

Opens the Score Actives dialog box, in which you can set the scoring parameters for scoring the active ligands, then start the scoring job.

Score Inactives button

Opens the Score Inactives dialog box, in which you can set the scoring parameters for scoring the inactive ligands, then start the scoring job. This button is not available until you have scored the active ligands.

Rescore button

Opens the Rescore Hypotheses dialog box, in which you can change the weights of the contributions to the scoring function and the inactive match weight, to generate a Post-hoc score.

Cluster button

Opens the Cluster Hypotheses panel, in which you can select a linkage method and run a job to cluster the hypotheses. This is useful when there are several hypotheses of a given variant that look very much alike and have very similar scores.

View Clusters button

Opens the View Clusters dialog box, in which you can select which hypotheses to display for each cluster, and set the clustering level.

Hypotheses table

The Hypotheses table displays the scores for each hypothesis. When you include a hypothesis in the Workspace (click the In column), the alignments of all the ligands that fit the hypothesis are displayed in the Alignments table. You can also add excluded volumes to the included hypothesis.

You can select multiple hypotheses with the usual shift-click and control-click actions. The selected hypotheses can be deleted, or carried forward to the building of a QSAR model or to a search for matches.

To sort the table by the values in a column, click the column heading. The data in the table is noneditable. The table columns are described below.

In   Workspace inclusion status of the hypothesis. You can only display a single hypothesis at a time, by clicking the diamond in this column. If you select hypotheses that do not include the displayed hypothesis, the hypothesis is undisplayed and the diamond is cleared. If you click this column for a hypothesis, the hypothesis is also selected. Hypothesis features are displayed as described in the Create Sites topic, and are labeled by the feature letter and index.
ID   Hypothesis ID, which consists of the variant name and an index.
Survival   Weighted combination of the vector, site, and volume scores, and terms for the number of matches, the relative energy of the reference ligand, and the activity of the reference ligand. The weights can be varied in the Score Actives dialog box. The minimum value of this score is 1.0. The selectivity score, relative energy and activity are excluded from this score by default.
Survival - inactive   Survival score penalized for matching inactive ligands. This score is generated when you click Score Inactives and run the job to score the inactive ligands.
Post-hoc   This score is the result of rescoring. You can set the weights in the Rescore Hypotheses dialog box, which you open by clicking Rescore.
Site   Site score. This score measures how closely the site points are superimposed in an alignment to the pharmacophore of the structures that contribute to this hypothesis. The site score for each ligand is calculated from the alignment score by

site_score(i) = 1.0 − alignScore(i)/alignCutoff

The site scores for the ligands are averaged to obtain the overall site score. The alignment score is the RMS deviation of the site points of a ligand from those of the reference ligand. The weight and cutoff are 1.0 and 1.2 Å by default, and can be specified in the Score Actives dialog box.
Vector   Vector alignment score. This score measures how well the vectors for acceptors and donors are aligned in the structures that contribute to this hypothesis, when the structures themselves are aligned to the pharmacophore. The vector score is the average of the cosines of the angles between the vectors of each aligned ligand and those of the reference ligand.
Volume   Measures how much the volumes of the contributing structures overlap when aligned on the pharmacophore. The volume score is the average of the individual volume scores. The individual volume score is the overlap of the volume of an aligned ligand with that of the reference ligand, divided by the total volume occupied by the two ligands.
Selectivity   Estimate of the rarity of the hypothesis, based on the World Drug Index. The selectivity is the negative logarithm of the fraction of molecules in the Index that match the hypothesis. A selectivity of 2 means that 1 in 100 molecules match. High selectivity means that the hypothesis is more likely to be unique to the active ligands.
# Matches   Number of active ligands that matched the hypothesis.
Energy   Energy of the reference conformer relative to the lowest-energy conformer.
Activity   Reference ligand activity.
Inactive   Inactive score
Cluster Number   Index of the cluster that the hypothesis belongs to. This column is only present if a clustering job has been run.
Cluster Size   Size of the cluster that the hypothesis belongs to. This column is only present if a clustering job has been run.
Average Similarity   Average similarity of the hypotheses in the cluster. This column is only present if a clustering job has been run.

Hypotheses Action Buttons

These buttons perform various actions on the hypotheses in the table.

  • Add to Project Table
  • Export to File
  • Excluded Volumes
  • Delete
Add to Project Table button

Adds the selected hypotheses with their reference ligands to the Project Table as project entries. Hypotheses are stored in the run as part of the project, but not as project entries. When the hypotheses are added, the Project Table panel is opened, and the hypotheses are selected.

Export to File button

Opens a file selector so that you can save the selected hypotheses to disk. This facility provides a way of using hypotheses outside the project.

Excluded Volumes button

Opens the Excluded Volumes dialog box, in which you can define volumes that should be excluded when searching for matches to the selected hypothesis. These excluded volumes become part of the hypothesis. If a ligand has atoms in any excluded volume, it is not considered to be a match. This button is not available if multiple hypotheses are selected.

Delete button

Delete the selected hypotheses.

Alignments Table

When you select a single hypothesis in the Hypotheses table, the ligands are displayed in this table, and data for the alignment of the ligands that match the hypothesis are displayed in this table. Table rows for ligands that are not aligned are dark gray. The table heading identifies the hypothesis.

You can calculate alignment data for all ligands in the Alignment Options dialog box, which you open by clicking Alignment Options, below the table. This dialog box also allows you to limit the list of ligands displayed to those that match selected sites. Information on which ligands are displayed is given next to the

The table has a shortcut menu for exporting information from the table, with the following items:

  • Export Table Data—Export the data in the table to a CSV file or an HTML table. Submenu with items for the format.

  • Add Alignments to Project Table —Add the aligned ligands to the Project Table. The activities and the QSAR set membership are added as properties.

  • Export Alignments to File—Export the aligned ligands to a Maestro file. The activities and the QSAR set membership are added as properties.

The columns of this table are mostly noneditable. The first four columns contain the same information or controls as the Ligands table in the Prepare Ligands step. The row for the reference ligand (the ligand that matches the hypothesis exactly) is colored. You can sort the table by the values in a column, by clicking the column heading.

In   Inclusion status of the ligand. The diamond has a cross in it if the ligand is included in the Workspace, and is empty if the ligand is excluded. You can include and exclude ligands with click, shift-click and control-click.
You can also display feedback information in the Workspace when you include a single ligand. This information can include data from the table. The setting and choice of properties is made in the Phase Workspace Feedback dialog box, which you open by choosing Display → Workspace Feedback.
Ligand Name   The name of the ligand.
Activity   The ligand's activity. You can edit the activity, but the change will not affect the inclusion of the ligand in the active set, regardless of the threshold specified for this set.
Pharm Set   Indicates whether a ligand is in the active set used to identify common pharmacophores.
Fitness   Measures how well the conformer matches the hypothesis. The fitness score is a linear combination of the site and vector alignment scores and the volume score, and is related to the default survival score. The reference ligand, which matches exactly, has a perfect fitness score.
# Sites Matched   Number of sites on the ligand that matched the hypothesis.
Relative Energy   Energy of the best matching conformer relative to the lowest conformer in kcal/mol. High relative energies indicate that the conformer is strained. A large proportion of high relative energies could indicate a poor hypothesis.

Ligand Display Options

There are three options for the display of ligands in the Workspace or in the Alignments table:

  • Keep ligands in Workspace—Keep the ligands that are currently included in the Workspace in the Workspace when you change the hypothesis that is displayed. You can include or exclude ligands, and the new list is used when you change the hypothesis. The exception is that, if any included ligand is not aligned in the new hypothesis, it is not displayed.

  • Include reference ligand—Include the reference ligand in the Workspace when you include the hypothesis in the Workspace.

  • Show unaligned ligands—Show the ligands that are not aligned to the current hypothesis in the table. The rows that contain these ligands are colored dark gray. When you turn this option off, only the aligned ligands are shown in the table.
