Summary

The purpose of the Visualization Setup panel is to cluster hypotheses and display information on the clusters in order to identify multiple binding modes.

Opening the Visualization Setup Panel

To open the Visualization Setup panel:

  • Choose Applications → Phase → Detect Multiple Binding Modes.

Using the Visualization Setup Panel

The clustering calculation identifies similar hypotheses by comparing which ligands match each hypothesis. For each hypothesis, a bit string is constructed that identifies the ligands that match the hypothesis, and this string is used to determine the similarity between hypotheses. Hypotheses that have more ligands in common that match are considered to be more similar. They do not have to have the all the same pharmacophore features, though they may do so.

To perform the clustering, you will need a large number of hypotheses based on the same set of ligands. These hypotheses can come from multiple runs, with different numbers of sites. You will also need to set the number of ligands to match in the Find Common Pharmacophores step to a value that is less than the maximum, but not so low that too many hypotheses are generated that do not account for a reasonable number of actives. You should also score the actives, to eliminate hypotheses with poor alignments.

To perform a clustering calculation:

  1. Generate hypotheses using the Develop Common Pharmacophore Hypotheses panel.

  2. Select the runs in the Project Data section that you want to use for clustering. All hypotheses from each run are used.

  3. Select options from the option menus in the Clustering Options section

  4. Click Calculate.

When the calculation finishes, a panel is displayed that shows the results as a map.

For more information, see Chapter 14 of the Phase User Manual.

Visualization Setup Panel Features

  • Project Data section
  • Clustering Options section
  • Calculate button

Project Data section

This section reports the name of the project, and provides a list of runs in the project from which you can select the runs to use in the clustering calculation. You can select multiple runs from the Runs list using shift-click and control-click.

Clustering Options section

Select options for the clustering calculation.

Number of Clusters option menu

Select the number of clusters that will be marked on the cluster visualization map. This should be equal to the number of likely binding modes.

Metric option menu

Select the method used to compute distances between bit strings representations:

  • Euclidean—Sum of the squares of the differences in the bit string values. This is just the number of positions at which two bit strings differ.

  • Tanimoto—Tanimoto metric, 1 − N(common)/N(total), where N(common) is the number of "on" bits shared by the two strings (bit intersection), and N(total) is the total number of "on" bits in either string (bit union).

  • Cosine—Cosine of the angle between the two bit string vectors.

Linkage option menu

Select the method used to compute distances between clusters.

  • Single—The distance between clusters is the smallest distance between any pair of objects (one object from each cluster). This option produces diffuse, elongated clusters.

  • Average—The distance between clusters is the average distance between all pairs of objects in the two clusters.

  • Complete—The distance between clusters is the largest distance between any pair of objects (one object from each cluster). This option produces compact, spherical clusters.

Calculate Button

Clicking this button performs the clustering calculation, then opens a display panel in which you can visualize the results. This panel shows a map, with ligands on the horizontal axis and hypotheses on the vertical axis. Each cell in the map is colored blue if the ligand matches the hypothesis, and red if the ligand does not match the hypothesis. The ligands and the hypotheses are ordered to optimally display the clustering. Perfect clustering is indicated by a set of rectangular blue areas that have no rows or columns in common. Yellow lines are drawn on the basis of the Number of Clusters setting to indicate the optimal clusters. The Show Legend button in the display panel opens a dialog box that shows the correspondence between the ligand numbers and the ligand titles.
