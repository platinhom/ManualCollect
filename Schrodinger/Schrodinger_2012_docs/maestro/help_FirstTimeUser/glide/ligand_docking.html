Summary

The Glide Ligand Docking panel is used to set up and run docking jobs using previously calculated receptor grids. To generate receptor grids, see the Receptor Grid Generation Panel topic.

Opening the Ligand Docking Panel

To open the Ligand Docking panel, do one of the following:

  • Choose Applications → Glide → Ligand Docking in the main window.

  • Choose Tasks → Docking → Glide Docking in the main window.

Using the Ligand Docking Panel

Glide ligand docking jobs require a set of previously calculated receptor grids and one or more ligand structures. Preparation of the ligands before docking is strongly recommended. LigPrep or MacroModel can be used to prepare ligands. If a correct Lewis structure cannot be generated for a ligand, it will be skipped by the docking job.

When you have chosen settings for the options in the tabs, you can click the Start button in the lower left corner of the panel. This opens the Ligand Docking - Start dialog box. Along with the standard controls, this dialog box includes controls for distributed processing: a text box for specifying the number of subjobs, and a table of hosts, in which you can select hosts and specify the number of processors to use on each. For details, see the Start Dialog Box topic. The number of subjobs cannot be set to fewer than the number of processors: if you specify more processors, the number of subjobs is adjusted upward to match. The ligands are divided evenly between the subjobs when the job is run.

You can also incorporate the docking results into the project by choosing from the Incorporate option menu; however, incorporating a large number of ligands can take a long time.

Default values are used for all other command-line options. For example, the range of ligands that is divided into subjobs is the entire range of ligands to dock as specified in the Range text boxes in the Ligands tab. If you are using the default range (dock all ligands in the file), glide creates j subjobs starting with the first ligand in the file and ending with the last. To use different settings for this and other options, run glide from the command line, as described in the Glide User Manual.

Alternatively, you can click Write to write the input files without starting the job. The dialog box that opens has the same Name text box and Standard Names option menu that appear in the Start dialog box. It has a Browse button that you can use to navigate to a location for the input files.

To restore the default settings in all tabs, click Reset.

Ligand Docking Panel Features

The Ligand Docking panel contains seven tabs, which are described in separate topics:

  • Settings Tab— Use this tab to select the type and precision of docking job you want to run and to specify the previously calculated receptor grids to use.

  • Ligands Tab— Use this tab to specify the source of ligand structures to be docked, set size thresholds for skipping ligands, and optionally to scale the van der Waals radii of ligand atoms.

  • Core Tab— Use this tab to define the core of a reference ligand for RMSD calculations or for restricting the docking to poses where the core coincides with the reference core to some tolerance.

  • Constraints Tab— Use this tab to view the Glide constraints that are defined in the receptor grids and select those you want to apply to the docking job.

  • Torsional Constraints Tab— Use this tab to set torsional constraints on the ligands, using SMARTS patterns.

  • Similarity Tab— Use this tab to incorporate similarity into ligand scoring, either to favor ligands that are similar to a set of probe molecules (typically known actives), or to favor ligands that are dissimilar. The tab includes options for creating and using trained (weighted) similarity models.

  • Output Tab— Use this tab to specify the output pose file type, the number of successful poses reported per run and per ligand, minimize the poses after docking, and optionally to change pose filtering and clustering settings.

In addition, there are three action buttons at the bottom of the panel:

  • Start—opens the Start dialog box, in which you make job-related settings and start the docking job. See the Start dialog box topic for more information.

  • Write—write a Glide input file to the current working directory, but do not run the job. You can then run the job from the command line with the glide command.

  • Reset—Resets the options in all the tabs to the default values.
