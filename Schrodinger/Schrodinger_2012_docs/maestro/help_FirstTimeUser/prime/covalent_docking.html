Summary

In this panel you can set up and run a job to dock ligands that are covalently bound to a receptor. The docking job runs a Prime loop prediction to sample the ligand and the parts of the receptor that you select for sampling. Docking can be done to nucleic acids as well as amino acids.

Opening the Covalent Docking Panel

To open the Covalent Docking panel, do one of the following:

  • Choose Applications → Prime → Covalent Docking in the main window.

  • Choose Tasks → Docking → Covalent Docking in the main window.

Using the Covalent Docking Panel

The covalent docking procedure works as follows:

  1. The side chains of the residues selected for refinement are temporarily removed.

  2. The rotatable bonds of the ligand (and, if the attachment residue is selected for sampling, the rotatable bonds in that residue's side chain) are exhaustively sampled, producing a large number of potential poses.

  3. These poses are then clustered into about clusters.

  4. For each cluster, the side chains of the residues selected for refinement are restored and predicted.

  5. For each cluster, the ligand, the attachment residue (if included), and any residues specified for refinement are minimized.

Covalent Docking Panel Features

  • Source of ligands

Source of ligands

These tools allow you to specify the source of the ligands to be docked.

Use ligands from option menu

Specify the source of the ligands:

  • Project Table (selected entries)—Use the entries that are currently selected in the Project Table.

  • File—Use the structures from the file specified in the Input file text box. This is the default. When you select this option, the Input file text box and Browse button become available.

Input file text box

Specify the input file in this text box. You can use an absolute path or a relative path to the file.

Browse button

Click this button to open a file selector, in which you can navigate to and select the input file. The file name is displayed in the Input file text box.

Ligand reactive groups section

In this section, you specify the reactive groups on the ligands, in which a bond is broken to form a bond with the receptor. Each group is specified by a SMARTS pattern and the index of the atom that leaves, with its attachments, when the ligand bond is broken. You can specify multiple reactive groups. For each ligand, all groups are tried for a match to the ligand, and each match is docked.

SMARTS pattern text box

Enter the SMARTS pattern in this text box that defines the reactive group on the ligand, or edit the pattern that is in the text box.

Get from selection button

Use the atoms selected in the Workspace to derive a SMARTS pattern, which is displayed in the SMARTS pattern text box. You can edit this pattern before adding it to the table.

Leaving atom text box

Specify the atom on the ligand that is removed, along with the atoms attached to it, when the ligand forms a bond with the receptor. The fragment with the smaller number of atoms is the part that is removed. The atom is specified by its index (position) in the SMARTS pattern; the first atom index is 1.

Add button

Add the SMARTS pattern in the SMARTS pattern text box to the table.

Remove button

Remove the selected SMARTS patterns from the table.

SMARTS pattern table

Displays a list of the SMARTS patterns to be used to define ligand reactive groups, along with the index of the leaving atom. You can select patterns and click Remove to remove them from the table.

Import button

Import the ligand reactive group definitions from a file (in CSV format). Opens a file selector, in which you can navigate to and select the file.

Export button

Export the ligand reactive group definitions to a file in CSV format. Opens a file selector, in which you can navigate to a location and name the file.

Receptor options section

In this section you specify the bond in the receptor that is broken to form a bond with the ligand, and choose residues that are allowed to adjust to the formation of the covalent complex. The receptor must be displayed in the Workspace.

Pick receptor bond to break option

Select this option to pick the bond in the receptor that will be broken to form the covalent bond with the ligand, then pick the bond in the Workspace. The atom that is attached to the smallest number of atoms is the atom that is deleted with its attached atoms, and defines the leaving group. The bond is marked in the Workspace with yellow markers.

Picking a bond to a metal is not permitted.

Staying atom and Leaving atom text boxes

These text boxes display the atom numbers of the atom on the receptor that remains when the bond is broken to form a bond to the ligand and the atom number of the atom that leaves. You can edit these text boxes to change the atom numbers.

Include attachment residue in the initial sampling option

Include the residue that the ligand is bonded to in the exhaustive side-chain sampling procedure when the ligand is docked.

Pick residues to refine option

Select this option to pick residues to include in the refinement procedure. You can pick any number of residues. The residues are listed in the Additional residues text box.

Include residues within text box

Enter a value to add residues for refinement that are within this distance of the additional residues that you pick. These residues are also added to the list in the Additional residues text box.

Residues to refine text box and Select button

Lists the residues that are included in the refinement. You can click Select to open the Atom Selection dialog box and specify the additional residues to refine. Residues are added to the list by picking as well. You can remove residues from the list by deleting them from this text box.

Output poses per ligand text box

By default, covalent docking only returns a single pose, which is the lowest-energy pose. If you want multiple poses to be returned, you can enter the number of poses you want in this text box.

Action buttons

When you have finished making settings, you can use the action buttons to start the job, write the input file, or reset the panel to its defaults.

  • Start—Open the Start dialog box to make job settings and start the job.
  • Write—Write the input file for the job. Opens a file selector, in which you can browse to the location and choose the file name. To run the job later, use the command:
  • Reset—Reset the panel to its defaults.
