Summary

Maestro is a general-purpose graphical user interface (GUI) for 3D molecular modeling. Tools are provided for generating and displaying molecular models, starting and monitoring calculations on these models, and analyzing and manipulating these models.

For information on getting help and technical support see the topic Getting Help.

Features

Maestro provides an array of features, including basic reading and writing of structure files. Supported formats include MacroModel, Mol2, PDB, SD, and Maestro file format. The Maestro format is free-format ASCII text and is substantially self-describing.

The current release provides the ability to import, export, construct, and modify molecular structures; to display structures in various forms; to set up, start, and monitor CombiGlide, Epik, Glide, Jaguar, Impact, Liaison, LigPrep, MacroModel, MCPRO+, Phase, Prime, PrimeX, QikProp, QSite, SiteMap, and Strike calculations; and to perform various analyses.

Maestro features include:

  • Reading/writing structure files
  • Extensible structure building
  • Hydrogen adding/deleting (based on the notion of a hydrogen treatment)
  • Scripting via Python and the Maestro command language
  • Command aliasing (similar to UNIX shell aliases)
  • Command script editing (allows you to read, write, modify, and run scripts of Maestro commands)
  • Macros (assigning a sequence of commands to a named macro or to F1...F12 keys)
  • Undoing (one level)
  • Atom labeling and coloring
  • Assigning molecular representation (CPK, Ball & Stick, etc.)
  • High-quality PostScript output
  • Various panels for setting up, initiating and monitoring jobs
  • Measuring - distance, angle, dihedral
  • Superimposing structures
  • Plotting
  • Online help facility (which provides various mechanisms for locating the appropriate help topic - search by name, by category, related topics, etc.)
  • Automatic help (similar to Balloon Help on the Mac or Tooltips under Windows)
  • Context-sensitive help (providing task completion hints to the user)

Interaction

Maestro functionality is organized into a collection of task-oriented panels that are accessible from the menus. Each panel allows access to some subset of the program's functionality, e.g reading a structure, defining a set of atoms, or selecting energy settings.

The panels in Maestro do not communicate directly with the rest of the program. They generate text commands that are supplied to the program and then executed. So at the lowest level Maestro is a completely command-driven program. This has two important consequences:

  1. All actions can be logged and replayed exactly as they were first performed. This makes the program scriptable, and tools are provided to allow users to create command scripts from scratch or based on editing a set of previously performed actions.

  2. Users can choose to interact with the program, entirely or in part by direct typing of commands instead of using the mouse.

Thus Maestro is a command-based, object-oriented GUI written mostly in C++. This means that users may accomplish tasks by either taking graphical actions (clicking buttons, selecting options, etc.) or by entering and executing textual commands. The equivalent to a user's graphical actions are recorded as a command or commands. These can be stored for use at a later point.

Interaction Paradigms

Maestro uses two interaction paradigms:

  • Selection First: select objects then choose an action
  • Action First: choose an action then select objects

By default, Maestro is in "selection first" mode. Said equivalently it is noun -- verb: you select objects then specify the action to take. For example, you can click an atom or drag to select several atoms, then open the Molecular Representation panel, choose a representation, and click Apply.

Alternatively, you can choose an action first. Following the above example, you can open the Molecular Representation panel, choose a representation, and then select the atoms to which to apply that representation.

The command language uses the "action first" paradigm. For example:

repatom rep=cpk
repatom atom. 1-14

These commands change the style of atoms 1-14 to the CPK representation. The first command (repatom rep=cpk) does not change anything on-screen. It only says that atom representations that are set from this point forward will be CPK. The second repatom command actually sets atoms 1-14 to CPK. From this last command one can see the "verb+noun" paradigm. The verb is repatom; the noun is atom. 1-14.

Dialog Boxes and Panels

There are two classes of "windows" that are displayed in Maestro. The first class is dialog boxes, used to display informational or warning messages. Dialog boxes are modal and must be dismissed by the user before further actions can be taken. Some dialog boxes allow you to perform actions in the Workspace, such as atom selection, but otherwise do not permit interaction with other windows.

The second class is panels. Panels are used to perform the real work in Maestro. Most panels in Maestro are amodal: they may be displayed for extended periods and any number of panels may be displayed simultaneously. You can move back and forth between these visible panels, doing as much or as little work in each as you wish. There is no requirement to hide a panel in Maestro before working in another panel. (Of course, you may wish to hide panels to reduce screen clutter.)

There are some groups of panels (such as the MacroModel panels) that display a kind of modal behavior in the group: you can only have one panel from the group open at any one time. If you open another panel from the group, the first panel is dismissed. These panels usually perform mutually exclusive tasks for a particular application.

In Maestro all changes are recorded immediately. For example, if you select an option, the value is changed immediately. There is no Accept button to commit the changes, nor is there a Cancel button to abort the changes. Whenever a setting is changed in the GUI, the internal state of the program is immediately updated.

In the lower right corner of every panel is a Close button. When you click Close, the panel is hidden from view. Closing a panel does not alter or remove any settings you may have specified. To redisplay the panel, select the corresponding item from the menus on the main menu bar.

Workflow Paradigm

The principal Maestro workflow paradigm is "build first, then perform task" (this is a different concept from the interaction model of "act first, pick second"). The "build first, then perform task" paradigm in Maestro means you first make all necessary modifications to the structures. Next, select settings in preparation for performing tasks. Finally, execute the task or tasks. Deletions to the current structures cause a "ripple effect". Any panel with a list of atoms (or atom-based items like atom pairs, torsions, etc.) has entries containing the just deleted atoms removed.

For example, suppose you select a number of constrained atoms: 1, 5, and 10. Then later you delete atoms 1 and 5. The list of constrained atoms no longer contains entries for atom 1 or atom 5.

Maestro also supports the "pick first, act second" paradigm for operations on structures, as well as the "act first, pick second" paradigm.

Mouse Buttons

For a full description of mouse button and keyboard combinations, see Mouse Interactions and Settings. Below is a quick summary:

LEFT Select atoms, menus, buttons, etc.
Drag to select atoms and bonds
MIDDLE Drag for rotation (XY, X, Y, Z)
RIGHT Drag for translation (XY, X, Y, Z)
Click and hold to display context-sensitive menu
MIDDLE+RIGHT Drag for zoom

Or equivalently:

Select: LEFT (click, drag)
Rotate: MIDDLE (drag)
Translate: RIGHT (drag)
Zoom: MIDDLE+RIGHT (drag)
Adjust: LEFT (drag, with Adjust button selected on toolbar)
Shortcut menu: RIGHT (click and hold)

GUI Layout

The Maestro GUI consists of a main window, with panels and dialog boxes that can be opened from the menus on this window. The main window consists of:

  • Menu bar: Located at the top of the window. Provides access to the various task-based panels in Maestro.

  • Workspace: The large molecular display area in the center. Used for drawing and interacting with 3D structures.

  • Toolbars: Located at the top by default. The toolbars contain buttons for manipulating the structures in the Workspace, controlling the appearance of structures and performance of common tasks.

  • Status bar: Located below Workspace. Provides information on objects displayed in the Workspace.

  • Sequence viewer: Located below the status bar. Displays the sequence of any displayed protein.

  • Command input area: Located below the sequence viewer. Allows you to enter and execute Maestro commands.

  • Context-sensitive help: Located at the lower edge of the main window. Displays task-specific help, including hints on how to complete the current task when selecting atoms is required.

Selecting Atoms

Maestro has a powerful set of tools for selecting atoms: toolbar buttons, picking tools in panels, and the Atom Selection dialog box. With all these tools, you can choose a structural unit for selecting atoms, from Atoms, Bonds, Residues, Chains, Molecules, or Entries, then click on an atom in the Workspace to perform the action on all the atoms in that structural unit.

For example, to select all the atoms in a residue, click and hold the Select button on the Edit toolbar, choose Residue from the button menu, and click on an atom in the desired residue. All the atoms in that residue are selected.

Each tool has a list of permitted structural units. The set of available structural units varies depending on the task represented by the button or panel.

Text Commands

Maestro has a command language that can be used to perform GUI functions and other operations. For information about the syntax of Maestro commands, see the topic Maestro Command Language Syntax.

All commands in Maestro are logged. The Command Script Editor Panel can be used to view these commands as well as to create, modify, save, and run command scripts.
