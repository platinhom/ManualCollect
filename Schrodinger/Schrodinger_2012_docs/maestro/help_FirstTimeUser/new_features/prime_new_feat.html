New Features in Prime 3.1

New and changed features in Prime are listed below.

    Structure Refinement

  • When refining a homology model, you can automatically select the loops, side chains, and atoms that did not come from the templates, by clicking buttons.

  • You can refine multiple structures in a single run by specifying a Maestro file with multiple structures as the structure input. By default all structures in the file are processed, but you can specify which structures to process with the ENTRY keyword.

  • Positional constraints can be applied to non-C-alpha atoms during loop refinement, by adding the appropriate CONSTRAINT_i keywords to the input file. The constraints are applied in the loop buildup stage.

    • Homology Modeling

  • When choosing to do homology modeling, a dialog box opens offering you the choice of using the Structure Prediction panel or the Multiple Sequence Viewer.

  • The toolbar has been reorganized, so that only the relevant buttons are shown.

  • Two more ways of importing a sequence have been added: from the PDB, by specifying the PDB ID and optional chain ID, and from text, which can be typed or pasted in.

  • Editing a sequence can only be done in the Input Sequence step. The toolbar button and menu item have been replaced with an Edit Sequence button below the Sequences table.

  • Multiple templates can be imported in the Find Homologs step from the Project Table, by selecting them and clicking From Project Table.

  • The Find Family button is now labeled Identify Globally Conserved Residues to indicate more accurately what is being done. Along with this change, only the conserved residues (upper case) are shown in the sequence viewer, colored if they match both the query and the template.

  • Two alignment programs can be used for alignment: ClustalW and Prime STA. The ClustalW alignment was available previously as Quick Align, but it is now the preferred method if the sequence identity is high. The ClustalW alignment does not use secondary structure information. The Prime STA option is the method that was formely used when you clicked Align, and is still used for GPCR alignment.

  • There are two methods available for building structures: knowledge-based and energy-based, which correspond to the former "preview model" and "build model". The options in the Build Structure - Options dialog box apply to both methods.

  • The template residue numbering is preserved by default in the built model, as far as possible, and there is an option in the Build Structure - Options dialog box to control this action, supported by an input file keyword.

  • You can select multiple ligands for inclusion in a model with the usual list-selection actions, shift-click and control-click.

  • The Ligands and cofactors table shows only the ligands and cofactors that are within 5 Å of the selected chain by default. You can list all of them by selecting Show all. (Update 2)

  • Secondary structure predictions can be imported in native (Maestro) format as well as FASTA format. (Update 2)

  • The full Pfam annotation can be displayed with a choice from the Display menu. (Update 2)

  • Fold recognition is no longer available.

    • MM-GBSA

  • You can adjust the flexible region of the protein by adding selected residues, excluding selected residues, or limiting the region to selected residues. An option menu and a set of picking tools has been added to the panel for this purpose.

  • The MM-GBSA program has been substantially rewritten, with new output. If you are relying on output in the earlier format, you can run the old version with the command

    $SCHRODINGER/run -FROM psp prime_mmgbsa2011.py

    • Covalent Docking

  • You can import and export ligand reactive group definitions. They are stored in a CSV file, with the SMARTS pattern as the first column and the atom index as the second.

    • Other New Features

  • Prime has been ported to MacOSX and is now available on Mac computers.

  • A unified script for structure building and refinement has been introduced, named prime. This script replaces bldstruct, refinestruct, and multirefine. It can be used directly in place of these scripts, with one addition to the input file: the keyword jobname, which specifies whether building or refinement is being done, with the values model and refine.

  • The RMSD and the score from structural alignment are now available as Maestro properties (r_psp_StructAlign_RMSD, r_psp_StructAlign_Score) with the aligned structures.

  • The automatic scanning alignment in the Protein Structure Alignment and the Align Binding Sites tools is always performed, so the option to run it has been removed from these panels.

  • The Protein Structure Alignment panel has been updated. It now provides a choice of the reference residues and the residues in the proteins that are to be aligned to the reference. It runs the structalign utility directly, which uses the optimized alignment. The option for this feature and the Options button have therefore been removed. You can also force an alignment for low similarity with a new option.

  • The results of a protein structure alignment have been moved out of the Protein Structure Alignment panel into a dialog box. (Update 2)
