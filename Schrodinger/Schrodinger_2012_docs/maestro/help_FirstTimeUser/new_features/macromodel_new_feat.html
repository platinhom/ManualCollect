New Features in MacroModel 9.9

The following features have been added to MacroModel 9.9 in Suite 2012:

  • The Constraints tab in the MacroModel energy panels has been redesigned. Instead of buttons, there are tabs for each type of constraint. The force constants and widths of the flat-bottomed potentials can be edited, and can be set for multiple constraints simultaneously. Constraints can be imported from and exported to .sbc files.

  • A script for conformational sampling of macrocycles has been added. This script is available from Maestro in the Macrocycle Conformational Sampling panel and from the command line as macrocycle_conformational_sampling.py.

  • The use of ASL1 and ASL2 relative constraints is supported for serial conformational searches.

  • The bmin command can be used for distributed jobs as well as serial jobs. The former utility for distributed jobs, para_bmin, is still available, but is essentially a synonym for bmin. With this change, three options of the former para_bmin are deprecated:
    -GZIP : This option is no longer needed because the compression of the output file is controlled by the file extension, and defaults to compressed (.maegz).
    -ROBUST : This option attempted to rerun subjobs. Its functionality is superseded by the -MAX_RETRIES option, which specifies the number of times to retry a failed subjob.
    -SUBINTERVAL : This option has been removed.

The following features were added to MacroModel 9.9 in Suite 2011:

  • You can do a Perform Automatic Setup for pure low-mode (LMOD) and large-scale low-mode (LLMOD) calculations.

  • In the Substructure tab, the number of substructure atoms, number of fixed atoms, and number of frozen atoms is displayed at the bottom of the tab.

  • The bmin command now supports distributed jobs, and can be used instead of para_bmin, which is deprecated.

  • Improvements have been made in the ring templating script.
