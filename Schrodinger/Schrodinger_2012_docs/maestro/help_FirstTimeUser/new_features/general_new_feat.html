General New Features

New features in Schrödinger Suite 2012 that apply to multiple or all products are listed below. These features include changes to Job Control, support scripts, and utilities. For information on graphical tools, see New Features of Graphical Tools.

Note: This suite is likely to be the last suite in which Linux-x86 32-bit software is supported.

    Installation

  • On Linux, a choice is offered to install all modules or to select modules for installation.

    • Job Control

  • Transfer of data to and from a cluster now takes place via an ssh tunnel by default. This mechanism should address many problems relating to firewalls.

    • Filters and Converters

  • Converting PDB files to Maestro includes reading of temperature and pH data from PDB files and adding them as Maestro properties, s_pdb_PDB_EXPDTA, r_pdb_PDB_EXPDTA_TEMPERATURE, and r_pdb_PDB_EXPDTA_PH. Converting Maestro files to PDB format includes EXPDTA records and the corresponding REMARK records.

  • A template has been added to mmpdb for cacodylate ion (CAC), as the OPLS force fields now include arsenic parameters.

  • Nonstandard residue names are converted to standard names when exporting to MOL2 format.

    • Tools and Utilities

  • A graphical tool (Diagnostics panel) has been added for running tests and reporting on licensing and host-related conditions. The tool can be run from the command line as $SCHRODINGER/diagnostics, from the Help menu in Maestro, or as an application on Windows or Mac. This tool should be used to gather information to send to technical support.

  • A utility has been added for validating Maestro files, named maevalidate.

  • A utility for zipping and unzipping files has been added, named ziputil. This utility can handle large zipped archives that other zip facilities fail on.

    • Force Fields

  • Improvements were made in the OPLS_2005 force field for tetrazoles, to correct the dependence of the partial charges on the Lewis structure.

  • Changes have been made to the glycine backbone torsions in OPLS_2005.
