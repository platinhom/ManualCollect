Summary

In this tab, you choose a source of structures for new cores, choose options for finding new cores, and run the job. The new cores are chosen based on the alignment of the attachment bonds on the new core to the attachment bonds on the original core. Up to 2 linkers are added to each new core to improve the alignment.

Find New Cores Tab Features

  • Use structures from option menu
  • Input file text box and Browse button
  • Use immediate results visualizer option
  • Use receptor option
  • Specify Receptor button
  • Database Filters button
  • Advanced button
  • Start button
  • Write button
Use structures from option menu

Choose the source of structures for the potential new cores from this option menu, from Project Table (selected entries) or File.

Input file text box and Browse button

Specify the file containing the structures for the potential new cores. Click Browse to navigate to the file, which can be a Maestro file (.mae*) or an SQLite database (.sqlite). The file name is displayed in the Input file text box.

In the file selector, under Look in, there is an item for the standard core libraries in the installation, labeled Schrodinger Cores—. The actual directory is combiglide-vversion/data/core_library. The libraries are SQLite databases.

Use immediate results visualizer option

Use receptor option

Use a receptor to guide the selection of cores, by requiring hydrogen bonds to the receptor and preventing steric clashes.

Specify Receptor button

Choose the receptor, then select hydrogen bonds and make settings for clashes with the receptor. Opens the Specify Receptor dialog box.

Database Filters button

Open the Database Filters dialog box, in which you can filter the protocores from the SQLite database that is specified as input. Only available when you choose an .sqlite file for input. The dialog box allows you to set the minimum and maximum numbers of heavy atoms, the minimum number of hydrogen-bond acceptors, hydrogen-bond donors, nitrogen and oxygen atoms, and chiral centers.

Advanced button

Open the Ligand-Based Advanced Options dialog box to control the parameters of the search, including options for running on a GPGPU.

Start button

Start the job to find new cores. Opens a Start dialog box, in which you can name the job, choose a host, and choose the number of CPUs for distribution of the job, then click Start to submit the job to the host.

Write button
Write button

Write out the input files and a shell script (.sh)to run the job. Opens a dialog box in which you can specify the job name, which is used for the script and associated files; and the host information, which is included in the commands that are written to the script. A directory named with the job name is created in the current working directory, and the files are written to this directory. The absolute paths to the files are written in the script, so you would have to edit the script if you moved the files. On Windows, you can run this script in a Schrodinger Command Prompt window after entering the sh command to start a Unix shell.
