Summary

In this step, the docking results are reduced to a small combinatorial library that contains the highest-scoring compounds. The reduction is achieved by filtering on various properties and by selection, and the final library is written out.

Working in the Analyze Library Step

The goal of this step is to select a small set of reagents that is then used to create a combinatorial library, by applying various selection strategies, filtering, and manual selection. The purpose of this selection step is to create reagent lists that are small enough and well enough chosen for practical synthesis and screening of the resulting real combinatorial library.

Selection and filtering is done in the Filter and Select dialog box, which you open by clicking Filter and Select. The selection strategies and the filters are described below. When you click OK or Apply in this dialog box, a job is run to perform the filtering and selection. At the end of the job, the tables are filled and you can examine the results. You can repeat the selection and filtering as many times as you like, save the results in the run, restore results from a previous selection, and compare results for different selections. When you are satisfied with a library, you can run a job to create and store the structures in the library, by clicking Create Library.

Selection Strategies

CombiGlide has three automated selection strategies, which can be supplemented with manual selection. The automated strategies use the GlideScore to select the reagents that produce the greatest number of structures that have a good GlideScore, and are therefore likely to be active.

The selection strategies work by filling lists of reagents at each position, until the limits on the minimum and maximum number of reagents and the overall library size are met. The molecules from the docking step are ordered according to their GlideScore or some derived quantity. Each molecule is considered in turn, and the reagents that were used at each position to build that molecule are added to the corresponding lists.

The first strategy selects reagents based on the single side-chain docking. In this strategy, the lists for each position can be filled independently from the docking results for each position.

The second strategy selects reagents based on the GlideScores from the combinatorial docking. In this strategy, reagents for each position are added from each molecule, unless they have already been added. As lists are filled, the remaining lists can be filled using molecules with reagents that are in the filled list, or using any molecule, regardless of whether the reagents are in the filled list. The first option ensures that all molecules used to create the library are in the library. This strategy ensures that the molecules in the library contain those with the best GlideScores.

The third strategy uses the GlideScore from the combinatorial docking as a free energy, and calculates the normalized Boltzmann population for each docked molecule. The populations are summed for each reagent, and reagents are added to the lists in order of population, from highest to lowest. This strategy samples the most frequently occurring reagents, weighted according to GlideScore: reagents coming from molecules with good GlideScores are weighted more heavily than those with poor GlideScores.

Finally, you can manually select reagents that must be included in the library or that must be excluded from the library, in the Include/Exclude tab of the Filter and Select dialog box. In addition, you can set the minimum and maximum number of reagents at each position, and set an overall library size, in the Selection Strategy tab of the Filter and Select dialog box.

Filtering

In addition to selecting reagents based on the GlideScore of the molecules that they are used to build, you can filter the combinatorial results based on various properties of druglike molecules. These properties are mostly generated by QikProp. Several filters are provided, and you can customize these filters by including or excluding any of the listed properties and by setting the limits on the acceptable range of properties. The filters are set up in the Filter tab of the Filter and Select dialog box.

Filtering is not available unless you have a QikProp license.

Analyze Library Step Features

  • Filter And Select button
  • Display Receptor option
  • Results table
  • Reagents at position table
  • Results status area
  • Buttons
  • Library size status area
  • Create Library button

Filter And Select Button

Opens the Filter and Select dialog box, in which you can select a strategy, set up filters, or perform manual selection of reagents, then run the job to create reagent lists for each position.

Display Receptor option

Displays the receptor in the Workspace. This is useful for viewing the poses of the various reagents. Select or deselect this option to display or undisplay the receptor.

Results Table

The first four columns of this table are identical to the corresponding columns of the Reagent retention table from the Dock Library step. The table columns are described below.

When you select a row in the table, the reagent list for that position is displayed in the Selected position table, once you have run a library selection job from the Filter and Select dialog box.

Name   Attachment position label
Reagent File   Name of the reagent file (minus the extension).
Single   Number of reagents from this file that survived the single-side-chain analysis.
Comb   Number of reagents from this file that survived the combinatorial screening. This number includes all reagents that contributed to any structure that docked well.
Library   Number of reagents selected for the library.
Best   Best GlideScore of the molecules used to select reagents at this position for the library. If combinatorial results are used in the selection, the best GlideScore is the same for all reagents.
Worst   Worst GlideScore of the molecules used to select reagents at this position for the library.

Reagents at Position Table

Displays the top reagents for the position selected in the attachment position table, once a library selection job has been run. The reagents not included in the library are highlighted in blue. When you right-click in a row, a menu is displayed, from which you can add the selected reagent to the Include list or the Exclude list. Selecting a menu item opens the Filter and Select dialog box so that you can re-run the library selection job with the newly included or excluded reagent.

In   Workspace inclusion status. You can use this column to display reagents in the Workspace. However, manually included reagents that did not contribute to any docked molecule in the docking stage cannot be included in the Workspace.
Rank   Numerical rank of the reagent. The reagents are listed in order of rank; the rank is determined by the lowest GlideScore of any molecule that contains this reagent. A + sign indicates that the reagent was manually selected for inclusion in the library definition. Manually included reagents are listed at the top and are assigned the top ranks.
Reagent Title   Title of the reagent. This is the title chosen in the Reagent Preparation panel.
GlideScore   Best GlideScore for any compound in the library that used this reagent.
Boltz Pop   Boltzmann population for best-scoring molecule containing this reagent. Boltzmann population strategy only.
Delta G Rel   Effective relative GlideScore for this reagent, derived from the Boltzmann populations. Boltzmann population strategy only.

Results Status Area

Indicates whether the results of the current library selection have been saved in the run or not. If they are saved, the name under which they are saved is shown.

Buttons

Save button

The Save button saves a library definition (the current filter and selection settings and the results from these settings) in the run. Opens the Save Results dialog box, in which you can name the library definition.

Restore button

The Restore button allows you to read in a library definition as the current library definition, and to delete library definitions. Opens the Restore Results dialog box, in which you can select a library definition and restore it as the current definition or delete it.

Compare button

Opens the Compare Results dialog box, in which you can compare different library definitions. This dialog box allows you to select multiple library definitions and display the Selected position table for each definition.

Chem Features button

Opens the Chem Features panel, in which you can visualize the chemical features of the library.

Actives button

Opens the Actives panel, in which you can examine data on the enrichment of actives.

Write Text File button

Writes the library definition to an external file, i.e. outside the project. The contents of the file include options used to run the library definition job, and the Selected position table for each position. Opens a file chooser, in which you can navigate to the desired location and name the file.

Library Size Status Area

Displays the size of the combinatorial library, both in terms of the number of reagents and the number of structures. These can differ because a given reagent might have more that one structure (ionization state, tautomer, conformer, stereoisomer) generated during reagent preparation.

Create Library Button

Run the job to create the library (structures) based on the current library definition. The library is a Maestro file that contains all the structures generated from the reagent variants that score better than the cutoff. In addition, a plain text file is written that contains a summary of the library definition. This is the same file as is written with the Write Text File button.
