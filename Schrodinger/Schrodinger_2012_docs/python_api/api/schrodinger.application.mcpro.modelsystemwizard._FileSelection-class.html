                                         _FileSelection
  • This widget includes three components: an entry field, a browse button, and a checkbutton.
  • Normally the entry field and browse button are on the top, and the checkbutton is put under with some indent space on the left. But if the user set the argument `invert_order' to true, then the widget-layout order is inverted.
  • When the widget-layout order is inverted, the checkbutton state will control the state of the entry and browse button. This is done internally by this class.
  • The three components are accessible as attributes: entry, browse, check_btn.
  • The state of the `check_btn' can be obtained and controlled by an `IntVar' object: the attribute `option'.
  • The construction arguments are of the following meanings, respectively:
    • master Master frame
    • filetypes A list, e.g., [('Maestro Files', '*.mae'), ('SD Files', '*.sdf'),], specifying the file types that will be supported by the file selection dialog.
    • label Entry label (string)
    • command Command to invoke when there is a new valid entry input. The command should take two arguments: the first is the entry's content (the file name), the second is the value of `option'.
    • check_label Label string for the checkbutton.
    • check_command Command to invoke when the state of the checkbutton is changed. The command should take 1 argument: the value of `option'.
    • invert_order As explained above
