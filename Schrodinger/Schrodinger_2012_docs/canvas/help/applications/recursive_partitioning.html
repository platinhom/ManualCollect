Summary

Use recursive partitioning to build a model that can be used to predict property ranges or categories.

Opening the Recursive Partitioning Dialog Box

To open the Recursive Partitioning dialog box:

  • Choose Applications → Recursive Partitioning in the main window.

Recursive Partitioning Dialog Box Features

  • Data set option menu
  • Save results as text box
  • Options tab
  • Training Set tab
  • Results tab
  • Action buttons
Data set option menu

Select the source of the data. The data source can involve both row and column selection. The choices are:

  • Selected rows—Use the rows that are selected in the master view. You must select the rows before opening the dialog box.

  • Visible rows—Use the rows that are displayed (visible) in the master view. You must ensure that the required rows are visible before opening the dialog box.

  • Saved view—Use the rows and columns in a chosen saved view. When you select this item, an option menu that lists the saved views in the project is displayed to the right. You can choose the desired view from this option menu. A second option menu, labeled Domain, allows you to choose whether to restrict the rows or the columns, or both, to those of the saved view. Otherwise the visible rows or columns of the Master View are used.

  • Original data set for jobname—Use the rows and columns that were used for the specified job. This item is only available when you are cloning a job or viewing the results of a job.

Save results as text box

Assign a name to the results. This name is listed under Recursive Partitioning in the Project View window.

Options tab

In this tab, you can set up the input and options for the job.

  • Model options
  • Y variable option menu and search box
  • X variables property selection tools
  • Y type options
  • Model building options
  • Advanced Options button
Model options

Select the option for the action to perform.

  • Build new model—Build a new model.
  • Apply saved model—Apply the model selected from the option menu.
  • Apply imported model—Import and apply a recursive partitioning model. Click Import and navigate to the desired model file (.rp).
Y variable option menu and search box

Select the Y variable to use in the recursive partitioning. You can type the name in the search box to restrict the menu to the matching properties. The values of the Y variable are displayed in the data set structures table.

X variables property selection tools

Select the properties to use as the X variables in the principal components regresssion, using the property selection tools.

Y type options

Select an option for the type of data defined by the Y variable.

  • Y is numeric—The Y variable has numeric values. Click Bins to assign the number and distribution of bins in the Bins dialog box.
  • Y is categorical—Y is treated as a list of categories. This option should be selected for Y variables that are integers or strings. Click Categories to view the list of categories and the number of structures in each category.
Model building options

Choose the type of model to build:

  • Build single tree model—Build only a single tree using recursive partitioning.
  • Build ensemble model—Build a number of trees and use the ensemble of all trees for prediction. This approach filters out noise and corrects the biases in a single tree.

Advanced Options button

Set options for model building. Opens the Recursive Partitioning - Advanced Options dialog box.

Training Set/Results tab

This tab is initially labeled Training Set, and is relabeled Results after a model is built. It contains controls for selecting the training set and test set, and displays the results.

  • Training set section
  • Data set structure table
  • Training tab
  • Test tab
Training set section

Assign the training set and the test set using an automated procedure, using the controls in this section, described below.

Assign randomly option and box

Select this option to assign the training set randomly, and specify the percentage of the rows to include in the training set. The rest are assigned to the test set.

Seed text box

Specify the seed for generating random numbers, used to assign the training set.

Generate Seed button

Click this button to generate a seed. The seed is displayed in the Seed text box.

Assign from partition option and menu

Select this option to assign the training set and the test set from a partition. One class value can be chosen for each set. The rows in the remaining classes are not assigned to any set.

Training set value option menu

Choose the partition class value from this option menu to assign the members of the class to the training set. The percentage of the total number of rows in this class is shown in the text box to the right.

Test set value option menu

Choose the partition class value from this option menu to assign the members of the class to the test set. The percentage of the number of rows in this class is shown in the text box to the right.

Apply button

Apply the choices in this section to generate the training set and the test set.

Dataset structure table

Table of structures in the dataset. The Structure column lists the title of each structure. The Model Set column indicates whether a structure is in the training set (training), the test set (test), or neither (none). When you select the Y variable, its values are added to the table.

If you select Y is numeric, a Bin column is added to the table. The values in this column indicate which bin the structure is assigned to.

When Build new model is selected, you can assign the training and test sets manually by changing the value in the Model Set column. If you double-click in a cell, an option menu is displayed, from which you can choose the set. To assign the set for multiple rows, select the rows, then right-click in a cell and choose the set from the Change Selected submenu of the shortcut menu.

When a model is applied, a column is added to the table for the predicted property value.

Training tab

Displays statistics for the training set on the number of correct and incorrect predictions for each category or bin.

Test tab

Displays statistics for the test set on the number of correct and incorrect predictions for each category or bin.

Action buttons

  • Export Model button
  • View Trees button
  • Build Model button
  • Apply Model button
Export Model button

Export the model to a file (.rp). Opens a file selector, in which you can navigate to a location and name the file.

View Trees button

View the decision trees for the model. Opens the Recursive Partitioning Trees panel.

Build Model button

Run the job to build the model.

Apply Model button

Apply the selected model to the dataset. Not available when Build new model is selected.
