Summary

In this dialog box you can set options for the building of a model by recursive partitioning.

Opening the Recursive Partitioning - Advanced Options Dialog Box

To open the Recursive Partitioning - Advanced Options dialog box:

  • Click Advanced Options in the Recursive Partitioning dialog box.

Recursive Partitioning - Advanced Options Dialog Box Features

  • Splitting section
  • Ensemble model section
Splitting section

In this section you can set various parameters that control how the splitting (partitioning) is performed at each node in the tree.

Splitting algorithm options

There are two choices for the splitting algorithm:

  • Gini impurity—Use the Gini impurity criterion for splitting the observations. This criterion measures the divergences between the probability distributions of the target values.
  • Information gain—Use the information gain for splitting the observations. This criterion uses an entropy measure.

Minimum leaf size box

Minimum size of a leaf node in the decision tree. If splitting a node would create nodes that are smaller than this size, the node is not split and is kept as a leaf node.

Minimum X:Y correlation option and text box

Minimum absolute value of the x:y Pearson correlation coefficient evaluated over the training set. Any x variable not meeting this threshold is ignored: x variables that do not have any correlation with the y variable are not likely to be of predictive value.

Use average of equivalent split values option

Use the average of the two training set property values on either side of the split point to define the split value. If this option is off, the lower of the two values is used. For example, if the values on either side of the split were 2.5 and 2.8, selecting this option would set the split point to 2.65, whereas deselecting this option would set the split point to 2.5. The test for this node would then be x ≤ 2.65 or x ≤ 2.5, where x is the property (or attribute) used for splitting.

Ensemble model section

In this section you can specify parameters for ensemble model building. This section is not available if you are building a single tree model.

Number of trees box

Specify the number of trees to generate for the ensemble model.

Training set sample size box

Percentage of the training set (selected at random) used to build a tree.

Use average leaf node probabilities option

Use the class populations in the leaf node to assign class probabilities to an observation. By default, the class populations in the leaf node are weighted by the class populations in the training set to arrive at a probability for each class.

Require unique root attributes option

If this option is selected, a different property is selected in each tree for the first partitioning of the training set.If it is not selected, no restriction is placed on the property used for the first partitioning of the training set.
