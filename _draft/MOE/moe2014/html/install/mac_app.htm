The following documentation describes certain graphical capabilities available on the Mac OS X platform.
NOTE: These options are only available on Mac OS X versions >= 10.6.

Opening a File From Finder

MOE will, by default, open all files with extensions .moe, .ph4, .pdb, .mdb, and .svl in a new MOE instance if none are currently open, or an existing MOE instance if one or more are currently open.

Opening One or Multiple Files From Drag-and-Drop

An alternate way to directly open a file in MOE is to select one or more files and then drag-and-drop these files onto the MOE icon on the Desktop or in the Dock. Performing this action will cause the specified file or files to be open in a new instance of MOE if none are currently open, or an existing MOE instance if one or more instances are currently open.

Opening MOE From Dock/Desktop Icon

Starting in MOE 2013.08, users have the option of launching MOE from a Desktop or Dock icon. These icons are aliases to MOE.app in the $MOE/bin directory, and can be created at install-time.

If these icons are deleted, to recreate either one of these icons,

  1. Navigate to $MOE (the directory in which MOE is installed).
  2. Navigate to $MOE/bin.
  3. Right-click (if using a multi-button mouse) or control-click MOE (MOE.app) and select Make Alias from the menu that appears. An alias should be created.

    4. NOTE: There are multiple files in $MOE/bin with the name moe or MOE. The correct file should have the Chemical Computing Group logo as its icon, as in the above image.

  4. Either copy and paste, or drag and drop the resulting alias to the desired location.

MOE can also be added to the Launchpad by copying and pasting the alias into the user's /Applications directory. Note that this may require Administrator privileges.

Dock Icon Capabilities

Starting in MOE 2013.08, the MOE application icon will stay open in the Dock for the duration of the session in which the user is working in MOE.

Additionally, the Dock icon contains a menu which can be accessed by right-clicking or control-clicking the icon in the Dock.

NOTE: Versions of MOE prior to MOE 2013.08 are not compatible with the following Dock icon menu items:

  • Hide
  • Show (only valid after Hide has been selected)
  • Quit

About Panel

To access the About Panel, right-click or control-click on the MOE icon in the Dock while MOE is running, and select the About MOE option from the Dock menu that appears.

The About Panel displays the current version number, as well as copyright information.

Configuring Defaults via Preferences Panel

To access the Preferences Panel, right-click or control-click on the MOE icon in the Dock while MOE is running, and select the Preferences… option from the Dock menu that appears.

  • MOE Version and Path.

    This option allows the user to change the version of MOE to be opened by default, previously solely configurable through a terminal command. The path to the chosen version of MOE can be selected through a graphical File Browser by clicking on the Choose Path to MOE button. Additionally, the current key (version of MOE) can be configured, or a new key can be made by selecting the Edit checkbox.

  • Run Configuration.

    Users have the ability to select whether to run MOE in 32- or 64- bit mode, or whether to allow MOE to auto-select its configuration, depending on the current machine's architecture, by choosing the appropriate radio button under Select Run Configuration.

  • Startup Options.

    Command line arguments to be appended to the startup script which launches MOE can be specified in this section of the panel. The descriptions of these switches can be found here.

    • Common Startup Options. Common command line switches can be turned on by enabling the the appropriate checkboxes.
    • Custom Startup Options. In the edit box, additional command line arguments can be specified.

NOTE: If a MOE launch error arises from an incorrect path issue (such as, 'No such file or directory'), an error panel is generated with a button that provides access to the Preferences panel, where you can correct the path.

Launching MOE on Login

In order to launch MOE automatically upon login, click on the Apple menu (in the top left corner of the menubar) and select System Preferences… | Users & Groups | Current User | Login Items, and add MOE to the list by pressing the + button while selecting the MOE application from the file browser which appears. Finally, ensure that the corresponding checkbox under Hide is not selected.

Alternatively, if the user has chosen to create an icon in the dock, setting MOE to launch upon login can be achieved by right-clicking or control-clicking the MOE application icon in the dock and selecting Options | Open at login.
