Please note that this function has been superseded by pro_SearchPDB.

Description

Searches the MOE homology databank for protein structures that are homologous to a specified amino acid sequence. The databank is a library of alignments that was generated by clustering the chains in the MOE protein database along with selected sequences from public domain sequence databases into families of related structures.

In batch mode, or when called with explicit arguments, a hitlist is returned. The hitlist is a vector which contains a sub-vector corresponding to each chain or family in the homology databank which met the search criteria when compared to the query. For each hit, a vector with the following information is returned: 

    [ Zscore, hfc, code, header, family_info, query_positions ]

If passed null, pro_Search opens the MOE-SearchPDB panel and allows many searches to be performed.

Arguments

chain_key or sequence_string
Key of chain in MOE or string of characters specifying the single-letter codes of the query sequence.

search_options
Vector of search parameters. Optional arguments are attribute-value colon-separated pairs:

ID_cutoff:real
Minimum percentage residue identity that must be achieved in order to be submitted to the Z-score (significance) test. The default is 20.

Z_cutoff:real
Minimum Z-score that must be achieved in order to retain a search candidate in the hitlist. The Z-score is estimated by generating a baseline score and standard deviation from the alignment of a number of random permutations of the query sequence against the candidate sequence. It is equal to the difference between the score of the optimal alignment, between the query sequence and the candidate sequence, and the estimated baseline score, expressed in terms of the standard deviation of the random scores. The default is 5.

mda:'token'
Name of the amino acid scoring matrix that is used in the sequence alignment. The name will be concatenated onto the value of the configuration parameter in the MOE configuration resource file (or using the Matrix Path field in the Protein Defaults page of the Configuration Options panel) to produce a full path specification. The default name is 'blosum62'.

gapstart:real
Penalty (a positive real number) to be levied for each gap start. The default is 3.

gapextend:real
Penalty (a real number) to be levied for each residue which matches a gap. The default is 1.

sec_struct:flag
Set to 1 to use the predicted secondary structure probabilities of the candidate and target sequence in the alignment calculation. The default is 0 (off).

hfc:flag
Set to 1 to calculate the hydrophobic fitness score of the alignments of the query sequence to the top-ranked candidate structures. The default is 0 (off).

Return Values

For each hit, a vector with the following information is returned:
Z-score
See explanation of Z-scores.

hfc
Hydrophobic fitness score (if requested).

code
Code of the PDB chain in the family most similar to the query.

header
Header of the PDB chain in the family most similar to query.

family_info
Sub-vector of the hitlist containing complete information on all sequences in the family:

db_ref Tokens specifying database sources ('PDB', 'SWS').
chain_codes Accession codes.
chain_headers Header information (PDB entries only).
sequences Vectors of 3-letter amino acid codes for each sequence.
res_positions Alignment positions of query sequence when aligned to family.

query_positions
Alignment positions of query when aligned to family.
