Syntax

    chain_key = pro_SeqCreate [ "residue_letter_names", options ]

    options = [
	name		: chain_name_token,
	tag		: chain_tag_token,
	header		: chain_header_token,
	conformation	: structure_token
    ]

Description

Creates a sequence of residues and returns the key of the newly-created chain. The residues are specified by their single-letter codes, supplied as a string. If the conformation option is not specified, a chain containing only residues, i.e. no atoms, is created; unrecognized residues are given the name UNK. If, however, the conformation is specified, residues are populated with atoms; in this case it is an error to include unknown residues in the residue sequence.

The options parameter is given as a list of attribute-value pairs:

name The name to be assigned to the new chain. The default is ''.
tag The tag to be assigned to the new chain. The default is ''.
header The header to be assigned to the new chain. chainheader will appear in the Sequence Editor once the chain is created. The default is ''.
conformation The dihedral geometry to be assigned to the created sequence. Specifying this option causes atoms to be created; if unspecified (default) a sequence-only chain (no atoms) is created. The legal values for structure_token are: 
    '3-10-helix'
    'helix'
    'pi-helix'
    'strand'
    'anti-strand'
    'poly pro'
    'extended'
    '3-10-helix-left'
    'helix-left'
    'pi-helix-left'

	// create TYR-ALA-LYS in helical conformation

    key = pro_SeqCreate["YAK", [header:'new', conformation:'helix']]
