The Project Search Panel

The Project Search Panel offers a multitude of features that allows the user to extract potentially complex data sets quickly and efficiently. The flexible setup of the application does not imply any predefined search modes for searching a database.

The hallmark feature of the Project Search application is that the results of each individual search are stored and may be combined with other search results that have been generated along the search process. This way, data sets of interest that are based on complex search parameters can be generated, stored, and used for downstream applications.

Additionally, a personalized MOE Project database can be created, which can be searched and filtered using the Project Search panel.

To open the Project Search Panel choose any of the databases listed below the Domain Motif menu item, in either of the following menus:

MOE | Protein | Search or
SE | Protein | Search.

Selecting any of the databases will open the relevant Project Search panel. Most Project Search panels will be similar to the Nuclear Receptor panel shown below. Details about the fields used in Project Databases can be found in the chapter about Protein Sequences and Structure Data.

As can be seen on the following screenshot, the Project Search panel shown when a protein kinase database is opened contains additional options for the ligand binding mode and for searching unique subfamilies (highlighted in red). It also displays additional data columns by default. Details about the fields used in Kinase Databases can be found in the chapter about Protein Sequences and Structure Data.

Upon opening the Project Search application the entire content of the database is shown in the result list. The results can be sorted on any of the data columns by clicking on its column heading. Click on the heading again to alternate the column sorting between ascending and descending sorts.

By default, the referred database is an mdb file located in the $MOE/lib/project directory. The default directory can be changed by selecting the Protein Defaults in the Configuration Options panel.

Listing and Marking Entries

The top portion of the Project Search panel is used to list and mark search entries.

List Select the filter to apply to the results. Filters include Full Database, only Marked Entries, and the results of previous searches.

Edit After a search has been performed (for example, searching on the Species of Human), press the Edit button to name the query results. The name given, such as human for example, will be displayed in the List results, replacing the generic name of Search 1. Using the Edit button, a query result can also be renamed or deleted, or the entire history of query results can be cleared.

Mark / Unmark Create a persistent subset of interest with the current entry selection by pressing the Mark button. These entries are then indicated in the Mark column with the "+" character, and are even available if the result set or the selection status has changed. It can be used as a permanent selection across multiple search result sets. The set of currently Marked Entries can be selected in the List drop-down. Pressing Unmark will remove the specifier from the selection.

General Search/Filter Parameters

In the section below the displayed results, the Project Search panel allows filtering the database by specifying one or more search terms. In addition to these search terms, parameters that contain information about the kinase regulatory conformation, family association, and structural parameters can be used as additional filters to reduce the search space.

Keywords Enter the search terms in the text field, with different terms separated by the space character. The search is case insensitive.

By default, wildcard ("*") characters are used on both sides of each search term. Use double quotes around a word to change this behavior. For an exact match to a phrase, enclose the phrase in double quotes.

A column header can be used for specializing the search on that column data. The format for such a specialization is ColumnHeader:SearchPattern.

The column character ":" matches exact values. Use greater than and less than operators (">" and "<", respectively) to match a range of values.

For matching dates, the expected input format is YYYY-MM-DD.

The Boolean operator to use between search terms is specified by the dropdown that follows Keywords.

The example in the following figure shows a search for Species:*yeast Date>2000-01-01 Res<3.0, using a Boolean operator of And. The returned results display both Fission yeast and Baker's yeast as species, which have a date after January 1, 2000, and which have a resolution less than 3.0 Å.
Species Specify the species the resulting compounds should originate from, as given in the PDB record. In addition to common names, scientific species names are also presented.

Family Specify a collection of families to be included in the search. An entire group can be selected by double-clicking any family in the group.

Ligand
Specify a MOE descriptor expression to match ligands in the database. If SMARTS strings are used as search criteria, they must be enclosed in double-quotes. A common set of descriptor expressions are given in the drop-down list.

Tip! To quickly find all entries with apo structures (complexes without ligands) use the descriptor expression !Weight, or Weight>0 if interested in structures with ligand.

Similar to Selected
If enabled, the currently selected atoms in the MOE main window will be used to perform a fingerprint similarity search of ligands in the database.

For the similarity calculation, the MACCS keys are applied at a Tanimoto-distance cutoff as specified in the drop-down field to the right. Only structures with a Tanimoto-distance above the specified value are retained as results. The Tanimoto-distances will be listed after the search in the result set column Sim.
Ligand Binding Mode

Ligands can be searched based on their binding profile to the kinase ATP-binding cleft. The regions within the pocket include the G-Loop, Hyd1, alphaC, Hinge, HRD, and DFG, and are abbreviated by the letters G, Y, A, H, R, D in the Bind column of the results display. This allows a search for inhibitors with mandatory interactions in the specified region.

This filter is available only for kinase databases.
Structure
Specify key interaction regions to use in the structure search. For kinases, valid residue sets include Core, Pocket, Canonical, Extended, or Complete. For non-kinases, structure searches are always done on the Complete structure. The structure search is based on superposition of the C-alpha backbone atoms in the specified regions. The input structure to be searched for in the database is selected from the drop-down list. RMSD values above a specified cutoff value are discarded from the results. The RMSD cutoff (in Å) is given in the RMSD field. The RMSD values will be listed after the search in the result set column RMSD.
Sequence
Select the sequence search method. Identity will calculate the %-identity to the specified sequence search string over the complete sequence. For kinases,

Core, Pocket, Canonical, or Extended will calculate the %-identity to the specified sequence search string over the kinase core, pocket, canonical, or extended portion of the sequence, as selected.

Specify the query sequence by either typing, pasting from the clipboard using Ctrl+V keys, or using one of the kinases already loaded in MOE. In the latter case, press to obtain a list of kinases currently present in MOE.

Specify the %-Identity Cutoff value in the drop-down to the right of the entered sequence. The %-identity values will be listed after the search in the result set column SeqID.

If the special search method Expression is selected, one can specify a sequence residue selection expression which is evaluated against the sequences in the database. If the selection expression matches a database entry's sequence, it is included in the the search result set.

Combine Search
Combine the current search parameters with a previous search. Any of the previous searches or the marked entries, shown in List can be used. Use the Boolean operators (And, Or, Not) to fine-tune the search.
Unique
Enable any of these options to display unique (first occurence) search results in respect to the selected options.

Main Actions

The bottom row of buttons in the Project Search panel contains the primary actions for displaying the filtered results, as well as performing actions upon those results.

Search Filter the database based on the current search parameters. This button remains inactive until search criteria are entered in the Project Search panel. After a search is performed, the button again becomes inactive until the search criteria change.
Clear Clear and reset all search parameters. When this button is pressed, a dialog box asks for confirmation before proceeding.
Load Load the selected entries into the MOE main window.

Copy Specify the result set to be loaded in MOE: either everything in the result list, or only the marked collection, or only those currently selected in the list.
Options

Center Molecule: the loaded structures will be centered in the MOE main window.

Apply Sequence Alignment: the loaded structures will have their sequences aligned using the alignment data from the database before being displayed in the MOE main window.

Load Ligand Only: only load ligand structures from the database.

Select New Objects: the loaded structures will be selected when displayed in the MOE main window.

Close Current System: all existing entities in MOE will be erased before displaying the loaded structures in the MOE main window.
Save Save the selected entries in a separate database for future use.

Copy Specify the result set to be loaded in MOE: either everything in the result list, or only the marked collection, or only those currently selected in the list.
Copy Scores Values If the current result set includes score columns from similarity, structure, or sequences searches, they can be included in the output database.
File Specify the path and file name of the database to be saved.
PLIF Open the Protein Ligand Interaction Fingerprints (PLIF) tool for the current database. The subset of entries currently shown in the search panel list will be preselected in the PLIF tool.

Stats Open the Project Database Statistics panel for the currently displayed results. Statistics for each of the result columns are calculated and displayed as a pie chart or as a table of results, depending on whether Chart or Table is selected from the panel drop-down.

To only show the statistics of the currently selected entries in the Project Search panel list, enable the Selected Only checkbox. The statistics panel is synchronized with the Project Search panel: any changes in the search result set list or its selection is immediately reflected in the Statictics panel.

Press Export to store the chart in commonly used graphics formats. To copy/export the numerical data from the Table view, select the table items of interest and copy them onto the clipboard by opening the list popup menu with a right mouse click and choose the option Copy To Clipboard.

The final set of data that can be displayed in the Project Database Statistics panel is the UID. This is not a column of data from the displayed results. Rather, it represents a specific position in the alignment of the reference structure. UID displays the residue statistics of all the displayed results for that selected position. The position can be moved using the arrows in the upper right of the panel, or a position number can be typed directly in the text box between the arrows.

Browse Open the Database Browser for the currently displayed results. The Project Search browser shows in the MOE main window the annotated pocket region of the currrent entry and the 2D diagram of the ligand in the Browser panel. Additionally, the entry's Compound name, Code, and Species are listed. If available, group and family information is automatically selected to be shown in the configurable data field section, where one can select to show additional data fields of the database.

The Database Browser panel is also synchronized with the Project Search panel and the set of browsed entries will be adjusted to the result set list in the search panel. If entries are selected in the list, one can enable to Browse Only Selected in the Browser panel.

Close Close the Project Search panel without saving any information.
Note: When this button is pressed, the panel closes immediately, without any confirmation.
