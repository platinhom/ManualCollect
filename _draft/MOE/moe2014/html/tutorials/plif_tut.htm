The Protein Ligand Interaction Fingerprints (PLIF) application is a method for summarizing the interactions between ligands and proteins using a fingerprint scheme. Interactions such as hydrogen bonds, ionic interactions, and surface contacts are classified according to the residue of origin, and built into a fingerprint scheme which is representative of a given database of protein-ligand complexes.

The input data for PLIF can come from a variety of sources, which most commonly include X-ray crystal structures and docking results. Fingerprints generated using PLIF are compatible with other fingerprint tools found in MOE. Standard fingerprint tools such as clustering and diverse subsets can be applied to fingerprints generated by PLIF. There is a PLIF visualization interface which illustrates the specific structural meaning of the fingerprints and reveals which binding site residues interact with which parts of the ligands throughout the database.

This tutorial provides an example of calculating, interpreting, and utilizing PLIF on a set of five structures where CDK2 is co-crystallized with different inhibitors. The tutorial has been broken down into the following steps:

  • Load and prepare five co-crystal CDK2 complexes in MOE
  • Generate and Analyze PLIFs
  • Generate a Pharmacophore Query from PLIFs

Load and prepare five co-crystal CDK2 complexes in MOE

  1. Close the current system using MOE | File | Close.

  2. Because we are working with a protein system, we will change the potential to Amber10:EHT.
    1. Click the potential button (its default is MMFF94x) in the bottom left of the MOE window. The Potential Setup panel will open.
    2. In the panel, click the Load menu, and select Amber10:EHT.
    3. In the Solvation menu, select the R-Field model.
    4. Click OK to accept the remaining default values and close the panel.

  3. Open the CDK2 complexes: MOE | File | Open

    Change to the $MOE/sample/mol/ directory and hold down the <Ctrl> key to select the following PDB files:

    1g5s.pdb
    1gih.pdb
    1gij.pdb
    1ke8.pdb
    1ke9.pdb

    Click OK. In the Load PDB File panel, leave the default options and press OK to load the five structures into MOE.

  4. To allow easy differentiation of the various structures, give each one a separate color.
    MOE | RHS | System

    Click on the gray box in the System line and choose Tag from the Color popup.

  5. Isolate the ligands and render the protein backbones. In the MOE Window, choose:
    1. MOE | RHS | Hide | All Atoms
    2. MOE | RHS | Show | Ligand

  6. Prepare the system. MOE | RHS | LigX

    Uncheck the Refine checkbox, then press OK to initiate the calculations.

    The process will take a few moments during which progress reports are written to the MOE Window. The messages disappear when the calculation is done.

    The LigX application prepares one or more protein-ligand complexes for use in active site analysis and ligand modification. The first step is correcting any problems with the structure using the Structure Preparation tool, then adding hydrogens and assigning protonation states, which is performed by default using the Protonate3D application. After this has completed, tethers are added to the receptor heavy atoms and distant atoms are fixed. (You can fix the receptor or allow it to move freely by modifying the default options.)

  7. Superpose the complexes. Open the Protein Superpose panel:
    MOE | Protein | Superpose

    Use the defaults in the Superpose panel and press the Superpose button. Click Yes when prompted to align the protein sequences prior to the superposition and Yes to unfix and refix the prepared atoms.

    Chains with the same tag will move together as a unit. In this way, protein-ligand complexes may be aligned. Close the Superpose window, then click

    MOE | RHS | Center

    Take a moment to view and rotate the aligned structures in MOE.

  8. Compute 2D Ligand Interaction diagrams for the five complexes with:
    MOE | Compute | Ligand Interactions

    Since the ligand interactions were computed for an aligned set of complexes, the application will first compute a consensus configuration of the binding site residues. In the Ligand Interactions panel, use the "up/down" arrows to scroll between complexes – you will see that the residues are displayed in the same orientation for each diagram. Close the Ligand Interactions panel.

  9. Create a database of complexes, which we will use for the PLIF calculations.
    MOE | File | New | Database

    Enter cdk_pdbs.mdb for the database name.

    Press OK to open the new database into a Database Viewer. The new database will be empty except for a mol field. Next, add the five complexes as entries in the database.

    DBV | Edit | New | Entry

    In the Add Entry panel set the Mode as Chain Tag and press OK. This will group together chains with the same tag into the same database entry.

    Tip: PLIF calculations can either use ligands from entries in the database, with a protein structure in the main MOE window, or protein-ligand complexes in each entry of the database. In this case, the protein structures should have consistent residue UID numbers.

Generate and Analyze PLIFs

The PLIF Setup panel is used to compute the interactions and generate the fingerprints for a database of complexes. Thresholds can be set for the 8 types of protein-ligand interaction which comprise the fingerprint (see the PLIF Documentation for details on these interactions).

If no interactions of a particular category are found, or none pass the thresholds, no bits are set for that category. If an interaction between the ligand and a protein residue passes the lower interaction threshold (Min Score 1), the low order fingerprint bit is set. If the interaction passes the higher interaction threshold (Min Score 2), then the low order and high order bits are both set. If there are several interactions between atoms in the ligand and the protein residue, only the strongest interactions are annotated in the fingerprint.

In the cdk_pdbs.mdb database created in the previous section, open the PLIF Setup panel.

DBV | Compute | PLIF | Generate...

  1. Specify the receptor and the ligand.

    Set the Receptor to Database. Enter ligand for the Ligand field name (ligands will be written to this new database field).

  2. Perform raw contact analysis. This scores the interactions for each protein-ligand complex.

    Using the default settings, press Prepare to perform the raw contact analysis. A new field PLIF_raw will be written to the database. Once the Prepare step is finished, a message will appear reporting that the preparation is complete. Press OK to close the message panel.

  3. Compute the PLIFs. This will select the most important interactions for the series of protein-ligand complexes. By default up to 250 fingerprint bits will be stored.

    Click Generate and then Yes when prompted to build the fingerprints. The PLIFs will be written to a new database field called FP:PLIF. The PLIF panel will automatically open with a visual display of the results.

The PLIF panel is used to visually analyze the fingerprints. From this panel, statistical analyses can be performed on the fingerprint bits and several operations such as generating a pharmacophore query can be performed. These fingerprints can also be used in the normal MOE Fingerprint tools.

By default, the Display Mode is set to Barcodes. In the graphical display at the top of the panel, you can view the barcodes for the five complexes along the y-axis. On the x-axis, you will see the single letter codes of the residues for which fingerprint bits were generated. For an example, the V18, shown in green, interacts with the ligands in 2 out of the 5 complexes. The residues of the x-axis are color-coded consistently throughout the different Display Modes.

The bottom portion of the PLIF panel contains the entry list, which shows all of the fingerprints and other fields in a listbox, allowing them to be selected. Various display options affect the fields shown in the entry list, but at a minimum, the composition of the visible fingerprint bits is shown. Each fingerprint bit is denoted by a character to indicate its meaning:

-bit not set
Dsidechain hydrogen bond donor
Asidechain hydrogen bond acceptor
dbackbone hydrogen bond donor
abackbone hydrogen bond acceptor
Osolvent hydrogen bond
Iionic attraction
Csurface contact

The Display Mode option buttons control the presentation and interaction with the underlying data.

Display Mode: Population. This displays a histogram showing the number of ligands with which each residue interacts.

Display Mode: Ligands. This displays 2D depictions of the ligands. The color highlights of the ligands correspond to the interactions shown in the Barcodes and Population modes. Notice that all interactions with a single protein residue will have the same color in all the display modes. For instance, the interactions with the Leucine/Valine at position 83 are common to three of the five complexes shown in the PLIF panel – as seen in the 2D ligand interaction diagrams.

Display Mode: Significance. When an activity threshold has been specified, this shows the interactions which are found more frequently in active ligands.

Generate a Pharmacophore Query from PLIFs

In the PLIF panel, open the Query Generator panel:

Tools: Query Generator

This panel enables the user to generate pharmacophore queries based on the frequencies of protein-ligand contacts.

  1. Generate the pharmacophore features. To use all ligands in the query generation, choose:
    Entries: All

    Keep the default values (including the Unified Scheme) and press Build to construct the pharmacophore.

    Two pharmacophore features are created by the Query Generator.

  2. Add receptor-generated excluded volumes to the query.

    Turn on the Excluded Volumes option.

  3. Create the pharmacophore query.

    Press Create in the Query Generator to convert the pharmacophore features into a searchable pharmacophore query. Press Yes when prompted. The query will be converted into a searchable pharmacophore and the Pharmacophore Query Editor containing this query will open.

Now that you have finished this example, generating and analyzing PLIFs, you may want to take some time to explore the other options in the PLIF panel. For example, the similarity calculator selects ligands that make similar interactions; the pose selector can be used to analyze docking results.
