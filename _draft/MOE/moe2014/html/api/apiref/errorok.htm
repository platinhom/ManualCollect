Syntax

    svl_error svl_ErrorOK(void);

Description

SVL API callback functions must not use long jumps (i.e. throw-catch style of control flow). Sometimes, it might however be desirable to terminate a sequence of callbacks prematurely. For instance, using the svl_UnaryOp() tree-traversal routine to search the contents of an SVL variable for the presence of a particular value, we want to terminate the search whenever such a value is found. This functionality is achievable by the callback function that returns a special error code. The calling routine then catches the thrown exception condition by checking explicitly for that error code.

svl_svl_ErrorOK() returns the error code to be used for this purpose.
