The Print command is used to send the content of the MOE Window, Sequence Editor, Database Viewer or SVL Text Editor to a printer. The Print command can be accessed using the File menu of the window to be printed. When invoked, a Print Preview panel allows you to verify and, if required, modify the print layout.

Print Preview Panel

A Print Preview panel comprises three sections: the Preview Area, the Printer Parameters section and the Content section. The parameters available in the Content section depend on which window is being printed.

Preview Area

The Preview Area occupies the left side of the Print Preview panel. There is a rendering area where pages are displayed. In the case of the MOE window, the document will always comprise a single page. The contents of the Sequence Editor, Database Viewer and SVL Text Editor may, however, span multiple pages. Beneath the rendering area, the preview area footer holds the necessary controls to browse the different pages of the document and zoom or translate the currently displayed page.

Button Legend
First Page Previous Page
Next Page Last Page
Reset View  

The Previous Page and Next Page buttons are used to cycle through the pages of the document while the First Page and Last Page button jump to the beginning and end of the document respectively. Alternatively, type a page number in the textfield and press Enter to go directly to that page. Note that the number of pages in the document is displayed to the right of the textfield.

The magnification level of the page preview can be set using a pulldown menu. The X and Y sliders are available to translate the page along the corresponding axes so that all areas are accessible. The Reset View button resets the magnification level to a 100% and centers the page in the rendering area.

Printer Parameters

In the top right corner of the Print Preview panel is the Printer Parameters section. In this section, a printing device can be selected and related settings, for example paper format, margins and print range, can be specified.

The Printer pulldown menu contains a list of all the printing devices found on the system. The available choices depend on the devices installed on the computer. To print into a PostScript file, select the printer named PostScriptFile in the menu. By default, a file named postscript.ps will be created in MOE current working directory. A different name can be specified using the Print to File checkbox.

The Print to File checkbox allows you to specify whether to print to a file or to send the data to a printing device. If you turn the checkbox on, the File textfield specifies the name of the file to write. You can specify the name of the file by either typing it in or using the Browse button which opens a file selection box.

The page size employed by the printing device is specified by selecting the appropriate paper format in the Page Size pulldown menu. The Landscape checkbox controls the page orientation. By default, Landscape is off and so portrait orientation is used. When turned on, landscape orientation is used. The desired width of the page margins are specified with the Left, Right, Top, and Bottom textfields. The unit of measure is either inches or centimeters, depending on the state of the Margins button bank.

If you wish to print only a subset of the document, enter the range of pages to print in the Pages textbox. When no pages are specified, the whole document is printed. Otherwise, only the specified pages are printed. Multiple pages can be separated by either , or ;. For convenience, a continuous sequence of pages is denoted by x-y where x is the first page to print and y the last page to print. For example, to print page 1,2,3,4,5 and 10 of a document, the following print range is used: 1-5; 10. To print on both sides of the page, on devices which are able to do so, enable the Double-Sided Printing checkbox. Additionally, choose whether Short Edge Binding or Long Edge Binding should be used, to determine the relative orientation of the two page sides. Double-sided printing is supported only on Windows platforms.

Content Parameters

The Content Parameters section of the Print Preview panel occupies the bottom right corner of the panel, underneath the Printer Parameters section. The various options it contains are specific to the window being printed.

Printing the MOE Window

To print the content of the MOE Window, use the MOE | File | Print menu command. This will open the MOE Window Print Preview panel.

The MOE Window Print Preview panel offers options for controlling the layout of the page and for adding a header and footer to the page.

  • Horizontal. Vertical. These button banks specify the placement of the drawing area on the page. The buttons let you select the general alignment of the image, Left, Center, or Right for horizontal, Top, Center, or Bottom for vertical.
  • Header. Footer. These parameters are used to add a line of text to the top and the bottom of the page respectively. Menus for Font Type, Font Style and Font Size are available to customize the text appearance. By default, the MOE window image is scaled to ensure that there is no overlap with either the header or the footer. Turning on the Overlay Header or Overlay Footer checkbox renders the text with no consideration for the image placement, potentially superimposing the text onto the image.
  • Resolution. This sets the print resolution of the image created from the scene in the MOE Window.
  • Options.
    • Anti-Alias. If on, the image will have high-quality anti-aliasing applied. If off, the default amount of anti-alias specified in the Visualization Setup panel will be applied.
    • Transparent Background. If off, the background is rendered; if on, the background is not rendered, i.e. it is set transparent.

Printing the SVL Text Editor

To print the content of an SVL Text Editor, use the TED | File | Print menu command. This will open the SVL Text Editor Print Preview panel.

The SVL Text Editor Print Preview panel provides controls to manage text appearance and page numbering.

  • Font Type. The Font Type pulldown menu is used to select the font used to render characters. Please note that it is recommended to select a fixed-width font like Teletype when printing SVL code in order to preserve indentation.
  • Size. Specifies the size of the text.
  • Syntax Coloring. If on, the text is colored according to SVL syntax. Color definitions are found in the MOE resource configuration file (moe-rc) under the prefix editor.color.svl. Note that using this setting with a device unable to display color may result in certain colors being indistinguishable from each other. In this case, use the Bold Keywords option instead.
  • Bold Keywords. If on, SVL keywords are rendered in bold. This is useful for devices unable to display colors.
  • Line Numbers. If on, every line is prefixed with the line number.
  • Numbering. The Numbering pulldown menu is used to activate page numbering and specify the position of the page number on the page.

Printing Sequence Data

You can print the content of the Sequence Editor using the SE | File | Print command. This opens the Sequence Editor Print Preview panel, which allows you to format the sequence data before sending it to a printing device.

There are a variety of settings that determine the look and feel of a SE print job. These include all of the components of the Consensus section such as Consensus Sequence, Color, Secondary Structure, RMSD Plots as well as the Identity/Similarity Plots. The settings in the SE panel will determine which consensus elements will be displayed in the printout. The user may determine if the individual components are actually being printed by checking the corresponding checkboxes in the Consensus section of the Print Preview Panel. The same principles are applied on a chain level controlling the parameters for coloring the residues, secondary structure and residue plots.

The size of the text can be specified by typing a value in the Font Size textfield, or by selecting one of the predefined font sizes in the shortcut menu. If you do not wish page numbers to be printed, turn off the Page Numbers checkbox.

The following options in the panel all concern sequence data display:

  • Wrap Sequences. If on, the sequences are wrapped such that the same sequence can appear many times on the same page. If off, each sequence will only appear once on each page.
  • Draw Segmented Background. If on, then the sequences are displayed every five residues with alternating white/gray background.
  • Residue Plots. If on, scalar plots displayed in the Sequence Editor are printed.
  • Selected Chains Only. If on, only the selected chains are considered for printing. If no chains are selected in the Sequence Editor, then this checkbox is not shown.
  • Secondary Structure. If on, secondary structure assignments are displayed as color-coded bars above the residues to which they apply. If the secondary structure display is not enabled in the Sequence Editor, then this checkbox is not shown.
  • Predicted Secondary Structure. If on, predicted secondary structure assignments are displayed as color-coded bars above the residues to which they apply. The checkbox is only shown if secondary structure predictions have have been calculated in the Sequence Editor.
  • Color Residues. If on, the residue colors displayed in the Sequence Editor are used. If you are not using a color printer, certain colors may be indistinguishable from each other. In this case, you may wish to adjust the residue colors before printing. The checkbox is only shown if residue color display is enabled in the Sequence Editor.
  • Compound Name. Controls display of the name and header of the chains, which can be printed in the selected position (either to the left or above each sequence).
  • Numbering. Controls residue numbering. If Index is selected, ordinal numbering of the residues within the chains is used. Selecting UID turns on display of the residue UID numbers.

The following options in the panel all control the consensus display:

  • Consensus Sequence If on, then the Consensus sequence based on majority is shown.
  • Clustal X Consensus If on, then the Consensus sequence based on ClustalX is shown.
  • Color If on, then the Consensus background color is shown. The displayed colors are equivalent to the ones seen in the Sequence Editor.
  • Secondary Structure. If on, then the Consensus secondary structure is displayed.
  • RMSD Plot If on, then the consensus RMSD plots are shown. The plot colors are identical to the SE plot colors.
  • Similarity Plot If on, then the consensus sequence similarity plots are shown.
  • Identity Plot If on, then the consensus sequence identity plots are shown. The color scheme for sequences and identity plots are identical to the SE colors.

Printing Databases

You can print the content of a Database Viewer using the DBV | File | Print menu command. This opens the Database Viewer Print Preview panel.

You can print just a subset of a database by selecting the entries and fields to print in the Database Viewer. Turning on the Use Selected Entries Only or the Use Selected Fields Only checkbox respectively restricts the printable set to only the selected entries or the selected fields of a database.

Page Header and Page Footer are used to specify the text to be displayed in the top and bottom margins of the page respectively. Special markers can be used to control the text alignment:

Tags Description
<L> Any text following this tag is left-aligned.
<C> Any text following this tag is centered.
<R> Any text following this tag is right-aligned.

The Page Header and Page Footer also recognize the following shortcut tags that are replaced at print time by their actual values:

Tags Description
<PNUM> Current page number.
<TIME> Current time in HH:MM format.
<DATE> Current date in YYYY/MM/DD format.
<FNAME> Name of the source database.
<ASCTIME> 26-characters English-language time/date stamp.

Font attributes and page layout are modified using the following controls:

  • Font. Selects the font used to render characters.

  • Size. Specifies the size of the text.

  • Fit Width. If on, ensure that no horizontal space goes unused on the page by scaling field widths to fill the page.

  • Fit Height. If on, ensure that no vertical space goes unused on the page by scaling entry heights to fill the page.

Mol Style determines how molecular diagrams are drawn in the printed document. Three drawing styles are possible:

Style Description
3D Draw the molecule as seen in the database viewer.
2D Draw a 2D diagram of the molecule from the atom coordinates.
Depicted Rebuild atom coordinates in 2D using a flat layout algorithm.

If using a color printer, color can be added to the molecular diagram by enabling the Color Het Atoms checkbox.

Tile mode is enabled by turning on the Tile Molecule Field checkbox. In tile mode, the page is subdivided into tiles (boxes). The number of tiles used is specified using the Grid textfield. Each tile contains a molecular image at its center and six annotation locations (one header, one footer and 4 legends) to present compound information provided in the database. The pulldown menu beside the Tile Molecule Field checkbox is used to select the database field that contains the molecular structures to be used.

Enabling the Display Entry Number checkbox writes the molecule source entry number in the top-left corner of its tile.

Header and Footer are used to specify the data displayed above and below the molecule respectively. They can display the name of the molecule, as stored in the molecule, or any fields present in the database, as selected by the option menu to the right.

Legends are displayed below the footer using a smaller font. They can display data from any fields in the database and can be prefixed by their source fieldname when the Display Field Name checkbox is enabled.

Printing Graphical Objects

You can print a grob with key key using the SVL command gr_print key. This opens the Graphical Object Print Preview panel.

Title and Footnote allow you to add text at the top and bottom of the page, respectively. The image will never overlap the text. You may select the Font Type, Font Style and Font Size for each from the corresponding drop down menus, as well as the justification (Left, Center or Right). If your changes do not take effect immediately in the preview window, click the Apply button to update it.

Scale can be used to make the image larger or smaller. Values larger than 100% will be treated as 100% and values smaller than 10% will be treated as 10%. The aspect ratio of the image is preserved. Selecting Fit to Page disables the Scale selector and expands the image to fit the page both horizontally and vertically. This may result in some distortion of the image. When Fit to Page is not selected, the Anchor control can be used to position the image within the page, justifying to one of the sides or corners of the page, or centering it.

The Frame option surrounds the graphics object with a thin black border when selected.


Exporting Graphical Objects

You can export a grob with key key to various graphics file formats using the SVL command gr_export key. This opens the Export Image panel.

Enter the Filename you wish to export to, or use the Browse... button if you want to find and select an existing file (which will be overwritten). Choose the graphics Format you wish to save as from the drop-down list. The filename extension will automatically be changed to match the format you selected. Each format has different format-specific options associated with it, and will also change the Color Space available (see below). Transparency is available with certain formats as well (see Background below). The two Clipboard formats (Windows only) export directly to the Windows clipboard and thus the Filename is ignored in these cases.

Below the image preview are a number of options concerning the image that will be exported. You can specify the desired Width and Height of the image, either in pixels, cm, inches or expressed as a percent of the size stored internally in the graphic object. Enter the desired values and hit enter. Note that a readout at the very bottom of the panel always reports the size of the image in pixels, regardless of the units setting.

By default, changing one of Width or Height also changes the other, so that the overall aspect ratio is preserved. If you wish to disable this behavior, uncheck the Constrain Width/Height option. This will allow you to change one dimension without affecting the other, stretching the graphic object as necessary. This will also make sensitive the Maintain Aspect Ratio box. When this is checked, the graphic object is shrunk in one dimension such that its aspect ratio is the same as when the panel was first opened. However, the overall size of the exported image is not changed; this is accomplished by filling the newly available space with background color. So for example, suppose the object was initially 6" x 4". Then it is stretched (with Constrain Width/Height unchecked) to 9" x 12". Thus it will appear stretched in the y-direction by a factor of 2. Now if the Maintain Aspect Ratio button is checked, the overall image remains at 9" x 12" but the graphic object is reduced to 9" x 6", centered vertically within the 9" x 12" image area. Now the object has the same aspect ratio, 3:2, as initially.

Resolution is used to specify the desired resolution in dpi or dpc of the image. Select the desired units, enter the value and hit enter. This only affects the final size in pixels of the image, and is also stored in some file formats such as JPG and TIF to determine the output size when printing the image from a graphics application. Resolution does not apply to EMF+ or Clipboard EMF as they are vector formats.

Color Space affects how the colors are represented in the exported file. All formats support the RGB color space which is that used by most computer monitors. Additionally, sRGB is a standard internet colorspace supported by PNG.

The Background color may be altered by simply selecting a new color from the drop-down box. Background color is used to fill in areas of the graphic object which have not been drawn on, as well as areas outside the graphic object if Maintain Aspect Ratio is selected. If Format is one of: PNG, BMP, Clipboard EMF then you will also be able to select a transparency level for the background color.

The bottom of the panel presents you with three options.

  • OK - Performs the export using the specified options. If the file already exists, you will be asked to confirm that you want to overwrite the existing file. If the extension does not match that normally used by the given file format, you will also be asked if you really want to proceed.
  • Print... - Opens the Graphical Object Print Preview panel so that you can print the graphical object.
  • Cancel - Closes the panel.

Printer Setup

In the unix environment, printing from MOE is generally performed via the /bin/lp or the /bin/lpr commands. If these commands are invalid, they can be substituted by specifying the RC variable print.postscript.lp in the MOE resource configuration file (moe-rc), or via the SVL command SetRC.

For example, if the printing command for the printer named pqr is /pub/bin/lp -dpqr, one would need to include the following line in the MOE resource configuration file:

print.postscript.lp=/pub/bin/lp -d{}
