Purpose and Scope of this Manual

This document has been written both as a user's guide and as a reference. Its purpose is to introduce the Scientific Vector Language (SVL) to new users, to act as a tutorial for intermediate-level users, and to serve as a reference for experienced SVL programmers.

This document is intended to give a comprehensive overview of the basic SVL package. The basic SVL package may be run on its own, but it is very likely that you will be executing SVL commands from within application-specific software in which SVL has been embedded. In this case, the application-specific software may have a library of user-callable SVL routines. These routines are beyond the scope of this manual, and for more information on them, please refer to their accompanying documentation.

A good grasp of vector concepts and a good SVL programming style can be achieved only through coding experience. Our aim has been to give users a sufficient understanding of the principles of vector programming and of SVL to be able to write their own code.

Although vector programming is a natural, efficient means for treating data, most people are familiar only with the scalar programming approach. We expect that many users embarking on this manual will have had little or no prior experience with vector languages, so we will begin with a proper introduction to vector concepts. It is our hope that users will come to see the vector as a natural unit for data manipulation.

Notational Conventions

Style/Label Indication
typewriter SVL input and output
svl> type what follows at the SVL command line
<Ctrl>-key hold down Control and press key
<Alt>-key hold down Alt and press key
<Return> press the Return key
<Del> press the Delete key
Up, Down, Left, Right arrow keys
PageUp, PageDown Page-Up and Page-Down keys
Home, End Home and End keys

In examples, typewriter style is used to indicate SVL input and output. The SVL prompt svl> signals that what immediately follows is what the user should type at the keyboard at the SVL command line. Do not type the letters svl> as part of the command. The result, as returned by SVL in interactive mode, is displayed in typewriter font without a prompt. When you are starting out, be sure to type exactly what is given; in particular, be scrupulous about reproducing brackets and uppercase letters.
