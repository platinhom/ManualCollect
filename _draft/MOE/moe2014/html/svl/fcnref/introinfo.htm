A number of functions exist for querying module and symbol information. The functions, consts, and globals functions search the symbol table for global functions, global constants, and global variables. The function task_info provides assorted symbol and other information about the queried task. where gives function, filename, and line number information for the latest SVL error that resulted in abnormal termination of an executing task (it returns the same information as presented in the Crash History window, where available).

Information about SVL modules, for example their filenames or the global functions defined therein, can be obtained using the mod_* functions. The dumpmodules function provides information about all loaded SVL modules. Several functions return the filename associated with a given module. For finding the filename associated with an explicit module, use mod_file. mod_find can be passed a pattern against which the module title and filename are matched, and returns filenames. modules behaves similarly, but returns the module key and title in addition to the filename.

Information about SVL symbols, for example their type, or the module containing them, can be queried using the sym_* functions. The symbol table can be searched for patterns using the sym_find* functions. These functions allow you to restrict the search to, for example, only global functions or global variables. sym_type returns the type of the symbol, e.g. global constant. The filename and module in which a symbol is used can be found by calling sym_file and sym_modules, respectively. For global functions, sym_line returns the line number in the SVL source file on which the function is defined. For global constants or global variables, the actual value of the symbol can be obtained using sym_value.
