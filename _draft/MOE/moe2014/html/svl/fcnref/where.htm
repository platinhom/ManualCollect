Syntax

    where []
    [fcn_names, counts, file_names, line_numbers] = where []

    where tkey
    [fcn_names, counts, file_names, line_numbers] = where tkey

Description

Prints an account of the latest error that resulted in abnormal termination of an executing task. This includes the name of the function, the filename, and the line number of its source code. This information is printed for all functions in the chain of nested calls (function stack trace). Only one line of information is printed for a chain of identical recursive calls, with recursion depth printed beside the function name. For example, 

    svl> where[]
    factorial       line 25, fact.svl
    factorial (8)   line 27, fact.svl
    my_func         line 32, fact.svl
indicates that the error occurred when function 'my_func' (on line 32) invoked function 'factorial', which (on line 27) called itself recursively 8 times, before finally terminating on error on line 25.

If given a task key, the function returns information about a given task. Task key -1 specifies the current task.

Note: when available, the CrashHistory window provides the same information. where is similar to the "where" command of the UNIX utility "dbx".

The functional version of where returns the crash history information (instead of printing it out).
