Syntax

    load 'filename'
    load ["source_string"]

    unload 'module_title'
    unload 'module_filename'
    unload module_key

Description

load 'filename'
load ["source_string"]

Loads (or installs) the module contained in the SVL file as expressed by the 'filename' token or the "source_string" statements. No tasks are initiated. After a module is loaded, the global functions defined within it become executable from the command line and by other modules.

If there is a function in the new module of the same name as a function from a currently loaded module, an error is generated and the new module is not loaded. This rule does not apply when re-loading a module, i.e. when the module has the same filename or title as an already loaded module. In this case, the entire module is replaced by the newer version. 

unload 'module_title'
unload 'module_filename'
unload module_key

Removes the module specified by its title 'module_title', full pathname 'module_filename' or key module_key.

If a module does not have its title set explicitly using the #set title environment variable, its title is implicitly set to the name of the file (no path included).

The key of the module is returned by mod_key and modules. Note that module_key changes every time the module is reloaded.
