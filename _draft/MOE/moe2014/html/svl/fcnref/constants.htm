Description

SVL includes a set of predefined constants. Most of these constants give the numerical limits of the machine on which the software is running.

Name Description
EPS_M Machine epsilon; floating-point precision
INT_MAX Maximum positive integer
INT_MIN Maximum negative integer
INT_BITS Number of bits (including sign bit) in an integer
MPU_HOSTCOUNT
MPU_PORTCOUNT
MPU_THISHOST
MPU_HOSTNAMES
MPU_DIGITBITS
MPU_DIGITCOUNT
MPU_NEIGHBORS 		Constants associated with the distributed computing environment.
REAL_BITS Number of bits in the mantissa of a floating-point number
REAL_DIGITS Number of significant digits in a floating-point number
REAL_IEEE 1 if IEEE standard used, 0 otherwise
REAL_INF Positive infinity (1/0)
Inf Positive infinity (1/0)
REAL_MAX Maximum positive floating-point number
REAL_MAXEXP Maximum exponent for positive numbers in e-notation
REAL_NAN Not a number (0/0)
NaN Not a number (0/0)
REAL_MIN Minimum full-precision positive floating-point number; under IEEE standard, smaller positive numbers can be created, but at the expense of precision
REAL_MINEXP Minimum exponent for normalized positive numbers in e-notation; under IEEE standard, the exponent may be further reduced, but at the expense of precision
PI pi (3.14159...)

In addition, there are the following predefined values:

Name Description
ARGC Number of command line arguments used to start the application
ARGV Token vector of command line arguments, of length ARGC
CLI File handle of the command line interface
'*cli*' Filename of the command line interface
STDERR File handle of the standard error device
'*stderr*' Filename of the standard error device
STDIN File handle of the standard input device
'*stdin*' Filename of the standard input device
STDOUT File handle of the standard output device
'*stdout*' Filename of the standard output device

Note that the CLI, STDERR, and STDOUT are for output only.
