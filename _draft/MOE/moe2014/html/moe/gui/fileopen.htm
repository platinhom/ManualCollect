Opening Files

The MOE open file system is used to perform a variety of "open" operations on files of different formats. File formats are recognized automatically, but forcing a particular format interpretation is possible if necessary. Where additional options are available, a secondary panel will open. For example, when molecular data from a PDB file are being loaded into MOE, the Load PDB File panel will be raised, or when an SKC file (sketch file) is opened, the Sketch View panel is raised.

In MOE, the notion of opening a file has been extended beyond such simple operations as opening a molecule file into the system or opening a database file into a database viewer. MOE's system for opening files allows you to browse files, to operate on multiple files, and to select from a variety of operations. For example, you can choose to either load a molecule into MOE or add it to a database file. The Open system is not limited to files containing molecular data. Opening an SVL file for example, allows you to load or run the SVL module.

To open a file, choose File | Open in either the MOE Window or the Sequence Editor, or use MOE | RHS | Open.

Filters

The panel is made up of different pages, for opening files from folders, or from obtaining structure files from PSILO or the PDB. Navigation controls are found at the top of the panel.

The first page of the Open panel presents a list of files from which you can select one or more to open, and a list of operations, common to all selected files, from which you can choose the operation to perform. For information on browsing files and directories, see the documentation on the File Selection Box.

Choose All Files to filter the list by file extensions of common types. In addition to the given file extensions, arbitrary file patterns can be entered in the filter text field.

Tip  File patterns are recognized only in the filter text field. File patterns will not be recognized if entered in the file text field. Items in the file text field must match an existing file to allow for any open operation to proceed.


To open files, select them in the list, or type their names into the file text field. If a single file is selected, its type will be shown in the Open As field, and the default action for that type will be shown in the Action field. If MOE does not correctly identify a file's type, you can enforce a file type by choosing from the Open As option list. Similarly, you can choose a non-default operation from the Action list.

If multiple files are selected, they will be shown in the text field as a comma-separated list. If all the selected files are the same type, that type will be shown in Open As option list, otherwise the type will be blank. The Action option list will initially show the default operation for the current type. If more than one type of file is selected, the Action option list will be populated with the operations (if any) common to all the selected files.

If the selected operation is Open Molecule, then the Close System checkbox will be available. If enabled, the current molecular system will be closed, and the new molecular data will be centered in the MOE Window.

To perform the currently chosen action on the selected files, press OK or Apply. Double-clicking in the list is the same as selecting a single file and pressing OK, in which case the default action associated with the selected file will be performed. Press Edit to load all the selected files into individual SVL Text Editors. When either Apply or Edit is pressed, the panel remains open.

Downloading PDB Files from PSILO or the RCSB

The PSILO and RCSB pages of the Open panel are direct links between MOE and a PSILO server and the RCSB Protein Data Bank website.

When selected, the PSILO and RCSB pages immediately check for an index file corresponding to the chosen resource in the DOWNLOADS directory and use its contents to initialize the Open panel file list. These index files will be created if necessary and automatically updated using web services. To download files, select their entries in the list or type their codes into the Codes text field, then press OK or Apply. Double-clicking in the list is the same as selecting a single entry and pressing OK. Note that files can be downloaded even if the index file is not available, or if its code is not in the current index file. There is no need to wait for the index to finish updating before downloading files this way.
Note:   Downloading files from the RCSB or a PSILO server requires that Java be configured to run with MOE. Please read instructions on installing Java if there are problems using the download tool.

The index list displays the code, (accession) date, resolution (blank if NMR) and the entry name (title). The Codes and Filter fields can be used to search for specific database entries. The Codes field restricts the visible entries to those that match at least one code or pattern entered in the text field. The Filter field searches the index file for words, word fragments or quoted phrases. If "Or" with Filter is on, filter matches will be added to the list of visible entries; otherwise the Filter field will be used to further restrict which entries are displayed in the list.

The Codes and Filter text fields both admit the use of wildcards:

* matches 0 or more of any character
? matches any single character
@ matches any single character in [a..z, A..Z, _ ]
# matches any single digit [0..9]
[...] matches any character in the set; ranges (using dash (-) symbol) are allowed
[^...] matches any character not in the set
Note that an implicit * is placed at the beginning and end of each word. For more on wildcards, see findmatch.

Browsing the MOE Protein Database

The PDB page is used to load molecules from MOE's internal protein database. Alternatively, the PDB page can be used to browse any molecular database file with a code (or chain name) field and a molecule field.

When selected, the PDB page immediately loads index data from the specified MDB file (use Browse to select a different file). To load molecules into MOE, select their entries in the list and press OK or Apply. Double-clicking in the list is the same as selecting a single entry and pressing OK.

The index list displays the code, (accession) date, resolution (blank if NMR) and the entry name (title). The Codes and Filter fields can be used to search for specific database entries. The Codes field restricts the visible entries to those that match each at least one code or code pattern entered in the text field. The Filter field searches the index file for words, word fragments or quoted phrases. If "Or" with Filter is on, filter matches will be added to visible list; otherwise the visible list will be further restricted to entries matching both fields.

The Codes and Filter text fields both admit the use of wildcards:

* matches 0 or more of any character
? matches any single character
@ matches any single character in [a..z, A..Z, _ ]
# matches any single digit [0..9]
[...] matches any character in the set; ranges (using dash (-) symbol) are allowed
[^...] matches any character not in the set
Note that an implicit * is placed at the beginning and end of each word. For more on wildcards, see findmatch.
