Introduction

The Atom Selector and Residue Selector panels offer an extensive array of selection controls and the ability to combine selections using logical operators. These panels may be called into play when the built-in menu commands of the the MOE Window and Sequence Editor are insufficient. For basic selection tasks, the MOE and SE menu commands provide a variety of pre-defined atom and residue selection commands such as ligand and pocket atom selection, atom selection by element, chirality, and protein properties, or chain, residue, and child atom selection with given properties. The Atom Selector panel additionally supports such selection criteria as ring connectivity, atom name, accessibility, and pharmacophoric type, among others, and the Residue Selector panel such criteria as amino acid classes and pattern matching. Additional flexibility is gained through the use of named sets, which are collections of molecular objects identified by a unique name. Both panels support the use of such sets.

Named Sets

In MOE, molecular data can be managed as named sets or collections. A collection comprises any mixture of atom, residue, and chain keys, and has a unique associated name. The selection panels allow named sets to be manipulated using the Named Set field. In the Atom Selector only atom keys will be added to a named set and in the Residue Selector only residue keys will be added to a named set. The atom set further allows for 10 predefined sets to be created.

Named sets are displayed in three locations in the selection panels.

  1. The option menu at the top.

    These menus define the set of objects to be included in the selection operations. If any named sets are defined then they will appear in the menu. When chosen, only the objects (and child objects) in the chosen set will be used for the selection.

  2. The shortcut menu in the Named Set text field.

    The Named Set field is used to create new or overwrite existing sets. The shortcut will show any currently named sets.

  3. The Sets: menu in the selection button area.

    The Sets menu located among the selection buttons of the panel is used to operate on the objects in the set itself. The operation performed will depend on the defined set and the operation chosen.

Named sets are stored with the system and will be reinstated when the molecule file is reopened. Empty sets are not allowed, thus, creating a named set with no objects is equivalent to deleting the set from the system.

Defining the Selection Set

The Atom Selector and Residue Selector panels generally operate on a restricted set of atoms or residues called the defined object set. When a button is pressed, only the objects (or child objects) will be used. If, say, a selection of all Carbon atoms is chosen, only those carbon atoms located in the defined set will be operated upon.

Defining this restricted set is done through the options at the top of the panel using a combination of the option menu and the checkboxes. The intersection of all chosen restrictions will define the set.

The Defined Atom Set

The defined atom set is specified by the following controls:

All Atoms Include all atoms in the system.
Selected Residues Restrict operations to atoms whose parent residue is selected.
Named Set Restrict operations to atoms in the set or to child atoms of residue and chain objects in the set.
Selected Chains Restricts operations to atoms whose parent chain is selected.
Visible Atoms Restricts operations to visible atoms.

If all checkboxes are turned off and All Atoms is specified then the defined atom set is all the atoms in the system.

The Defined Residue Set

The defined residue set is specified using the following controls:

All Chains Include all residues in all chains.
Selected Chains Restricts operations to residues whose parent chain is selected.
Named Set Restricts operations to residues in the set and to child residues of chain objects in the set.
Residues with
Visible Atoms		 Restricts operations to residues with at least one atom visible in the MOE Window.
Nonempty Residues Restricts operations to residues with at least one atom.

If all checkboxes are turned off and All Residues is chosen, then the defined residue set is all the residues in the system.

The Selection Operation

In both selection panels, the Operation radio buttons are used to combine consecutive selections.

or Add the objects satisfying the new criteria to the current selection set.
and Select the intersection between the objects satisfying the new criteria and the previous selection set. Deselect any objects not in this set.
deselect Deselect the objects satisfying the new criteria.
replace Clear the current selection and replace it with the objects satisfying the new criteria.

The Atom Selector

The Atom Selector allows complex selection operations to be performed on sets of atoms. From this panel you can perform logical operations on selection sets, as well as select atoms based on specific properties, substructure or proximity. The Atom Selector is opened from the MOE Window Selection Prompter. Choose

MOE | Select | Selector
and in the Selection Prompter interface, press the UI button. The following panel will open:

The buttons to the right of the Atom definition controls operate upon the defined atom set, as follows:

All Select all atoms within the defined atom set.
Clear Deselect all atom within the defined atom set.
Fixed Select all fixed atoms within the defined atom set.
Hidden Select all hidden atoms within the defined atom set.
Invert Toggle the selection state of all atom within the defined atom set.
Visible Select all visible atoms within the defined atom set.

The Current Set label displays the number of atoms currently selected in the system.

The buttons at the bottom of the panel, View, Hide, Show, operate on the entire current selection set ignoring the defined atom set. View centers all the selected atoms in the 3D rendering area. Hide hides all the selected atoms and Show makes the selected atoms visible.

Predefined Atom Sets

You can save and restore up to ten different system atom sets using the Save Set/Load Set buttons. Loading an atom set will restore the selection set and combine it with the previous selection set according to the chosen logical operation. The saved selection set remains in memory as long as it is not overwritten. A bold label indicates the set currently has a value.

Atom Selection Buttons

The selection buttons are grouped together based on property. You can select by Element, Name, SMILES or use a selection from one of the pages located in the pager: Accessibility, Chiral, Connectivity, Geometry, General, Pharmacophore, and Protein.

Element Select atoms based on their element. Table contains the periodic table, as well as entries enabling you to select based on row, column, heavy atom (Q), transition metal (T), or lone pair (LP).
Name To select atoms based on their atom name you can use one of the buttons (A0..A6) or turn on the Other toggle which will change the display to allow you to enter the name in the text field. Press Apply for the selection to be performed.
SMILES Enter a molecule input language string representation. Enter the string and press All to select all the atoms within matching substructures or Root to select just the root atom of the substructures which matched.
Proximity Select atoms within a given radius of any currently selected atom. Enter the desired radius (in angstroms) in the Radius text field and then press the Proximity button. Repeatedly pressing proximity will cease to have an effect if the operation selected no new atoms. In this case you may wish to increase the radius and try again.
Accessibility Selects atoms which are buried (inaccessible) or exposed (accessible) to a probe sphere of the given radius. The probe must touch the accessible atoms without intersecting any other atoms. See ialpha_complex3 for more information on this function.
Chiral Centers: select R and S chiral atoms (aRSChirality). Constraints: select atoms which have chiral constraints (aForceRS) imposed on them.
Connectivity Select atoms based on the size of ring they are located within or atoms which are spiro centers (i.e. which constitute the articulation point between two rings that are connected through a single atom). See aInRing.
Geometry Select atoms based on the selected hybridization. See aGeometry.
General Select inert atoms (atoms marked for omission from potential energy calculation, e.g. using MOE | Edit | Potential | Inert or aSetInert) or ?? MM Type atoms (atoms with an unknown MM Type). Select polar or non-polar hydrogens.
Pharmacophore Select atoms of the chosen pharmacophoric type.
Protein Select alpha carbon atoms (these have atom name "CA"), backbone atoms (atoms with the backbone bit set), sidechain atoms (atoms not on the backbone), or peptide bonds (atoms in a peptide bond).
Extend To extend the selection based on the current selection use the Extend buttons. All of these selections select within the defined atom set using the chosen logical operation.
  • Bonded To. Atoms bonded to any currently selected atom.
  • Chain. Atoms within the same chain as any currently selected atom.
  • Domain. Atoms within the same protein domain as any currently selected atom.
  • Hydrogens. Hydrogens attached to any currently selected atom.
  • Light Atoms. Hydrogen and lone pairs attached to any currently selected atom.
  • Molecule. Atoms within the same connected component as any currently selected atom.
  • Residue. Atoms within the same residue as any currently selected atom.
  • Sidechain. Sidechain atoms belonging to the same residue as any currently selected atom.
  • Chain Tag. Atoms belonging to a chain which has the same chain tag as any currently selected atom.
  • Secondary Structure. Atoms within the same secondary structure segment as any currently selected atom.
  • Topologically Equivalent. Atoms matching the graphical structure of any currently selected atom. See aPrioZQH for more information on this operation.

The Residue Selector Panel

The Residue Selector allows the selection of residues based on specific criteria. Selections can be made on residue name (e.g. Alanine), structure (e.g. helix, 4-turn), chemical class (e.g. hydrophobic, acidic), search strings, as well as other methods. The selection operations permit the intersection or union of two selection sets or entire replacement of the current selection set.

The Residue Selector is opened from the Sequence Editor Selection Prompter. Choose

SE | Select | Selector
and in the Selection Prompter interface, press the UI button. The following panel will open:

The buttons to the right of the residue definition controls operate upon the defined residue set, as follows:

All Select all residues within the defined residue set.
Clear Deselect all residues within the defined residue set.
Invert Toggle the selection state of all residues within the defined residue set.
Atoms Select all visible atoms that are children of residues within the defined residue set.

The Select Atoms button at the bottom of the panel is independent of the defined residue set. Pressing this button will select the atoms of the currently selected residues.

Extending the Selection

Extending a selection means expanding the current set of selected residues according to particular rules defined by the extend operations. The currently selected residues are the basis for the extend operation, and any additional residues can only be selected if they are within the defined residue set. Each extend operation defines a rule that determines how additional residues will be selected:

Column All residues that are in the defined residue set and are also in the same column as at least one currently selected residue.
Secondary Structure Residues that are in the defined residue set and are also in the same secondary structure segment as a currently selected residue.
Domain Residues that are in the defined residue set and are also in the same protein domain as a currently selected residue.
Radius Residues that are in the defined residue set and that are located within the specified radius (in angstroms) of a currently selected residue. The text field contains a shortcut for common values, however, any value may be entered. Press Proximity to perform the extend operation.
Neighbor Residues that are in the defined residue set and that are neighboring a currently selected residue in terms of the residue sequence. The text field permits specifying by how many neighbors on each side to extend. If Columns is on, gaps are taken into account, otherwise they are ignored.

Residue Selection Operation

In addition to setting the defined residue set, the operation may also be changed when using any of the buttons in the Select section of the panel.

Residue Selection Buttons

The buttons available for selecting and deselecting residues are grouped into several categories as outlined in the following sections.

Selecting by Residue Name

To select residues by name use the groups of buttons located in the Name pager. If none of the buttons contain the desired residue name, the Other text field can be used to enter an arbitrary name. The Name pager contains the following pages of buttons:

Amino Acid Selecting by amino acid types can be done using either the single letter or the three letter amino acid names. The Single Letter Residues toggle controls which buttons will be shown. Included in this list is UNK for unknown three letter codes. The three letter codes correspond to the values returned by the svl function rName.

The single letter codes correspond to the values returned by the svl function Seq_NameToLetter.

Nucleic Acid Nucleic acid residues are selected based on name alone. For example, if a non-nucleic acid residue has the name 'a', then it will also be selected.
Cofactors Residues with common cofactor names.
Metals Residues with common metal names.
Other This field can be used to enter a single name which will be searched for in the defined residue set. Often it is desirable to combine a residue name with its UID value in order to more precisely specify a residue. Several examples of how input will be interpreted are as follows:
Text Name UID INS
ARG30A ARG 30 A
ARG30a ARG 30 a
Ar309 AR 309

* *    

Note that in the last example, the star is the name itself, not a pattern matching function. To select based on a pattern string, see selecting by pattern below.

Selecting by Pattern

Residues can be selected based on an amino acid residue search pattern. To do so, enter the search string, a sequence of characters designating the sub-sequence to be found within the current system, in the text field and press Pattern. Residues matching the pattern and within the defined residue set are selected or deselected (depending on the Operation). For example, the sequence gx will select all pairs of consecutive residues in which the first residue is a glycine.

The standard IUPAC one-letter codes for amino acids are used. The patterns are case-insensitive.

Wildcards and variations of a sequence can be specified using the following characters:

Wildcard Description
x
?		 Wildcard matching exactly 1 character.
[] Indicates a set of matchable characters. For example, [ALT] can match Ala or Leu or Thr.
{}
[^ ] 		Set of characters which must not be matched. For example, {AM} or [^AM] both mean any amino acid except Ala and Met.
- Allows each character to be separated from its neighbor by a dash (-).
* Wildcard matching 0 or more characters.
< Pattern must be in the N-terminal. The < must be at the beginning of the pattern.
> Pattern must be in the C-terminal. The > must be at the end of the pattern.

Examples of search strings:

Pattern Interpretation
[AC]-x-V-x-{ED} [ALA or CYS]-any-VAL-any-{any but Glu or Asp}
<A*N> ALA in the N-terminal and ASN in the C-terminal
ASF ALA-SER-PHE located anywhere in the chain sequence

Selecting a Range

Selecting a range of columns can be done quite easily directly in the Sequence Editor by using the columns headers and the Shift-click keystroke to extend a range. This, however, will select all the residues within the specified columns. Using the Range field in the Residue Selector, the residues can be restricted and the operation can be varied.

The range is used to specify multiple residues, not necessarily consecutive. Multiple values are entered separated by blanks or commas. Ranges are specified by separating two numbers with a hyphen (-). The range is expanded to include all integer values in-between the two numbers.

Three buttons are available to use on the specified range:

Pos The residue position (column number) is determined by the SVL function rPos.
Index The residue index is determined by the SVL function rNumber. This is the index of the residue in its parent chain.
UID The residue UID is determined by the SVL function rUID (an integer typically equal to the ID field in PDB files).

Examples:

Text Button Pressed Result
1 5 7-10 Index All residues with an index (rNumber) value of one of: 1, 5, 7, 8, 9, or 10.
1, 6, 7 - 10, 12 Pos All residues with a position (rPos) value of one of: 1, 6, 7, 8, 9, 10, or 12.

Selecting by Scalar

Residues can be selected based on their scalar values. The sliders below the buttons allow a range to be specified.

Set the desired range and then press one of the scalar buttons. Note that the name on the buttons will change automatically when the scalar is recalculated.

Selecting by Atom Data

The Atom Data page has functions to select residues based on their internal atom data. For example, selecting by atom data can be used to discover residues that should be deleted prior to attempting a homology model.

Missing Atoms Nonempty amino acid residues with one or more standard atoms missing.
Incomplete Backbone Nonempty amino acid residues with one or more standard backbone atoms missing.
Nonstandard Atom Names Nonempty amino acid residues with one or more atoms with nonstandard names.
No Atoms Amino acid residues with no atoms.

Selecting by Amino Acid Functional Class

Residues can be selected based on their amino acid functional classes. The following table gives the amino acids associated with each class:

Hydrophobic: VAL, ILE, LEU, MET, TRP, PHE, TYR, ALA
Hydrophilic: SER, THR, ASN, GLN, ASP, GLU, ARG, LYS, HIS
Acidic: GLU, ASP
Basic: ARG, LYS, HIS
Polar: ASN, SER, THR, GLU
Hydroxyl: SER, THR
Aliphatic: ALA, VAL, LEU, GLY, ILE, MET
Aromatic: PHE, TYR, HIS, TRP

Selecting by Structure

To select residues based on their underlying structure, use the Secondary Structure or DSSP Label pages. The SVL function rActualSegment is used to determine these residues.

Helix: 3-10-helix, 3-10-helix-left, helix, helix-left, pi-helix, pi-helix-left
Sheet: strand, anti-strand
Turn: 3-turn, 4-turn, 5-turn
None: residues whose rActualSegment token is 'none'.

Selecting by Ribbon Mode

The Ribbon Style page buttons identify residues whose ribbon mode is set to the specified token as determined by the SVL function rRibbonMode. Whether or not the ribbon is currently drawn in the MOE Window does not affect these buttons.

Selecting by Residue Type

There are four buttons available on the Residue Type page allowing the selection of specific kinds of residues. The residue types identified are: Amino Acid (as determined by the SVL function rType), Nucleic Acid (as determined by the SVL function rType), Salts & Solvent (waters and some simple ions, i.e. PO3, PO4, SO3, SO4), and Ligand. A ligand is defined as a residue that is not a solvent, not an amino acid and not a nucleic acid yet has more than one atom (i.e. none of the above types).
