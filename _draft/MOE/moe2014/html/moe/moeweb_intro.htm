The MOE distribution includes a number of MOE/web applications that provide web access to built-in MOE applications. A tutorial is also provided which describes how to develop a new MOE/web application. To run the MOE/web applications, you must first have installed MOE/web. In your browser, visit the web page 

   http://SERVER:8888/

Homology Search

The Protein Homology Search web application provides web access to MOE's built-in Homology Searching application. The web application accepts an amino acid sequence and returns the results of a homology search. The output can be filtered by applying a cutoff to the statistical significance Z-scores of the search results. More details of the methodology can be found in the chapter on Homology Searching. SD Filter

The SD Filter web application provides web access to the sdwash and sdfilter tools of MOE's suite of SD tools. The purpose of the SD Filter application is to standardize and correct molecular data stored in SD files, rejecting records that violate such rules as Lipinski's rule-of-five [Lipinski 1999], Oprea lead-like [Oprea 2000], or whether a molecule contains reactive groups.

The washing step normalizes the molecular data for each record in the input SD file. Such operations as hydrogen adjustment, salt (small component) removal, acid-base normalization, ylide-bond standardization, wedge-bond regeneration for correctness and aesthetics, and flat depiction using MOE's 2D depictor are performed. These operations are optional and can be enabled or disabled as desired.

The filtering step removes records from the input SD files based on rules defined by programmable filters. In addition to the Lipinski druglike and Oprea leadlike rules, there are filters for reactive group content, restriction on ring size, molecular weight, number of chiral centers, number of rotatable bonds, and element composition, among others. There are also filters for substructure requirements that are specified using SMARTS strings.

All records that pass the filters are written to the output file. The rejected records are written to a separate file, and both files are available for download.

Small Molecule Properties

The Small Molecule Properties application provides a drawing area in which to sketch the small molecule for which properties will be calculated. The sketching tool is Peter Ertl's JME Molecular Editor, available at www.molinspiration.com/jme. This is a Java applet and therefore Java must be installed in order for the Small Molecule Properties application to run.

Various molecular properties are calculated, including number of H-bond donors and acceptors, number of chiral centers, number of rotatable bonds, logP (octanol/water), molecular weight, and van der Waals surface area and volume. These calculated values correspond to molecular descriptors available in MOE's Calculate Descriptors application.

Structure Factor Check

The Structure Factor Check application is based on the SFCHECK program [Vaguine 1999] from the CCP4 suite, and also computes the Real Space R Factor [Jones 1991]; Using experimental structure factor and model molecular coordinates, it computes various electron density maps and assesses the quality of the fit to the model. The results may highlight possible errors in the model.

The application takes as input a structure file containing molecular coordinates, and a structure factor file. The structure files can be provided in PDB, mmCIF, MOE, or MOL2 formats. The structure factor files can be provided in CIF, MTZ, or CNS formats.

The space group, cell dimensions, and resolution limit can be provided optionally as additional parameters, when they are missing or when you wish to override them.

After the input files and parameters have been specified, press Launch Check. At this point, the application will read the input files and, if there is more than one intensity/magnitude or sigma column, the input page will be regenerated with Specify Columns options. Press Launch Check again to do the calculations.

The output page generated by the application is divided into several areas, including an Input area that summarizes the inputs, and a Plots area.

The plots give a visual indication of how well the model matches the electron density. Included are such plots as density correlation [Vaguine 1999], where low values indicate poor agreement, Real Space R Factor [Jones 1991], which measures the agreement between the observed and the calculated electron densities, B-Factor, which is a measure of atom mobility, and the Wilson Plot, which assesses data quality.

Structure-Activity Report

The Structure-Activity Report web application provides web access to MOE's built-in Structure-Activity Report application. The input of the web application is a molecular database containing small molecules with their activity data. The input molecules are assumed to be mostly based on a small number of scaffolds which are sufficiently analogous that the implied R-groups have common meaning throughout the series.

The input database, provided in either MOE MDB (.mdb) or MDL SD (.sdf) file formats, is read and parsed when Submit Input Data is pressed. A new page containing various options pertaining to the input data is then generated.

This second page provides a means of specifying the meaning of the numeric data fields contained in the database, as well as of providing molecular scaffolds. Note that activity data must be present in order for the application to run successfully.

If scaffolds are not provided, they will be automatically detected. Scaffolds can be provided in a molecular structure file or using the molecular sketcher provided, Alex Clark's SketchEl molecular drawing tool. This is a Java applet, therefore Java must be installed to be able to input a scaffold using the sketcher.

Antibody Homology Modeling

The Antibody Homology Modeling web application provides web access to MOE's built-in Antibody Modeler application. It takes as input query sequences for light and heavy antibody chains and a set of template searching and forcefield parameters. The VL and VH chains can be supplied as amino acid sequences, or as a FastA (.fst) file.

The outputs are presented over several pages: CDR Clusters (Complementarity Determining Regions), Hydrogen Bonds, ASA (Accessible Surface Area) and RMSD (Root Mean Square Distance). An example of the generated CDR Clusters and H-Bond Profiles output pages are as follows:

Database Browser

The Database Browser web application provides a web equivalent of MOE's Database Browser application. It generates a 2D snapshot of each compound in a molecular database, which can be viewed in either table or tile format.

The application generates 3 pages of output: Database Info, Tile Layout, and Table Layout. The Database Info gives information about the input database. The remaining two pages are launch points for generating the tile and table layouts, respectively.

In both Tile and Table mode, MOE's 2D depictor can be chosen to produce aesthetically pleasing 2D renderings of the molecules. The Table view can displays each row in the database as a row in the page. Numeric fields can be shown. In Tile view, multiple molecules can be placed on each row. Tiles can be annotated.

Configure Project Database

Configure Project Database is a tool that can be used to prepare an XML Configuration File for use with the MOE Project application. It allows the fields in the output database for MOE Project to be defined, named, sorted, and so on, and all of the options pertaining to those fields can be modified as well. The tool includes complete online documentation, where more details on the various options can be found.
