Syntax

    current_mode_token = Stereo []
    Stereo new_mode_token

Description

The Stereo function controls the stereo projection modes. The mode is specified by the argument token and can be one of

Mode Description
'none' Disables stereo projection.
'quad-buffer' Enables quad buffer stereo video. Each eye is rendered into a separate image buffer and special hardware (e.g. NVidia 3DVision Kit, Nuvision or StereoGraphics shutter glasses) is used to rapidly alternate the display of each image. On some computer platforms, the graphics driver can be configured for passive stereo equipment (polarized glasses) by outputting the images for both eyes to separate outputs (e.g. two projectors).
This requires appropriate OpenGL or other graphics drivers which may not exist on all computer platforms.
'isl' Enables autostereoscopic rendering. Each eye is rendered simultaneously and special filters in the autostereo monitor ensure that each eye view can only be seen by the matching viewer eye. Autostereoscopic viewing requires the Interactive Stereo Library from SGI Japan, which may not be available for all computer platforms, and a supported autostereoscopic monitor.
'over-under' Enables over/under stereo video. The left eye is rendered on the top half of the screen and the right eye on the bottom half. Special hardware (e.g. NuVision or StereoGraphics) is used to overlay the two images resulting in a stereo composite. No operating system drivers are required.
'left-right' Enables left/right or split pair stereo. The left eye is rendered on the left half of the screen and the right eye on the right. By relaxing one's eyes ("looking through the monitor") one can overlay the two images and see stereo.
'interlace' Enables line-by-line interleaving of left and right eye images used by some polarized filter stereo systems (e.g. Zalman 2D/3D monitors).
'anaglyph' Enables anaglyph stereo. Anaglyph stereo colors the left and right eye images complementary colors and superposes the two images on the screen. The 3D effect is then achieved by wearing glasses with one lens of each complementary color. MOE supports two types of colored glasses: green-magenta (green on the left eye) and red-cyan (red on the left eye). The video card you are using must support GLSL shaders.
'parallel' Enables parallel projection mode. This is not really a stereo mode but a mode in which the molecular system is rendered without perspective.

The ViewSetup options eye_exchange, eye_location and eye_separation apply to all stereo modes that display two images (i.e. other than none or parallel). The ViewSetup option stereo_pairsep controls how far apart the images are in left-right stereo mode. The ViewSetup options stereo_ou_scale and stereo_ou_offset allow for calibration of the over-under image size and overlay. The anaglyph stereo color mode (green-magenta or red-cyan) can be set with the option anaglyph_mode (1 for green-magenta, the default, and 0 for red-cyan). 

    svl> Stereo 'none';           // turn off stereo mode
    svl> Stereo 'anaglyph';       // enable green-magenta anaglyph stereo
    svl> Stereo [];               // return current stereo mode
    'anaglyph'

Obsolete

The SVL functions SplitPairStereo and ParallelProj were used to control the left-right and parallel projection modes in verions of MOE prior to (and including) 2001.01. These functions are now obsolete.
