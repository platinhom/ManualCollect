Syntax

    db_ValueCluster [ 
        src_mdb,
        ['field_names'],
        options
    ]

    where options is:

    [
        esel          : flag,
        write_cluster : flag,
        cfield        : 'field_name',
        write_weight  : flag,
        cwfield       : 'field_name'
    ]

Description

Assigns cluster codes to a collection of fields based on the values in the specified fields. (See Value Cluster Panel for more information on cluster codes.) This function takes the following parameters as input:
src_mdb
Filename or database key of the database to cluster.

['field_names']
Vector of field name tokens specifying the fields to be used in the cluster analysis.

options
Vector of attribute-value colon-separated pairs. Supported options are:

esel:flag
If flag is non-zero, the calculation will be restricted to selected entries in the source database (provided that the source database is open in a Database Viewer). If the database is not open in a Database Viewer or flag is zero, then this option is ignored and all entries are used in the calculation. The default is 0.

write_cluster:flag
If flag is non-zero, then a new field of type integer (named based on cfield) will be created in the database and each entry's cluster code written to it. If flag is zero, then this output is suppressed. The default is 1.

cfield:'field_name'
Name of the field (a token) into which the cluster codes will be written. The default is '$CLUSTER'.

write_weight:flag
If flag is non-zero, then a new field (named based on cwfield) will be created in the database and the row weights written to it. If flag is zero, then this output is suppressed. The default is 0.

cwfield:'field_name'
Name of the field into which the cluster weights will be written. The default is '$WEIGHT'.
