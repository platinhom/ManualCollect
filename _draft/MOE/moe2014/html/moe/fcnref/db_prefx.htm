Syntax

    db_Prefix [ mdb, entry_keys ]
    db_Prefix [ 'filename', entry_keys ]

    old_field_order = db_FieldPrefix [ mdb, field_names ]
    old_field_order = db_FieldPrefix [ 'filename', field_names ]

    old_field_order = db_ReorderFields [ mdb, field_names, field_pos ]
    old_field_order = db_ReorderFields [ 'filename', field_names, field_pos ]

Description

The following commands are used to modify the order of entries and fields in a database. Please note that all database modifications, including entry and field re-ordering, are saved to disk.

In all cases, the database mdb is identified by filename or by key as returned by db_Open. 

db_Prefix [ mdb, entry_keys ]
db_Prefix [ 'filename', entry_keys ]

Changes the order of entries in the database.

Entries are specified by keys in entry_keys and placed at the beginning of the database. Their order in entry_keys determines their order in the database. If an entry key appears twice in entry_keys, the first occurrence is used.

The following code moves the last entry to the top of the database file specified by db_key: 

    db_Prefix [ db_key, last db_Entries db_key ]


old_field_order = db_FieldPrefix [ mdb, field_names ]
old_field_order = db_FieldPrefix [ 'filename', field_names ]

Changes the order of fields in the database.

Fields are specified by names in field_names and placed at the beginning of the database. Their order in field_names determines their order in the database. If a field name appears twice in field_names, the first occurrence is used. The field names in the original field order are returned.

The following code moves the last field to the beginning of the database file specified by db_key: 

    fields = first db_Fields db_key;
    db_FieldPrefix [ db_key, last fields ];


old_field_order = db_ReorderFields [ mdb, field_names, field_pos ]
old_field_order = db_ReorderFields [ 'filename', field_names, field_pos ]

Changes the order of fields in the database. The field names in the original field order are returned.

Fields are specified by names in field_names placed in the position specified in field_pos. Indices in field_pos must be integers greater than or equal to 1.

field_pos and field_names must be of the same length, and no duplicates are allowed in either vector.

The following code moves fields mol and name to the 5th and 3rd column respectively, of the database file specified by db_key: 

    db_ReorderFields [ db_key, ['mol','name'] , [5,3] ]
