Syntax

    ph4_ClusterMDB [
	'src_database',
	'fp_code',
	'fp_sim',
	options
    ]

    where options are:

    [
        esel    : flag
        overlap : number
        cfield  : 'field_name'
        sim     : number
        mfield  : 'field_name'
        matfile : 'filename'
    ]

Description

Assigns cluster codes to a database using fingerprint similarity and Jarvis Patrick clustering. This function takes the following parameters as input:

'src_database'
Token indicating the filename of the database.

'fp_code'
Token indicating the fingerprint code to use in the calculation. The codes of the available fingerprinting methods are the first element of the return value of ph4_FingerprintList.

'fp_sim'
Token indicating the similarity metric. The metrics available are the fourth element of the return value of ph4_FingerprintList.

options
Vector of attribute-value colon-separated pairs:

esel:flag
If flag is true (one), the calculation will be restricted to selected entries in the source database (provided that the source database is opened in the Database Viewer). If no molecular database viewer is opened on the source database or flag is false (zero), then this option is ignored and all entries are used in the calculation. The default is 0.

overlap:number
Percentage overlap which is used to determine whether two molecules belong in the same cluster. Used to compare neighbor lists of molecules. Default is 0.85.

cfield:'field_name'
Token indicating the name of the cluster field into which the cluster codes will be written. The default is '$CLUSTER'.

sim:number
Percentage similarity which is used to determine whether two molecules are similar. Used to compare fingerprints. Default is 0.85.

mfield:'field_name'
Token indicating the name of molecule field to use.

matfile:'filename'
Token indicating the matrix filename. If this is specified, the similarity matrix will be written to 'filename'; otherwise, it will not be written.
