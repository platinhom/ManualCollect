Syntax

Obsolete Syntax

    GSurface [ obj_key, color, origin, P1, P2, spacing, level ]

Description

Obsolete Usage

Writes into the object specified by obj_key an iso-surface at the height specified in level. The surface is drawn in the color color, and is calculated for the volume of space sandwiched between two x-y planes (Plane 1 and Plane 2).

P1 and P2 are matrices of function values evaluated at the grid points of the x-y planes Plane 1 and Plane 2. P1 and P2 are each passed as a vector of rows of values.

For example, if P1 is the matrix 

    a b c d
    e f g h
    i j k l

it should be passed as: 

    [ [a,b,c,d], [e,f,g,h], [i,j,k,l] ]

The origin of Plane 1 is given by the 3-vector [x,y,z]. The separation between Planes 1 and 2 is spacing, which is also the grid spacing. Unit extension applies to all arguments.

Color is specified as RGB intensities:

    2562r + 256g + b

i.e. one byte per color.

To calculate the iso-surface of a given level for a particular function, slice the volume of space of interest into x-y oriented slabs. Each pair of x-y slab surfaces correspond to Planes 1 and 2. Calculate the function values in each plane over a grid, and pass them as rows of values to GSurface.
