A number of useful databases are distributed with MOE. Some are designed for use with specific MOE applications while others are for more general use.

Small Molecule Databases

Leadlike Conformer Database. The MOE Conformation Database contains 3D conformations in MOE Molecular Database (.mdb) format for approximately 650,000 (Oprea) lead-like chemical structures. These are ready to be used for pharmacophore searching, fingerprint analysis, and so on. The structures were obtained from chemical suppliers' public catalogs, including the following:

The conformations were generated with the MMFF94x forcefield using Conformation Import in MOE. A limit of 4.5 kcal/mol strain energy was imposed with a 500 conformation limit for each molecule. Duplicate conformations were removed with a heavy atom RMSD tolerance of 0.6 angstroms (0.75 angstroms for conformations with strain greater than 3.5 kcal/mol). These databases (leadlike_conf_###.mdb) can be obtained from the MOE download page, or by contacting CCG Support.

Linker Database. The MOE Linker Database contains conformations for approximately 750,000 fragments suitable for scaffold replacement methodologies. The fragments were generated using MOE's SD Pipeline Command Tools from 4.3 million unique structures culled from medicinal chemistry literature and various vendor catalogs. On average there are 24 conformations per molecule. The database (linker_2010_10.mdb) can be obtained from the MOE download page, or by contacting CCG Support.

ChEMBL r14 Database. The ChEMBL r14 Database contains conformations for approximately 830,000 fragments suitable for scaffold replacement or combinatorial chemistry methodologies (labeled linkers). The fragments were generated using MOE's SD Pipeline Command Tools from 1.2 million structures in Release 14 of the ChEMBL database of bioactive drug-like small molecules. The database (chemblr14_frag.mdb) can be obtained from the MOE download page, or by contacting CCG Support. Subset libraries are available in $MOE/lib.

RECAP Fragments. $MOE/lib/recaplib.mdb is the output of RECAP analysis on a collection of ~750,000 diverse lead- and drug-like chemical structures from vendor catalogs and medicinal chemistry journals. It can be used with the RECAP Panel.

Protein Databases

Non-redundant PDB. $MOE/lib/pdb.mdb is a database of non-redundant chains derived from the PDB. Chains were clustered such that all pairs of chains within a cluster had 90% pairwise sequence identity as well as similar structure. One representative was chosen from each cluster based on resolution, R free, and number of missing residues not at the ends of chains. Note that ligands and solvent are not included in this database, only the protein chains are stored. This database may be searched with the the MOE-PDB page of the Open panel.

Protein Families. The non-redundant set of PDB chains is further clustered structurally to form families. These structure-based sequence alignments are stored in the $MOE/lib/pdb.fam file in plain text. Small families are enhanced with additional sequences from UniRef 50. This file can be used during Homology Searching to identify possible templates to a query.

Antibody Modeling Databases. The central building block for knowledge-based antibody homology modeling is the antibody structure database, which constitutes the information pool of the antibody homology modeling procedures. The current antibody structure collection (MOE 2014.09) is composed of over 3000 VL and 3000 VH chains stored in $MOE/lib/fab.mdb.

For more information, see the MOE Antibody Database section.

Kinase Modeling Databases. The MOE kinase database is part of the MOE Project databases described in the previous section, and is found at $MOE/lib/project/Protein Kinase.mdb. After their identification, kinase chains are further classified based on their sequence identity within the kinase core. If this is at least 45% within one of the classes in the reference file $MOE/lib/kin_ref.fam, that class is assigned; otherwise a check is conducted to verify if the sequence is sufficiently close to one of the known pseudo kinases in the pseudo reference file ($MOE/lib/kin_refpseudo.fam). If the sequence identity is higher than 35%, the sequence group is labeled as Pseudo-Kinase. Sequences that are positive with respect to kinase pattern detection (G-Loop, DFG and HRD) but lack sufficient homology with the reference sequences are not given any classification for a kinase group, family, or subfamily. This might apply for distant species like bacteria, parasites, nematodes etc., since the reference collection is biased towards the detection of human kinases. In addition to classifications based on sequence, the conformational states of distinct topologies such as the DFG segment or alpha-C helix are also stored in the database.

For more information, see the MOE Kinase Database section.

GPCR Structure Database. The G protein-coupled receptors (GPCRs) comprise the largest family of integral membrane proteins and one of the most important classes of drug targets. In an effort to assist in 3D structure modeling and function annotation, the GPCR Class A structure repository is provided with the other MOE Project databases in $MOE/lib/project/GPCR CLASS A.mdb.

For more information, see the MOE GPCR Database section.

MOE Project Databases. A number of protein family databases — including kinases and GPCRs — are provided in the $MOE/lib/project directory. These databases have been created using the MOE Project framework, and are all very similar in construction and appearance. Each is superposed into a common frame of reference based on the core region of the reference .moe files, which are also found in $MOE/lib/project. The databases can all be used with the Project Search panel as the repository for all relevant information used in this application.

For more information, see the MOE Project Database section.

Rotamer Library. The Rotamer Library in MOE is a collection of all-atom sidechain conformations created using LowModeMD. These conformations differ from traditional rotamer libraries in that bond length and angle variations are permitted. The intent is to provide adequate coverage of the conformational space available to a sidechain and not just encode dihedral preferences. The rotamers are located in $MOE/lib/amber10.mdb in a packed conformation field associated with each sidechain.

Protein-Ligand Complex Database. $MOE/sample/mol/complex.mdb contains a set of structures from the PDB with experimental binding affinities. For each entry, ligand stores the ligand for which binding was measured, and receptor, the nearby chains. image contains symmetry-related molecules which are near the ligand, if any, and water contains any water chains. dG contains the experimental binding affinity of the ligand in kcal/mol, obtained from various sources in the literature. degeneracy indicates the number of copies of the ligand found in the PDB file; each copy will have its own row in the database. The other fields contain summary information about the structure from the PDB file. This database can be used for validation studies.
