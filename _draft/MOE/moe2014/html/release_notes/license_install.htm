Important License Information

MOE 2014 is not compatible with the license manager from MOE 2012 and earlier versions.

If you are currently serving MOE licenses using a license manager application from MOE 2012 or earlier, you must disable the old license manager and use the new one, otherwise MOE 2014 will not start properly.

On Windows systems, if you are currently using a license service from MOE 2012 or earlier, you must upgrade to the new license service, otherwise MOE 2014 will not start properly.

Note: MOE 2014 is compatible with the license manager from MOE 2013.

Installing on Unix Systems (Unix, Linux, Mac)

In the instructions, $MOE refers to the full path to the MOE root directory. For example, $MOE would be similar to /usr/local/moe2014 on Linux systems, or /Applications/moe2014 on Mac OS.

You can consider setting MOE environment variables to allow copying the commands below as-is. Please refer to your system documentation for the correct syntax for creating environment variables.

To create the environment variable in Unix Bourne Shell or BASH:

export MOE=/usr/local/moe2014

To create the environment variable in CSH:

setenv MOE /usr/local/moe2014

In the instructions, the following sample license file is used for demonstration purposes:

SERVER moeserver.chemcomp.com 001122334455 7788
DAEMON chemcompd $MOE/bin/chemcompd
FEATURE moe chemcompd 2014.09 31-dec-2014 3 \
     SIGN="0123 4567 89AB CDEF 0123 4567 89AB CDEF 0123 4567 89AB \
     0123 4567 89AB CDEF 0123 4567 89AB CDEF 0123 4567"

Your actual license will be different.

Updating the License File on Unix

Replace the contents of the file $MOE/license.dat with the new license key. Replace $MOE in the DAEMON line with the absolute path to the MOE root directory. Assuming that the MOE installation directory is /usr/local/moe2014, the sample license file should look like:

SERVER moeserver.chemcomp.com 001122334455 7788
DAEMON chemcompd /usr/local/moe2014/bin/chemcompd
FEATURE moe chemcompd 2014.09 31-dec-2014 3 \
     SIGN="0123 4567 89AB CDEF 0123 4567 89AB CDEF 0123 4567 89AB \
     0123 4567 89AB CDEF 0123 4567 89AB CDEF 0123 4567"

Attention: For Mac users especially, please ensure that the license file is saved as plain text, instead of Rich Text Format (RTF) or any other format.

Configuring the License Manager on Unix

The following steps are necessary to configure the License Manager on Unix:

  1. Beginning the License Installation on Unix.
  2. Verifying the License Server Status on Unix.
  3. Stopping the License Server on Unix.
  4. Starting the License Server on Unix.

Beginning the License Installation on Unix

Log onto the license server computer and open a Unix shell.

If this is the first time that the MOE license server application is installed on the license server computer, you can directly proceed with Starting the License Server on Unix.

However, if a license server application is currently running on the license server computer, proceed with Verifying the License Server Status on Unix.

If you are not sure whether a license server application is currently running, see Verifying the License Server Status on Unix.

Verifying the License Server Status on Unix

Verify whether the MOE license server application is running by typing:

$MOE/bin/lmutil lmstat -c $MOE/license.dat

If the license server (lmgrd) is running, proceed with Stopping the License Server on Unix.

For all other messages (errors, or the license server is not running), proceed with Starting the License Server on Unix.

Stopping the License Server on Unix

Stop the license server application by typing:

$MOE/bin/lmutil lmdown -c $MOE/license.dat

Proceed with Starting the License Server on Unix.

Starting the License Server on Unix

Start the license server by typing:

$MOE/bin/lmgrd -c $MOE/license.dat -l <PATH_TO_LOG>/flex.log

<PATH_TO_LOG>/flex.log is the path for writing the FlexLM log file, such as $MOE/flex.log or /tmp/flex.log. The log file will be created when the license server application starts, and will be overwritten if it already exists.

 

Installation of the MOE 2014 License Manager is now complete on the license server computer.

Continue with Launching the License Manager Automatically on Unix.

Launching the License Manager Automatically on Unix

On Mac and Linux systems, it is useful to have the License Manager launch automatically during the boot sequence of the license server computer, so that you don’t have to start it manually every time the computer reboots.

To do so, follow the set of instructions that corresponds to the platform of your license server.

  • Configuring the Startup on Mac OS X
  • Configuring the Startup on Linux

Configuring the Startup on Mac OS X

On Mac, the license manager process can be started automatically during the boot sequence by registering a launched startup configuration file, located in:

$MOE/bin-mx86/com.chemcomp.flexlm.plist

To use the script, the following procedure must be followed:

  1. Edit the script file to match your environment. Modify each instance of the path to MOE (the default is /Application/moe).
  2. As root, copy this script to /System/Library/LaunchDaemons, which is the System-wide daemon provided by Mac OS X.
  3. Test the license server configuration with:

    4. user% launchctl load /System/Library/LaunchDaemons/com.chemcomp.flexlm.plist

  4. Verify whether the configuration was properly loaded with:

    5. user% launchctl list

  5. You can shut down the server with:

    6. user% launchctl unload –w /System/Library/LaunchDaemons/com.chemcomp.flexlm.plist

 

The License Manager is now set to automatically start on the license server computer.

Continue with Installing on Client Computers Running MOE.

Configuring the Startup on Linux

On Linux, the license manager can be started automatically using the init script located in:

$MOE/bin-lnux/lmgrd_init

or

$MOE/bin-lnux/lmgrd_init_ubuntu  (Ubuntu, Linux Mint, or similar distributions)

To use the script, the following procedure must be followed:

  1. Edit the script file to match your environment:

    2.         MOE=/usr/local/moe2014

            LOGFILE=/usr/local/moe2014/flex.log

            USERNAME=ccg

    where:

            MOE is the root of the MOE installation directory, such as /usr/local/moe2014

            LOGFILE is the name of the log file created by the license manager, such as /usr/local/moe2014/flex.log

            USERNAME is login name of the user owning the license daemon process.

  2. Login as root.

  3. Copy the script file into the init.d directory:

    4. user% cp lmgrd_init /etc/init.d/lmgrd_init

    or

    user% cp lmgrd_init_ubuntu /etc/init.d/lmgrd_init  (Ubuntu, Linux Mint, or similar distributions)

  4. Ensure that the script is executable:

    5. user% chmod +x /etc/init.d/lmgrd_init

  5. Register the lmgrd_init script so that it is run when the machine starts and stops:

    6. user% chkconfig --add lmgrd_init

    user% chkconfig lmgrd_init on

         If chkconfig is not available, the links can be created manually using the following commands:

         user% ln -s /etc/init.d/lmgrd_init /etc/rc2.d/S99moe_lmstart

         user% ln -s /etc/init.d/lmgrd_init /etc/rc3.d/S99moe_lmstart

         user% ln -s /etc/init.d/lmgrd_init /etc/rc4.d/S99moe_lmstart

         user% ln -s /etc/init.d/lmgrd_init /etc/rc5.d/S99moe_lmstart

         user% ln -s /etc/init.d/lmgrd_init /etc/rc0.d/K10moe_lmstop

         user% ln -s /etc/init.d/lmgrd_init /etc/rc1.d/K10moe_lmstop

         user% ln -s /etc/init.d/lmgrd_init /etc/rc6.d/K10moe_lmstop

  6. Start the MOE license server:

    7. user% /sbin/service lmgrd_init start

         If /sbin/service is not available, the script can be invoked directly using the following command:

         user% /etc/init.d/lmgrd_init start

 

The License Manager is now set to automatically start on the license server computer.

Continue with Installing on Client Computers Running MOE.

You can consider setting MOE environment variables to allow copying the commands below as-is. Please refer to your system documentation for the correct syntax for creating environment variables.

Updating the License File on Windows

Replace the contents of the file %MOE%\license.dat with the new license key. Replace '$MOE/bin' in the DAEMON line with the absolute path to the \bin directory. Assuming that the MOE installation directory is c:\moe2014, the license file should look like:

SERVER moeserver.chemcomp.com 001122334455 7788
DAEMON chemcompd c:\moe2014\bin\chemcompd
FEATURE moe chemcompd 2014.09 31-dec-2014 3 \
     SIGN="0123 4567 89AB CDEF 0123 4567 89AB CDEF 0123 4567 89AB \
     0123 4567 89AB CDEF 0123 4567 89AB CDEF 0123 4567"

Attention: Please ensure that the license file is saved as plain text and encoded in ANSI, instead of UTF-8, Unicode, or any other encoding. To verify the encoding, open the license file using Notepad, and click Save As from the File menu. The encoding information is displayed in the drop-down list at the bottom of the Save As dialog box.

Windows users are provided with a convenient GUI tool, LMTOOLS, for the license installation. It installs the license manager application as a Windows Service, which starts automatically when Windows boots.

Windows users can use the LMTOOLS application to set up the MOE 2014 license service by proceeding to Configuring the License Service on Windows Using LMTOOLS.

Alternatively, Windows users preferring command line tools can set up the MOE 2014 license service by proceeding to Configuring the License Service on Windows Using Command Line Tools.

Configuring the License Service on Windows Using LMTOOLS

IMPORTANT: If the license service from MOE 2012 or previous versions is already running, it must be upgraded to the MOE 2014 license service, otherwise MOE 2014 will not start properly. 

This is done by first removing the existing old license service, and then setting up the MOE 2014 license service as a fresh installation.

To remove an old license service using LMTOOLS, see Removing a MOE 2012 (or Older) License Service Using LMTOOLS.

To set up the MOE 2014 license service (or to update an existing MOE 2013 license service) using LMTOOLS, see Installing the MOE 2014 License Service Using LMTOOLS.

Removing a MOE 2012 (or Older) License Service Using LMTOOLS

The following steps are necessary to remove an old license service:

  1. Starting the 32-bit LMTOOLS.
  2. Stopping the Old License Service Using LMTOOLS.
  3. Removing the Old License Service Using LMTOOLS.
Starting the 32-bit LMTOOLS

In Windows Explorer, navigate to %MOE%\bin-i4w9\ (c:\moe2014\bin-i4w9\, for example).

For Windows 8, Windows 7 and Windows Vista: Right-click on lmtools, and select Run as administrator. Click OK on the confirmation dialog.

For Windows XP: Double-click on lmtools to launch the LMTOOLS application.

 

Proceed to Stopping the Old License Service Using LMTOOLS.

Stopping the Old License Service Using LMTOOLS

In the LMTOOLS application (see Starting the 32-bit LMTOOLS), select the Start/Stop/Reread tab, and then click the Stop Server button.

 

Proceed to Removing the Old License Service Using LMTOOLS.

Removing the Old License Service Using LMTOOLS

In the LMTOOLS application (see Starting the 32-bit LMTOOLS), select the Config Service tab, and the following form will show the existing configuration:

 

Click the Remove Service button, and click Yes on the pop-up confirmation box.

Continue with Installing the MOE 2014 License Service Using LMTOOLS.

Installing the MOE 2014 License Service Using LMTOOLS

The following steps are necessary to install the new license service:

  1. Starting LMTOOLS.
  2. Stopping the License Service Using LMTOOLS.
  3. Preparing and Starting the License Service Using LMTOOLS.
Starting LMTOOLS

Two versions of LMTOOLS exist: 64-bit, and 32-bit. You are encouraged to use the version that corresponds to the architecture of the computer that will run the license server.

64-bit LMTOOLS

Users of 64-bit computers are strongly recommended to run the 64-bit version of the MOE 2014 license service.

For 64-bit systems, navigate to %MOE%\bin-win64 (c:\moe2014\bin-win64\, for example) in Windows Explorer.

 

For Windows 8, Windows 7, and Windows Vista: Right-click on lmtools, and select Run as administrator. Click OK on the confirmation dialog.

For Windows XP: Double-click on lmtools to launch the LMTOOLS application.

Proceed to Stopping the License Service Using LMTOOLS.

32-bit LMTOOLS

For 32-bit systems, navigate to %MOE%\bin-i4w9 (c:\moe2014\bin-i4w9\, for example) in Windows Explorer.

 

For Windows 8, Windows 7, and Windows Vista: Right-click on lmtools, and select Run as administrator. Click OK on the confirmation dialog.

For Windows XP: Double-click on lmtools to launch the LMTOOLS application.

Proceed to Stopping the License Service Using LMTOOLS.

Stopping the License Service Using LMTOOLS

In the LMTOOLS application (see Starting LMTOOLS), select the Start/Stop/Reread tab, and then click the Stop Server button.

 

Proceed to Preparing and Starting the License Service Using LMTOOLS.

Preparing and Starting the License Service Using LMTOOLS

In the LMTOOLS application (see Starting LMTOOLS), select the Config Service tab, then fill in the form with the correct information for your MOE installation:


Service Name: Type a recognizable name such as MOE2014 License

IMPORTANT: If a MOE2013 License Service already exists, do not change the Service Name, otherwise a duplicate service will be created. Instead, just update the Path fields below.

Alternatively, you can remove the existing MOE2013 License Service, and then create a MOE2014 License Service by updating the Service Name field.

Path to the lmgrd.exe file: Use the Browse button to locate lmgrd.exe:

  • For 64-bit systems, use lmgrd.exe in the %MOE%\bin-win64\ directory.
  • For 32-bit systems, use lmgrd.exe in the %MOE%\bin-i4w9\ directory.
IMPORTANT: Make sure the correct 64-bit or 32-bit version of lmgrd.exe is selected, otherwise the license service may not start properly.

Path to the license file: Use the Browse button to locate license.dat. You might need to modify the filter from License Files(*.lic) to License Files(*.dat) to find the license file.

Path to the debug log file:  Enter the path and file name where the log file can be saved. Typically, the log file is saved in the %MOE% directory, such as c:\moe2014\flex.log. If the log file does not exist, it will be created when the license service is launched, otherwise it will be overwritten.

Then, enable both Use Services and Start Server at Power Up, and finally click Save Service.

Note: The MOE paths in the screenshots are samples only. The paths that you use must reflect your actual MOE installation.

Select the Start/Stop/Reread tab, and click the Start Server button. You should see Server Start Successful in the status line.

The License Server is now up and running.

Launch MOE to test the license.

 

Installation of the MOE 2014 License Manager is now complete on the license server computer.

Continue with Installing on Client Computers Running MOE.

Installing on Client Computers Running MOE

Replace the contents of the license.dat file on all other MOE installations with the SERVER line in the license key followed by a USE_SERVER line. For example, the license.dat file on all MOE installations (other than the server machine) should look like:

SERVER moeserver.chemcomp.com 001122334455 7788
USE_SERVER

Note: When using Node-lock licenses, MOE cannot run on other computers.
