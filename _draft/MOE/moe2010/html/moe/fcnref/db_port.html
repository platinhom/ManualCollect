MOE Database Import/Export Functions

Syntax

    db_ImportDB [
	'dst_db',
	'src_db',
	fields,
	names,
	types,
	options
    ]

    where options is

    [
	append:		flag,
	file_field:	flag,
	start_entry:	startnum,
	end_entry:	end_num
    ]

    db_ExportDB [ 'dst_db', 'src_db', field_names, entry_keys ]

Description

db_ImportDB imports specified fields and entries from a source MOE molecular database to a destination MOE molecular database.

'dst_db'
Name of the destination MOE molecular database file to import to, specified as a token.

'src_db'
Name of the source MOE molecular database file to import from, specified as a token.

fields
The names of the fields to import, specified as a vector of tokens. New fields are created in the destination database if necessary. If fields is unspecified (i.e, []), then all fields are imported from the source database, under the original field names (i.e. the names option is ignored).

names
The names to be assigned to the imported fields in the destination database, specified as a vector of tokens. The names must be unique, and the names vector must be the same length as the fields vector.

types
The destination database field types, specified as a vector of tokens. Valid types are given in DB_FIELD_TYPES. The types vector must unit extend to the length of the fields vector.

options
The import options are specified as a tagged vector:

append:flag
If flag is non-zero, database entries are appended to the destination database. Otherwise, data is "merged" into the database starting from the first entry, i.e. the source data overwrites the destination, beginning at the first entry of the destination database. The default is 1.

file_field:flag
If flag is non-zero, then the name of the source database file for each entry is written to the destination database file. The default is 0.

start_entry:startnum
Specifies the entry number (n.b. this is not the entry key but the cardinal number of the entry in the database) at which to begin importing. startnum is an integer >= 1. If start_entry is unspecified (i.e. []), the default is 1 (begin importing from the first entry in the source database).

end_entry:endnum
Specifies the entry number of the last entry to import. endnum is an integer >= start_entry. If end_entry is unspecified (i.e. []), the default is the number of entries in the source database (import up to and including the last entry in the source database). 

db_ExportDB [ 'dst_db', 'src_db', field_names, entry_keys ]

Exports the specified fields and entries from a source MOE molecular database to a destination MOE molecular database which is overwritten, if it exists.

src_db
The name of the source MOE molecular database to export, specified as a token.

dst_db
The name of the destination MOE molecular database file, specified as a token.

field_names
The names of the fields to export from the source database, specified as a vector of tokens. The same names are used in the destination database.

entry_keys
The entry keys of the entries to export from the source database.
