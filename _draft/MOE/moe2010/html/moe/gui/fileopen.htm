Introduction

The MOE open file system is used to perform a variety of of "open" operations on files of different formats. File formats are recognized automatically, but forcing a particular format interpretation is possible in rare cases of need. Where additional options are available, a secondary panel will open. For example, when molecular data from a PDB file are being loaded into MOE, the Load PDB File panel will be raised, or when an SKC file (sketch file) is opened, the Sketch View panel is raised.

In MOE, the notion of opening a file has been extended beyond such simple operations as opening a molecule file into the system or opening a database file into a database viewer. MOE's system for opening files allows you to browse files, to operate on multiple files, and to select from a variety of operations. For example, you can choose either to load a molecule into MOE or to add it to a database file. The Open system is not limited to files containing molecular data. Opening an SVL file for example, allows you to load or run the SVL module.

The set of operations that can be performed on a particular file is determined by the file's "type". Each file is typed automatically. You can, however, force a file to be treated as being of a certain type. This becomes a necessity in the case of, for example, the ASCII database type which cannot be automatically detected.

To open a file, choose File | Open in either the MOE Window or the Sequence Editor.

Using the Open Panel

The Open panel contains a list of files from which you can select one or more files to open, and a list of operations, common to all selected files, from which you can choose the operation to perform.

Selecting Files

To select a file, either select it in the file list or enter its name in the text field and press Enter. A path specification may be included as part of the filename.

To make it easier to find the files you want to select, you can restrict the list of displayed files to those satisfying a given pattern; i.e. you can filter the list.

Note that only those files that are displayed in the file list can be selected. If, when filtering the file list, some of the selected files become hidden (are omitted from the displayed list), those files become unselected. The number of currently selected files is reported in the header of the file list.

Current Working Directory

MOE has a concept of the current working directory. If no path is given when specifying a file then the current working directory is the directory used. The value of the current working directory will affect where files are saved to and read from.

You can change the current working directory from the command line or by using the CWD button in the Open panel. From the Open panel follow these steps to modify the current working directory:

  • Bring up the Open panel.
  • Change to the desired directory.
  • Press the CWD button.
  • Close the panel.

Alternatively, you can change the current working directory from the command line by using the cd function.

Changing Directories

When the Open panel is first raised, the file listing of the current working directory is displayed. To open files from a different directory, you can change the directory of the Open panel. Note that this will have no effect on the current MOE working directory. The Open panel directory is shown in the Path field.

There are several ways to change directories in the Open panel:

  • Enter the name of the directory path in the text field and press Enter
  • Click on the arrow in the text field to open the shortcut list of directory paths; choose a directory
  • Press the Up button to go to the parent directory
  • Double-click on the directory in the file list
  • Select the directory in the file list then press OK

Selecting an Operation

When you have one or more files selected, the available open operations are presented as a set of buttons on the right-hand side of the panel. Only the operations that are common to all selected files will be available. For example, both MOE molecule files and MDLMOL files can be either opened into MOE or appended to a database, but neither file type has any operations in common with SVL files.

If a file was not recognized to be of a given type, you can force its type by using the Force Type option menu. All recognized file types are listed in this menu. When Force Type is set to any type other than Auto, the automatically detected file types are ignored. All selected files are treated as though they were of the specified type so that the operations on that file type are available on all selected files, regardless of whether they are actually applicable.

There are several ways in which to select an open operation:

  • With one or more files selected, press one of the Operations buttons.
  • With one or more files selected, press OK.
  • Double-click on a file in the file list (opens exactly one file).

The latter two cases use the notion of a "default action". For file types with only one possible open operation, that operation is the default and will be performed on double-click or OK. For file types with several possible operations, there is usually one operation that is the default. For example, for SVL files, Run SVL File is the default operation that will be performed on double-click or OK.

Once you have selected an operation, the Open panel closes.

File Information

You can browse the contents of the files in a directory by opening the File Information panel (press File Info). When a single file is selected, information about the file is displayed in this panel, including the first 1000 bytes of the file.

To be able to read/edit an entire file, press Text Edit. This will open each selected file into an SVL Text Editor. The Open panel remains up for further operations.

Changing the File List Display

The file list can display the type, date, size and name of each file in the displayed directory. The files that are shown are the ones that match the current filter and settings. The type, date and size fields are optional and may be turned off in the Open Settings panel. The filter is shown in the Path at the top of the panel and may be altered through the name/filter text field.

Filtering

The files to be displayed may be restricted using a filter, which is a pattern containing a wildcard. The allowable wildcard characters are as follows:

* matches 0 or more of any character
? matches any single character
@ matches any single character in [a..z, A..Z, _ ]
# matches any single digit [0..9]
[...] matches any character in the set; ranges (using dash (-) symbol) are allowed
[^...] matches any character not in the set

For more on wildcards, see findmatch.

There are several ways in which to apply a filter:

  • Type the filter in the text field and press Enter.
  • Click on the arrow in the text field and select one of the filters from the shortcut list.
  • Press Clear to clear the current filter (reset it to *).

To show all files, set the filter to be the asterisk (*). The current filter is displayed as a suffix to the directory in the Path field. The current number of files hidden because of the filter is reported in the header of the file list.

You can specify whether or not the filter is case-sensitive from the Open Settings panel. If the filter is not case-sensitive, then an uppercase letter is not distinguished from its lowercase form.

Note: Directories are always displayed no matter what the filter is set to.

Display Order

You have several ways of controlling the order in which files are listed. First, you can choose how the file list is to be sorted, for example, by name or by type. Second, you can choose how directories are to be listed. Next, you can decide whether to include in the list those files and directories beginning with a dot (.). Finally, you can specify a preferred set of file types to be always given at the top of the file list.

Sorting

Files can be sorted by name, type, size, or date. In the Sort Files By controls, there are two sorting keys for the file list. These apply to both filenames and directory names. The first key is the primary sort key, which means it takes precedence over the second sort key. For example, if you sort first on type and then on name, the files will be clustered by type, with the files of each type being sorted by name. Note that dates displayed in the panel are abbreviated, whereas the dates used for sorting are the full dates. Check the file information (File Info) for the exact last modification date of a file.

In the Open Settings panel, the Case Precedence menu allows you to determine how upper and lowercase letters are handled during name sorting. The following options are available:

a > A Uppercase letters come before lowercase letters
a = A Case is ignored
a < A Lowercase letters come before uppercase letters

The method of sorting may be specified in a local MOE resource file using the following four resources: 

    fileopen.sort_key1=
    fileopen.sort_key2=
    fileopen.sortdown1=
    fileopen.sortdown2=

sort_key1 and sort_key2 can be one of the following: Name, Type, Size or Date. sortdown1, sortdown2 can be either 1 for reverse and 0 for not reverse.

Directories

Directories can be listed together at the top or bottom of the file list, or interspersed among the non-directory files. The controls are in the Open Settings panel. Note that the directories '.' and '..' are always listed at the top of the file list.

Dot Files

Files beginning with the character '.' are usually system files and are less commonly opened from within a MOE session. For this reason, there are special options to omit files and directories with names beginning with a dot ("dot files", "dot directories") from the file list in the Open panel.

Note that on Unix systems, the user menu file is called .moe-menus. In order to open (load) this file from the Open panel, you must unhide dot files in the Open Settings panel.

The number of hidden dot files and dot directories is not included in the reported number of hidden files in the header of the file list.

Preferred File Types

If there are files of a certain file type that are of particular interest to you, you may wish to have them listed first above all other files in the file list (but after directories).

In the Open Settings panel, the Preferred Types text field allows you to enter one or more patterns that will be applied to file types. When Preferred Files is set to Top, the files of any types matching these patterns will be listed first in the file list. The patterns must be space or comma separated. The files of these preferred types will be listed in the same order as the types in the Preferred Types text field.

For example, the Preferred Types 

    moe, m*

will result in all files of type moe being listed first, followed by all files whose types begin with the letter 'm', and finally by the remaining files.

The File Open Settings Panel

Which files to display and how to display them in the file list depends not only on the current filter and directory specified in the Open panel but also on the options given in the Open Settings panel. Here you may set your preferences such as whether or not you wish to see files beginning with dot or whether you prefer directories to be displayed at the top of the list or the bottom.

The settings in an Open Settings panel apply only to the Open panel from which it was opened. To put the settings into effect, press Apply. If you want the settings to apply to all subsequent Open panels in this session of MOE, as well as to subsequent sessions of MOE, press the Save as Default button. Your settings will be written to the user MOE resource configuration (RC) file.

If you have made some changes and wish to revert to the default settings, press Reset. The defaults in the RC file will be retrieved and applied to the current Open panel.

This section describes the options in the Open Settings panel. The panel is opened from the Open Panel using the Settings button.

File Typing Specifies how files are typed. If on, then files are typed strictly by name pattern and extension when entering a new directory. Full file typing is still performed when a file is selected. If off, full file typing is always done.
Directories Specifies whether directories are to be always listed together at the top or bottom of the file list or whether they are to be given no special preference. Note that the current directory (.) and the parent directory (..) are always displayed at the top of the list regardless of this setting. If Ignore is chosen then the directories are listed in among with the non-directory files.
Preferred Files Specifies whether files of the Preferred Types are to be always listed at the top of the file list (after directories), or whether they will be given no special preference.
Preferred Types The file type patterns, space- or comma-separated, that specify which files are to be listed first in the file list. Use double quotes to delimit patterns containing spaces or commas. The files are listed in the same order as the types in the text field.
Case Precedence Specifies the case precedence used to calculate the sort order of the files in the file list.
a > A Uppercase letters come before lowercase letters
a = A Case is ignored
a < A Lowercase letters come before uppercase letters
Name Filter Specifies whether the filename filter is case-sensitive.
Dot Files Specifies whether to include non-directory files beginning with a dot (.) in the file list.
Dot Directories Specifies whether to include directories beginning with a dot (.) in the file list.
Display Specifies what information (type, last modification date, size) to display for each file in the file list. The date information is shown in abbreviated form.
Paths If Do Backslash Processing is turned on, backslashes in filenames or directory names will be interpreted as separator characters in directory paths. Certain Asian character sets require that this feature be turned off. The default is 0, i.e. the backslash is not recognized as a separator character.
Apply Puts the settings into effect for the current Open panel.
Reset Resets the settings to their default values.
Save as Default Applies and saves the settings as the default.

Open Panel Reference

This section describes the options in the Open panel. The panel is opened using File | Open from either the MOE Window or the Sequence Editor.

Double-clicking on a file in the list causes the default open operation (if any) to be performed on that file.

Path The path of the currently displayed directory as well as the currently applied filter. The filter pattern is appended to the directory path.
Name/Filter Text field for entering a filename, directory path, or filename filter. Press Enter to put the typed text into effect.

Entering a filename causes the filename to be selected. If a directory path is included as part of the filename specification, the current directory of the Open panel is first changed to that directory and then the file is selected.

Entering a directory causes the current directory of the Open panel to be changed to that directory.

Entering a filter pattern causes the file list to be filtered by that pattern. Only filenames (not directories) are filtered. The pattern * results in all files being displayed.

Note: The filter remains in effect when the Open panel current directory is changed.

Clear Clears the current file selection and sets the current filter to *.
Up Changes the currently displayed directory to the parent directory.
CWD Sets the MOE current working directory to the directory displayed in the Path field. If the current directory is the same as the directory displayed then the operation is ignored.
File List The file listing of the directory shown in the Path field. The number of selected files and the number of files hidden by the current filter are reported in the header.
