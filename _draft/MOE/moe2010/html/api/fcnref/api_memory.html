MOE 2010.10

API Memory Functions

Syntax

    svl_error svl_MallocE(void *ptr_addr, int n);
    svl_error svl_ReallocE(void *ptr_addr, int n);
    svl_error svl_CallocE(void *ptr_addr, int n);

    void svl_Free(void *ptr);

Description

The memory allocation routines all return an svl_error in the case that the memory could not be allocated. Otherwise they return NULL. If memory allocation was successful then *ptr_addr will point to the newly allocated memory. When n is 0, ptr_addr is set to the address of a buffer of size zero.

Note that since *ptr_addr is compatible with type (void*), the type of ptr_addr should really be of type (void**) and not (void*), used in the declaration. The casting rules of C, however, force us to declare the argument as (void*) in order to avoid compulsory casting of every pointer allocated. 

svl_error svl_MallocE(void *ptr_addr, int n);

Allocates a block of memory of size n and returns the pointer to the buffer in argument *ptr_addr. 

svl_error svl_ReallocE(void *ptr_addr, int n);

Resizes the memory buffer referred to by pointer *ptr_addr to size n bytes. *ptr_addr must either be NULL or point to an already-allocated buffer. If the buffer cannot be reallocated, *ptr_addr will keep its original value. 

svl_error svl_CallocE(void *ptr_addr, int n);

Allocates a block of memory of size n and clears the contents of the buffer (sets it to 0's). 

void svl_Free(void *ptr);

Assumes that pointer ptr points to a previously allocated buffer. The function releases the memory of that buffer. If the argument is NULL, the function does nothing.

A freed buffer must not be accessed again. If the variable ptr is to be used further in the code, it should be explicitly set to NULL after svl_Free: 

    svl_Free(ptr), ptr = NULL;
