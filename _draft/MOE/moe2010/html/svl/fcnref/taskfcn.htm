MOE 2010.10

Task Functions

Syntax

    tkey      = task_key tkey
    curr_tkey = task_key -1
    tkeys     = task_keylist []

    task_kill tkey

    tname = task_title tkey
    task_settitle [tkey, 'new_tname']

    env_value = task_getenv env_name
    name_value_pairs = task_getenv []
    old_env_value = task_setenv [ env_name, new_env_value ]

    task_info []
    task_info tkey
    info = task_info []
    info = task_info tkey

Description

tkey      = task_key tkey
curr_tkey = task_key -1

Replaces all invalid task keys in its argument with zeros. If -1 is passed as the argument then the return value is task key of the current task. 

tkeys = task_keylist []

Returns task keys of all tasks. 

task_kill tkey
Destroys all tasks present in its argument. Invalid task keys are ignored. A task key of -1 specifies the current task. 

tname = task_title tkey

Replaces all valid task keys with the name of the corresponding task. Task key -1 specifies the current task. Zeros are replaced with the empty token, ''. Any other invalid task key produces an error. 

task_settitle [ tkey, 'new_tname']

Sets the name of tasks of all valid task keys to their corresponding new token names. Task key -1 specifies the current task. Task key 0 is ignored. Any other invalid task key produces an error. 

env_value = task_getenv env_name
name_value_pairs = task_getenv []

Returns the value of the task environment variable name env_name (a token). If passed a null argument, task_getenv returns a list of all task environment variables and their associated values. name_value_pairs is a tagged vector, with the variable names as tags.

old_env_value = task_setenv [env_name, new_env_value]

Sets the value of the task environment variable env_name (a token) to env_value. The previous value of the specified environment variable is always returned. 

task_info []
task_info tkey
info = task_info []
info = task_info tkey

Reports useful information about SVL tasks. When called as a procedure (i.e. the value is not assigned), task_info prints the task information to the SVL Commands window. Otherwise, when called as a function, the information is returned as a tagged vector.

When passed a null argument, information about all tasks in the system is returned. If passed a valid task key, only information pertaining to that specific task is returned.

The returned information is as follows:

Name Description
key Task key
title Task title
prio Task running priority
parent Key of parent task
status Task status
stack Return value of the where function
