Overview of Pharmacophore Applications

The purpose of the MOE Pharmacophore Application suite is to effect 3D searches of conformation databases using molecule annotations related to ligand-receptor binding (like, H-bond donor, acceptor, hydrophobe etc.). A pharmacophore is a set of structural features in a ligand that are related to the ligand's recognition at a receptor site and its biological activity. In MOE, pharmacophoric structural features are represented by labeled points in space about a conformation of the ligand. Each ligand conformation is assigned a set annotation points, which is a set of structural features that may contribute to the ligand's pharmacophore. A database of conformations can be searched with a query that represents a pharmacophore hypothesis. The result of such a search is a set of matches that align the pharmacophoric features of the query to the pharmacophoric features present in the ligands of the searched database.

In this document we present an overview of the Pharmacophore Discovery applications and the steps required to conduct 3D Pharmacophore searches. First, however, some definition of terms is required.

  • A ligand annotation is a set of points in space with attached labels that reflect the presence or absence of pharmacophore features at those locations. Each ligand conformation in a database is annotated at search time (on the fly). The ligand annotation is what the Pharmacophore Search procedure "sees" during a search.
  • A pharmacophore annotation scheme, or more simply a scheme, is an automatic procedure to compute pharmacophore annotation points (such as h-bond donor, h-bond acceptor, hydrophobe, etc.) that make up a ligand annotation. MOE contains a number of schemes encoding different methods of ligand annotation.
  • A pharmacophore query is a collection of restrictions imposed upon a ligand annotation and the atoms of a molecule. The purpose of the query is to select a limited subset from a given set of ligand conformations such that all restrictions of the query are satisfied. Queries can be created interactively using the Pharmacophore Query Editor. Queries can be created automatically using the Pharmacophore Elucidator.
  • A match (or hit) is the occurrence of a ligand conformation satisfying a query. A match is the successful mapping of the query features to ligand annotation points, including the corresponding alignment of the ligand with the query such that all restrictions of the query are satisfied. There can be several different ways in which a given query can match a ligand. Hits are found and reported by the Pharmacophore Search application.

The intended usage of the Pharmacophore Applications is as follows.

  1. Generate Conformations. Generate a database of 3D conformations for all molecules that will be the subject of future pharmacophore searches. Existing conformation databases may be searched (such as those distributed with MOE or generated by other programs). See High Throughput Conformation Generation for more information.
  2. Create a Pharmacophore Query. Using one of the Pharmacophore Annotation Schemes and the interactive Pharmacophore Query Editor or the automatic Pharmacophore Elucidator create a pharmacophore query.
  3. Search the Conformation Database. Using the Pharmacophore Search application, search the conformation database using the created query and obtain a database of hits.
  4. Refine the Query. If required, modify the query and use Pharmacophore Search on the same database, or search the generated database of hits with a more restrictive query.

Pharmacophore Conformation Databases. The Pharmacophore Applications operate on databases of 3D conformations. Typically, these databases are stored in MOE Molecular Database format (for speed). Conformations may be stored in a one-conformation-per entry format or a one-molecule-per entry format. The latter format allows for compression of the conformation data. Alternatively, externally generated conformation databases can be searched (such as OMEGA databases, for more information see Pharmacophore Search). MOE will annotate molecule conformations during the search; consequently, no preprocessing or indexing of the conformation databases is required. In addition, the conformation databases may be distributed over the nodes of a cluster of computers allowing for parallel search capabilities.

Pharmacophore Query Editor. The purpose of the Pharmacophore Query Editor is to manually create a query consisting of a set of constraints on the location and type of ligand annotations which can then be used to search and filter a conformation database. The main characteristics of the Pharmacophore Query Editor are:

  • Interactive Modification. The positions, radii, as well as other characteristics of the pharmacophore query can be interactively adjusted.
  • Template Ligand. A ligand (possibly in complex with a receptor) can optionally be used as a "template" to simplify defining the features of a query.
  • Template Receptor. A receptor can optionally be used to help identify key interactions or as a template to simplify defining excluded volume constraints or other shape constraints.
  • Consensus Features. A set of aligned molecules can be used to suggest a consensus set of pharmacophore query features. Suggested features can be visually examined on the screen and selectively converted into features of an edited query.
  • Custom Rendering. The visual characteristics (like color) of the displayed query features and volumes can be individually adjusted. The annotation points of template molecules can be customized.

Pharmacophore Elucidator. The purpose of the Pharmacophore Elucidator is to generate a collection of pharmacophore queries from a collection of compounds some or all of which are active against a particular biological target such that all or most of the active compounds satisfy the generated queries. In the absence of receptor information, it is hoped that feature geometries common to many of the actives will contain information related to the important interactions between the bound conformations of the actives and the receptor. The fundamental approach for Pharmacophore Elucidation is to exhaustively search for all pharmacophore queries that induce a good overlay of most of the active molecules and separate actives from inactives (when inactives are presented). Thus, the plausibility of the pharmacophore is measured by overlay of actives (common binding mode) and the relationship to activity is measured by classification accuracy.

Pharmacophore Search. The purpose of the Pharmacophore Search is to use a pharmacophore query to filter databases of 3D conformations based on the positions of annotation points derived from each of the conformations. The main characteristics of the Pharmacophore Search are:

  • Boolean Expressions. Query features can contain boolean expressions to encode concepts such as "Donor and not Cationic".
  • Substructure Matching. Pharmacophore query features can be augmented with a SMARTS patterns to search for particular functional groups. The substructure tests are efficiently performed during the search itself (not as a post-filter).
  • Volume and Shape Filtering. The query can be focused by adding restrictions on the shape of the matched ligands in the form of a union of spheres. Alternatively, ligand hits can be filtered to enforce shape occupancy. SMARTS patterns can be applied to individual volume and shape constraints.
  • Partial Matching. Only those features of the query that are marked as "essential" must be matched to a corresponding ligand annotation point. Other query features may be left unmatched, resulting in a "partial match". The search outputs all partial matches of the query that contain at least a given minimum number of matched features.
  • Group Constraints. Match constraints can be applied to, for example, match "exactly one of" a subset of features, or to impose that a hydrogen bond heavy atom matches with its projected feature simultaneously.
  • Existing Conformations. External conformation databases can be searched efficiently since no indexing or prior preparation steps are required.
  • Systematic Matching. All possible matches of a ligand conformation and the query are systematically examined (and are reproducible).

Pharmacophore Annotation Schemes. Ligand annotations are performed by general purpose SVL functions encoding the atom typing and geometric rules to annotate 3D conformations. Consequently, the Pharmacophore Applications can be customized a great deal: a) by modifying the atom typing rules; or b) by modifying the SVL functions that implement the schemes; or c) by writing new custom schemes in SVL.
