The 2D Molecules panel is a repository for a small set of molecular structures, which are rendered onscreen in 2D, one at a time. The manner in which they are displayed can be adjusted with a number of options, which are applied to the whole set.

The molecule collection can be acquired from a number of sources, such as the MOE main window, the clipboard, or various file formats. The molecules can be transferred or exported to various destinations, such as the MOE main window, database files, raster or vector graphics files, or various clipboard flavors.

The molecules can be displayed using their original layout coordinates, or they can be redepicted automatically, in various modes, including unconstrained aesthetics, flattening to match original coordinates, flattening with group alignment, or alignment according to designated common scaffolds. Various display options can be changed, such as atom coloring, background, bond style, hydrogen rendering, stereo labels, and supplementary annotations.

The 2D molecule viewing panel can be invoked from the menu MOE | Compute | 2D Molecules, MOE | RHS | Ligand | 2D Molecules or from the command line by run 'view2d.svl':

Structure Depiction

The molecules loaded into the current set are drawn in 2D diagram style, using either the original source coordinates, or calculated 2D layout coordinates. The various options for the Depict mode are as follows:

Yes The coordinates for each molecule will be replaced by those calculated by the automatic 2D layout algorithm, as implemented by the DepictionCoordinates function.
No The original coordinates will always be used, even if they are unsuitable for 2D depiction.
Maybe The original coordinates will be replaced with automatic layout coordinates for molecules which appear to be 3D (i.e. the Z-coordinate is not always zero).
Flatten The 2D coordinates will be recalculated using the Flatten3DStructure function, which produces an aesthetically viable diagram layout, which matches the original inter-atomic distances as closely as possible without violating any rules of aesthetics.
Align The molecules are rendered collectively. This option applies only in cases where the original molecules are all aligned in 3D space. First, each molecule is "flattened", in order to produce a 2D layout which resembles the original 3D embedding. Then, each molecule is superimposed onto its original coordinates, and these are collectively rotated onto the page. The result is that the 2D coordinates of the molecule set are directly comparable, in an approximate way.
Scaffold Each of the molecules is aligned on the page according to a notion of common scaffold. Specification of the scaffold is done in a sub-section of the panel:

If the Auto checkbox is on, then the scaffolds are calculated, analyzed and depicted automatically, using the same algorithm as is used by the Structure Activity Report. Otherwise, when the Auto checkbox is off, the text box to its right is used to provide the means of ascertaining the common scaffold. If a SMARTS pattern is entered, it will be used to identify the scaffolds.

Pressing the mouse popup button on the text box brings down a list of recently-used SMARTS patterns, and also two special options. user-selection examines the selected atoms, and computes a SMILES string, which is used to identify the scaffold. The predefined option applies only when the input molecules already have their scaffold definitions encoded within the atom number field, which can be imported from MDL MOL and SD files.

To change the depiction mode, select the new option, and press the Apply button. All molecules will be updated.

Display Options

A variety of options can be selected which influence the way in which molecules are displayed on the screen, and also how they are exported in graphics formats.

Rotate All To rotate all of the current molecules by a selected value in degrees (anti-clockwise).
Invert If checked, then the coordinates of the 2D molecules will be inverted. Any wedge bonds will be adjusted accordingly.
Bonds The rendition of bond orders for certain functional groups may be modified. The options are:
  • Default
  • Kekulé
  • Resonance
  • Skeleton

By default, all bonds are displayed in their incoming style. For formats which require bond orders to be calculated, these consist of single, double and triple bonds, with Kekulé forms for aromatic rings. When the Kekulé option is selected, any aromatic rings which were originally specified in resonance form are converted to alternating single/double bonds. Conversely, the Resonance option will convert Kekulé style aromatic rings to the "ring" resonance notation.

The Skeleton option displays all bonds as single lines, regardless of their bond order.

Font There are 3 settings for font size, with the default being the smallest.
Hydrogen Hydrogen atoms for incoming molecules may be specified explicitly as separate atoms, or implicitly as part of a heteroatom heavy atom label, or not provided at all. The options for hydrogen display are as follows:
Inline Hydrogen counts are incorporated into heteroatom heavy atom labels. They are not displayed when attached to carbon atoms. In some cases, such as dihydrogen, or hydrogens attached to chiral ring junctions, explicit hydrogens are included in the diagram as distinct atoms. This is the most common convention for complete-molecule sketches.
Explicit All explicit hydrogens are drawn as distinct atoms. Note that this will not cause explicit hydrogens to be added to the structure, i.e. if the incoming structure is hydrogen-suppressed, then this option will have no effect.
None All hydrogen representation is excluded from the diagram, whether explicit or implicit.
Chiral Any explicit hydrogen atom attached to a chiral center is drawn as a distinct atom, while other hydrogen atoms will be treated in the default way. Note: To show chirality in the window, the chirality constraint must be set to the current geometry.
Stereo Chirality and double bond stereochemistry are traditionally annotated by using wedges and planar layout, respectively, but are ideally also represented by parity-based schemes. If the R/S button is selected, then the diagrams will be annotated with Cahn-Ingold-Prelog (CIP) notation, which is R or S for chiral centers, and E or Z for stereoactive double bonds.

Alternatively, if the Parity button is selected, then the diagrams are annotated with odd/even notation, where one dot encodes odd, and two dots encodes even. This notation has cheminformatic rather than chemical meaning.

Note: To show chirality in the window, the chirality constraint must be set to the current geometry.

Color Atoms When checked, the heteroatoms are drawn according to a predefined color scheme, which corresponds to that used for atoms in the MOE main window, e.g. oxygen is red, nitrogen is blue, etc.
Foreground The default foreground color may be changed, which is black by default. The foreground color is used to draw all atoms and bonds, unless specified otherwise, e.g. if the atoms are explicitly colored, or if scaffolds are defined.
Background The paper color for rendering the molecules may be changed. The default paper color is transparent, i.e. white when displayed onscreen, but when exported using any graphics format which supports transparency, the bounding box of the molecule will not obscure any underlying content.
Annotations The annotation options include a color, which defaults to blue, and a choice of additional annotation features is presented. The default, None, does not add any of these features to the diagram.

For incoming molecule formats which support an atom number field, such as MDL MOL, the Acol option can be selected, which causes any atom with an atom number of other than zero to be drawn using the annotation color, which overrides any other coloring scheme. This is particularly useful for showing reaction centers, or precomputed scaffolds.

Alternatively, the atom number field can be shown explicitly with drawn-in labels, by selecting the Anum option. The labels are arranged around the atoms themselves.

To display the index of each atom, select the Index options. This is useful for debugging cheminformatics algorithms, which refer to atoms by indices. Note that if the input molecule contains explicit hydrogens which are not being displayed, the original atom numbers are preserved, so the numbers will be non-consecutive where hydrogen atoms are removed.

Scaffolds When the Depict mode is set to Scaffolds, the drawing color for scaffold atoms can be selected, and the method of scaffold selection can be specified, as described previously.
Name Molecule formats generally provide some way to record a name for an explicit molecule species, and this is displayed in the Name text field for the selected molecule. To change the name of a molecule, edit the text, and press ENTER, or click Apply.
Rotate The rotation of just the currently selected molecule can be modified using this input control, and is applied immediately to the diagram. An explicit number can be entered, or the wheel control may be used.

All of the options, with the exception of Rotate, do not take effect until the Apply button is pressed.

Acquiring Molecules

When the panel opens, the default behavior is to examine the contents of the MOE main window, and for each chain which is not too large, nor considered to be solvent, convert the input into a sketch representation. This collection of molecules comprises the initial starting set, and may be empty. If there are atoms selected in the main window, then only components for which at least one atom is selected are included in the set.

Additional molecules can be added to the current set by clicking on the Acquire radio button, which opens up a set of action buttons:

FromMOE The contents of the MOE main window will be examined, in the same way as when the panel starts. Any new molecular species will be appended to the current list of molecules. No duplicate checking is done, so pressing this button multiple times will continually grow the set.
Clipboard The system clipboard will be examined for formats which are readily translated into chemical sketch representations. Such formats include MOE, MDL MOL and SD files, and ISIS/ChemDraw sketch data. Non-molecular data, such as pictures or text, will not be processed.

For single-sketch formats, such as MOL or ISIS/ChemDraw, the whole chemical species will be added to the set as a single entity. For MOE data, the appended molecules will be split by chain, one entity each. For MDL SD files, one entity is added for each record.

Entities with more than 1000 heavy atoms are excluded, and a maximum of 100 molecules may be loaded at any time.

ClipComp The same as the Clipboard button, except that each entity is further separated into individual connected components. This is useful when a number of molecules are drawn into the same sketch.
File... Prompts for an input file to be selected, then reads that file and adds each entity to the current set. Supported file formats include MOE, MDL MOL, MDL SDF, ISIS/ChemDraw, and MDB.

Managing Molecules

The 2D Molecules panel stores a set of between 0 and 100 molecules, one of which is displayed at a given time. When there are currently 2 or molecules loaded, part of the molecule display is taken up by a button-bank:

Each of the buttons shows a thumbnail depiction of the molecule that it represents. Clicking on any of the buttons makes that molecule the current molecule. The button for the currently selected molecule is shown with the selected background color (or white if transparent), while the non-selected molecules have standard gray buttons. The images shown in the buttons reflect the display settings in the same way as for the larger display underneath. When there are more than 10 molecules loaded, navigation buttons are added to their right.

As previously mentioned, the initial set of molecules is derived from the content of the MOE main window. New molecules can be added to the set using the Acquire buttons, as described above.

The currently selected molecule from the loaded set is shown in the main molecule display area. This will be blank if there are no molecules loaded.

If there is more than one compound in the loaded set, it can be selected by clicking on the miniature icon button, above the main molecule display. If there are more icons than will fit on one page, the left, right, start and end buttons can be used to navigate through the set.

To remove the currently selected molecule from the loaded set, click the Remove button. To clear out all of the loaded molecules, and optional replace them with the contents of the MOE main window, click the Reset button.

Output Graphics

The molecules shown in the panel may be exported to a variety of destinations in various formats, and to the extent that the format allows, will resemble what is seen in the panel.

Molecules can be exported as molecular object formats, raster graphics and vector graphics. They can be exported to files or copied to the clipboard, where platform capabilities allow. Molecular formats which allow multiple entries incorporate all loaded molecules, while those which do not provide the option of exporting one file per molecule. Molecules can also be transferred back to the MOE main window, and printed in tiled form.

Clipboard

Select the Copy radio button to bring up the clipboard submenu, which consists of action buttons, which correspond to the respective format. All clipboard copy operations function on the currently selected molecule only.

The following formats can be used to copy data onto the clipboard:

  • MOE
  • MOL
  • EMF+
  • SVG
  • BMP

Note that EMF+ and BMP formats are only supported for the clipboard on Microsoft Windows platforms. On POSIX systems (Linux, Mac OS X, etc.), these formats can be generated, but only by writing to files, as is described below.

When copying EMF+ graphics to the clipboard on Microsoft Windows platforms, it is possible with many presentation applications to paste the graphic into a document. With some applications, such as older versions of Microsoft Office, it may be necessary to use a menu action such as "Paste Special", and select the EMF+ portion of the clipboard data. The resulting graphic will be rendered with anti-aliased lines and fonts, scale seamlessly, and print at maximum resolution.

Export

The current set of molecules can be written to one file, or a group of files, or in some cases, the currently selected molecule may be written to a single file, depending on the format. The following export formats are supported:

  • MOE
  • MOL
  • SDF
  • EMF+
  • SVG
  • PNG
  • JPG
  • GIF
  • BMP

The MOE and SDF formats are naturally capable of holding multiple molecules (separate chains and records, respectively), and so when the currently loaded set has more than one entry, only a single file is required to export them.

All of the other formats are not readily compatible with multiple entries, and thus a series of files will be written, one for each molecule. The output filenames will be a series based on the filename prefix that is entered in the textfield of the export panel:

Pressing <Return> or OK will populate the lower part of the panel with the names of the files that are to be written. Pressing OK once more will initiate the export operation.

If only the current molecule is to be exported, turn on the Current Molecule Only checkbox. Just a single file will be created.

Print

The loaded molecules can be arranged in a printable format, which arranges the molecules on pages in tiled mode, in a print preview dialog:

The molecules are rendered with display options at the time the print panel was launched. The display of molecule names, numbers and borders between tiles can be toggled with the respective options.

Graphics Formats

There are a number of formats which can be used to capture the visual representation of a molecule. These formats are divided between raster and vector formats. Raster formats, often described collectively as "bitmaps" or "pixmaps", are matrices of pixels, each of which have a specific color value. They tend to be well supported and easy to deploy using the largest possible variety of image presentation software, but they use up an amount of disk space proportional to the resolution, cannot be scaled to a larger size than that at which they were generated, and not all software is capable of gracefully shrinking raster graphics. Raster graphics also do not print well unless they are generated at a particularly high resolution.

Vector graphics, on the other hand, can be grown or shrunk to any resolution without penalty, and so in principle can be displayed on any device without prior knowledge of the destination. There are few universally supported vector formats, however, and the output quality is heavily dependent on the properties of the rendering software, if it supports the format at all. Many vector formats are designed for printers, but do not support features such as alpha blending, and may be rendered poorly on-screen due to lack of anti-aliasing. Problems are also often encountered with portability of fonts, and the absence of a vector graphics standard for web browsers can be an issue.

The 2D Molecules panel can generate graphics in the following formats:

PNG Portable Network Graphics. Raster. Supports alpha transparency levels and lossless compression. Compatible with almost all graphics manipulation and presentation software, including all modern browsers.
JPG Joint Photographic Experts Graphics format. Raster. Does not support any kind of transparency, and does not support lossless compression, which makes it a relatively poor choice for molecular diagrams, but is useful for some compatibility purposes.
GIF Graphical Interchange Format. Raster. Supports binary transparency and lossless compression, but is restricted to 8-bit paletted color. This format is adequate for molecular diagram graphics, but use of transparent backgrounds it not recommended.
BMP Microsoft Windows Bitmap format. Raster. Supports 24-bit color and is lossless, but even though modern variants of the format include compression, this is not used, in order to maximize compatibility. Lack of well supported transparency, unnecessarily large output files, and lack of support from web browsers make this format mainly useful when interacting with certain Windows applications.
EMF+ Enhanced Metafile Plus format. Vector. This is a Windows-specific variant of the Enhanced Metafile format, which has been extended to include advanced features such as alpha-level transparency and anti-aliasing, which are critical to rendering molecular graphics equally well on all devices, at any level of magnification. While EMF+ graphics can be generated from MOE on any supported platform, support for rendering EMF+ is largely confined to Microsoft Windows, from 2000 onwards. This format should not be confused with EMF, which is closely related, but not as sophisticated.
SVG Scalable Vector Graphics format. Vector. XML-based. This is an international standard for sophisticated vector graphics, the features of which are a superset of those needed to render molecules in 2D. Support for this format should not be taken for granted, and is often incomplete. Rendering will not necessarily take advantage of high quality anti-aliasing, and inconsistent support for embedded fonts may cause atom labels to vary in appearance depending on the device. However, a fully compliant SVG renderer can reproduce graphics of a quality at least as good as is seen in the 2D Molecules panel.

Before exporting graphics or copying to the clipboard, it is often useful to control the properties of the output, in particular the bounding box or resolution of the resulting images. For raster formats, this is particularly important, because the resolution cannot be increased after the file is written. For vector formats, it can be convenient to specify the default graphic size, even though the images can subsequently be magnified without penalty.

When the Image radio button is pressed, the following options are provided:

When the property of Boundary is set to None, the size of each molecular graphic is determined by using industry standard settings for the number of angstroms (in the units of the diagram) per millimeter (units of the output graphic). The information which is required is the number of dots the intended rendering device can display per unit of length, which is typically given as dots-per-inch (DPI). A value of 300 dpi is sufficient for a low budget printer, but far too high for contemporary monitors. It is safer to overestimate the resolution than to underestimate it, because most software is capable of scaling raster images down, but it is not possible to scale it up with adequate results. Unfortunately, target resolution specifications within graphics formats are not always reliable: many rendering engines will display a raster graphic at 1:1 scale by default, which can be inconvenient. For many purposes, it is advisable to select the correct resolution for the device. The same advice applies to vector formats, though it is only for convenience.

If the Boundary is set to Box, then the parameters which are requested are width (W) and height (H). For raster formats, this measurement is in pixels, and for vector formats it is an arbitrary unit of dimension roughly equivalent to screen pixels. When a bounding box is used, instead of a resolution, the dimensions of the output graphics are guaranteed to be that which is specified, for all molecules. This is particularly useful for a series, e.g. when rendering in order to place inside a table which needs to have a uniform cell size.
