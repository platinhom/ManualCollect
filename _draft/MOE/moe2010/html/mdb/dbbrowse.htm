The Database Browser

The Database Browser is used to view fields within a MOE database, one entry at a time. Usually fields are viewed individually using the default field viewers, but specially-created databases may install additional viewing modes.

To open the browser, choose DBV | File | Browse in the Database Viewer.

By default, each field in the MOE database is bound to either one or two viewers, which present the content of a single field in the Subject option. Databases which have been created by special applications may have their own viewers installed in addition to these.

In most cases the 3D molecule is the default subject, as shown above. The 3D viewer is always available for fields of type molecule. The browser displays molecule data by instantiating the molecular species in the MOE Window.

Ordinarily when browsing in 3D, the molecule will be removed from the MOE Window when the database browser is closed, or a different entry is selected. Pressing the Keep button causes the molecule to remain in the MOE Window when it would otherwise be deleted.

2D molecule views, and scalar fields, are rendered within the Database Browser panel rather than in the MOE Window.

Customized views may manifest themselves in the MOE Window or the Database Browser panel (for example, see the section on on animating molecular dynamics below), or a combination of both.

The Database Browser supports the following intrinsic features:

Database The filename of the current database being browsed.
Subject The identifier name for the viewer instance. All databases provide 3D and 2D viewer instances for each of their fields for which the type is molecule, and a simple scalar viewer instance for each of the fields of type byte, int, float, double and char.

Additional viewers may be available if the database was created by a custom application, such as Molecular Dynamics or Pharmacophore Elucidation.

Entry The entry number in the database may be selected using the slider control, or by entering the number in the text field to the right. The navigation buttons at the bottom of the panel can also be used.
Selected If the database is also open with the Database Viewer, the selection state of the presently browsed entry is reflected by the state of the Selected checkbox. If it is changed in the Database Browser, the Database Viewer will also note the change.
Browse Selected The domain of entries to which the browser is restricted is one of the following three possibilities:

  • If the database being browsed is not also open with the Database Viewer, then all of the records are browsed.

  • If the Browse Selected box is not checked, or there are no selected entries, the set of browsed entries is all which are visible in the Database Viewer.

  • If the Browse Selected box is checked and at least one entry is selected, the entries browsed are restricted to selected entries only.

Center View (3D viewer only) If checked, the MOE Window will be centered and aligned to show the browsed molecule.
Layout Coordinates (2D viewer only) If checked, the molecule will be redrawn in 2D by calling the DepictionCoordinates function prior to displaying it in the panel. Otherwise, the (X,Y) coordinates from the structure in the database will be used.

Animating Molecular Dynamics

Databases created by the Molecular Dynamics application support a special dynamics animation viewing mode in the Database Browser. In such databases, the Database Browser opens, by default, with Subject set to Dynamics. The basic mode features of the Database Browser are still available, as appropriate, when Subject is changed to some other field.

In Dynamics mode, the panel appears as follows:

The additional options are:

Animation The animation wheel controls the speed of the animation, i.e. the speed at which the different conformations are loaded into MOE.
Loop If on, the animation will run continuously, returning to the beginning of the database when the end of the database has been reached, or vice-versa if Reverse is on. If off, the animation will stop when the end of the database is reached.
Reverse If on, the animation will run from the end of the database backwards to the beginning, i.e. time will run in reverse. If off, the animation will run forwards from the start of the database.
