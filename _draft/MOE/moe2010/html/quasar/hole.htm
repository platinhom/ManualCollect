QuaSAR-Hole Filler

MOE's QuaSAR-HoleFiller is used to generate a new database of compounds which spans the diversity of molecules in multiple source databases.

Methodology

QuaSAR-HoleFiller clusters its source databases based on the set of descriptors chosen for diversity calculation. The application is called a "hole filler" because it fills gaps in coverage of the first source database.

The source databases are processed in the order they are specified in the interface. The first source database is clustered, and a unique representative from each cluster written to the output. The subsequent databases are clustered in succession, with representatives drawn only from clusters with codes different from what is already represented in the output database. The resulting output database will contain molecules each with a unique cluster code.

Running QuaSAR-HoleFiller

To create a HoleFiller database, open the QuaSAR-HoleFiller panel by choosing: MOE | Compute | QuaSAR | QuaSAR-HoleFiller. The following panel appears:

The QuaSAR-HoleFiller panel includes the following options:

Output Database Name of the database will contain the molecular structure and the source library filename of every unique compound, as determined by the HoleFiller clustering. The default name for the Output Database is holefill.mdb, although any filename may be used.
Source Database 1-4 A maximum of four source databases, whose paths and filenames must be typed into the Source Database text fields, may be searched in any given HoleFiller calculation. The first molecular field in each source databases will be used by QuaSAR-HoleFiller.
Smoothing Standard deviation of the normal distribution used to smooth accumulated distributions during clustering.
Code Count Controls the number of equiprobable subdivisions along each principle component axis, and is equal to the number of letter codes used per axis. Small code counts lead to fewer clusters.
Component Limit Sets the maximum number of principle components; a value of zero sets no limit to the number of principle components.
Condition Limit Sets the maximum condition number for the principle component transformation; the largest eigenvalue of the generated covariance matrix divided by the smallest eigenvalue will be smaller than this parameter.
Descriptors List of descriptors. Select the one or more descriptors to be used by the HoleFiller operation. These descriptors will automatically be calculated for each compound in all the source databases. There is no limit to the number or types of descriptors which may be used in any given HoleFiller calculation.

The diversity of the database generated by QuaSAR-HoleFiller will depend on the descriptors used in the calculation. For example, if the number of carbon atoms (a_nC) is the only descriptor chosen for a HoleFiller calculation, each compound in the output database will contain a different number of carbon atoms, and no other properties or characteristics will be used to differentiate the compounds.

The QuaSAR-HoleFiller calculation is initiated by pressing OK.
