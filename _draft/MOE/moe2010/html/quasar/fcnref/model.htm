MOE 2010.10

model_EvaluateMDB

Syntax

    model_EvaluateMDB [mdb_file, model, outfield, opt]

    opt = [
	esel:	flag,
	evis:	flag,
	usedbi:	flag
    ]

Description

model_EvaluateMDB evaluates a model over a database. The function takes the following parameters:

'mdb_file'
Molecular database filename token.

'model'
Name of the model file supplied as a token. The model can be of any of the recognized model types: linear or binary QSAR, pharmacophore, fingerprint, or composite (MOE model).

'outfield'
The name of the field that will contain the predicted values from the model.

esel: flag
If flag is non-zero then only the selected entries will be used in the calculations. The default is 0. This option is ignored in moebatch (all entries are evaluated).

evis: flag
If flag is non-zero then only the visible entries will be used in the calculations. The default is 0. This option is ignored in moebatch (all entries are evaluated).

usedbi flag
If flag is non-zero then if the database contains fields required by the model, those field values are used in the evaluation (any missing fields are calculated). Otherwise, all fields required by the model will be recalculated. The default is 1.

Note: Any fields that cannot be recalculated must be present in the database, in which case usedbi must be turned on.
